# Zoom Agent Tool

Standalone интеграция Zoom для AI-агентов на платформе Chatium.

## Структура проекта

```
/zoom-agent-tool/
├── tables/
│   ├── zoom_settings.table      # Таблица настроек API (секреты + дефолты)
│   └── zoom_meetings.table      # Таблица созданных встреч
├── api/
│   ├── zoom-api.ts              # Core Zoom API функции
│   ├── datetime.ts              # Конвертация даты/времени в timestamp
│   ├── zoom/create.ts           # API: создание встречи
│   ├── zoom/recordings.ts       # API: список записей
│   ├── zoom/test.ts             # API: проверка подключения
│   ├── settings/get.ts          # API: получение настроек
│   └── settings/save.ts         # API: сохранение настроек
├── tools/
│   └── zoom.ts                  # Tools для AI-агента
├── pages/
│   └── SettingsPage.vue         # Страница настроек (admin only)
├── index.tsx                    # Главная страница
└── settings.tsx                 # Страница настроек
```

## Функционал

### Для AI-агента (Tools)

1. **zoomAgentCreateMeeting** — создание Zoom-встречи
   - Параметры: topic, agenda, duration, auto_recording
   - **Время начала**: start_timestamp (Unix timestamp в секундах), timezone (IANA формат)
   - Если время не указано — создаётся "прямо сейчас" с часовым поясом из настроек
   - Возвращает ссылки для организатора и участников
   - ⚠️ **ВАЖНО**: Для запланированных встреч сначала вызовите `zoomAgentParseDateTime`, а затем передайте полученный timestamp в `zoomAgentCreateMeeting`

2. **zoomAgentParseDateTime** — конвертация удобочитаемой даты в timestamp
   - Принимает: date_time (например, "завтра 20:00", "2026-05-03 20:00", "понедельник 10:00")
   - Возвращает: timestamp, formatted_date, iso_string
   - Используется перед созданием встречи для точного расчёта времени

3. **zoomAgentListRecordings** — список облачных записей
   - Записи за последние 30 дней
   - Ссылки для скачивания

### Настройки (только для администраторов)

Страница `/settings` доступна только пользователям с ролью Admin.

**API Credentials:**
- Account ID — из Zoom Marketplace
- Client ID — из Zoom Marketplace  
- Client Secret — из Zoom Marketplace
- Secret Token — для вебхуков (опционально)

**Дефолтные настройки встреч:**
- Тема встречи (default_topic)
- Повестка (default_agenda)
- Автоматическая запись: none / local / cloud
- Часовой пояс по умолчанию (IANA формат: Europe/Moscow, America/New_York и т.д.)

### Как получить API-ключи

1. Перейдите в [Zoom Marketplace](https://marketplace.zoom.us/user/build)
2. Создайте новое приложение типа "Server-to-Server OAuth"
3. Скопируйте Account ID, Client ID, Client Secret
4. Вставьте в настройки на странице `/settings`

## API Endpoints

| Endpoint | Method | Описание |
|----------|--------|----------|
| `/api/zoom/create` | POST | Создание встречи |
| `/api/zoom/recordings` | GET | Список записей |
| `/api/zoom/test` | GET | Проверка подключения |
| `/api/settings/get` | GET | Получение настроек |
| `/api/settings/save` | POST | Сохранение настроек |
| `/api/datetime` | - | Конвертация даты/времени в timestamp (shared функция) |

## Диагностика: агент не видит тул

Если агент иногда не видит тул, хотя он включён в настройках:

### Возможные причины

1. **Конфликт accountHook** — если в аккаунте несколько workspace'ов с `@start/agent/tools`, они могут перезаписывать друг друга. Проверьте, нет ли других workspace'ов с такими же хуками.

2. **Кэширование** — система может кэшировать список тулов. Попробуйте пересохранить файл с тулхуками или перезапустить агента.

3. **Структура возврата из хука** — должен возвращаться объект с тулхуками, не массив:
   ```typescript
   return {
     toolName: toolFunction,
     anotherTool: anotherToolFunction
   }
   ```

4. **Должны быть exports** — каждый тул должен быть экспортирован (`export const`).

5. **Поля meta** — обязательны поля `name` и `description`, рекомендуется `llmDescription` для AI-агента.

### Как проверить

Добавьте логирование в accountHook для диагностики:
```typescript
app.accountHook('@start/agent/tools', async (ctx, params) => {
  ctx.account.log('🔧 Tools hook called', { 
    level: 'info',
    json: { params, tools: Object.keys(toolsObject) } 
  })
  return toolsObject
})
```

Проверьте логи аккаунта — вызывается ли хук при запросе к агенту.
