# Команды Claude Code — индекс

Эта папка содержит slash-команды для Claude Code в этом workspace.

## Доступные команды

| Файл | Команда | Назначение |
|------|---------|------------|
| `pipeline.md` | `/pipeline` | Авто-конвейер: квалифицирует задачу, сам выбирает уровень 1–10 и запускает его. Для случая «выбери глубину за меня» |
| `pp1.md`…`pp10.md` | `/pp1`…`/pp10` | Масштабируемый конвейер разработки с явным уровнем. Уровень 1–10 задаёт модели, число шагов, параллелизм и бюджет: pp1 — быстро/дёшево (без агентов), pp10 — максимальная автономная проработка (дорого, ~1.5–2 ч) |
| `check.md` | `/check` | Технические проверки: типы (vue-tsc), стиль (Prettier), стандарты, роутинг, рантайм, тесты |
| `toprod.md` | `/toprod` | Деплой ассистента из dev-workspace в prod-workspace |

## Выбор уровня конвейера

Уровень = сложность задачи. Это и есть оптимизация по токенам и времени.

- **pp1–2** — опечатки, однострочники, переименования (правит основной чат, без агентов).
- **pp3–4** — локальная фича/багфикс в 1–3 файлах (implementer + одно консолидированное ревью на Sonnet).
- **pp5–6** — фича со связями api+lib+ui (формализация, план с ревью, checker'ы, платформенный гейт).
- **pp7–8** — сложная фича, платформенные тонкости (Opus на плане/ревью/гейтах, полный fan-out).
- **pp9–10** — крупные/рискованные задачи, миграции (adversarial, loop-until-clean, мульти-план).

```
/pp4 добавь поле email в форму регистрации
/pp9 переработай систему платёжных виджетов под единый поток
```

Команды автоматически подхватываются Claude Code из этой директории.

## Связь с агентами

Все уровни `/ppN` исполняются основным чатом по единому плейбуку
`.claude/agents/pp-orchestrator.md` (матрица уровней, модели, фазы, anti-patterns).
Плейбук делегирует роли субагентам из `.claude/agents/`. Уровень определяет, какие
роли запускаются и на какой модели.
