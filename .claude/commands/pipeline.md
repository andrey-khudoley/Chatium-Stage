# /pipeline

Активируй **полный конвейер разработки**. **Этот чат сам становится оркестратором** — он следует плейбуку `.claude/agents/pipeline-orchestrator.md` и делегирует шаги субагентам напрямую.

> **Принцип.** Оркестрацию ведёт основной чат, а не субагент. Оркестратору нужно порождать субагентов на шагах 2–5, 8–9, 10, а субагент вложенных субагентов порождать не может — это умеет только основной чат. Поэтому `pipeline-orchestrator` запускать через Agent tool нельзя.

## Что произойдёт

Конвейер пройдёт 12 шагов в 5 фазах (модели закреплены в субагентах: Opus — рассуждение, Sonnet — генерация, Haiku — механика):

**Фаза 1 — Постановка**
1. Принять задачу + зафиксировать START-таймстемп
2. Формализация (Agent → `task-formalizer`, sonnet)

**Фаза 2 — План**
3. План (Agent → `planner`, **opus**)
4. Ревью плана **параллельно**: `plan-reviewer` (opus) + `chatium-platform-checker` mode=plan (opus)
5. Полнота ревью плана (Agent → `completeness-reviewer`, haiku)
6. Решение по плану (цикл 3→4→5, ≤ 3 итерации)

**Фаза 3 — Реализация**
7. Написание кода по плану (Agent → `implementer`, **sonnet**)

**Фаза 4 — Верификация**
8. Параллельный fan-out: `code-reviewer` (opus), `chatium-platform-checker` mode=code (opus), `standards-checker` (haiku), `file-based-routing-checker` (haiku), `runtime-architecture-checker` (opus) + типы/стиль/тесты (Bash) — по плейбуку `verification-runner`
9. Полнота ревью кода (Agent → `completeness-reviewer`, haiku)
10. Цикл исправлений (правки делегируются `implementer`, 8→9, ≤ 3 итерации)

**Фаза 5 — Финал**
11. Документация (Agent → `docs-keeper`, sonnet, опционально)
12. Метрики цикла (`scripts/pipeline-usage.mjs`: время + токены по моделям) + финальный отчёт

## Инструкция запуска

1. Прочитай плейбук `.claude/agents/pipeline-orchestrator.md` через Read.
2. **Прими роль оркестратора в этом же чате** и веди задачу строго по плейбуку — все 12 шагов.
3. **Не** вызывай Agent с `subagent_type: pipeline-orchestrator` — оркестрация ведётся здесь, в основном чате.
4. Контекст для оркестрации:
   - **Формулировка задачи:** $ARGUMENTS (если пусто — возьми последнюю формулировку из чата).
   - Переписка текущего чата — твой контекст, ты уже им владеешь.

Инварианты (из плейбука):

- Шаги 2–5, 8–9, 10 — **только** через Agent tool (ты, основной чат, порождаешь этих субагентов напрямую).
- Все 12 шагов обязательны; конвейер не считается завершённым без шага 12.
- Цикл правок ≤ 3 итерации.

## Если конвейер прерывается на вопросах формализации

`task-formalizer` может вернуть уточняющие вопросы. Передай их пользователю **дословно** и остановись. Дождись ответа пользователя в новом сообщении и продолжи конвейер с шага 2.
