# /pipeline

Активируй **полный конвейер разработки** через субагента `pipeline-orchestrator`.

## Что произойдёт

Конвейер пройдёт 12 шагов:

1. Принять задачу
2. Формализация (Agent → `task-formalizer`)
3. План (Agent → `planner`)
4. Ревью плана (Agent → `plan-reviewer`)
5. Полнота ревью плана (Agent → `completeness-reviewer`)
6. Решение по плану (цикл 3→4→5 при правках)
7. Реализация кода (этот чат)
8. Ревью кода (Agent → `code-reviewer`)
9. Полнота ревью кода (Agent → `completeness-reviewer`)
10. Технические проверки (Agent → `verification-runner`)
11. Цикл исправлений (8→9→10 при критичных замечаниях, ≤ 3 итерации)
12. Финальный отчёт + опционально `docs-keeper`

## Инструкция запуска

Сейчас вызови **Agent** с `subagent_type: pipeline-orchestrator` и передай в промпт:

1. **Формулировка задачи:** $ARGUMENTS (если пусто — возьми последнюю формулировку из чата).
2. **Полная переписка текущего чата** — она нужна оркестратору для контекста.
3. **Явное напоминание инвариантов:**
   - Шаги 2–5, 8–9, 10 — **только** через Agent tool, не выполнять самостоятельно.
   - Все 12 шагов обязательны; конвейер не считается завершённым без шага 12.
   - Цикл правок ≤ 3 итерации.

После запуска оркестратора **не выполняй** работу субагентов сам в этом чате — оркестратор сделает делегирование.

## Если конвейер прерывается на вопросах формализации

`task-formalizer` может вернуть уточняющие вопросы. Оркестратор передаст их пользователю и остановится. Дождись ответа пользователя в новом сообщении и продолжи конвейер с шага 2.
