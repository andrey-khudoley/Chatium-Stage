# /commit

Вызови навык **`auto-commit`** через **Skill tool** и передай ему `$ARGUMENTS`.

Навык работает **асинхронно** через Haiku-субагентов:

1. **Основной поток** соберёт незакоммиченные изменения и определит **границы
   задач** — группы коммитов и точный список файлов в каждой.
2. На каждую группу основной поток породит **отдельного субагента** на модели
   **Haiku**, все — **параллельно** (fan-out одним сообщением).
3. Каждый субагент читает дифф своих файлов, формулирует сообщение в формате
   `type(scope): описание` + список изменений, и **сам коммитит** свою группу
   явными путями (без `git add -A`). Реальный `git commit` сериализуется через
   lock-файл — гонки за git-индексом нет.
4. **Основной поток** делает **один `git push`** в конце (если не `--no-push`).
5. Вернёт список созданных коммитов и результат пуша.

## Аргументы (`$ARGUMENTS`)

- `--no-push` — только коммиты, без пуша.
- свободный текст — подсказка по группировке или формулировкам (учесть при работе навыка).

## Важно

- Не запускай `/check` автоматически — это решает пользователь до коммита (CLAUDE.md §5).
- Не вмешивайся в работу навыка вручную: вся логика группировки и формулировок — в `auto-commit`.
