# Проект p/saas/gateways/gc_api

## Что это
Gateway GetCourse на Chatium: приложение `p/saas/gateways/gc_api` (на базе `template_project`). Публичный контракт **`/api/v1/invoke`** (Bearer по школе), **`/api/v1/operations`**, **`/health`**, онбординг и ротация токена. Heap: settings, logs, **gatewaySchool**, **opCatalog**, **requestLog**, **openapiCache**. Реестр операций — `shared/opRegistry.ts` (+ OpenAPI для контура new). Входящие webhook от GC на это приложение **не** настраиваются ([ADR-0003](docs/ADR/0003-gateway-outgoing-only.md)). Детали — `README.md`, `docs/architecture.md`, `docs/api.md`.

Страница тестов `/web/tests`: вкладки юнит и интеграция дергают сервер через `route.run(ctx)` к `api/tests/unit` и `api/tests/integration` (`@shared-route`), не через самособранный `fetch` URL.

`lib/tests/templateUnitSuite.ts`: тестовые master key в base64 — строковые литералы, без `Buffer` при загрузке модуля (в UGC нет Node Buffer).
