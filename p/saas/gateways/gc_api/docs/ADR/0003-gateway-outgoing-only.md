# ADR-0003: Gateway только для исходящих вызовов в GetCourse

## Status
Accepted

## Context
Тонкие клиентские приложения Chatium должны вызывать GetCourse API через единый gateway (`op` + `args`), не храня у себя dev-ключи и не дублируя маппинг контуров new/Legacy.

Входящие POST от GetCourse (события, колбэки) по продуктовой модели идут **напрямую** в приложения-потребители (URL с секретом в query), а не через этот gateway.

## Decision
- Gateway реализует **только исходящий** путь: клиент → `POST /api/v1/invoke` → GC API.
- Отдельного эндпоинта «приём webhook от GC» на gateway **нет**.
- Регистрация URI приёма событий в GC выполняется обычной операцией реестра (`setUri`) через тот же `/v1/invoke`.

## Consequences
- Проще модель безопасности: gateway не обрабатывает подписи входящих тел от GC.
- Владельцы приложений C/D настраивают вебхуки в GC на свои URL; gateway остаётся общей прокладкой для API-вызовов.
