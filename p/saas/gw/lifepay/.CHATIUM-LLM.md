# Проект lifepay (LifePay payments-gateway)

## Что это
LifePay payments-gateway: реализация публичного контура `bills_v1` (`/api/v1/createBill`,
`getBillStatus`, `cancelBill`, `operations`) с маршрутизацией к LifePay через `@app/request`.
На `/` — панель оператора с журналами входящих/исходящих запросов (PII-маска, raw-просмотр).

## Логирование (актуализация 2026-04-05)
- **SSR logLevel**: все 5 страниц (главная, админка, профиль, тесты, логин) читают уровень из настроек через `getLogLevelForPage` и инжектят `window.__BOOT__.logLevel` через `getLogLevelScript`.
- **browserRemoteLogger**: подключён на главной, админке, профиле и тестах. Перехватывает console.*, window.onerror, unhandledrejection; логи из `setLogSink` также передаются. Пакетная отправка на POST /api/logger/browser (требует авторизации). На логине — не подключён (пользователь не аутентифицирован).
- **Info vs Debug**: trace-логи (карта вызовов: entry/exit функций) имеют severity 6 (Info). Payload (сырые данные) автоматически отсекается при уровне != Debug: на сервере — `shouldIncludePayload` в `lib/logger.lib.ts`, в браузере — `emitLog` фильтрует non-string args в `shared/logger.ts`. При Debug видна полная карта с данными, при Info — только текст вызовов.
- **Подробные этапы**: каждая страница логирует монтирование с сырыми props (URL-ы, auth state, user data), состояние boot loader, инициализацию remote logger, анимации, размонтирование. Все функции и ветки покрыты trace-логами.
- **Управление уровнем**: настройка `log_level` (Debug/Info/Warn/Error/Disable) в админке; порог применяется и на клиенте (`shouldLog` в shared/logger), и на сервере (`shouldLogByLevel` в lib/logger.lib).

Ширина: `w-full` / `width:100%` у корня, main и `.ap`/`.tp` (контент до 1440px по центру).

## Панель оператора (актуализация 2026-05-16)
- **Главная `/` — это панель** (`pages/HomePage.vue`), Admin-only через `requireAccountRole(ctx, 'Admin')` в `index.tsx`. Неадмин → 302 на `/web/login?back=/`.
- KPI за 24ч + журналы:
  - `gatewayRequestLog` (`tables/gatewayRequestLog.table.ts`, UID `t__saas-gw-lifepay__greq__G4n2Lp`) — входящие запросы клиентов, поля включая `rawArgs` (PII-маска) и `rawHeadersSafe` (без секретов);
  - `gatewayUpstreamLog` (`tables/gatewayUpstreamLog.table.ts`, UID `t__saas-gw-lifepay__gups__U7q3Mn`) — исходящие к LifePay, поле `rawLpJson` (PII-маска).
- Запись организована через `finally`-обёртку в `lib/gateway/handleV1Op.ts`: локальный аккумулятор `GatewayLogCtx` заполняется по ходу, в `finally` блока `handleV1Op` функция `writeGatewayLogs` записывает обе таблицы (request — всегда; upstream — если был реальный вызов).
- Утилита `shared/redactRaw.ts` (`redactRawDeep`) — рекурсивная PII-маска и удаление секретов, лимит 64 KB с усечением. Файл синхронизирован с клиентским `p/units/aayakovleva/sbp-client/shared/redactRaw.ts`.
- API панели (Admin-only): `GET /api/admin/raw/{requests,upstream}/recent`, `GET /api/admin/raw/{requests,upstream}/get?id=`, `GET /api/admin/dashboard/gatewayCounts`.
