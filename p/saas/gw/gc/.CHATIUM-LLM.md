# Проект `p/saas/gw/gc` — GetCourse Gateway

## Что это

Chatium-приложение: gateway к API GetCourse на каркасе `template_project` (страницы, админка, логи, тесты). Путь в воркспэйсе: **`p/saas/gw/gc`** (`PROJECT_ROOT` в `config/routes.tsx`).

## Норматив разработки gateway

Единый техдок слоя (маршруты `/v1/{op}`, заголовки, ошибки, маппинг на GC) и JSON-артефакты лежат в **`docs/SPEC/`** приложения. Ключевые файлы:

- `docs/SPEC/gateway-operation-manual.md` — SSOT разработки (§0–§13: жизненный цикл `/v1/{op}`, каталог `GET /v1/operations`, контуры, авторизация, наблюдаемость, коды ошибок; §9.3 версионирование, §12 журнал решений, §13 «История изменений»). Ключевые нормативы: таймаут к GC = **10 000 мс** (§8.1, `GW_OUTBOUND_TIMEOUT_MS`); лимит входящего тела `POST /v1/{op}` = **1 MiB** (§8.7, `GW_MAX_REQUEST_BODY_BYTES`, превышение → `INVOKE_BODY_TOO_LARGE` HTTP 413); идемпотентность и серверные ретраи в gateway **запрещены** (§8.6); единственная версия публичного API — `/v1/` (§9.3).
- `docs/SPEC/gateway-testing-strategy.md` — стратегия юнит- и интеграционного тестирования (фазы прогона, цепочки `op`, 1 rps, страница тестов в админке).
- `docs/SPEC/implementation-plan.md` — пошаговый план реализации (Прототип → MVP → Прод). **Точка старта разработки**; закрытие всех чек-боксов = готовый продукт.
- `docs/SPEC/gc-unified-op-registry-v0.md`, `gc-op-http-mapping.json`, `gc-required-fields-by-op.json`, `new_api_schema.json`, `legacy_api_schema.json`, `extract-gc-required-fields.py` — машиночитаемые контракты.

## Критические правила

- Не хардкодить базовый путь приложения: `config/routes.tsx`, `withProjectRoot`.
- Клиентский бандл: не тянуть `lib/`, `repos/`, `tables/` из Vue — только `shared/` с `// @shared` и контракты `api/*/route`.
- Логирование в прикладном коде: через `lib/logger.lib` (`writeServerLog`), не `console.log`.
