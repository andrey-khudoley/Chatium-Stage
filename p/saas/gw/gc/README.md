# GetCourse Gateway (`p/saas/gw/gc`)

## Назначение

Приложение Chatium — экземпляр **gateway** к API GetCourse: публичные маршруты вида `/v1/{op}`, каталог операций, проксирование на школу по заголовкам `X-Gc-School-Host` и `X-Gc-School-Api-Key`, настройки и логи в Heap (каркас унаследован от `p/template_project`).

Полный норматив разработки слоя gateway и машиночитаемые артефакты (`gc-op-http-mapping.json`, OpenAPI JSON, реестр `op` и т.д.) лежат в каталоге **`docs/gateway/`** (точка входа: `docs/gateway/gateway-operation-manual.md`). Код в этом приложении должен согласовываться с этим manual на этапе активной разработки.

## Точка старта разработки

Пошаговый план реализации (Прототип до эфира → MVP → Прод) — **`docs/gateway/implementation-plan.md`**. Закрытие всех чек-боксов = реализация продукта; каждый пункт ссылается на конкретный `§N` manual или файл проекта.

## Платформа и деплой

- Платформа: Chatium; локальный сервер не требуется; зависимости фиксированы платформой.
- Деплой: закоммитить и запушить изменения — Chatium подхватит обновления.
- URL приложения: `https://<домен>/p/saas/gw/gc/` (и подмаршруты из `config/routes.tsx`).

## Текущее состояние

Сохранены возможности шаблона: админка, профиль, логин, страница тестов, API настроек, Heap-таблицы settings/logs (с **отдельными** ключами таблиц для этого проекта), серверные логи, дашборд админки.

Реализованы публичные **`GET`/`POST /v1/{op}`** для всех 59 операций. **Архитектура (этап 2 рефакторинга, финал):** общий обработчик **`lib/gateway/handleV1Op.ts`** (`handleV1Op(ctx, req, op, handler)`: requestId → каталог → метод → availability → размер тела → заголовки школы → `gc_developer_api_key` → content-type/body → валидация `args` → path-template → вызов per-op handler → классификация ответа GC → raw-журналы в finally) + per-op handler в каждом **`api/v1/{op}.ts`** (явный выбор клиента: `invokeNewGcApi` / `invokeLegacyGcImportPost` / `invokeLegacyGcExportGet`) + реестр **`lib/gateway/v1OpHandlers.ts`** (единый путь для роутов и тест-раннера). Каталог **`lib/gateway/operationsCatalog.ts`** — рукописный массив 59 записей (`argsValidator` + `argsSchema.fields[]`), без `*.generated`. **`GET /v1/operations`** через `toOperationSummaries()` — ответ в формате `fields[]` (публичный контракт изменён). Raw-журналы **`gatewayRequestLog`** / **`gatewayUpstreamLog`** (PII-маска `shared/redactRaw.ts`), admin-роуты **`api/admin/raw/{requests,upstream}/{recent,get}.ts`** + **`api/admin/dashboard/gatewayCounts.ts`**. Ответ **`TuneHttpHeadersResponse`** + CORS. Совместимостный shim **`lib/gateway/handleV1OpRoute.ts`** (через реестр) обслуживает `v1AddUserHandler.ts` и тест-раннер. Codegen (`scripts/`, `*.generated`), `gatewayWorkspaceEvents.ts` и каталог `api/gateway-analytics/` удалены. Детали — **`docs/api.md`**, **`docs/gateway/implementation-plan.md`**.

**Панель (`PanelHomePage.vue`):** главная `/` рендерит панель мониторинга. Вкладки: KPI (24ч через `gatewayCounts`), Входящие (raw-журналы запросов), К GetCourse (raw-журналы upstream), Доступы (Admin: управление грантами и инвайтами). Данные через fetch по `props.apiUrls`. Toolbar фильтра по дате. Страница тестов ограничена только ролью Admin.

**Система доступов к панели:** Heap-таблицы `panelAccess` и `panelInvites`; `lib/access/` (requireInternalAccess, guardInternalApi, invites); API `api/access/` (6 эндпоинтов); страницы `/web/forbidden` и `/web/access/invite`. Доступ: Admin или активный грант.

**Фильтр по дате:** настройка `panel_date_filter` в Heap; эндпоинт `api/admin/analytics/filter-save`; SSR-проп `initialDateFilter` передаётся в `PanelHomePage.vue`.
