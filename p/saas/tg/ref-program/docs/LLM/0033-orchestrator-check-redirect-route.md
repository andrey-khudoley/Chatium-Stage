### Пользователь

Реализовать пункт плана (план приложен): to-do с ID route-r — роут r.tsx (GET /r~:linkId, визит и редирект). План не редактировать. Задачи уже созданы, отметить в работе и выполнить до конца.

### Ассистент

**Сделано:**
- Создан файл p/saas/ref/r.tsx: обработчик GET /:linkId (file-based URL /r~linkSlug). Логика: получение linkId из params, поиск ссылки через linkRepo.findLinkByPublicSlug, при отсутствии — 404; из ссылки извлекаются campaignId, partnerId, pageId, partnerLinkId; загрузка страницы через pageRepo.getPageById, при отсутствии или пустом urlTemplate — 404; вычисление fingerprint (computeFingerprint), создание визита (visitRepo.createVisit), подстановка ref в urlTemplate (substituteRef), редирект ctx.resp.redirect(finalUrl).
- В docs/imports.md добавлена секция для r.tsx с перечислением импортов и зависимостей.

**Итог:** Роут редиректа реализован, документация импортов обновлена. Предложено при необходимости запустить проверки оркестратором.
