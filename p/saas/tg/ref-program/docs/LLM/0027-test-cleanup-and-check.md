# 0027 — Очистка тестовых данных и проверка оркестратором

### Пользователь

Для всех тестов в каталоге p/saas/ref нужно убедиться, что когда что-то создаётся в таблицах для тестов, в конце теста оно удаляется и не создаётся мусора.

---

### Ассистент

Проверены все тесты в p/saas/ref. Внесены изменения только там, где очистки не было.

**Сделано:**

- **logs-repo.ts:** тест создавал запись в таблице логов и не удалял её. Добавлено сохранение идентификатора созданной записи и удаление через Logs.delete в конце обработчика перед возвратом ответа. Добавлен импорт таблицы Logs.
- **member-repo.ts и campaign-repo.ts:** очистка уже была реализована в блоке finally (удаление участников и кампании). Изменений не потребовалось.
- **Тест API кампаний на фронте (TestsPage.vue):** при прогоне «POST /api/campaigns/create (успех)» создавалась кампания и не удалялась. Добавлен серверный эндпоинт POST /api/tests/cleanup-campaign (файл api/tests/cleanup-campaign.ts): принимает campaignId в теле запроса, проверяет, что кампания принадлежит текущему пользователю, удаляет участников и кампанию. На странице тестов после блока «API кампаний» в блоке finally при наличии созданного campaignId вызывается этот эндпоинт, чтобы не оставлять тестовые кампании в БД.
- **Остальные тесты:** ref-generator не пишет в таблицы; settings-repo создаёт и удаляет запись в том же тесте; dashboard-lib, logger-lib, settings-lib и др. не создают тестовых записей в Heap.

**Итог:** везде, где тесты создают данные в таблицах, в конце теста эти данные удаляются. Предложено при необходимости запустить проверки (@check).
