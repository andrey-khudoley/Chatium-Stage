# LLM 0011 — индикаторы сохранения в админке, debounce полей, @check

Диалог: добавление зелёной/красной надписи «Сохранено»/«Ошибка» в админке после ответа сервера; автосохранение инпутов с debounce 2 с без blur; проверка @check.

---

### Пользователь

В компонентах DOM Path: div[1] > div.app-layout … > div.admin-card[1] (Настройки проекта) и div.admin-card[2] (Уровень логирования) необходимо отображать зелёную надпись «Сохранено» или красную надпись «Ошибка» на несколько секунд в правом верхнем углу после того, как пришёл ответ от сервера с результатом запроса на обновление.

### Ассистент

**Сделано:**
- Добавлены ref `projectNameSaveStatus` и `logLevelSaveStatus` для индикации результата сохранения.
- В карточках «Настройки проекта» и «Уровень логирования» — блок с надписью «Сохранено» (зелёный) или «Ошибка» (красный) в правом верхнем углу.
- Функция `showSaveStatus` с таймером 3 с для скрытия надписи.
- Вызов `showSaveStatus` в `saveProjectName` и `setLogLevel` после ответа сервера.
- Очистка таймеров в `onBeforeUnmount`.

---

### Пользователь

Также для числовых, строковых или многострочных инпут полей не надо дожидаться, пока пропадёт фокус для отправки. Если содержимое инпута изменилось и в течение двух секунд не меняется снова — изменения надо отправить на сервер.

### Ассистент

**Сделано:**
- Добавлен `watch` на `projectName` с debounce 2 с.
- Удалены `@blur` и `@keydown.enter` с инпута «Название проекта».
- Переменная `lastSavedProjectName` для сравнения с текущим значением; сохранение только при отличии.
- Константа `INPUT_DEBOUNCE_MS = 2000`.
- Очистка debounce-таймера при размонтировании.
