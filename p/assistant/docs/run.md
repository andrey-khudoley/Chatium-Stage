# Run

## Требования
- Проект работает внутри Chatium, локальный сервер не нужен.
- Зависимости фиксированы платформой.

## После копирования шаблона (новый проект)

Чтобы использовать скопированный шаблон как отдельный реальный проект, выполните следующие шаги.

### 1. Скопировать каталог проекта

Скопируйте весь каталог `p/template_project` в новое место, например `p/my_project`. Имя каталога станет путём проекта (от корня воркспэйса).

### 2. Обязательно: путь проекта

**`config/routes.tsx`**

- Замените значение `PROJECT_ROOT` на путь к новому каталогу (от корня воркспэйса, без ведущего слеша).
- Пример: если проект лежит в `p/my_project`, укажите `export const PROJECT_ROOT = 'p/my_project'`.
- От этого зависят все ссылки, редиректы и формирование URL в приложении.

**`.dir.json`**

- В поле `name` укажите путь к новому проекту (как в `PROJECT_ROOT`).
- Пример: `"name": "[INWORK] p/my_project"`.
- Нужно для корректной работы инструментов и навигации по воркспейсу.

### 3. По желанию: название и дефолты

**`config/project.tsx`**

- Константа `DEFAULT_PROJECT_TITLE` — название проекта по умолчанию (если в настройках ещё ничего не сохранено). Можно заменить на название нового проекта.

**`lib/settings.lib.ts`**

- В объекте `DEFAULTS` поля `project_name` и `project_title` задают значения по умолчанию для настроек. Их можно изменить под новый проект или оставить и позже задать через админку (поле «Название проекта»).

### 4. По желанию: отдельные Heap-таблицы на проект

По умолчанию в шаблоне используются таблицы с ключами `t__template-project__setting__7Fk2Qw` и `t__template-project__log__9Xm3Kp`. Если несколько скопированных проектов работают в одном воркспейсе и не менять эти ключи, они будут использовать одни и те же таблицы (общие настройки и логи).

Чтобы у нового проекта были свои данные:

- В **`tables/settings.table.ts`** замените первый аргумент `Heap.Table(...)` на новый уникальный ключ таблицы (по соглашениям платформы/Heap).
- В **`tables/logs.table.ts`** — аналогично.
- Обновите описание таблиц в **`docs/data.md`**, чтобы документация соответствовала коду.

Если каждый проект деплоится в отдельный воркспейс или общие настройки/логи допустимы, этот шаг можно не выполнять.

### 5. Деплой

- Закоммитьте и запушьте изменения в репозиторий.
- Chatium задеплоит обновления автоматически.
- Откройте приложение по новому пути (например `https://<домен>/p/my_project/`) и при необходимости задайте название проекта в админке.

## Запуск/деплой
- Внести изменения в репозиторий.
- Запушить изменения.
- Chatium автоматически задеплоит обновления.

## Конфигурация
- `config.json` пока не используется.

## Проверки
- Открыть главную страницу и убедиться, что страницы открываются.
- Проверить редирект на логин при входе в профиль/админку без авторизации.
