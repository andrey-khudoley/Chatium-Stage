## Общая архитектура проекта

Проект представляет собой полнофункциональную **вебинарную комнату** на базе Chatium с поддержкой:
- **Прямых эфиров** (live episodes) через Kinescope
- **Автовебинаров** (scheduled autowebinars) с воспроизведением записанного видео
- **Чата в реальном времени** с поддержкой сообщений, реакций, модерации
- **CTA-форм** (lead generation, оплата) с поддержкой платежей
- **Аналитики** просмотров, конверсии, вовлеченности
- **Сценариев автовебинаров** (показ форм, сообщения в чате, реакции по таймлайну)

### Два основных режима работы:

#### 1. **Episodes (Прямые эфиры)**
- Создаются в админке (`AdminCreatePage.vue`)
- Интегрируются с Kinescope для стриминга
- Имеют статусы: `scheduled` → `waiting_room` → `live` → `finished`
- Поддерживают чат в реальном времени
- Формы показываются вручную администратором

#### 2. **Autowebinars (Автовебинары)**
- Создаются с нуля (`AdminAutowebinarCreatePage.vue`)
- Используют загруженное видео (Kinescope/Muuvee)
- Имеют сценарий событий (сообщения чата, формы, реакции)
- Поддерживают расписание запусков (scheduled mode)
- Fake online счетчик для создания эффекта живого эфира

## Система тем и CSS переменные

### Основная структура тем

Проект поддерживает **две темы**: `dark` (по умолчанию) и `light`.  
Переключение темы происходит через `shared/theme.ts`.

#### Основной файл: `styles.tsx`

В этом файле объявлены все CSS переменные для обеих тем. Переменные начинаются с префикса `--wr-` (webinar-room).

## Точечная кастомизация

### 1. Кастомизация через переопределение CSS переменных

Если нужно изменить цвета только для конкретного клиента, создайте новый файл `custom-theme.tsx`:

```tsx
// custom-theme.tsx
import { jsx } from '@app/html-jsx'

export const customThemeStyles = (
  <style>{`
    :root {
      --wr-primary: #0066ff;           /* Новый основной цвет */
      --wr-primary-hover: #0052cc;
      --wr-accent: #0066ff;
    }
    
    .theme-dark {
      --wr-bg: #000000;                /* Более темный фон */
      --wr-chat-bg-image: url('https://custom-bg.jpg');
    }
  `}</style>
)
```

Затем импортируйте в `styles.tsx` или непосредственно в `index.tsx`:
```tsx
import { customThemeStyles } from './custom-theme'

// В headContent:
{headContent}
{customThemeStyles}
```

### 2. Кастомизация форм по ID

#### Пример: Разные цвета кнопок для разных форм

В `components/episode/FormButtons.vue`:
```vue
<template>
  <button
    v-for="form in forms"
    :key="form.id"
    class="form-btn"
    :class="getFormClass(form.id)"
    :style="getFormStyle(form)"
    @click="handleFormClick(form)"
  >
    {{ form.buttonText }}
  </button>
</template>

<script setup>
function getFormClass(formId) {
  // Кастомные классы для конкретных форм
  const customClasses = {
    'form-123': 'form-btn--premium',
    'form-456': 'form-btn--limited',
  }
  return customClasses[formId] || ''
}

function getFormStyle(form) {
  // Базовый стиль из формы
  const baseStyle = form.buttonColor
    ? { background: `linear-gradient(135deg, ${form.buttonColor} 0%, ${darken(form.buttonColor)} 100%)` }
    : {}
  
  // Переопределения для конкретных форм
  const customStyles = {
    'form-123': { boxShadow: '0 0 30px rgba(255, 215, 0, 0.5)' },
  }
  
  return { ...baseStyle, ...(customStyles[form.id] || {}) }
}
</script>

<style scoped>
.form-btn--premium {
  border: 2px solid gold;
  animation: premium-pulse 2s infinite;
}

@keyframes premium-pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.05); }
}
</style>
```

<template>

### 3. Условная кастомизация по query параметрам

Если нужно показывать разные стили в зависимости от того, откуда пришел пользователь:

В `EpisodeLivePage.vue` или `AutowebinarPage.vue`:
```vue
<template>
  <div class="page" :class="pageClass">
    <!-- Контент -->
  </div>
</template>

<script setup>
import { computed } from 'vue'

const queryParams = new URLSearchParams(window.location.search)
const source = queryParams.get('source') // ?source=vip

const pageClass = computed(() => {
  if (source === 'vip') return 'page--vip'
  if (source === 'partner') return 'page--partner'
  return ''
})
</script>

<style scoped>
.page--vip {
  --wr-primary: #ffd700;
  --wr-accent: #ffd700;
}

.page--partner {
  --wr-primary: #00ff00;
}
</style>
```

### 4. Кастомизация на уровне episode/autowebinar

Если нужна кастомизация на уровне конкретного вебинара, можно:

**Вариант A:** Добавить поле `customTheme` в таблицу `episodes.table` или `autowebinars.table`:
```json
{
  "name": "customTheme",
  "kind": "StringKind",
  "title": "Кастомная тема (CSS)"
}
```

Затем в компоненте:
```vue
<template>
  <div class="episode-page">
    <component :is="'style'" v-if="episode.customTheme">
      {{ episode.customTheme }}
    </component>
    <!-- Остальной контент -->
  </div>
</template>
```

**Вариант B:** Использовать inline стили через props:
```vue
<template>
  <div :style="themeStyles">
    <!-- Контент -->
  </div>
</template>

<script setup>
import { computed } from 'vue'

const props = defineProps({
  episode: Object
})

const themeStyles = computed(() => {
  if (!props.episode.customColors) return {}
  
  return {
    '--wr-primary': props.episode.customColors.primary,
    '--wr-accent': props.episode.customColors.accent,
  }
})
</script>
```

## Работа с формами

### Типы форм (submitAction)

Формы имеют три возможных действия после отправки:

1. **thank_you** — просто показать "Спасибо"
2. **redirect** — редирект на внешний URL
3. **payment** — перейти к оплате

### Структура формы (episode_forms.table)

```json
{
  "title": "Название формы",
  "subtitle": "Подзаголовок",
  "buttonText": "Текст кнопки",
  "buttonColor": "#f8005b",
  "fields": [
    {
      "id": "field1",
      "type": "text",
      "label": "Имя",
      "placeholder": "Введите имя",
      "required": true
    }
  ],
  "submitAction": "payment",
  "paymentAmount": 1990,
  "paymentCurrency": "RUB",
  "paymentDescription": "Доступ к вебинару",
  "paymentOldPrice": 2990,
  "bonuses": [
    { "text": "Бонус 1" },
    { "text": "Бонус 2" }
  ]
}
```

### Типы полей формы

- `text` — обычный текст
- `email` — email с валидацией
- `phone` — телефон
- `firstName`, `lastName`, `fullName` — автозаполнение из ctx.user
- `textarea` — многострочный текст
- `select` — выпадающий список
- `radio` — радио-кнопки
- `checkbox` — чекбокс
- `number` — число
- `url` — URL

### Показ форм

#### В прямых эфирах (Episodes):
Формы показываются **вручную** администратором через админку:
```typescript
// api/forms.ts -> apiFormShowRoute
await apiFormShowRoute.run(ctx, {
  formId: 'form-123',
  episodeId: 'episode-456'
})
```

Скрытие формы:
```typescript
await apiFormHideRoute.run(ctx, {
  formId: 'form-123',
  episodeId: 'episode-456'
})
```

#### В автовебинарах (Autowebinars):
Формы показываются **автоматически по сценарию**:
```json
// scenario_events.table
{
  "eventType": "show_form",
  "offsetSeconds": 300,
  "formId": "form-123"
}
```

### Отправка формы

```typescript
// api/forms.ts -> apiFormSubmitRoute
const result = await apiFormSubmitRoute.run(ctx, {
  formId: 'form-123',
  episodeId: 'episode-456', // или autowebinarId
  data: {
    name: 'Иван',
    email: 'ivan@example.com'
  }
})

// result.action может быть:
// - 'thank_you'
// - 'redirect' (+ redirectUrl)
// - 'payment' (+ paymentMethodUrl)
// - 'direct_payment' (+ paymentLink) — если только 1 провайдер
```

### Кастомизация форм

#### 1. Цвет кнопки формы:
```json
{
  "buttonColor": "#00ff00" // зеленая кнопка
}
```

#### 2. Кастомный попап по ID формы (см. секцию "Точечная кастомизация")

#### 3. Автозаполнение полей:
В `EpisodeFormPopup.vue` поля автоматически заполняются из `ctx.user`:
```javascript
if (field.type === 'email' && ctx.user.confirmedEmail) {
  initialValue = ctx.user.confirmedEmail
}
```

## Основные модули

### 1. Episodes (Прямые эфиры)

**Lifecycle:**
```
scheduled → waiting_room → live → finished
```

**Ключевые компоненты:**
- `pages/EpisodeLivePage.vue` — основная страница
- `components/episode/LiveState.vue` — состояние live
- `components/episode/WaitingRoomState.vue` — waiting room
- `components/episode/FinishedState.vue` — finished

**API:**
- `apiEpisodeCreateRoute` — создать эфир
- `apiEpisodeOpenRoomRoute` — открыть waiting room
- `apiEpisodeStartRoute` — запустить live
- `apiEpisodeFinishRoute` — завершить

**Интеграция с Kinescope:**
- Создание event: `createEvent(ctx, params)`
- Запуск стриминга: `enableEvent(ctx, kinescopeId)`
- Завершение: `completeEvent(ctx, kinescopeId)`

### 2. Autowebinars (Автовебинары)

**Режимы:**
- `scheduled` — запуск по расписанию (поддерживается recurrence)

**Lifecycle расписания:**
```
scheduled → waiting_room → live → finished
```

**Ключевые компоненты:**
- `pages/AutowebinarPage.vue` — основная страница
- `components/autowebinar/AwLiveState.vue` — состояние live
- `components/autowebinar/HybridChatScreen.vue` — гибридный чат (сценарий + реальные сообщения)

**Сценарий автовебинара:**
События хранятся в `scenario_events.table` и обрабатываются клиентски:
- `show_form` — показать форму
- `hide_form` — скрыть форму
- `chat_message` — фейковое сообщение в чате
- `reaction` — фейковая реакция
- `sale_banner` — баннер продажи

**Fake Online:**
Рассчитывается на клиенте через `calculateFakeOnline()` из `shared/fake-online.ts`.

**Движок автовебинаров:**
- `api/autowebinar-engine.ts` — обработка запусков по расписанию
- `api/autowebinar-recurrence.ts` — создание будущих расписаний для recurring автовебинаров

### 3. Чат

**Два типа чата:**

#### A. Обычный чат (Episodes)
- Использует `ChatScreen.vue`
- Все сообщения реальные (Feed API)
- Поддержка replies, reactions, модерация

#### B. Гибридный чат (Autowebinars)
- Использует `HybridChatScreen.vue`
- Комбинирует фейковые сообщения из сценария + реальные из Feed
- Синхронизируется по `currentElapsedSeconds`

**Модерация:**
- `api/chat-admin-routes.ts`
- Бан пользователя: `apiChatBanUserRoute`
- Удаление сообщения: `apiChatDeleteMessageRoute`

### 4. Формы и платежи

**Workflow оплаты:**
```
1. Пользователь заполняет форму
2. apiFormSubmitRoute создает submission
3. Проверяется количество доступных провайдеров
4. Если 1 провайдер → direct_payment (сразу переход)
5. Если > 1 → payment (страница выбора провайдера)
6. runAttemptPayment создает платеж
7. Редирект на payment gateway
8. После оплаты → formPaymentSuccessRoute
```

**Ключевые файлы:**
- `api/forms-payment.ts`
- `payment-method.tsx` — выбор способа оплаты
- `payment-success.tsx` — успешная оплата

## API и роуты

### Episodes API

```typescript
// Список эфиров
apiEpisodesListRoute.run(ctx) // GET /api/episodes~list

// Получить эфир
apiEpisodeByIdRoute({ id }).run(ctx) // GET /api/episodes~:id

// Создать
apiEpisodeCreateRoute.run(ctx, { title, scheduledDate, ... })

// Обновить
apiEpisodeUpdateRoute({ id }).run(ctx, { title, ... })

// Открыть waiting room
apiEpisodeOpenRoomRoute({ id }).run(ctx, {})

// Запустить live
apiEpisodeStartRoute({ id }).run(ctx, {})

// Завершить
apiEpisodeFinishRoute({ id }).run(ctx, {})

// Удалить
apiEpisodeDeleteRoute({ id }).run(ctx, {})
```

### Autowebinars API

```typescript
// Список автовебинаров
apiAutowebinarsListRoute.run(ctx)

// Создать
apiAutowebinarCreateRoute.run(ctx, { title, muuveeVideoId, ... })

// Обновить
apiAutowebinarUpdateRoute({ id }).run(ctx, { ... })

// Активировать
apiAutowebinarActivateRoute({ id }).run(ctx, {})

// Архивировать
apiAutowebinarArchiveRoute({ id }).run(ctx, {})

// Получить данные для просмотра
apiAutowebinarViewerDataRoute({ id }).run(ctx)

// Получить сценарий
apiAutowebinarScenarioRoute({ autowebinarId })
  .query({ mode: 'past', elapsedSeconds: '120' })
  .run(ctx)
```

### Forms API

```typescript
// Показать форму (только для episodes)
apiFormShowRoute.run(ctx, { formId, episodeId })

// Скрыть форму
apiFormHideRoute.run(ctx, { formId, episodeId })

// Получить показанные формы
apiFormGetShownRoute({ episodeId }).run(ctx)

// Отправить форму
apiFormSubmitRoute.run(ctx, { formId, episodeId, data })
```

### Chat API

```typescript
// Получить чат эфира
getEpisodeChatRoute({ id }).run(ctx)

// Получить чат автовебинара (schedule)
apiAutowebinarChatRoute({ scheduleId }).run(ctx)

// Отправить сообщение
episodeChatMessagesAddRoute({ feedId }).run(ctx, { text })

// Модерация: забанить
apiChatBanUserRoute({ userId }).run(ctx, { episodeId, duration })

// Удалить сообщение
apiChatDeleteMessageRoute({ messageId }).run(ctx, {})
```
