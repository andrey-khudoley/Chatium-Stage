# CRM Design System — Спецификация v2

> Полностью переработанная дизайн-система для CRM-проекта.  
> Сохранены: цветовая палитра, общий эмоциональный вайб.  
> Создано заново: визуальные решения, компоненты, структура.

---

## 1. Концепция и визуальный вайб

### Светлая тема «Солнечная листва»
- **Ассоциации:** светлый солнечный день, свежесть, воздух, цвет листвы как акцент
- **Ощущение:** лёгкость, чистота, современность, спокойствие
- **Интерфейс:** визуально «дышащий», не перегруженный, много воздуха

### Тёмная тема «Ночной лес»
- **Ассоциации:** ночной лес, глубина, тишина, мягкий контраст
- **Ощущение:** премиальность, концентрация, светлячки как акцент, комфорт для длительной работы
- **Ограничения:** без агрессивных контрастов, предпочтение мягким переходам

## 3. Типографика

- **Шрифт:** Mulish (sans-serif) — основной текст и интерфейс
- **Шрифт акцентный:** Old Standard TT (serif) — заголовки, брендинг
- **Масштаб:** чёткая иерархия H1 > H2 > H3 > body > caption
- **Line-height:** 1.4–1.6 для читаемости

---

## 4. Радиусы и отступы

- `--radius-sm`: 8px — теги, мелкие элементы
- `--radius-md`: 12px — кнопки, инпуты
- `--radius-lg`: 16px — карточки
- `--radius-xl`: 24px — модалки, крупные блоки
- Отступы: 4, 8, 12, 16, 24, 32, 48px

## 5. Компоненты (каталог)

### 5.1 Кнопки
- **Primary** — заливка акцентом, основной призыв
- **Secondary** — нейтральный фон
- **Ghost** — прозрачный, только текст
- **Outline** — обводка
- **Disabled** — неактивное состояние
- **Loading** — со спиннером

### 5.2 Формы
- Поля ввода с иконками (слева/справа)
- Валидация: focus, error (обводка + сообщение)
- Подсказки под полем

### 5.3 Фильтрация и поиск
- Поле поиска с иконкой
- Табы-фильтры (Все, Непрочитанные, Открытые, Мои)
- Теги состояний (В работе, Приоритет, Архив)

### 5.4 Уведомления
- Success — зелёная иконка, кнопка «Открыть»
- Warning — жёлтая иконка, кнопка «Исправить»
- Info — синяя иконка, кнопка «Обновить»

### 5.5 Списки и таблицы
- Элемент списка: тег + заголовок + описание + время
- Таблица: заголовки + строки с бейджами статусов

### 5.6 Состояния
- **Loading** — спиннер + текст
- **Skeleton** — плейсхолдер из полос
- **Error** — текст ошибки + кнопка «Повторить»
- **Empty** — текст + кнопка действия

### 5.7 Модальные окна
- Оверлей + карточка с заголовком
- Поля ввода, кнопки Отмена / Создать

### 5.8 Навигация
- Сайдбар: логотип, пункты меню, пользователь
- **Header (DcPageHeader):**
  - Первая строка — название страницы (`title`).
  - Вторая строка — хлебные крошки через « / » (`breadcrumbs`: массив строк).
  - Название проекта (например, «А/Ley Services») в хэдере не отображается.
  - Класс второй строки: `dc-page-breadcrumbs`; для обратной совместимости также вешается `dc-page-subtitle` (при смене с `subtitle` на `breadcrumbs` стили не ломаются).

## 6. Демо-страницы и сборка страниц из компонентов

Страницы выполняют роль **витрины дизайн-системы**:
- Все компоненты представлены в одном месте
- Используются как эталон при создании новых страниц CRM
- Покрытие: навигация, формы, кнопки, состояния, списки, таблицы, модалки, уведомления, фильтрация, пустые/loading/error-состояния

**Сборка страниц как конструктор:** все страницы (кроме шаблонных, например лендинга выбора темы) собираются из компонентов: **DcAppShell** (оболочка с фоном, сайдбаром, хэдером) + слоты `#sidebar`, `#header` + **DcMain** → **DcContent** → **DcPageSection** (hero, stats, default) и блоки контента (DcHeroCard, DcStatCard, DcCard, DcPageTable и т.д.). Общие стили страницы (только токены темы, база html/body, скроллбар — без стилей компонентов) подключаются через компонент **DcThemeGlobalStyles** (внутри DcAppShell) и дублируются в SSR из **shared/themeStyles.ts** для быстрой отрисовки.

---

## 7. Ограничения

- Старый дизайн не использовать (кроме цветов и вайба)
- Компоненты — логически завершённые, легко копируемые
- Desktop-first
- Светлая и тёмная темы — равноправные

---

## 8. История изменений (релевантные для стилей)

- **Хэдер (DcPageHeader):** Вместо одной строки «Проект / Страница» и опционального `subtitle` сделаны две строки: (1) только название страницы, (2) хлебные крошки через « / ». Проп `subtitle` заменён на `breadcrumbs` (массив строк). Вторая строка раньше имела класс `dc-page-subtitle`; при переименовании в `dc-page-breadcrumbs` селекторы по `.dc-page-subtitle` переставали срабатывать. Для обратной совместимости на элемент второй строки вешаются оба класса (`dc-page-breadcrumbs` и `dc-page-subtitle`), в стилях заданы оба селектора.

- **Фон и оболочка:** Компонент **DcPageBackground** (`components/DcPageBackground.vue`) — базовый градиент/фон, фоновый рисунок и градиентный оверлей по теме (dark/light). Порядок слоёв: **DcPageBackground** (z-index 0) → **DcGridLayer** → **DcGlowDots** (три светящиеся точки + свечение в правом верхнем и левом нижнем углах) → контент.

- **Сетка и точки:** **DcGridLayer** — лёгкая линейная сетка 48×48px. **DcGlowDots** — три маленьких светящихся точки (орба) и угловые свечения (правый верх, левый низ). См. п. 5.8 и раздел 8.

- **Хэдер:** Кнопки и бейдж в слоте `#actions` стилизуются внутри **DcPageHeader** (классы `.dc-action-btn`, `.dc-action-btn-icon`, `.dc-action-btn-primary`, `.dc-badge`).

- **LIVE:** Компонент **DcLive** (`components/DcLive.vue`) — бейдж «LIVE» с пульсирующей точкой; слот по умолчанию для стрима событий. Режим `badge-only` для использования в header-extra карточки.

- **Layout:** Папка **layout/** — **DcAppShell** (оболочка страницы: фон, grid, DcGlowDots, overlay, CSS-переменные темы, ready/sidebarCollapsed), **DcMain**, **DcContent**, **DcPageSection** (variant: hero | stats), **DcMainGrid**, **DcSidebarOverlay**.

- **Демо-страницы:** Страницы максимально вычищены: только данные и композиция. **DcHeaderActions** — кнопки хэдера (поиск, уведомления с бейджем, CTA). **DcDemoDashboard** — весь контент дашборда (hero, stats, быстрые сценарии, журнал с LIVE, график, таблица); принимает данные пропсами, табы графика — внутреннее состояние.
