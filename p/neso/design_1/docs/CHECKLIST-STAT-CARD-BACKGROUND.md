# Чек-лист: почему фон .dc-stat-card остаётся прозрачным

## Проверенные варианты

### 1. Страница рендерится не из того компонента
**Проверка:** В design_1 для URL `/p/neso/design_1/web/dark` используется только `web/dark/index.tsx` → `DesignDemoDarkPage.vue` → `DcDemoDashboard` → `DcStatCard`. Отдельной страницы с inline-разметкой (как DesignDemoPage.vue в design_2) в design_1 нет.  
**Вердикт:** Страница рендерится именно из компонента DcStatCard.

### 2. Другой CSS перекрывает стили компонента
**Проверка:** В проекте p/neso/design_1 нет правил по селектору `.dc-stat-card` или `background` для карточек ни в themeStyles, ни в других Vue-файлах (кроме самого DcStatCard.vue). Tailwind preflight не задаёт background для div. :deep() в layout/компонентах не затрагивает .dc-stat-card.  
**Вердикт:** В коде design_1 нет перекрывающего правила.

### 3. Стили из таблицы стилей приходят позже и выигрывают по каскаду
**Проверка:** В коде добавлен `!important` к background в DcStatCard, DcCard, DcHeroCard. Если бы на страницу попадал актуальный CSS, он бы перебивал любые другие правила.  
**Вердикт:** Если на проде всё ещё прозрачно, то до браузера доходит старая версия CSS (см. п. 5).

### 4. Переменные (--glass-soft и т.п.) переопределены родителем
**Проверка:** В тёмной теме DcStatCard больше не использует var(--glass-strong/--glass-soft) для фона — задан явный градиент. DcAppShell задаёт только --accent, --text, --border и не трогает background карточек.  
**Вердикт:** Переопределение переменных родителем не причина.

### 5. На сервер/браузер отдаётся старый бандл (кэш сборки или CDN)
**Проверка:** В DevTools видно значение `linear-gradient(165deg, rgb(12 16 16 / 94%), var(--glass-soft))` — это старая версия правил из DcStatCard (с 94% и переменной). В текущем коде такого уже нет.  
**Вердикт:** На проде отдаётся закэшированная/старая сборка CSS. Нужна пересборка без кэша и повторный деплой.

### 6. Стили Vue (scoped) не попадают в бандл
**Проверка:** Селектор в DevTools — `.dc-stat-card[data-v-xxx]`, т.е. стили компонента до браузера доходят, но с устаревшим содержимым.  
**Вердикт:** Стили в бандл попадают; проблема в версии бандла (п. 5).

---

## Что сделано в коде

1. **Инлайн-стиль фона** в DcStatCard, DcCard и DcHeroCard для тёмной темы:  
   `:style="cardBgStyle"` с `background: linear-gradient(165deg, #0c1416, #0e1618)`.  
   Инлайн даёт приоритет над любыми правилами из таблиц стилей (кроме !important в CSS) и не зависит от того, какой CSS-файл загрузился.

2. В `<style scoped>` по-прежнему задан непрозрачный фон с `!important` на случай, если инлайн по какой-то причине не применится.

После деплоя актуальной сборки фон должен быть сплошным. Если после деплоя инлайн-стиль виден в DevTools, но фон всё равно прозрачный — значит, что-то перебивает уже инлайн (крайне редко); тогда следующий шаг — смотреть в DevTools, какое правило даёт итоговый background.
