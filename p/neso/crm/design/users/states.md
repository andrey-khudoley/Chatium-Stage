# Users · States

| State | UX Intent | Visual Pattern |
|---|---|---|
| empty | Explain zero-data state and next action | Icon + guidance copy + primary CTA |
| loading | Preserve structure while data hydrates | Skeleton rows/cards and progress indicator |
| error | Provide recoverability and diagnostics | Error surface + retry action + context id |
| filled | Support dense daily operations | Cards/tables/charts with contextual controls |

## Responsive Notes
- Desktop: full split-layout with secondary panels.
- Mobile: card mode enabled, low-priority columns collapsed.
