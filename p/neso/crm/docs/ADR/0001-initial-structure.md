# ADR-0001: Базовая структура проекта

Дата: 2026-01-31

## Контекст
Проект разворачивается в Chatium: серверная инфраструктура фиксирована, зависимости менять нельзя, деплой происходит автоматически при пуше. Нужна понятная структура каталогов и документации, чтобы любой разработчик быстро понял устройство проекта.

## Решение
Использовать стандартную структуру Chatium:
- корневой `index.tsx` как единственная главная точка входа;
- папки `web/`, `pages/`, `components/`, `api/`, `tables/`, `shared/`, `lib/`;
- документация в `docs/` с разделами `ADR/` и `LLM/`.

## Последствия
Плюсы:
- структура соответствует стандартам и удобна для навигации;
- проще избегать циклических импортов;
- документация разделена по назначению.

Минусы:
- больше файлов и папок даже на минимальном этапе;
- требует дисциплины по обновлению документации.
