# Гайдлайн по дизайну CRM

Современный, минималистичный интерфейс для SaaS с множеством разделов и функций. **Тёмная тема**: тёмный фон, светло-зелёные и золотые акценты (брендбук Tarot Vision). Документ задаёт структуру блоков, расположение элементов и визуальные правила.

---

## 1. Принципы

- **Современность** — плоские акценты, мягкие тени, чёткая типографика, единая шкала отступов и скруглений.
- **Минимализм** — только необходимые элементы, без декора; пустое пространство как часть композиции. Тактильная чистота: ограниченная палитра, «воздух» между блоками, без визуального шума и лишних деталей.
- **Удобство для SaaS** — быстрый доступ к разделам, понятная иерархия, контент важнее chrome.
- **Консистентность** — одни и те же паттерны для списков, карточек, форм и навигации во всех разделах.

---

## 2. Глобальная структура (shell)

Один и тот же каркас на всех страницах.

```
┌─────────────────────────────────────────────────────────────┐
│  HEADER (фиксированная полоса)                                │
│  [Лого] [Название] ······ [Навигация] [Профиль] [Войти/Выйти] │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  MAIN (прокручиваемая область)                                │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │  Контент страницы (max-width, центрирован)              │  │
│  │  — заголовок страницы                                    │  │
│  │  — блоки/карточки/таблицы                               │  │
│  └─────────────────────────────────────────────────────────┘  │
│                                                               │
├─────────────────────────────────────────────────────────────┤
│  FOOTER (одна строка: копирайт, ссылка на платформу)          │
└─────────────────────────────────────────────────────────────┘
```

- **Header** — всегда виден, одна строка. Слева: логотип (если есть) + название проекта. Справа: навигация (иконки + подписи на десктопе), затем кнопка «Войти» или «Профиль»/«Выйти».
- **Main** — единственная прокручиваемая зона; внутренний контент ограничен по ширине (например, 72rem) и центрирован.
- **Footer** — минимальный: копирайт и одна ссылка. Не отвлекает от контента.

Без боковой панели (sidebar): навигация по разделам в header. Так интерфейс остаётся простым и подходит для небольшого числа разделов (Главная, Настройки, Тесты, Профиль).

**Опционально** для страниц с большим числом подразделов: комбинация **горизонтальных закладок (folder tabs)** и **вертикального бокового меню**. Закладки задают основные разделы (как вкладки в папке), боковое меню — пункты внутри выбранного раздела. См. разделы 3.6 и 8 (меню-закладки).

## 3. Структура по типам страниц

### 3.1. Главная (лендинг)

- **Hero-зона** — один блок по центру (или две колонки на десктопе: текст слева, визуал справа).
- Содержимое: один заголовок (H1), короткий подзаголовок, 1–2 кнопки (например, «Войти», «Тесты»).
- Визуал: абстрактная форма, иллюстрация или фото, без перегруженной графики.
- Ниже hero при необходимости: один ряд карточек-превью разделов (опционально).

### 3.2. Страницы с контентом (Настройки, Тесты, Профиль)

- **Заголовок страницы** — один H1 вверху (например, «Настройки», «Тесты», «Профиль»).
- **Блоки контента** — вертикальный стек карточек. Каждая карточка:
  - заголовок блока (H2) с опциональной иконкой слева;
  - тело: форма, список, таблица, лог и т.д.
- Между карточками — фиксированный вертикальный отступ (например, 24px).
- Действия (кнопки «Сохранить», «Запустить тесты») — внутри карточки, внизу блока или справа от заголовка, в зависимости от контекста.

### 3.3. Админка / Настройки

- Порядок блоков сверху вниз: краткий дашборд (счётчики) → настройки проекта (поля) → уровень логов → лог-вывод.
- Каждый блок — одна карточка с чётким H2.
- Формы: лейбл над полем, один столбец; кнопки выравниваются по левому краю или в одну линию с полем.

### 3.4. Тесты

- Заголовок страницы, затем карточки по группам тестов (например, «Эндпоинты», «Настройки», «Логи»).
- Внутри карточки: заголовок группы, кнопка «Запустить», список тестов с статусом (иконка + текст). Логи — в отдельной карточке с фиксированной высотой и прокруткой.

### 3.5. Вход / Авторизация

- Один центр экрана: одна карточка с заголовком «Вход», кратким текстом и одной основной кнопкой («Войти»). Без лишних полей на самой странице, если авторизация вынесена на платформу.

### 3.6. Страницы с закладками и боковым меню

Для разделов с множеством подразделов (например, Настройки по категориям, отчёты, справочники): комбинация **горизонтальных закладок** и **вертикального бокового меню**.

- **Расположение**: под заголовком страницы (или под header) — горизонтальный ряд закладок (folder tabs). Слева от основного контента — вертикальное боковое меню (узкая колонка с пунктами).
- **Иерархия**: закладки переключают **основные разделы** (например: «Динамика», «Визиты», «Назначения», «Анализы»). Боковое меню показывает **пункты внутри выбранной закладки** (подразделы, фильтры, типы отчётов).
- **Контент**: основная область справа от бокового меню — карточки, таблицы, формы в зависимости от выбранного пункта. Заголовок контента (H2) при необходимости уточняет контекст.
- **Адаптив**: на узких экранах закладки могут переноситься или сворачиваться в выпадающий список; боковое меню — в бургер/выдвижную панель. Сохранять понятную иерархию: сначала выбор раздела (закладка), затем подпункт (боковое меню).

## 4. Типографика

- **Шкала размеров** (база 16px): 12, 14, 16, 18, 20, 24, 32 px (и при необходимости 40 для крупных заголовков).
- **Основной текст**: один sans-serif (например, Mulish или Inter) — 14–16px для тела, 12–14px для подписей и меток.
- **Заголовки**: тот же шрифт с увеличенным весом (600–700) или отдельный шрифт для H1 (например, акцентный serif по брендбуку). H1 — 24–32px, H2 — 18–20px, H3 — 16–18px.
- **Межстрочный интервал**: тело 1.5, заголовки 1.2–1.3.
- **Цвет текста**: основной — белый/светло-серый на тёмном фоне; вторичный — приглушённый серый для подписей и меток; третичный — для подсказок и неактивного. Акценты (ссылки, активные состояния) — светло-зелёный и золотой.

---

## 5. Цвета (тёмная тема)

Сайт в тёмной теме: контраст между тёмным фоном и светло-зелёными и золотыми акцентами (брендбук Tarot Vision).

- **Фон**: тёмный основной (глубокий тёмно-зелёный или тёмно-синий/charcoal, например `#131321`, `#0f1a17`); вторичный — чуть светлее для карточек и блоков; третичный — для hover и выделенных областей.
- **Акценты**: светло-зелёный/бирюзовый (primary, например `#46F0D2`) — основные кнопки, активные состояния навигации, ключевые данные; золотой (secondary, например `#FBE2B4`, `#FDC760`) — второстепенные акценты, декоративные элементы, премиальный тон по брендбуку.
- **Текст**: светлый на тёмном — основной белый или почти белый; вторичный — светло-серый; для ссылок и hover — светло-зелёный или золотой. Высокая контрастность для читаемости.
- **Границы**: приглушённые, тёмные (тёмно-серый или тёмно-зелёный), чтобы не резать глаз на тёмном фоне.
- **Семантика**: успех — светло-зелёный, ошибка — красный/розовый, предупреждение — жёлтый/золотой, информационный — бирюзовый. В рамках тёмной палитры.

### 5.1. Градиенты и плавные переходы (тёмный → акценты)

Плавные переходы от тёмного фона к более светлым оттенкам или золоту для глубины.

- **Фоновые градиенты**  
  В секциях (hero, блоки под заголовками) — линейные градиенты: от глубокого тёмно-зелёного к чуть светлому тёмно-зелёному или к золотистому оттенку. Примеры: `linear-gradient(180deg, #0f1a17 0%, #1a2a24 100%)` или с золотым: `linear-gradient(160deg, #131e1b 0%, #1e2a22 50%, rgba(251,226,180,0.08) 100%)`. Один градиент на секцию.

- **Световые акценты и размытые формы**  
  На тёмном фоне допускаются мягкие, размытые светлые пятна (светло-зелёный, золотой, белый с низкой непрозрачностью) — круги, овалы с большим `blur`. Они создают глубину и «свечение» за контентом. Не перегружать: 1–2 таких элемента на секцию.

- **Карточки на тёмном фоне**  
  Блоки контента — фон чуть светлее основного (вторичный фон) или полупрозрачный светлый при glassmorphism. Мягкие тени (`0 8px 32px rgba(0,0,0,0.3)`), скруглённые углы (12–16px). Карточки визуально «парят» над тёмным фоном. Границы — приглушённые или отсутствуют.

### 5.2. Размытие фона под блоками (glassmorphism / frosted glass)

Для карточек и панелей поверх фона или градиента — эффект «матового стекла»: фон под блоком размыт, блок полупрозрачный.

- **Размытие фона**  
  `backdrop-filter: blur(12px)` (или 16–24px). При необходимости — `-webkit-backdrop-filter` и запасной непрозрачный фон для старых браузеров.

- **Полупрозрачность блока в тёмной теме**  
  Фон блока — полупрозрачный тёмный, например `rgba(19, 30, 27, 0.85)` или `rgba(30, 42, 38, 0.9)`, чтобы тёмный фон слегка просвечивал. Либо полупрозрачный светлый `rgba(255,255,255,0.06)` для тонкого выделения. Не опускать читаемость: текст остаётся светлым (белый/светло-серый).

- **Скругления и тени**  
  Скругления 12–20px. Тень для тёмной темы: `0 8px 32px rgba(0,0,0,0.4)` — отделяет блок от фона.

- **Контрастность текста**  
  Текст на полупрозрачном блоке — белый или светло-серый; акценты — светло-зелёный/золотой. Достаточный контраст для доступности.

Использовать выборочно: hero-карточки, блоки поверх градиентов, модальные панели.

### 5.3. Соотношение тёмного и акцентов

Сохранять сильный контраст между тёмным фоном и акцентами — основа визуальной иерархии.

- **Палитра**: тёмный фон (глубокий зелёный/charcoal) — основа; светло-зелёный/бирюзовый и золотой — акценты для кнопок, активных состояний, ключевых данных. Избегать грязных промежуточных оттенков; акценты должны «читаться» на тёмном.
- **Иконки**: на тёмном фоне — белые для неактивных, светло-зелёный или золотой для активных. Единый стиль (например, line или solid).
- **Карточки**: фон карточек чуть светлее основного фона, но остаётся в тёмной гамме; границы приглушённые. Так сохраняется многослойность без перехода в светлую тему.

## 6. Отступы и сетка

- **Базовая сетка**: 4px. Все отступы и размеры — кратные 4 (4, 8, 12, 16, 24, 32, 40, 48).
- **Внутри карточки**: padding 24px (16 на мобиле).
- **Между карточками**: 24px по вертикали.
- **Контент по ширине**: max-width 72rem (1152px), центрирование, горизонтальный padding 24px (16 на мобиле).

---

## 7. Скругления и тени

- **Скругления**: единая шкала. Рекомендация: 8px для кнопок и полей ввода, 12px для карточек, 16px для модальных окон. Не смешивать случайные значения (6, 10, 14).
- **Плитки (карточки)**: сильно скруглённые углы (12–20px) для мягкого, дружелюбного вида; один стиль скругления для всех плиток в одном контексте.
- **Тени** (в тёмной теме — затемняющие, для отделения слоёв):
  - Карточки: мягкая тень, например `0 4px 24px rgba(0,0,0,0.2)` или `0 8px 32px rgba(0,0,0,0.25)`.
  - Кнопки при hover: лёгкое усиление, например `0 4px 12px rgba(70,240,210,0.25)` (светло-зелёный).
  - Модальные окна: чуть сильнее карточек.
- Избегать множественных сложных теней; предпочитать одну понятную тень на элемент (или пару: внешняя + внутренняя по правилам ниже).

### 7.1. Объёмные элементы (Soft UI / тактильный объём)

Элементы интерфейса выглядят объёмными за счёт мягких теней и подсветок: кажется, что кнопки и карточки слегка приподняты над фоном или, наоборот, «вдавлены» в него. Стиль минималистичный, без резких границ и перегруженной графики.

- **Приподнятые элементы (raised)**  
  Кнопки, карточки, чипы отбрасывают **мягкую размытую внешнюю тень** вниз и в сторону (например, `0 4px 12px rgba(0,0,0,0.15)` в тёмной теме или `0 2px 8px rgba(0,0,0,0.08)` на светлом фоне). Тень не жёсткая: большой blur, умеренная прозрачность — элемент визуально «парит» над поверхностью. Опционально: очень лёгкая **внутренняя подсветка** по верхнему и левому краю (`inset 1px 1px 0 rgba(255,255,255,0.06)` в тёмной теме), чтобы подчеркнуть выпуклость; не перегружать.

- **Вдавленные элементы (recessed / inset)**  
  Поля ввода, треки слайдеров, области под переключатели могут выглядеть «утопленными»: **внутренняя тень** по краям, создающая ощущение углубления. Пример: `inset 0 2px 4px rgba(0,0,0,0.2)` в тёмной теме. Фон элемента чуть темнее окружения или с лёгкой границей — так форма читается как объёмная «выемка».

- **Состояние «нажато» (pressed / active)**  
  При нажатии кнопка может переходить в **debossed**-состояние: внешняя тень уменьшается или убирается, добавляется **внутренняя тень** (`inset 0 2px 6px rgba(0,0,0,0.25)`), элемент визуально «вдавливается» в фон. Переход короткий (100–150ms). Так пользователь получает тактильную обратную связь.

- **Многослойность**  
  Внутри карточки допускается вложенный слой (например, блок с полем ввода или список пунктов): у вложенного блока — свой лёгкий объём (небольшая внешняя тень или тонкая внутренняя тень), скругления согласованы (например, 8px внутри карточки с radius 12px). Не более двух уровней глубины в одном блоке.

- **Единообразие**  
  Все интерактивные элементы одного типа (все кнопки, все карточки, все поля) строятся по одним правилам: либо приподнятые, либо вдавленные. Смешивать оба приёма в одном компоненте только осознанно (например, кнопка приподнята, поле ввода вдавлено).

### 7.2. Объём и многослойность (карточки и контент)

- **Парящие плитки**: карточки отбрасывают мягкие, достаточно длинные тени (`0 8px 24px rgba(0,0,0,0.2)` или `0 12px 40px rgba(0,0,0,0.15)` в тёмной теме), читаются как отдельный слой над фоном.
- **Объёмные картинки поверх плиток**: изображения внутри карточек — в скруглённой рамке с собственной лёгкой тенью; радиус 8–12px внутри плитки 12–16px.
- **Иконки**: допускается лёгкий рельеф или мягкая тень у иконок внутри плиток; не перегружать.
- Сохранять минимализм: один-два уровня глубины (фон → плитка → опционально медиа в рамке).

### 7.3. Игра света и тени

Единая модель освещения делает интерфейс цельным и тактильным: элементы выглядят объёмными, а не плоскими. Свет и тень согласованы по направлению и характеру.

- **Единое направление света**  
  Условный источник света — **сверху и слева** (как у окна или верхнего освещения). Все тени от приподнятых элементов смещены **вниз и вправо**; подсветки (highlights) у вдавленных элементов — с верхнего и левого края, тень внутри «выемки» — с нижнего и правого. Одна и та же логика на всей странице: не смешивать тени «от разных источников».

- **Мягкий, размытый свет**  
  Тени **без жёстких границ**: большой blur (8–24px и больше для крупных карточек), умеренная прозрачность (opacity 0.15–0.3 в тёмной теме). Тень наиболее плотная прямо под элементом и плавно расходится, создавая ощущение «парения». Жёсткие тени (малый blur, 100% opacity) не использовать — они выглядят плоскими и грубыми.

- **Приподнятые панели и карточки**  
  Внешняя тень смещена вниз и вправо, например `4px 8px 24px rgba(0,0,0,0.2)` или `6px 12px 32px rgba(0,0,0,0.18)`. Опционально очень лёгкая внутренняя подсветка по верхнему и левому краю (`inset 1px 1px 0 rgba(255,255,255,0.04)` в тёмной теме) — усиливает ощущение выпуклости и «освещённости» сверху-слева.

- **Вдавленные элементы (поля, треки)**  
  Внутренняя тень создаёт «углубление»: тень с верхнего и левого края (как бы от того же источника света), подсветка по нижнему и правому — тоньше. Пример: `inset 2px 2px 4px rgba(0,0,0,0.2), inset -1px -1px 0 rgba(255,255,255,0.03)` в тёмной теме. Элемент читается как выемка в поверхности.

- **Мелкие объёмные элементы (радио, чипы, кнопки-кружки)**  
  Мягкая тень по контуру (равномерный blur вокруг) или лёгкая тень снизу-справа — элемент кажется выпуклым, «подушкообразным». Не перегружать: один небольшой blur (4–8px) и низкая прозрачность.

- **Контраст и читаемость**  
  Игра света и тени не должна снижать контраст текста и иконок. На тёмном фоне тени затемняют; подсветки — очень слабые. Акцентные кнопки (primary) остаются читаемыми за счёт насыщенного фона и контрастного текста; тень лишь подчёркивает объём.

- **Фон (опционально)**  
  Очень тонкая текстура или лёгкий шум на фоне страницы может усилить ощущение «реального» падения тени и глубины. Не обязательно; при использовании — ненавязчиво, без отвлечения от контента.

---

## 8. Компоненты

Все интерактивные и контентные блоки — **объёмные** по правилам раздела 7.1: кнопки и карточки приподняты (мягкая внешняя тень), поля ввода при необходимости — с лёгким inset-эффектом.

### Кнопки

- **Primary**: заливка светло-зелёным акцентом (`#46F0D2` и аналоги), текст тёмный для контраста. Высота 40px, padding по горизонтали 20px, border-radius 8px. Объём: мягкая внешняя тень в состоянии default, усиление тени при hover, при active — переход в «вдавленное» состояние (inset-тень, см. 7.1).
- **Secondary**: фон приглушённый, текст и граница — золотой или светло-зелёный. Тот же размер и radius, тот же объёмный стиль.
- **Ghost/Text**: без фона, цвет текста светлый; при hover — фон слегка светлее (или акцентный оттенок), при необходимости лёгкая тень.
- Все кнопки: один размер шрифта (14–16px), font-weight 600 для primary/secondary.

### Карточки

- Фон чуть светлее основного тёмного (вторичный фон); граница приглушённая или только тень. **Объём**: приподнятые — мягкая размытая тень под карточкой; опционально тонкая внутренняя подсветка по верхнему/левому краю. Верхняя или левая граница может быть акцентной полосой (светло-зелёный/золотой).
- Внутри: заголовок (H2) сверху, текст светлый. Отступ от края карточки 24px. Вложенные блоки (списки, формы) — один уровень глубины, скругления 8px.

### Поля ввода

- Высота 40px, фон тёмный, border приглушённый, radius 8px. Текст светлый. **Объём**: допускается лёгкий inset-эффект (внутренняя тень), чтобы поле читалось как «утопленное»; либо приподнятая рамка с мягкой тенью — единообразно с остальными полями на странице. При focus — обводка светло-зелёным или золотым (2px), без сдвига контента.
- Лейбл над полем, 12–14px, цвет вторичного (светло-серый) текста.

### Навигация в header

- Ссылки: иконка + текст (текст скрывать на мобиле при нехватке места). Цвет по умолчанию — вторичный текст, при hover и активном — акцент.
- Кнопка «Войти»: как primary. «Выйти»: как secondary или outline.

### Меню-закладки (folder tabs)

Горизонтальное меню в виде **закладок в папке**: переключение между основными разделами страницы. Визуально и тактильно напоминает листы с вкладками — удобно сочетается с вертикальным боковым меню (закладки = разделы, боковое меню = подпункты).

- **Внешний вид**  
  Каждый пункт — вытянутый прямоугольник с **сильно скруглёнными углами** (8–12px, при необходимости pill-форма). Фон — чуть светлее основного (вторичный фон) или приглушённый; **активная закладка** — темнее (фон вторичный/третичный или акцентный оттенок), визуально сливается с областью контента под ней (как «выбранный лист»). Между закладками — небольшой зазор (4–8px) или лёгкое визуальное разделение.

- **Объём**  
  Закладки слегка **приподняты** над контентом: мягкая внешняя тень (`0 2px 8px rgba(0,0,0,0.15)` в тёмной теме). Активная закладка может не отбрасывать тень вниз или иметь меньшую тень, чтобы читаться как «прилегающая» к контенту. Соответствует правилам объёмных элементов (раздел 7.1).

- **Содержимое пункта**  
  Иконка + текст в одной строке. Иконка слева (16–20px), текст 14px, font-weight 500 для активного, 400 для неактивного. Цвет: неактивные — вторичный текст; активная закладка — основной текст или акцент (светло-зелёный/золотой).

- **Второй уровень закладок**  
  Под основным рядом закладок допускается **второй ряд** — более мелкие табы для фильтров или подкатегорий (например, «Офисные визиты», «Назначения», «Анализы»). Стиль тот же: скруглённые, приподнятые, активный пункт выделен фоном. Размер и отступы чуть меньше, чтобы не конкурировать с основными закладками.

- **Сочетание с вертикальным боковым меню**  
  Слева от контента — вертикальная панель бокового меню (см. ниже). Горизонтальные закладки задают контекст (какой раздел открыт); боковое меню — что именно показано внутри раздела. Визуально: закладки и боковое меню в одной стилистике.

### Боковое меню (drawer / панель)

Вертикальное меню слева от контента или выезжающая панель (drawer) на мобиле. **Лёгкий, полупрозрачный** стиль: панель не перегружает экран, создаёт ощущение слоя поверх контента. Пункты могут **раскрываться как спойлеры** (аккордеон) — иерархическая навигация без перехода на другую страницу.

- **Полупрозрачность и размытие (glassmorphism)**  
  Фон панели — полупрозрачный: в тёмной теме `background: rgba(26, 30, 40, 0.92)` или светлее; опционально `backdrop-filter: blur(12px)` и `-webkit-backdrop-filter: blur(12px)`, чтобы контент за панелью мягко размывался. Панель визуально «лёгкая», не сплошная стена. Запасной вариант для старых браузеров — непрозрачный фон вторичного цвета. Сторона панели, обращённая к контенту, может иметь лёгкую тень (`box-shadow: 4px 0 24px rgba(0,0,0,0.15)`), чтобы отделить слой.

- **Ширина и скругления**  
  Ширина 12–16rem (или около 2/3 экрана на мобиле в режиме drawer). У панели, если она выезжает поверх контента, — скруглённые правые углы (12–16px) для мягкого вида.

- **Шапка панели**  
  Вверху: логотип или название приложения слева; справа — иконки (уведомления, профиль и т.п.). Высота шапки согласована с header (например, 56–64px). Цвет текста и иконок — основной или акцентный.

- **Пункты меню и раскрытие (спойлеры)**  
  Основные пункты — списком: иконка + текст в одну строку, отступы по вертикали 8–12px, padding по горизонтали 16px. Пункт с подпунктами — **раскрывающийся (спойлер/аккордеон)**: по клику под основным пунктом появляются вложенные пункты с увеличенным отступом слева (например, padding-left 2rem). Иконка-индикатор (стрелка вниз/вправо или chevron) поворачивается или меняется при раскрытии. Одновременно может быть раскрыто несколько групп или только одна — по контексту продукта.

- **Активный пункт и бейджи**  
  Выбранный пункт выделяется: фон (вторичный или `rgba(70, 240, 210, 0.12)` в тёмной теме), скругление 8px, опционально акцентная полоса слева (2–4px, цвет акцента). Подпункты — шрифт чуть мельче или вес легче. Справа у пункта допускается **бейдж** (число, метка) в виде небольшого кружка или плашки контрастного цвета для счётчиков или уведомлений.

- **Типографика и иконки**  
  Основные пункты: 14–16px, font-weight 500 для активного, 400 для остальных. Подпункты: 13–14px. Иконки 18–20px, цвет по умолчанию вторичный текст, у активного — акцент. Всё в одной стилистике с закладками и header.

- **Drawer на мобиле**  
  На узких экранах боковое меню может открываться поверх контента по кнопке (бургер). Затемнение оставшейся области (overlay) с лёгкой прозрачностью; панель выезжает слева с короткой анимацией (200–300ms). Закрытие — по клику на overlay или по кнопке закрытия в шапке панели.

## 9. Иконки

- Один набор на весь продукт (например, Font Awesome или Lucide). Один вес (regular или solid), размеры: 16px в навигации, 20px в заголовках карточек, 24px в крупных кнопках.
- Цвет: наследуют от текста или акцента; не смешивать с другим набором иконок.

---

## 10. Состояния и анимация

- **Hover**: смена фона/цвета и при необходимости лёгкое усиление внешней тени (элемент ещё чуть «приподнимается»). Transition 150–200ms ease.
- **Focus**: видимая обводка (outline или box-shadow) для доступности, без изменения размера элемента.
- **Active/нажатие**: переход в объёмное состояние «вдавлено» (см. 7.1): уменьшение или снятие внешней тени, добавление inset-тени; опционально лёгкое уменьшение scale (0.98). Длительность 100–150ms, единообразно для всех кнопок.
- Глобальные переходы (появление страницы, модалки): короткий fade или лёгкий сдвиг по Y (4–8px), длительность 200–300ms.

---

## 11. Адаптивность

- **Breakpoints**: ориентир 640px (мобильный/компактный), 768px, 1024px. Header переходит в компактный вид: меньше подписей, возможно бургер-меню при большом числе пунктов.
- Контент: на малых экранах один столбец, padding 16px, карточки на всю ширину с сохранением внутренних отступов.
- Таблицы и длинные списки: горизонтальный скролл или карточный вид на мобиле.

## 12. Чек-лист при верстке

- [ ] Все отступы кратны 4px.
- [ ] Одна шкала скруглений (8/12/16).
- [ ] Одна шкала размеров шрифта.
- [ ] Объёмные элементы: кнопки/карточки — приподняты (мягкая тень), при нажатии — вдавленное состояние; поля при необходимости — inset; единый стиль по разделу 7.1.
- [ ] Кнопки и ссылки имеют понятный hover и focus.
- [ ] Карточки и блоки не прижаты к краям; есть max-width и центрирование.
- [ ] Заголовок страницы один (H1), структура заголовков без пропусков уровней (H1 → H2 → H3).
- [ ] Цвета кнопок и акцентов из палитры (зелёный/золотой + нейтрали).
