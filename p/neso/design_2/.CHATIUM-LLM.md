# Проект CRM Design System

## Что это
Дизайн-система для CRM-проекта A/Ley Services. Переработана с нуля: сохранены цветовая палитра и вайб, созданы новые компоненты и визуальные решения.

## Концепция дизайна
**Спецификация:** `DESIGN_SPEC.md`

### 🌙 Тёмная тема «Ночной лес»
- Фон: #05080a, поверхность: #11191b
- Акцент: #afc45f (светлячки)
- Глубина, мягкий контраст, фокус на задаче

### ☀️ Светлая тема «Солнечная листва»
- Фон: #f8f6eb, поверхность: #ffffff
- Акцент: #4f6f2f (листва)
- Воздух, чистота, лёгкая иерархия

## Структура файлов
```
/p/neso/design_1/
├── index.tsx                    # Главная: 4 карточки (библиотека Dark/Light, пример Dark/Light)
├── pages/
│   ├── DesignComponentsDarkPage.vue   # Библиотека компонентов · тёмная тема
│   ├── DesignComponentsLightPage.vue  # Библиотека компонентов · светлая тема
│   ├── DesignDemoDarkPage.vue         # Пример страницы (dashboard) · тёмная тема
│   └── DesignDemoLightPage.vue        # Пример страницы (dashboard) · светлая тема
├── web/
│   ├── dark/index.tsx              # Пример страницы · Dark
│   ├── dark/components/index.tsx   # Библиотека компонентов · Dark
│   ├── light/index.tsx             # Пример страницы · Light
│   └── light/components/index.tsx  # Библиотека компонентов · Light
├── config/
│   ├── project.tsx   # Настройки проекта
│   └── routes.tsx    # PROJECT_ROOT p/neso/design_1; ROUTES.componentsDark/Light, pageDark/Light
└── shared/
    ├── logger.ts     # Логгер
    ├── logLevel.ts   # Уровни логирования
    └── preloader.ts  # Прелоадер
```

## Критические правила
- Не хардкодить URL, использовать `config/routes.tsx`
- Цвета только через CSS переменные
- Без border (кроме glassmorphism границ)
- Консистентность между темами (одинаковая структура)
