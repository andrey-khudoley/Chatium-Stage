# design/

Workspace for BPM screen templates, showcase states and mock datasets.

## Principles

- `design/` imports production components from root folders.
- Production components must not depend on `design/`.
- Mock data lives only for visual demos and scenario testing.

## Folders

- `templates/` - reusable screen/module compositions.
- `showcase/` - documented visual states for review.
- `mock/` - mock datasets for preview and UX stress checks.
