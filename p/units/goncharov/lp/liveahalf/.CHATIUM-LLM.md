# Жизнь вполсилы — Лендинг вебинара

## Описание
Премиальный лендинг для регистрации на вебинар «Жизнь вполсилы» — онлайн-встреча с врачом и практиком ТКМ Алексеем Волковым. Дизайн глубоко отражает философию традиционной китайской медицины через природные формы, успокаивающую палитру и органичную анимацию.

## Архитектура

### Таблицы
- `tables/registrations.table` — регистрации на вебинар (name, email, phone, полный набор UTM-меток: utm_source, utm_medium, utm_campaign, utm_content, utm_term)
- `tables/settings.table` — настройки интеграций (key, value):
  - `getcourse_account_name` — имя аккаунта GetCourse
  - `getcourse_api_key` — API-ключ
  - `getcourse_offer_code` — код оффера
  - `getcourse_price` — стоимость (руб.)
  - `getcourse_utm_source_field` — ID дополнительного поля для utm_source
  - `getcourse_utm_medium_field` — ID дополнительного поля для utm_medium
  - `getcourse_utm_campaign_field` — ID дополнительного поля для utm_campaign
  - `getcourse_utm_content_field` — ID дополнительного поля для utm_content
  - `getcourse_utm_term_field` — ID дополнительного поля для utm_term

### API
- `api/register.ts` — POST-эндпоинт регистрации с валидацией (email, phone, name), CRM-интеграция, интеграция с GetCourse (передача всех UTM-меток), уведомления, события аналитики
- `components/RegistrationModal.vue` — клиентская форма регистрации с тремя обязательными чекбоксами согласия (оферта, политика ПДн, маркетинговые рассылки); до подтверждения всех чекбоксов отправка формы заблокирована, значения чекбоксов не отправляются в API
- `web/admin/getcourseApi.ts` — API для сохранения/сброса настроек GetCourse

### Страницы
- `index.tsx` — точка входа с SEO-мета-тегами
- `pages/MainPage.vue` — главная страница-сборщик секций
- `index.tsx` + `pages/ThankYouPage.vue` — отдельный роут `GET /thanks` со страницей «Спасибо за регистрацию», таймером до `28.03.2026 19:00 МСК`, выбором мессенджера и внешними ссылками на Telegram/MAX-ботов
- `web/admin/getcourse.tsx` — панель настроек GetCourse (только для Admin)
- `web/admin/analytics.tsx` — аналитика регистраций с просмотром таблицы (только для Admin)

### Компоненты (components/)
- `HeroSection.vue` — первый экран с атмосферным overlay, органическими blob-формами
- `SignsSection.vue` — 7 признаков снижения энергии с природными фоновыми элементами
- `WhySection.vue` — почему привычные усилия не работают
- `ResultsSection.vue` — что получит участник
- `ProgramSection.vue` — программа встречи
- `EnergyLevels.vue` — 3 уровня состояния с плавным цветовым градиентом (серо-коричневый → охра → моховой зелёный)
- `PracticalTips.vue` — режим / питание / движение
- `ImportanceSection.vue` — тёмный блок с усиленным контрастом, пульсирующими свечениями, градиентной подсветкой
- `ApproachSection.vue` — основа подхода ТКМ с органическими формами на заднем плане
- `SpeakerSection.vue` — о спикере Алексее Волкове
- `AudienceSection.vue` — кому подойдёт встреча
- `FormatSection.vue` — формат, цена, CTA
- `FinalCTA.vue` — финальный блок дожима
- `RegistrationModal.vue` — модальное окно с glassmorphism-эффектом, backdrop-filter blur

### Важные связи событий (CTA)
- `HeroSection.vue`, `AudienceSection.vue` и `FormatSection.vue` эмитят событие `openModal`.
- В `pages/MainPage.vue` все CTA-секции подключены через `@open-modal="showModal = true"`, чтобы любая кнопка «Принять участие» открывала единое модальное окно `RegistrationModal`.

### Стили
- `styles.tsx` — расширенная палитра (moss, ochre, slate), анимации "breatheIn", органические blob-формы

## Визуальная концепция (обновлена)

### Цветовая палитра
- **Moss** (моховой зелёный): акценты, витальность, активное состояние
- **Ochre** (охра): CTA-кнопки, переходные состояния, привлечение внимания
- **Slate** (тёмный сланец): основной текст, контрастные блоки
- **Sage** (шалфей): вспомогательные элементы, мягкие акценты
- **Cream/Warm**: фоновые градиенты, тепло, уют

### Типографика
- **Cormorant Garamond**: увеличенный letter-spacing (0.015–0.02em), жирность 600–700, премиальность
- **Source Sans 3**: body-текст, чистота, читаемость

### Анимации
- `breatheIn` — появление элементов как "вдох", плавное масштабирование + движение
- `flowInLeft/Right` — боковое появление с микромасштабированием
- `expandIn` — расширение блоков с эффектом глубины
- `float` — медленная левитация blob-форм (20s цикл)
- Cubic-bezier (0.34, 1.56, 0.64, 1) для упругих, естественных переходов

### Графические элементы
- **Organic blobs**: размытые формы с border-radius 40%/60%/70%/30%, blur(60px), opacity 0.08
- **Градиенты**: многослойные (3+ цвета), создают глубину вместо плоских заливок
- **Glassmorphism**: backdrop-filter blur(20px), полупрозрачные белые панели с лёгкой границей
- **Свечения**: пульсирующие blur-эффекты в ImportanceSection для фокусировки внимания

### UI-компоненты
- **CTA-кнопки**: охра-градиент с анимированным background-position, тени с ochre-тонами, hover-эффект масштабирования
- **Модальное окно**: glassmorphism, скруглённые углы 28px, мягкая тень
- **Карточки**: backdrop-blur, 2px границы с низкой opacity, hover-трансформации
- **Шкала энергии** (EnergyLevels): визуальный градиент от slate → ochre → moss с маркерами-индикаторами

## Интеграция с GetCourse

### Настройка
1. Перейти на страницу `/p/units/goncharov/lp/liveahalf/web/admin/getcourse` (только для Admin)
2. **Основные параметры:**
   - Имя аккаунта GetCourse (поддомен)
   - API-ключ
   - Код оффера
   - Стоимость (руб.) — если указана стоимость больше 0, пользователь будет автоматически перенаправлен на оплату
3. **Дополнительные настройки** (опционально):
   - ID дополнительных полей GetCourse для передачи UTM-меток в кастомные поля пользователя
   - Найти ID полей можно в разделе «Настройки → Дополнительные поля» GetCourse
4. Сохранить настройки

### Процесс интеграции (формат API GetCourse)
При каждой регистрации:
1. **Валидация данных**: проверка email (формат), phone (формат), name (минимум 2 символа)
2. Данные сохраняются в `registrations.table` с нормализацией (trim, toLowerCase для email)
3. Проверяется наличие всех параметров GetCourse: `account_name`, `api_key`, `offer_code`
4. Формируется динамический URL: `https://{account_name}.getcourse.ru/pl/api/deals`
5. **Формат запроса** — `x-www-form-urlencoded` с тремя полями:
   - `action`: `add`
   - `key`: API-ключ
   - `params`: JSON-объект, закодированный в **base64** через встроенный Chatium-глобал `base64Encode(json)`, содержащий:
     - `user`: `email`, `first_name`, `last_name`, `phone`
     - `system`: `refresh_if_exists: 1`, `multiple_offers: 1`, `return_payment_link: 1`
     - `session`: UTM-метки (`utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`)
     - `deal`: `offer_code`, `deal_cost` (если > 0), `addfields` (UTM в кастомные поля)
6. **Обработка ответа:**
   - Проверка одновременно `success === "true"` и `result.success === "true"` (строковые значения в JSON-ответе)
   - Дополнительно учитывается `result.error !== "true"`
   - Если `deal_cost > 0`: извлекается `result.payment_link` для редиректа на оплату
   - Если стоимость = 0 или ссылка отсутствует → показывается сообщение об успешной регистрации
7. **Логирование:**
   - Технические диагностические логи в `ctx.log` по шагам:
     - `liveahalf:getcourse:start`
     - `liveahalf:getcourse:config-missing`
     - `liveahalf:getcourse:config-loaded`
     - `liveahalf:getcourse:request-prepared`
     - `liveahalf:getcourse:response-received`
     - `liveahalf:getcourse:success`
     - `liveahalf:getcourse:api-error`
     - `liveahalf:getcourse:exception`
   - Дебаг: событие `getcourse_api_debug` с телом запроса и ответа
   - При успехе: событие `getcourse_integration_success` с деталями (email, offer_code, price, payment_url)
   - При ошибке API: событие `getcourse_api_error` с `error_message` и телом ответа
   - При исключении: событие `getcourse_integration_exception`

### Безопасность
- API-ключ и имя аккаунта хранятся только на сервере
- Доступ к настройкам только для Admin
- Отправка данных через защищённый метод `.run()` вместо обычных HTML-форм
- Динамическое формирование URL на основе имени аккаунта из настроек

## Ключевые решения
- Философия ТКМ отражена через природные формы и цвета
- Глубина через многослойные градиенты вместо плоских цветов
- "Дыхание" в анимациях — элементы появляются как естественный вдох
- Контраст тёмных и светлых секций для визуальных пауз
- **Бэкенд-архитектура:**
  - Валидация данных на сервере (email, phone, name)
  - Нормализация данных перед сохранением (trim, toLowerCase)
  - Полный захват UTM-меток (5 параметров) для аналитики
  - Интеграции: CRM, GetCourse, уведомления админу, writeWorkspaceEvent
  - Обработка ошибок с логированием в аналитику
- **Админ-панель:**
  - Vue-компоненты вместо HTML-форм для безопасности и удобства
  - Отправка данных через `fetch()` API (POST-запросы к `/web/admin/getcourseApi~api/save` и `~api/reset`)
  - Glassmorphism-дизайн с органическими blob-формами
  - Toast-уведомления для обратной связи
