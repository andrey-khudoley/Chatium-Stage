# Half (жизнь вполсилы — в записи)

Лендинг и backend регистрации на вебинар «Жизнь вполсилы». Копия сценария `liveahalf` с отдельными Heap-таблицами и формулировками про формат «в записи» (не «онлайн» / «живой эфир»).

## Что делает проект

- Показывает посадочную страницу вебинара и форму регистрации.
- Перед отправкой формы требует обязательные юридические согласия (оферта, политика ПДн, маркетинговые рассылки).
- Сохраняет регистрации в `tables/registrations.table.ts`.
- Отправляет регистрацию в CRM и аналитические события workspace.
- Интегрируется с GetCourse через Import API (`/pl/api/deals`) и при платном оффере возвращает ссылку на оплату.

## Основные точки входа

- `index.tsx` — вход приложения.
- `pages/MainPage.vue` — основная страница лендинга.
- `pages/ThankYouPage.vue` — отдельная страница благодарности после регистрации с таймером и выбором мессенджера.
- `api/register.ts` — API регистрации и серверные интеграции.
- `web/admin/getcourse.tsx` и `web/admin/getcourseApi.ts` — админка настроек GetCourse.

## Жизнь вполсилы — лендинг вебинара

Проект: лендинг регистрации на вебинар «Жизнь вполсилы».

### Что внутри
- **UI**: `index.tsx`, `pages/MainPage.vue`, компоненты в `components/`.
- **Регистрация**: `api/register.ts` — валидация входных данных, создание записи в `tables/registrations.table.ts`, workspace-аналитика, уведомления, а также best-effort вызов интеграции при настройках.
- **Интеграция GetCourse API**: best-effort создание/обновление пользователя при регистрации (ошибки не ломают регистрацию).
- **Админка GetCourse**: `web/admin/getcourse/*` — страница для установки/замены/сброса ключа `gcApiKey` (доступ только `Admin`). На главной странице есть фиксированные элементы: “Аналитика” (видна всем, ведёт на `/web/admin/analytics`) и для `Admin` — фиксированная кнопка “Настройки” (ведёт на `/web/admin/getcourse`).
- **Админ-аналитика**: `web/admin/analytics/*` — заглушка страницы аналитики.

### GetCourse — какие настройки нужны
Настройки хранятся в KV-таблице `tables/settings.table.ts` (ключ → значение):
Основной параметр — `gcApiKey` (при необходимости используется legacy `getcourse.apiKey` для совместимости). Сброс ключа (через админку) удаляет оба значения: и `gcApiKey`, и `getcourse.apiKey`. Дополнительно настраиваются параметры соединения и поведение интеграции; полный список ключей — в `docs/data.md`.

## Текущее состояние
Лендинг работает как точка входа для регистрации (`POST /api/register`). При наличии ключа интеграции выполняется best-effort синхронизация пользователя в GetCourse, а также в режиме best-effort выполняются CRM-событие, workspace-event и отправка уведомлений. Ошибки этих шагов логируются и не влияют на успешность регистрации (пока не упал критичный шаг создания записи в Heap).

Админка вынесена отдельно: `GET /web/admin/getcourse` + `POST /admin/getcourse/api/save|reset` (только `Admin`). Страница `GET /web/admin/analytics` — заглушка.
