# Проект lava_gc_integration (Интеграция Lava.top и GC)

## Что это
Приложение Chatium по пути `p/units/aom/lava_gc_integration`: интеграция Lava.top и GetCourse. Структура унаследована от шаблона (страницы, API настроек и логов, Heap). Heap: помимо `settings` и `logs` — таблицы `lava_payment_contract`, `lava_webhook_event`, `lava_lock_log` (файлы в `tables/`), репозитории в `repos/` (`lava_payment_contract.repo.ts`, `lava_webhook_event.repo.ts`, `lava_lock_log.repo.ts`); `create` в этих репозиториях: публичный вход `HeapCreateInput` / экспорт `*CreateInput`, внутри — приведение к `Parameters<typeof …Table.create>[1]` (typings `@app/heap`); в `lava-payment.service` — именованные `create`, `findActiveByGcOrderAmountAndCurrency` и `deactivateActiveContractsForGcOrderId` из `lava_payment_contract.repo`; в каждом из трёх `lava_*` репозиториев — `writeServerLog` severity 7 на вход/выход операций (без рекурсии с `logs.repo.create`, в отличие от записи в таблицу `logs` из `logger.lib`). В `lib/settings.lib.ts` объявлены ключи и геттеры для секретов/URL Lava и GetCourse (`lava_api_key`, `lava_base_url`, `lava_product_id`, `lava_offer_id`, `lava_webhook_secret`, `gc_api_key`, `gc_account_domain`). Типы внешних контрактов (валюта, webhook, payment-link) — `lib/lava-types.ts`. Мастер на админке Lava: ввод ключа и URL → POST `/api/admin/lava/catalog` (каталог продуктов/офферов) → выбор → сохранение id в настройки; поле `lava_webhook_secret` — автогенерация hex (кнопки «Сгенерировать»/«Обновить»), показ по «Показать», сохранение в Heap через `POST /api/settings/save`; блок GetCourse: поля `gc_api_key` и `gc_account_domain` (`shared/gcSettingKeys.ts`) → POST `/api/admin/getcourse/verify` (`verifyGcPlApiAccess`) → «Сохранить в настройки»; `shared/lavaBaseUrl.ts` и `lib/lava-api.client.ts` (исходящий HTTP к Lava — `request({…})` без `ctx`, не `request(ctx,…)`): `updateOfferPrice` (PATCH цены оффера), `createContract` (POST invoice v3), `getProducts` (GET первая страница / диагностика), `fetchLavaProductsCatalog` (пагинация для каталога админки). Основной поток ссылки на оплату — `lib/lava-payment.service.ts`: `createPaymentLink` (идемпотентность по заказу GC + сумма + валюта; при смене суммы — отмена прочих активных контрактов по заказу; `runWithExclusiveLock` + `lava_lock_log`, PATCH/POST Lava, запись `lava_payment_contract`). Приём webhook Lava — `lib/lava-webhook.service.ts`: `processWebhook` (секрет `lava_webhook_secret` в заголовке `X-Api-Key`, дедупликация по `dedupe_key`, повтор обработки при сохранённой ошибке `contract_not_found`, маппинг статусов в Heap, вызов `updateDealStatus` для `paid`/`failed`). HTTP-вход для webhook — `POST /api/integrations/lava/webhook` (`api/integrations/lava/webhook/index.ts`, `lavaWebhookRoute`): тело через `.body` (`@app/schema`), вызов `processWebhook`; тот же путь **GET** — `lavaWebhookInfoRoute`, JSON-проба доступности для браузера. Исходящие вызовы в GetCourse PL API — `lib/getcourse-api.client.ts`: `normalizeGcAccountDomain`, `verifyGcPlApiAccess` (проверка ключа/домена для админки), `updateDealStatus` (POST form `action`/`key`/Base64 `params` на `/pl/api/deals`, без throw при ошибке ответа); **environment-agnostic Base64**: функция `encodeBase64()` использует `Buffer` (Node.js) / `btoa` (браузер) / ручную реализацию (sandbox) — полностью самодостаточная, работает во всех средах выполнения Chatium (UTF-8, суррогатные пары, RFC 4648), критична для компиляции `// @shared-route` в клиентский бандл. Входящий запрос ссылки на оплату — `POST /api/integrations/lava/payment-link` (`api/integrations/lava/payment-link/index.ts`, `lavaPaymentLinkRoute`): **без** заголовков авторизации (в т.ч. вызов из браузера/JS на странице оплаты), тело через `.body` (`@app/schema`); опционально `integrationTestDryRun: true` — ответ успеха без Lava/Heap. Тесты: GET `payment-link-dry-run-unit` (`lavaPaymentLinkRoute.run(ctx, { gcOrderId, … })` — поля тела на верхнем уровне, не `{ body: { … } }`), POST `payment-link-http-integration` (`request()` на тот же URL, тело с опциональным `paymentLinkOverrides`), лайв: GET `payment-link-heap-settings-read`, GET `payment-link-full-route-run`, POST `payment-link-full-http-integration` (`payment-link-live-test.lib`, сброс контрактов `gc_order_id=test` перед полным путём), `lib/app-public-url.lib.ts`.

## Тесты интеграции Lava
`lib/settings-save-credentials.lib.ts` — чистые функции слияния полей для `POST /api/settings/save` (`resolveGcCredentialsForSave`, `resolveLavaCredentialsForSave`, `shouldVerifyCredentialPair`). Категории в `GET /api/tests/list`: `unit-save-credentials`, `unit-page-routes`, `integration-credentials-both`, `integration-check`, `payment-link-integration` (dry-run юнит + HTTP POST), `payment-link-live` (чтение Heap для Lava, полный `route.run` и HTTP без dry-run — см. `lib/payment-link-live-test.lib.ts`, `deactivateActiveContractsForGcOrderId` в `lava_payment_contract.repo`; плюс POST/GET `webhook-live-test-arm` / `webhook-live-test-status` и `lib/webhook-live-test.lib.ts` для лайв-проверки webhook после оплаты), прочие — см. `docs/api.md`. Страница `./web/tests` (`TestsPage.vue`): вкладка «Интеграция» — блоки payment-link dry-run, отдельная карточка «Лайв: payment-link» (триада Heap + route.run + HTTP), карточка «Лайв: webhook после оплаты» (URL POST webhook, ссылка на оплату, вооружение по `lava_contract_id`); «Запустить все тесты» на интеграции **не** запускает лайв-триаду (отдельная кнопка). Слоёвые GET — query `testId=<id>`. Отдельно: GET `integration-credentials-both`.

## Временные HTTP-пробы (`temp/`)
Каталог **`temp/`** (не прод): маршруты для ручной проверки, как Chatium отдаёт разные статусы и тела (200/3xx/4xx/5xx, JSON при 200 с `ok: false`, редиректы, `throw`). Оглавление: **`./temp`** относительно `PROJECT_ROOT`. См. `docs/testing.md`. Сводка по фактическим HTTP-ответам (curl): **`inner/docs/048-chatium-http-response-probes.md`**.

## Маршруты интеграции (константы)
`PROJECT_ROOT` вынесен в **`shared/projectRoot.ts`** — Vue (`TestsPage.vue`, …) и `web/login` импортируют **только** его, не весь `config/routes.tsx` (иначе в клиентский бандл могли попадать пути/API-модули → ошибка при оценке `payment-link/index.ts`). В `config/routes.tsx`: **`ROUTES`** / **`ROUTE_PATHS`** только страницы; URL интеграций — в `docs/api.md` (`/api/integrations/lava/payment-link`, webhook). `config/project.tsx` — `// @shared`.

## Критические правила
- Не хардкодить URL, использовать `config/routes.tsx`.
- Документацию обновлять при любых изменениях структуры/логики.
