# Слайды эфира — спецификация

Документ описывает 23 слайда презентации до момента «форма доступа». После формы спикер выключает презентацию и переходит к Q&A в формате камера + чат.

Каждый слайд описан с уровнем детализации, достаточным, чтобы собрать его в Figma / Keynote / PowerPoint / HTML без дополнительных вопросов. Сначала — глобальная визуальная система, потом по слайду.

---

## Часть I · Визуальная система

### Палитра

Тёмная схема, без чёрного-в-ноль (он выглядит мёртво на проекторе и в стриме).

| Назначение         | Hex                      | Где применяется                                                     |
| ------------------ | ------------------------ | ------------------------------------------------------------------- |
| **Bg-base**        | `#0B0F19`                | Основной фон каждого слайда (тёмный сине-чёрный, с лёгким оттенком) |
| **Bg-surface**     | `#131826`                | Поверхности карточек, блоков кода — на 1 тон светлее фона           |
| **Bg-elevated**    | `#1B2336`                | Подсвеченные/выбранные блоки (gateway-карточка, активная вкладка)   |
| **Text-primary**   | `#F5F7FA`                | Основной текст, заголовки                                           |
| **Text-secondary** | `#8B95A7`                | Подписи, второстепенный текст                                       |
| **Text-tertiary**  | `#5A6478`                | Метаданные, нумерация фоном, ослабленные слои                       |
| **Accent-cyan**    | `#00D9FF`                | Главный акцент: подчёркивания, активные элементы, ключевые инсайты  |
| **Accent-amber**   | `#FFB347`                | Внимание / предупреждение / «горячие» точки                         |
| **Status-success** | `#4ADE80`                | Статус-код 200, «уже работает»                                      |
| **Status-warning** | `#FFB347`                | Статус-код 4xx, «в процессе»                                        |
| **Status-error**   | `#F87171`                | Статус-код 5xx, проблемные точки                                    |
| **Border-subtle**  | `rgba(255,255,255,0.08)` | Тонкие границы карточек                                             |
| **Border-accent**  | `rgba(0,217,255,0.4)`    | Границы выделенных карточек                                         |

**Градиенты:** только два, оба очень плавные.

- **Vertical bg-gradient:** от `#0B0F19` (top) до `#070A12` (bottom). Применяется к фону каждого слайда — даёт лёгкое визуальное углубление.
- **Accent glow:** `radial-gradient(circle, rgba(0,217,255,0.15) 0%, transparent 70%)` — для подсветки ключевых элементов (gateway-карточка, anchor-слайды).

### Типографика

Один основной шрифт + один моно. Оба бесплатные, доступны в Google Fonts.

| Стиль             | Шрифт          | Вес | Назначение                                             |
| ----------------- | -------------- | --- | ------------------------------------------------------ |
| **Display**       | Manrope        | 800 | Заголовки слайдов (огромные anchor-фразы)              |
| **H1**            | Manrope        | 700 | Заголовки слайдов                                      |
| **H2**            | Manrope        | 600 | Подзаголовки, заголовки карточек                       |
| **Body**          | Inter          | 400 | Основной текст                                         |
| **Body-emphasis** | Inter          | 600 | Выделенный текст внутри основного                      |
| **Caption**       | Inter          | 500 | Подписи, метаданные (uppercase, letter-spacing 0.05em) |
| **Mono**          | JetBrains Mono | 400 | Код, имена операций, URL, JSON                         |
| **Mono-bold**     | JetBrains Mono | 600 | Ключи в коде, выделенные операции                      |

**Размеры (для 1920×1080 канвы):**

- Display: 120pt
- H1: 64pt
- H2: 36pt
- Body: 28pt
- Caption: 20pt
- Mono: 26pt

### Сетка и отступы

- **Канвас:** 1920×1080 (16:9).
- **Безопасные поля:** 96pt со всех сторон. Ничего важного ближе к краю.
- **Колонки:** 12-колоночная сетка, gutter 32pt.
- **Вертикальный ритм:** базовая единица 8pt; все отступы кратны 8.

### Сквозные визуальные элементы

Эти штуки повторяются от слайда к слайду и держат единство:

1. **Дот-грид фон.** Очень тонкая сетка из точек 2pt диаметром, шаг 80pt, цвет `rgba(255,255,255,0.03)`. Едва заметен, добавляет «технологичности» без шума.
2. **Номер слайда.** Внизу справа, JetBrains Mono 16pt, цвет text-tertiary, формат `02 / 23`.
3. **Прогресс-бар.** Внизу под слайдом, тонкая (2pt) полоска во всю ширину; проявленная часть — accent-cyan, остальная — `rgba(255,255,255,0.06)`. Длина проявленной части = текущий слайд / всего слайдов.
4. **Микро-логотип/инициалы.** Внизу слева, моно `АХ · CHATIUM × GETCOURSE`, 14pt, text-tertiary. Не отвлекает, но напоминает.
5. **Заголовок-стрипка.** Под заголовком слайда — короткая горизонтальная линия 4pt высотой, 80pt шириной, цвет accent-cyan. Единый «штамп» бренда.
6. **Number badges (01, 02, 03…).** Круг 56pt диаметром, без заливки, бордер 2pt accent-cyan, внутри JetBrains Mono 22pt accent-cyan. Используются для нумерации концептов.
7. **Хайлайт-карточки.** Rounded rectangle радиус 16pt, fill bg-surface, бордер 1pt border-subtle. Активная — fill bg-elevated, бордер 2pt border-accent + accent-glow позади.

### Анимация (если соберёшь в Keynote/HTML)

Принципы:

- **Никаких** «плёночных переходов», вращений, отскоков.
- **Только два движения:** fade (200ms ease-out) и slide-up (8pt смещение вверх + fade, 240ms).
- Между слайдами — **fade through** на чёрный 100ms, чтобы глаз отдыхал.
- Ничего не мерцает, не пульсирует. Один accent-glow может медленно «дышать» (opacity 0.85→1.0, 4 сек цикл) — на gateway-слайде.

## Часть II · Слайды по порядку

### Блок 1 · Открытие (слайды 1–2)

---

#### Слайд 1 · Title

**Назначение:** держать на экране, пока заходят зрители; задаёт тон.

**Layout:** centered, vertical.

**Текст:**

- **Главный:** «Chatium + GetCourse»
- **Подзаголовок:** «Что случилось и что с этим делать»
- **Подвал:** «Эфир для клуба Chatium · Андрей Худолей · 6 мая 2026»

**Композиция:**

- Заголовок в Manrope 800, 120pt, text-primary. По центру горизонтально, чуть выше центра вертикально (на 1/3 от верха).
- Под заголовком — accent-cyan стрипка 4pt × 200pt, по центру, отступ 32pt.
- Подзаголовок: Manrope 600, 36pt, text-secondary. По центру, отступ 32pt от стрипки.
- Подвал: Inter 500, 20pt, uppercase, letter-spacing 0.1em, text-tertiary. По центру, отступ 96pt вниз от подзаголовка.

**Визуальные элементы:**

- Дот-грид фон (как описано).
- В правом верхнем углу — небольшая декоративная фигура: круг 240pt диаметром, без заливки, бордер 1pt `rgba(0,217,255,0.2)`, и внутри него ещё один круг 160pt с тем же стилем. Концентрические круги. Очень субтильно.
- В левом нижнем углу — то же зеркально, но круги меньше (120pt и 80pt). Создаёт визуальный баланс.

**Чего тут НЕТ:** прогресс-бара, номера слайда, логотипа. Title-слайд — единственный без них.

---

#### Слайд 2 · Agenda — о чём поговорим

**Назначение:** показать дугу часа; задать ожидание «не лекция, разговор».

**Layout:** двух-колоночный — слева список тем, справа таймлайн.

**Текст:**

- **Заголовок:** «О чём поговорим»
- **Подзаголовок (subtle):** «Час, без пауз, с вопросами по ходу»

**Список (левая колонка, 8 из 12 колонок сетки):**

| №   | Текст                | Подпись (caption под пунктом) |
| --- | -------------------- | ----------------------------- |
| 01  | Что такое интеграция | вайб-кодинг и его границы     |
| 02  | Как устроен HTTPS    | четыре вещи в каждом запросе  |
| 03  | Как читать API-спеку | пять минут на проверку        |
| 04  | Что я для вас собрал | gateway + тонкий SDK          |
| 05  | Демо                 | несколько готовых методов     |

**Композиция:**

- Заголовок «О чём поговорим» — H1 64pt, text-primary, верхний левый угол блока, после safe-area.
- Стрипка 4pt × 80pt, accent-cyan, 24pt ниже заголовка.
- Подзаголовок — Body 28pt, text-secondary, 16pt ниже стрипки.
- Список начинается на 96pt ниже подзаголовка.
- Каждый пункт — горизонтальная строка:
  - Number badge (как описано) слева.
  - Через 32pt — заголовок пункта (Manrope 600, 36pt, text-primary).
  - Под заголовком (8pt отступ) — caption (Inter 400, 22pt, text-secondary).
- Между пунктами — отступ 40pt, плюс hairline divider 1pt `rgba(255,255,255,0.06)` посередине отступа (по 20pt с каждой стороны).

**Правая колонка (4 из 12 колонок):**

- Вертикальный таймлайн.
- Тонкая вертикальная линия 2pt, accent-cyan opacity 0.4, длина = высоте списка слева.
- На линии — 5 точек (соответствуют 5 пунктам), круги 12pt, fill accent-cyan, ровно напротив каждого пункта.
- Слева от каждой точки (между линией и точкой) — короткая горизонтальная линия 1pt 24pt длиной, чтобы соединить с пунктом списка справа.
- В верху колонки — caption «00:00», в низу — «01:00». Между — caption «60 минут».

**Визуальные элементы:**

- Дот-грид фон.
- Прогресс-бар 1/23.
- Номер слайда «02 / 23».

### Блок 2 · Вайб-кодинг (слайды 3–5)

---

#### Слайд 3 · Вайб-кодинг — это нормально

**Назначение:** заявить тему так, чтобы зал узнал в ней себя.

**Layout:** centered, эмоциональный.

**Текст:**

- **Pull-quote (огромный):** «Мы все сейчас вайб-кодим»
- **Под ним:** «И это нормально. У этого есть граница, на которую вы уже наступили»

**Композиция:**

- Pull-quote: Manrope 800, 96pt, text-primary, по центру, чуть выше центра.
- Слева от pull-quote — accent-cyan вертикальная стрипка 4pt × 240pt, начинается за 24pt до текста, выровнена по центру pull-quote.
- Под pull-quote (отступ 48pt) — текст-продолжение, Manrope 400, 32pt, text-secondary, по центру, две строки.

**Визуальные элементы:**

- Дот-грид фон.
- В фоне за pull-quote — огромная цифра «01» (Mono 480pt, цвет `rgba(255,255,255,0.025)`), почти невидимая, обозначает первый «концептуальный блок».
- Прогресс-бар 2/23.

---

#### Слайд 4 · Декларативный vs императивный

**Назначение:** показать различие двух подходов на конкретных примерах кода.

**Layout:** две равные колонки, разделитель посередине.

**Текст и структура:**

**Левая колонка — «ДЕКЛАРАТИВНО»**

- Caption-метка (uppercase, letter-spacing): «ДЕКЛАРАТИВНО», 20pt, accent-cyan.
- H2: «Описываем результат».
- Code block (Mono 24pt, на bg-surface, padding 24pt, radius 12pt):
  ```
  «Сделай форму, которая собирает заявки
  и кладёт их в GetCourse»
  ```
- Под кодом — короткий список (Inter 22pt, text-secondary), 3 пункта с маркером ✓ (accent-cyan):
  - Быстро
  - Удобно
  - Естественно для AI

**Правая колонка — «ИМПЕРАТИВНО»**

- Caption-метка: «ИМПЕРАТИВНО», 20pt, accent-amber.
- H2: «Описываем шаги».
- Code block:
  ```
  «Возьми email из формы.
  Сделай POST на /api/users.
  Передай Authorization: Bearer <ключ>.
  Если 401 — обнови токен и повтори»
  ```
- Под кодом — список с маркером ✓ (accent-amber):
  - Скучнее
  - Длиннее
  - AI ошибается реже

**Композиция:**

- Заголовок слайда — мелкий, в верхнем левом углу: «Два способа разговаривать с AI про код», H2 36pt.
- Стрипка не нужна (две акцентные метки внутри колонок служат якорями).
- Между колонками — вертикальная hairline линия 1pt `rgba(255,255,255,0.08)`, во всю высоту контента.
- Колонки симметричные, по центру слайда. Каждая получает 5 колонок из 12, gutter 32pt в центре.

**Визуальные элементы:**

- Дот-грид фон.
- Прогресс-бар 3/23.
- Номер «03 / 23».

---

#### Слайд 5 · Где декларативный ломается

**Назначение:** именованная боль аудитории — три конкретные точки.

**Layout:** заголовок сверху, три равные карточки в ряд, callout снизу.

**Текст:**

- **Заголовок:** «Декларативно ломается на интеграциях»
- **Подзаголовок:** «AI не знает, как именно сейчас устроен конкретный API»

**Три карточки (в ряд):**

| Карточка | Заголовок                | Подпись                                                 |
| -------- | ------------------------ | ------------------------------------------------------- |
| 1        | Endpoints, которых нет   | AI генерирует правдоподобные URL, которых не существует |
| 2        | Старые названия полей    | Передаёт `name`, когда API ждёт `full_name`             |
| 3        | Перепутанная авторизация | Кладёт ключ в заголовок, который API не читает          |

**Композиция карточки:**

- Размер: 480 × 320pt.
- Fill: bg-surface, radius 16pt, border 1pt border-subtle.
- Сверху — абстрактная иконка-маркер: квадрат 64×64pt, состоящий из 4 точек/тире (как «битый» pattern), цвет accent-amber, 80% opacity.
- Заголовок карточки — Manrope 600, 28pt, text-primary, отступ 32pt от верха иконки.
- Подпись — Inter 400, 20pt, text-secondary, отступ 16pt от заголовка.

**Callout снизу:**

- Полоса во всю ширину между карточками и нижним полем.
- Fill: `rgba(255,179,71,0.08)` (легчайший amber-tint), border-left 4pt accent-amber.
- Padding 32pt, radius 12pt.
- Текст: «Декабрь 2025 у GetCourse сделал всё это особенно плохо», Inter 600, 28pt, text-primary.

**Визуальные элементы:**

- Дот-грид фон.
- Прогресс-бар 4/23.

### Блок 3 · Anchor: HTTPS = разговор (слайды 6–7)

---

#### Слайд 6 · ⭐ Anchor — интеграция = HTTPS

**Назначение:** ключевой anchor-слайд. Якорная фраза, к которой можно вернуться весь эфир.

**Layout:** почти пустой слайд, центральная фраза.

**Текст:**

- **Главное:** «Интеграция — это разговор по HTTPS»
- **Капелька:** «(запомните эту фразу)»

**Композиция:**

- Главная фраза: Display 120pt Manrope 800, text-primary, по центру по обоим осям.
- Под фразой (отступ 48pt) — accent-cyan стрипка 4pt × 280pt, по центру.
- Под стрипкой (отступ 32pt) — каплочка: Inter 400 italic, 28pt, text-secondary, по центру.

**Визуальные элементы:**

- Дот-грид фон.
- Accent-glow позади центральной фразы: огромный круг 1200pt диаметра, `radial-gradient(rgba(0,217,255,0.12) 0%, transparent 70%)`, центр совпадает с центром фразы.
- Этот glow медленно «дышит» (opacity 0.85→1.0, 4 сек цикл), если делаешь анимацию.
- Прогресс-бар 5/23.
- Номер слайда — но в этот раз справа сверху (вместо снизу), Mono 16pt, text-tertiary. Просто чтобы не мешать центру.

**Что не так с этим слайдом, специально:** он почти пустой. Это намеренно. Аудитория должна остановиться на нём вместе со спикером.

#### Слайд 7 · От кабеля к HTTPS — одна история

**Назначение:** за пару минут дать интуицию «наслоение», без отдельной лекции про «канал»; подвести к тому, что эфир сфокусируется на верхнем уровне — HTTPS.

**Layout:** заголовок сверху; основной блок — **горизонтальная цепочка из 6+1 шагов** (слева направо), как эволюция схемы; под цепочкой — одна строка-якорь.

**Текст:**

- **Заголовок:** «От кабеля к HTTPS»

**Шаги цепочки (каждый — вертикальная «колонка» 200–240pt шириной: маленькая пиктограмма сверху, заголовок, подпись):**

|   № | Заголовок      | Подпись (caption, 2 строки max)                                                              |
| --: | -------------- | -------------------------------------------------------------------------------------------- |
|   1 | Два компьютера | Сначала — сильный кабель напрямую: два узла, один линк                                       |
|   2 | Коммутатор     | Появляется свитч: в одном сегменте много машин — простейшая локалка → **протоколы 2 уровня** |
|   3 | Маршрутизаторы | Сети стыкуются, нужны маршруты → **протоколы 3 уровня**                                      |
|   4 | Надёжность     | Нужно понимать, дошло ли — **4 уровень**                                                     |
|   5 | Сеанс          | Долгий разговор, в т.ч. роль провайдера — **5 уровень**                                      |
|   6 | Сжатие         | Данные ужимают и приводят к виду для передачи — **6 уровень**                                |
|   7 | **HTTPS**      | Когда всё ниже уже «есть», сверху один привычный протокол — **про него эфир**                |

**Композиция шага-колонки:**

- Вверху — абстрактная иконка 48×48pt (монохром accent-cyan / text-secondary): для шагов 1–6 — условные «два узла», «крест свитча», «две сети + стрелка», и т.д.; для шага 7 — замок/TLS-намёк или буквы `HTTPS` в рамке.
- Заголовок: Manrope 700, 22–24pt, text-primary, по центру колонки.
- Подпись: Inter 400, 18–20pt, text-secondary, по центру, line-height 1.35.
- Между колонками — тонкая вертикальная стрелка → (1pt, text-tertiary, 32pt gap), читается как «потом».

**Якорная строка под цепочкой (во всю ширину контента):**

- «Номера слоёв — ориентир; запоминать не надо. Важно: интеграции вы видите как **HTTPS**», Inter 600, 24pt, text-secondary, по центру; слово **HTTPS** — accent-cyan, Mono-bold.

**Визуальные элементы:**

- Дот-грид фон.
- Прогресс-бар 6/23.
- При нехватке ширины на мобильном превью: перенос цепочки в **две строки** (1–4 сверху, 5–7 снизу), стрелки между строками — изгиб вниз.

### Блок 4 · Сетевые слои (слайды 8–9)

---

#### Слайд 8 · Слои протоколов (та же лестница, вид сверху)

**Назначение:** закрепить ту же историю в виде **вертикального стека** — от сегмента к HTTPS; зрительно связать со слайдом 7.

**Layout:** стек горизонтальных полос (снизу вверх), 6 полос + верхняя выделенная.

## Часть III · Сборка и техника

### Если собирать в Keynote / PowerPoint

- Создать master-slide с фоном, дот-гридом, прогресс-баром, номером слайда, мини-логотипом — единым шаблоном.
- Установить шрифты Manrope, Inter, JetBrains Mono локально перед сборкой.
- Все цвета из палитры выше — внести в палитру документа (Color → Custom).
- Прогресс-бар легче всего сделать руками: одна полоса 1920pt длиной, поверх неё — `(N/23) × 1920pt` accent-cyan полоса. Менять для каждого слайда.

### Если собирать в HTML / Reveal.js

- Tailwind/CSS variables по списку выше.
- Шрифты из Google Fonts.
- Дот-грид — `background-image: radial-gradient()`, шаг 80px.
- Анимации — CSS transitions, fade 200ms / slide-up 240ms.
- Reveal.js theme — кастомная, на базе black, с переписанной палитрой.

### Если делегировать дизайнеру

- Этого документа должно хватить как ТЗ. Если будут вопросы — пусть смотрит в [[elvira-broadcast-script|скрипт первых 10 минут]] и [[elvira-broadcast-cheatsheet|шпаргалку]] для понимания, что именно говорится на каждом слайде.

### Тайминг сборки

- На скоростную сборку в Keynote по этому ТЗ — ~3–4 часа.
- На дизайн с нуля в Figma + полировка — день.
- На минимальный «сделать чтобы было» — за 90 минут можно собрать функциональную версию: дот-грид, шрифты, цвета, основная вёрстка. Без украшательств: концентрические круги, glow, обратной маски QR. Это «чисто чтобы по делу».

## Часть IV · Что показывать поверх слайдов

В тех местах, где спецификация не требует слайда (вход в каждый блок, завершение блока, переходные «вдох-выдох» паузы) — **спикер просто говорит в камеру**, слайд при этом стоит на актуальном содержательном.

Не показывайте слайд там, где он мешает связи с залом. Якорные слайды (6, 9, 18) — те, на которых не страшно «постоять» 30–60 секунд молча, давая аудитории додумать. Все остальные — фон под рассказ.
