# Проект Flow (`p/units/k/flow`)

## Тесты (актуализация 2026-04-04)
- `shared/testCatalog.ts` (`// @shared`) — блоки и id тестов для списка и `/api/tests/list`.
- Реализация прогонов: `lib/tests/templateUnitSuite.ts` (юнит), `lib/tests/integrationSuite.ts` (интеграция), `lib/tests/logTestRunFailures.ts` — каждый FAIL в лог с severity 3; роуты `api/tests/unit/index.ts`, `api/tests/integration/index.ts`, `api/tests/list.ts`.
- `TestsPage.vue`: три вкладки — Юнит, Интеграция, HTTP; метрики по активной вкладке; прогон текущей вкладки; HTTP-проверки с фрагментами SSR и учётом редиректов; точечный запуск (play); блоки категорий сворачиваются по клику на заголовок (первая категория развёрнута по умолчанию); загрузка UI после `bootloader-complete`.

## Логирование (актуализация 2026-04-05)
- **SSR logLevel**: все 5 страниц (главная, админка, профиль, тесты, логин) читают уровень из настроек через `getLogLevelForPage` и инжектят `window.__BOOT__.logLevel` через `getLogLevelScript`.
- **browserRemoteLogger**: подключён на главной, админке, профиле и тестах. Перехватывает console.*, window.onerror, unhandledrejection; логи из `setLogSink` также передаются. Пакетная отправка на POST /api/logger/browser (требует авторизации). На логине — не подключён (пользователь не аутентифицирован).
- **Info vs Debug**: trace-логи (карта вызовов: entry/exit функций) имеют severity 6 (Info). Payload (сырые данные) автоматически отсекается при уровне != Debug: на сервере — `shouldIncludePayload` в `lib/logger.lib.ts`, в браузере — `emitLog` фильтрует non-string args в `shared/logger.ts`. При Debug видна полная карта с данными, при Info — только текст вызовов.
- **Подробные этапы**: каждая страница логирует монтирование с сырыми props (URL-ы, auth state, user data), состояние boot loader, инициализацию remote logger, анимации, размонтирование. Все функции и ветки покрыты trace-логами.
- **Управление уровнем**: настройка `log_level` (Debug/Info/Warn/Error/Disable) в админке; порог применяется и на клиенте (`shouldLog` в shared/logger), и на сервере (`shouldLogByLevel` в lib/logger.lib).

## UI/UX (актуализация 2026-04-05)
- Скроллбар (`styles.tsx` → `customScrollbarStyles`): приглушённый, низкий контраст (тёмно-серый ползунок на почти чёрной дорожке), ширина 6px, без акцентного свечения; Firefox — `scrollbar-color`.
- Админка и страница тестов: `.app-layout` с высотой viewport и `overflow: hidden`; `<main>` без вертикального скролла; левая колонка `.ap-main` / `.tp-main` с `content-wrapper` и `overflow-y: auto`; сетка по высоте окна (`minmax(0,1fr)` в строках), в двух колонках первая колонка `minmax(240px,1fr)`; скролл только в `.ap-log-out` / `.tp-log-out`. Ширина: `w-full` / `width:100%` у корня, main и `.ap`/`.tp` (контент до 1440px по центру).

## UI/UX (актуализация 2026-04-02)
- `AdminPage.vue` и `TestsPage.vue` — премиальный CRT-дизайн: пиксельные clip-path кнопки, scanline-текстуры, staggered-анимации входа, CRT-терминальный монитор логов.
- **Иконки**: FontAwesome 6 (CDN), иконки у каждого заголовка секции, кнопки, метрики, индикатора. Классы `ap-icon-muted`, `ap-icon-hd`, `tp-icon-muted`, `tp-icon-hd`, `tp-icon-tab`, `tp-icon-block` для единообразного размера и прозрачности.
- **Цветовая палитра**: нейтрально-тёмные фоны без розового отлива (`rgba(12,11,14,...)`, `rgba(16,15,19,...)`). Текст: primary `#dedadd`, secondary `#918b8e`, muted `#5c5659`. Акцент `#c4213f` / `#d95672` — только для активных состояний и CTA. Статусы: error `#d97a8a`, warn `#c9a660`, ok `#6aaf7e` — мягкие, десатурированные. Свечение (text-shadow/box-shadow) убрано везде кроме LIVE-индикатора.
- Границы: `rgba(50,44,54,0.55)` — структурные, минимально заметные. Hover `rgba(75,62,78,0.6)`.
- Карточные `::before` линии красного акцента — `opacity: 0.2` (приглушены). Логи — `box-shadow: inset 0 0 40px rgba(0,0,0,0.25)` без красного отлива.
- `AdminPage`: статус-бар + grid (счётчики, настройки) + sticky лог-сайдбар. Иконки: `fa-terminal`, `fa-layer-group`, `fa-chart-bar`, `fa-times-circle`, `fa-exclamation-triangle`, `fa-pen`, `fa-sliders-h`, `fa-stream`, `fa-chevron-down`, `fa-trash-alt`.
- `TestsPage`: тулбар с табами + метрики + test-строки + лог-сайдбар. Иконки: `fa-flask`, `fa-cube`, `fa-network-wired`, `fa-list-ol`, `fa-check-circle`, `fa-times-circle`, `fa-minus-circle`, `fa-vial`, `fa-server`, `fa-globe`, `fa-folder` / `fa-folder-open` у категорий, `fa-play`, `fa-bolt`, `fa-clock`.
- Script-логика обоих компонентов сохранена полностью, переработаны только template + style scoped.
- В шаблонах Vue нельзя писать `as const` — используется `LOG_STREAM_KEYS` в script.
