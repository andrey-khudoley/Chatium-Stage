### Ассистент

Сделаны три отдельных блока с разной логикой и кнопками.

**Блок 1 — Проверка эндпоинтов:** проверка маршрутов приложения (/, /web/admin, /web/profile, /web/login, /web/tests) по HTTP 200; для каждого пункта [OK] или [FAIL]; кнопка «Запустить проверку эндпоинтов».

**Блок 2 — Проверка системы настроек:** тесты по одной на каждую функцию библиотеки настроек (getSettingString, getLogLevel, getLogsLimit, getLogWebhook, getDashboardResetAt, getAllSettings); один запрос к GET /api/tests/endpoints-check/settings-lib возвращает массив results; кнопка «Запустить проверку настроек».

**Блок 3 — Проверка репозитория логов:** тесты по одной на каждую функцию репозитория логов (findAll, findBeforeTimestamp, countErrorsAfter, countWarningsAfter); один запрос к GET /api/tests/endpoints-check/logs-repo; кнопка «Запустить проверку репозитория логов».

Бэкенд: settings-lib и logs-repo возвращают массив results (id, title, passed, error). Дашборд «Запустить все тесты» по-прежнему запускает все три блока и обновляет общие метрики.
