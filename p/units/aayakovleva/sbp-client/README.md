# sbp-client (LifePay SBP Client)

## Назначение

Клиентская панель LifePay (implementation-plan §1.8). Одно Chatium-приложение совмещает три роли: хранилище секретов магазина, серверная прокладка `POST /api/lp/invoke` между UI и payments-gateway с журналом запросов, приёмник входящего webhook LifePay.

## Важно

- Платформа: Chatium. Серверная часть управляется платформой.
- Стек фиксирован платформой, новые зависимости не добавляем.
- Деплой происходит автоматически после пуша.

## Текущее состояние

- Heap-настройки: `lp_apikey`, `lp_login`, `lp_webhook_token`, `gateway_base_url` (с валидациями); редактируются на `/web/admin`.
- Heap-таблицы: `request_log`, `webhook_log`, `webhook_idempotency`, `panel_access`, `panel_invites`, `settings`, `logs`.
- Серверная прокладка `POST /api/lp/invoke`: один вызов через `@app/request` к gateway, без ретраев; журнал в `request_log`.
- Приёмник webhook `POST /web/webhook?token=...`: анонимный + токен; `req.formData()` + фоллбэки; дедупликация через `runWithExclusiveLock`.
- Главная страница `/` — `pages/HomePage.vue` (компонент `ClientHomePage`), оркестратор: вкладки «Обзор», «Запросы», «Webhook», «Поиск», «Создать счёт», «Доступ». Реализована как оркестратор с подкомпонентами в `components/home/` (10 файлов) и mixin-логикой в `pages/sbpHomePageMixin.ts`.
- Страница `/web/admin` (`pages/AdminPage.vue`) — оркестратор с подкомпонентами в `components/admin/` (4 файла). Содержит карточки настроек LifePay, уровня логирования, счётчиков и лог-панель.
- Страница `/web/tests` (`pages/TestsPage.vue`) — оркестратор с подкомпонентами в `components/tests/` (3 файла). Composable `shared/useTestSuites.ts`.
- CSS вынесен в `pagecss/` (18 файлов, каждый ≤350 строк). Vue-компоненты не содержат `<style scoped>`; стили инжектируются через `<style>{...}</style>` в соответствующих TSX-роутах.
- Связка webhook ↔ request_log через `correlationId` (`shared/correlation.ts`).
- Внутренняя авторизация панели: `requireRealUser` + `requireInternalAccess` (ADR 0003). Инвайт-ссылки (`lib/access/`).
- Глобальный фильтр по дате `panel_date_filter`; composable `shared/useLogsSocket.ts` для AdminPage.
- Тест-наборы разнесены по фазам: `lib/tests/lifepayUnitSuite.ts` + вспомогательные `Phase1..5`; `lib/tests/integrationSuite.ts` + разделы `Core/Api/E2e/DateFilter/Access`; `lib/tests/templateUnitSuite.ts` + helpers + routesChecks. Каталог тестов: `shared/testCatalog.ts` + `testCatalogUnit.ts` + `testCatalogIntegration.ts`.
- `lib/settings.lib.ts` + `lib/settings.mutations.ts` (геттеры/валидаторы/SETTING_KEYS отдельно от `setSetting`).
- Строгая типизация vue-tsc (strict + noUncheckedIndexedAccess): 0 ошибок. Ни одного файла >400 строк.
- Форматирование: Prettier чисто.

Результат подаётся в `unwrapWebhookBody` обёрткой `{data: "<json>"}`, дальше работает существующий парсер. Диагностический лог `body_received` расширен полями `hasFiles/filesKeys/filesDataShape/hasFields/fieldsKeys/fieldsDataShape/reqKeys` — чтобы по первому реальному webhook точно увидеть, в каком shape Chatium отдал multipart. 5 новых юнит-тестов `lp_webhook_multipart_*`.
