# sbp-client (LifePay SBP Client)

## Назначение

Клиентская панель LifePay (implementation-plan §1.8). Одно Chatium-приложение совмещает три роли: хранилище секретов магазина, серверная прокладка `POST /api/lp/invoke` между UI и payments-gateway с журналом запросов, приёмник входящего webhook LifePay.

## Важно

- Платформа: Chatium. Серверная часть управляется платформой.
- Стек фиксирован платформой, новые зависимости не добавляем.
- Деплой происходит автоматически после пуша.

- Heap-таблицы: `request_log` (журнал исходящих вызовов gateway), `webhook_log` (журнал webhook), `webhook_idempotency` (дедупликация по transaction number; гонки защищены `runWithExclusiveLock` из `@app/sync`).
- Серверная прокладка `POST /api/lp/invoke` (Admin-only) реализована: `lib/gateway/invokeClient.ts` делает один исходящий вызов `@app/request` к `<gateway_base_url>/api/v1/<op>` (префикс `/api/` соответствует file-based роутингу gateway) с заголовками `X-Lp-Apikey`, `X-Lp-Login`. Тело ответа gateway возвращается клиенту **без изменений**; `requestId` — из заголовка `X-Gateway-Request-Id`. Без серверных ретраев, без `Idempotency-Key`. Журнал — `lib/gateway/recordRequestLog.ts` с `argsRedacted` (без секретов, маска email/phone через `shared/redact.ts`).
- Приёмник webhook `POST /web/webhook?token=...` (`web/webhook/index.tsx`) — анонимный, с обязательной сверкой токена в query. MD5-подпись webhook **не** проверяется (LifePay её не публикует). Расхождение/отсутствие токена → 401/403 без тела и без записи. Тело читается каноническим `req.formData()` (Chatium native multipart с патча 18-05-2026) — LifePay шлёт `multipart/form-data` с полем `data` (JSON-строка); фоллбэки на `req.body` и сырое multipart-тело сохранены защитно. Запись в `webhook_log` + дедупликация по `number` через `webhook_idempotency`; бизнес-флаг `eligibleForOrderUpdate` (только `type=payment`+`status=success`) пишется в лог для будущего обновления заказа GetCourse (MVP §2.8).
- Главная страница панели `pages/PanelHomePage.vue` (доступна **по `/`**, `requireRealUser` + `requireInternalAccess`): вкладки «Обзор» (аналитика с глобальным фильтром по дате/времени — datetime-local + кнопка «Сброс», блок «Live» без фильтра), «Запросы» (журнал `request_log`), «Webhook» (журнал `webhook_log`), «Поиск» (по requestId с связанными webhook), «Создать счёт» (форма с полями orderNumber/amount/customerEmail/description/callbackUrl/customerPhone, QR через CDN qrcode.js, кнопка «Открыть в браузере»), **«Доступ»** (только Admin: управление инвайтами и грантами, ADR 0003). Вкладки «Настройки» нет — настройки LifePay (`lp_apikey`, `lp_login`, `lp_webhook_token`, `gateway_base_url`) редактируются на странице `/web/admin` (`pages/AdminPage.vue`), которая доступна только Admin; индикатор статуса конфигурации (полоска «Настройки выполнены успешно / Требуется настройка» + Base URL) оставлен на главной read-only. Старый роут `/web/panel` оставлен как редирект на `/` для совместимости закладок. В строках журналов запросов и webhook есть кнопка «raw» (`fa-code`) — открывает модалку с полным JSON (`rawResponseBody` или `rawBody`+`rawQuery`). Тулбар главной страницы — двухрядный sticky: строка A (`.toolbar-row--tabs`) содержит только навигационные вкладки на всю ширину; строка B (`.toolbar-row--tools`) содержит фильтр дат, LIVE-переключатель и поиск по requestId, отделена горизонтальным разделителем.
