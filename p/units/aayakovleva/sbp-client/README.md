# sbp-client (LifePay SBP Client)

## Назначение
Клиентская панель LifePay (implementation-plan §1.8). Одно Chatium-приложение совмещает три роли: хранилище секретов магазина, серверная прокладка `POST /api/lp/invoke` между UI и payments-gateway с журналом запросов, приёмник входящего webhook LifePay.

## Важно
- Платформа: Chatium. Серверная часть управляется платформой.
- Стек фиксирован платформой, новые зависимости не добавляем.
- Деплой происходит автоматически после пуша.

## Текущее состояние (LifePay panel)
- Каталог скопирован из `p/template_project`; переименован (`PROJECT_ROOT = 'p/units/aayakovleva/sbp-client'`, `.dir.json`, Heap-ключи settings/logs, `DEFAULT_PROJECT_TITLE = 'LifePay SBP Client'`). Файл `docs/run.md` удалён.
- Heap-настройки расширены ключами `lp_apikey`, `lp_login`, `lp_webhook_token`, `gateway_base_url` (с валидациями: `lp_login` — `^7\d{10}$`, `lp_webhook_token` ≥ 32 символа, `gateway_base_url` — `http(s)://...`).
- Heap-таблицы: `request_log` (журнал исходящих вызовов gateway), `webhook_log` (журнал webhook), `webhook_idempotency` (дедупликация по transaction number; гонки защищены `runWithExclusiveLock` из `@app/sync`).
- Серверная прокладка `POST /api/lp/invoke` (Admin-only) реализована: `lib/gateway/invokeClient.ts` делает один исходящий вызов `@app/request` к `<gateway_base_url>/api/v1/<op>` (префикс `/api/` соответствует file-based роутингу gateway) с заголовками `X-Lp-Apikey`, `X-Lp-Login`. Тело ответа gateway возвращается клиенту **без изменений**; `requestId` — из заголовка `X-Gateway-Request-Id`. Без серверных ретраев, без `Idempotency-Key`. Журнал — `lib/gateway/recordRequestLog.ts` с `argsRedacted` (без секретов, маска email/phone через `shared/redact.ts`).
- Приёмник webhook `POST /web/webhook?token=...` (`web/webhook/index.tsx`) — анонимный, с обязательной сверкой токена в query. MD5-подпись webhook **не** проверяется (LifePay её не публикует). Расхождение/отсутствие токена → 401/403 без тела и без записи. Запись в `webhook_log` + дедупликация по `number` через `webhook_idempotency`.
- Главная страница панели `pages/PanelHomePage.vue` (доступна **по `/`**, Admin-only): вкладки «Настройки», «Аналитика» (24ч), «Запросы» (журнал `request_log`), «Webhook» (журнал `webhook_log`), «Поиск» (по requestId с связанными webhook), «Создать счёт» (форма с полями orderNumber/amount/customerEmail/description/callbackUrl/customerPhone, QR через CDN qrcode.js, кнопка «Открыть в браузере»). Старый роут `/web/panel` оставлен как редирект на `/` для совместимости закладок. В строках журналов запросов и webhook есть кнопка «raw» (`fa-code`) — открывает модалку с полным JSON (`rawResponseBody` или `rawBody`+`rawQuery`).
- Сырые payload в журналах: `request_log.rawResponseBody` (полное тело ответа gateway), `webhook_log.rawBody` (полное тело webhook) и `webhook_log.rawQuery` (query параметры; token замаскирован) — записываются в виде JSON-объекта с PII-маской (`shared/redactRaw.redactRawDeep`: удаление apikey/login/token/authorization/cookie; маскирование email/phone/passport/inn/snils/fio/address/...; усечение > 64KB). Получение полной записи — через `GET /api/lp/raw-request?id=<heapId>|requestId=<rid>` и `GET /api/lp/raw-webhook?id=<heapId>` (Admin-only).
- Аналитические API: `GET /api/lp/recent-requests`, `/recent-webhooks`, `/analytics/summary`, `/search-by-request-id` (все Admin-only, без секретов и полных тел).
- Юнит-набор `lib/tests/lifepayUnitSuite.ts` — 31 проверка: каталог операций, сборка URL invoke (POST/GET, trailing slash, op_unknown, base_url_invalid), маскировка email/phone в argsRedacted, отсутствие apikey/login/token в argsRedacted, валидации настроек (login/url/токен), парсинг webhook (включая вложенный `order.number`), проверка токена (not_configured/missing/mismatch/valid). Подключён в `api/tests/unit` и `shared/testCatalog.ts` (блок `unit-lifepay`).
- Документация: обновлены `docs/architecture.md`, `docs/api.md`, `docs/data.md`.

## Changelog
- 2026-05-16: webhook-роут переведён на цепочечную форму `app.post('/').body(s => ({ data: s.string().optional() })).handle(...)` (`inner/docs/002-routing.md:400-427`). Диагностика прошлого webhook показала, что в простой форме `app.post('/', handler)` Chatium-платформа для `multipart/form-data` не парсит ни `req.body`, ни `req.files`/`req.fields` — `reqKeys` содержит только `body/method/path/query/headers/getSchema/params/url`. Цепочечная форма со схемой тела — попытка побудить платформу к разбору multipart-формы в `req.body.data`. На следующем webhook от LifePay по логу `body_received.bodyType` будет видно, помогло ли.
- 2026-05-16: добавлена поддержка `multipart/form-data` для webhook LifePay. По свежим серверным логам LifePay фактически шлёт `multipart/form-data; boundary=…` (хотя в собственной документации указан JSON), а Chatium-платформа в этом случае не заполняет `req.body` — текстовые поля попадают в `req.files`/`req.fields` (см. `inner/docs/009-files.md`). Добавлен `extractMultipartTextPayload(files, fields)` в `lib/webhook/processWebhook.ts`: ищет поле `data` (по контракту LifePay) среди `req.files`/`req.fields`, поддерживает значения как строку, как `{content/value/text/data/body/string: "<json>"}` или как Buffer-like `{type:"Buffer", data:[..bytes..]}`. Результат подаётся в `unwrapWebhookBody` обёрткой `{data: "<json>"}`, дальше работает существующий парсер. Диагностический лог `body_received` расширен полями `hasFiles/filesKeys/filesDataShape/hasFields/fieldsKeys/fieldsDataShape/reqKeys` — чтобы по первому реальному webhook точно увидеть, в каком shape Chatium отдал multipart. 5 новых юнит-тестов `lp_webhook_multipart_*`.
- 2026-05-16: исправлен парсинг тела webhook LifePay. По логам с реальным webhook (`webhook_payload_parsed` отдавал все пустые строки) выявлено, что LifePay шлёт payload завёрнутым в ключ `data` (apidoc.life-pay.ru/notification: «POST, JSON, data: {...}»). Добавлен `lib/webhook/processWebhook.unwrapWebhookBody`, разворачивающий обёртку для всех практических вариантов: `{"data":{...}}` JSON, `application/x-www-form-urlencoded` с `data=<urlencoded-json>`, чистый JSON-объект, JSON-строка, плоский form-encoded. Стратегия разворачивания пишется в лог `webhook_payload_parsed.unwrapStrategy`. Добавлен диагностический лог `body_received` с `bodyType/bodyKeys/contentType`. 6 новых юнит-тестов `lp_webhook_unwrap_*`. Информация о ретраях LifePay (1/3/5/10 мин, далее раз в час, до 10 попыток) и полный контракт payload зафиксированы в `docs/api.md` и в шапочном JSDoc приёмника.
