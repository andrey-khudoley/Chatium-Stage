# gw-client (Payments Client)

## Назначение

Универсальный клиентский слой к нескольким платёжным гейтвеям. Исходно — клиентская панель LifePay (implementation-plan §1.8), с 2026-05-28 расширен до многогейтвейной архитектуры. Одно Chatium-приложение совмещает: хранилище секретов магазинов (LifePay и Lava.Top), серверную прокладку `POST /api/lp/invoke` (исторический URL) между UI/JS и нужным payments-gateway с журналом запросов, приёмники входящих webhook от обоих гейтвеев.

## Поддерживаемые гейтвеи

- **LifePay** — серверный gateway `p/saas/gw/lifepay` (контур `bills_v1`). Заголовки авторизации: `X-Lp-Apikey`, `X-Lp-Login`. Операции: `createBill`, `getBillStatus`, `cancelBill`. Webhook: `/web/webhook` (multipart, токен в query).
- **Lava.Top** — серверный gateway `p/saas/gw/lavatop` (контур `invoices_v1`). Заголовок авторизации: `X-Lava-Apikey`. Операции: `createInvoice`, `getInvoiceStatus`, `listProducts`, `updateOfferPrice`. Webhook: `/web/webhook-lavatop` (JSON, секрет в `X-Api-Key` / Basic).
- **GetCourse (GC)** — серверный gateway `p/saas/gw/gc`. Заголовки авторизации: `X-Gc-School-Api-Key`, `X-Gc-School-Host`. Каталог операций динамический: список enabled-операций загружается с `GET /api/v1/operations` GC-гейтвея при SSR; изменение набора операций на стороне `p/saas/gw/gc` не требует правок клиента. Для операций GC в теле `POST /api/lp/invoke` обязателен `httpMethod` (`'GET'` или `'POST'`).

## Важно

- Платформа: Chatium. Серверная часть управляется платформой.
- Стек фиксирован платформой, новые зависимости не добавляем.
- Деплой происходит автоматически после пуша.

## Текущее состояние

- Heap-настройки LifePay: `lp_apikey`, `lp_login`, `lp_webhook_token`, `gateway_base_url` (с валидациями).
- Heap-настройки Lava.Top: `lava_test_apikey`, `lava_base_url` (default `https://gate.lava.top`), `lava_webhook_secret` (≥ 16 символов).
- Heap-настройки GetCourse: `gc_base_url`, `gc_test_school_api_key`, `gc_test_school_host`, `gc_enabled` (с валидациями; `gc_test_school_api_key` маскируется в exit-логах).
- Все настройки редактируются на `/web/admin`.
- Heap-таблицы: `request_log` (с опциональным полем `gatewayId`), `webhook_log` (с опциональными `gatewayId`, `gatewayExternalId`), `webhook_idempotency`, `panel_access`, `panel_invites`, `settings`, `logs`. Ключи таблиц сохранены исторически (`t__lifepay-sbp-client__*`) — переименование сломает данные; технический долг, не блокирующий.
- Серверная прокладка `POST /api/lp/invoke`: тело `{ gatewayId: 'lifepay' | 'lavatop' | 'gc', op, args, httpMethod? }`. **`gatewayId` обязательно**. Для GC дополнительно обязателен `httpMethod` (`'GET'` или `'POST'`). Диспатчер по `gatewayId` (`lib/gateway/invokeDispatcher.ts`) выбирает клиент: LifePay (`lib/gateway/invokeClient.ts`), Lava.Top (`lib/gateway/lavatopClient.ts`) или GC (`lib/gateway/gcClient.ts`). Один вызов через `@app/request` к соответствующему gateway, без ретраев; журнал в `request_log` с `gatewayId`.
- Приёмники webhook:
  - `POST /web/webhook?token=...` — LifePay; анонимный + токен в query; `req.formData()` + фоллбэки; дедупликация через `runWithExclusiveLock` по `number`; запись в `webhook_log` с `gatewayId='lifepay'`.
  - `POST /web/webhook-lavatop` — Lava.Top; анонимный + секрет в `X-Api-Key` / Basic Authorization (значение `lava_webhook_secret`); JSON-тело `PurchaseWebhookLog` (`contractId`, `eventType`, `status`); дедупликация по композитному ключу `lavatop:contractId:eventType:status`; запись в `webhook_log` с `gatewayId='lavatop'` и `gatewayExternalId=contractId`. GET на тот же URL возвращает статус доступности (без раскрытия секрета).
- Главная страница `/` — `pages/HomePage.vue` (компонент `ClientHomePage`), доступ — сотрудник или Admin (`requireRealUser` + `requireInternalAccess`). Оркестратор: вкладки «Обзор», «Создать запрос», «Формат запросов», «Запросы», «Webhook», «Настройки» (operational/business — `HomeSettingsTab.vue` + `HomeWidgetSettings.vue` + `HomeWidgetOfferList.vue`; доступна сотрудникам), «Доступ» (Admin-only). Вкладка «Создать запрос» (`components/home/HomeCreateRequestTab.vue`) универсальная: дропдаун операций сгруппирован по гейтвеям (LifePay / Lava.Top / GetCourse), форма перестраивается динамически из каталога `shared/operationsClientCatalog.ts` (хелперы — `shared/operationsClientForm.ts`); для GC-операций форма строится иерархически из `argsTree` (вложенные fieldset с отступом), вычисляется в `pages/sbpHomePageMixin.ts` как computed `gcFormRows` (`shared/gcArgsForm.ts:buildFormRows`) — единственный источник истины для иерархии полей; список GC-операций с `argsTree` загружается SSR-функцией `fetchGcOperations(ctx)` и передаётся пропом `gcOperations` в `ClientHomePage`; схема полей прозрачно наследуется от `p/saas/gw/gc` без правок клиента; QR рендерится только если у операции есть `paymentUrl` в ответе. Вкладка «Формат запросов» (`components/home/HomeRequestFormatTab.vue`) — декларативный рендер контрактов и атомарных сниппетов для магазинного JS (LifePay createBill, Lava.Top createInvoice, подписка на socket уведомлений об оплате), источник истины — `shared/requestFormatSamples.ts`. На вкладке «Обзор» в превью «Последние запросы» / «Последние webhook» добавлена колонка «GW» с бейджем гейтвея. Реализована как оркестратор с подкомпонентами в `components/home/` и mixin-логикой в `pages/sbpHomePageMixin.ts`.
- Страница `/web/admin` — системные настройки, **только Admin** (`requireAccountRole('Admin')`). Оркестратор `pages/AdminPage.vue` с подкомпонентами в `components/admin/`: счётчики, project_name, log_level, карточки LifePay/Lava.Top/GC (только секреты, URL, host). Operational-настройки (gc_enabled, виджеты) сюда не попадают — они на вкладке «Настройки» главной.
- **Публичные виджеты оплаты** (`userscripts/`): `lifepay-widget.user.js` (QR-модалка по клику) и `lavatop-widget.user.js` (редирект на форму). Общие утилиты — `userscripts/common.js` (namespace `window.GwWidgetCommon`). CORS-стратегия: запросы в `text/plain` без preflight (Chatium не поддерживает `app.options`); сервер парсит тело вручную.
- **API виджетов** (`api/widgets/`): 6 эндпоинтов — `GET /api/widgets/config` (публичный, CORS-whitelist), `POST /api/widgets/intent-lifepay`, `POST /api/widgets/intent-lavatop` (публичные, серверный hard-limit 500 000 ₽), `GET /api/widgets/settings-get`, `POST /api/widgets/settings-save`, `GET /api/widgets/offers` (`// @shared-route`, доступ — сотрудник или Admin через `guardInternalApi`; виджет-настройки/офферы — operational, не секреты). Маршруты зарегистрированы в `config/routes.tsx` (6 записей).
- **API operational-настроек**: `POST /api/settings/save-operational` (`// @shared-route`, `guardInternalApi`, whitelist `gc_enabled`) — для тоггла GC на вкладке «Настройки» главной.
- **Настройки виджетов** (8 новых ключей в `lib/settings.lib.ts` и `lib/settings.mutations.ts`): `widget_lifepay_enabled`, `widget_lifepay_domains`, `widget_lifepay_max_amount`, `widget_lavatop_enabled`, `widget_lavatop_domains`, `widget_lavatop_max_amount`, `widget_offer_list_type`, `widget_offer_ids`. Типы — `shared/widgetSettingsTypes.ts`; CORS-логика — `shared/widgetCorsCheck.ts`; агрегатор для SSR — `lib/widget/widgetSettings.lib.ts`.
- Hard-limit намерений (intent) — 500 000 ₽ (`WIDGET_INTENT_HARD_LIMIT_RUB`, `shared/widgetSettingsTypes.ts`); применяется на сервере до пользовательского лимита. Audit-лог каждого intent-вызова — через `loggerLib.writeServerLog`.
- Страница `/web/tests` (`pages/TestsPage.vue`) — оркестратор с подкомпонентами в `components/tests/` (3 файла). Composable `shared/useTestSuites.ts`.
- CSS вынесен в `pagecss/` (18 файлов, каждый ≤350 строк). Vue-компоненты не содержат `<style scoped>`; стили инжектируются через `<style>{...}</style>` в соответствующих TSX-роутах.
- Связка webhook ↔ request_log через `correlationId` (`shared/correlation.ts`).
- Внутренняя авторизация панели: `requireRealUser` + `requireInternalAccess` (ADR 0003). Инвайт-ссылки (`lib/access/`).
- Глобальный фильтр по дате `panel_date_filter`; composable `shared/useLogsSocket.ts` для AdminPage.
- Тест-наборы разнесены по фазам: `lib/tests/lifepayUnitSuite.ts` + вспомогательные `Phase1..5`; `lib/tests/integrationSuite.ts` + разделы `Core/Api/E2e/DateFilter/Access`; `lib/tests/templateUnitSuite.ts` + helpers + routesChecks. Каталог тестов: `shared/testCatalog.ts` + `testCatalogUnit.ts` + `testCatalogIntegration.ts`.
- `lib/settings.lib.ts` + `lib/settings.mutations.ts` (геттеры/валидаторы/SETTING_KEYS отдельно от `setSetting`).
- Строгая типизация vue-tsc (strict + noUncheckedIndexedAccess): 0 ошибок. Ни одного файла >400 строк.
- Форматирование: Prettier чисто.
