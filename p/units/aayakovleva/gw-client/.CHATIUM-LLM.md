# Проект gw-client (Payments Client)

## Что это

Универсальный клиент платёжных гейтвеев (с 2026-05-28 — многогейтвейная архитектура; ранее — `sbp-client`, моно-клиент LifePay по implementation-plan §1.8). Одно Chatium-приложение совмещает: хранилище секретов магазинов LifePay и Lava.Top, серверную прокладку `POST /api/lp/invoke` (исторический URL) к выбранному upstream gateway, журналы в Heap, приёмники webhook от обоих гейтвеев (`/web/webhook` — LifePay, `/web/webhook-lavatop` — Lava.Top). Upstream-гейтвеи: `p/saas/gw/lifepay` (контур `bills_v1`) и `p/saas/gw/lavatop` (контур `invoices_v1`).

## Ключевые инварианты (для LLM)

- **Многогейтвейность.** Тело `POST /api/lp/invoke` обязано содержать `gatewayId: 'lifepay' | 'lavatop'`. Без него — 400 `INVOKE_GATEWAY_REQUIRED`. Неизвестное значение — 400 `INVOKE_GATEWAY_UNKNOWN`. Каталог операций разнесён по гейтвеям (`shared/gatewayContract.ts` → `LIFEPAY_OPERATIONS` / `LAVATOP_OPERATIONS`); операция, отсутствующая в каталоге указанного гейтвея — 400 `INVOKE_OP_UNKNOWN`. Внутренние JS-вызовы (`pages/sbpHomePageMixin.ts → submitRequest`) передают `gatewayId`, выбранный пользователем во вкладке «Создать запрос». Внешний JS-код, ходящий к клиенту, должен быть мигрирован вручную.
- **UI-каталог операций.** Декларативное описание операций в терминах формы (label, type, required, hint, defaults) — в `shared/operationsClientCatalog.ts` (`OPERATIONS_CLIENT_CATALOG`). Хелперы группировки/валидации/сборки `args` — в `shared/operationsClientForm.ts` (ре-экспортируются из `operationsClientCatalog` для удобства импорта). При добавлении новой операции upstream-гейтвея — синхронно править этот каталог, иначе форма не сможет её отправить.
- **Диспатчер.** Точка входа диспатча — `lib/gateway/invokeDispatcher.ts → invokeByGateway(ctx, gatewayId, op, args)`. Выбирает клиент: `invokeLifepayGateway` или `invokeLavatopGateway`. Общие хелперы — в `lib/gateway/invokeResult.ts`.
- **Главная `/` — это панель оператора** (`pages/HomePage.vue`, компонент `ClientHomePage`), защищена `requireRealUser(ctx)` + `requireInternalAccess(ctx)` в `index.tsx`. Аноним → редирект на `/s/auth/signin`; авторизованный без гранта → `/web/forbidden` (403); Admin или активный грант → доступ. Старый `/web/panel` редиректит на `/` (легаси-совместимость).
- `guardInternalApi(ctx)` (= requireRealUser + requireInternalAccess) первой строкой всех `/api/lp/*`. Admin Chatium-аккаунта проходит `requireInternalAccess` всегда; не-Admin — только при активной записи в `panel_access` (ADR 0003, реализовано 2026-05-24).
- Доступ для не-Admin выдаётся через одноразовые пригласительные ссылки: `api/access/generate-invite` → `/web/access/invite?token=` → `api/access/consume-invite` (под `runWithExclusiveLock`). Переход по URL не расходует инвайт.
- Новые каталоги (ADR 0003): `lib/access/` (requireInternalAccess, invites, constants, apiGuard), `api/access/` (6 эндпоинтов), `web/access/invite/`, `web/forbidden/`, `shared/accessPages.tsx`, `pages/InviteAcceptPage.vue`.
- В журналах LifePay-операций (`request_log.argsRedacted`/`rawResponseBody`, `webhook_log.rawBody`/`rawQuery`/`email`) хранятся **сырые** payload — клиент является оператором персональных данных по 152-ФЗ. Структурная гигиена (циклы/несериализуемое/усечение > 64KB) — через `shared/prepareRawLog.prepareRawLog`. Маскирующие утилиты `shared/redact.ts` и `shared/redactRaw.ts` оставлены в коде с тестами, но в production-пути не используются.
- Webhook `/web/webhook` — анонимный, единственная аутентификация — токен в query. MD5-подпись webhook **не** проверяется (LifePay её не публикует).
- Серверные ретраи в прокладке **запрещены**: один входящий → один исходящий вызов к gateway.
- `requestId` берётся из заголовка `X-Gateway-Request-Id` ответа gateway, не генерируется заново.
- Секреты (`lp_apikey`, `lp_login`, `lp_webhook_token`) **не** попадают в `argsRedacted`, в `writeServerLog.payload`, в ответы аналитики и в журналы.
- В Vue (`.vue`) **запрещены** импорты `tables/`, `repos/`, `lib/` — данные только через SSR-пропсы и fetch к собственным API.
- Heap-схема не выражает unique constraint; дедупликация webhook — через `runWithExclusiveLock` из `@app/sync`.
- QR-код во вкладке «Создать запрос» — рендерится только если у выбранной операции в каталоге задан `paymentUrlPath` и в ответе по этому пути есть строка-URL (`LifePay.createBill`, `Lava.Top.createInvoice`). Используется CDN `cdn.jsdelivr.net/npm/qrcode/...`; при недоступности — текст paymentUrl.

## Логирование

Без изменений из шаблона. Сервер: `lib/logger.lib.ts` → syslog RFC 5424. Клиент: `shared/logger`, `browserRemoteLogger` (пакетная отправка на POST /api/logger/browser). Уровень `log_level` (Debug/Info/Warn/Error/Disable). Секреты автоматически не попадают (`shouldIncludePayload` при level != Debug). PII в LifePay-журналах **не маскируется** — клиент-оператор, см. инвариант выше.

## Критические правила

- Не хардкодить URL: использовать `config/routes.tsx` (`PROJECT_ROOT = 'p/units/aayakovleva/gw-client'`, `withProjectRoot`, `withProjectRootAndSubroute`).
- В Vue (`*.vue`) запрещены импорты `tables/`, `repos/`, `lib/`, `pagecss/` — только `shared/*` с `// @shared`.
- CSS-файлы из `pagecss/` импортируются только в TSX-роутах через `<style>{...}</style>`.
- Документацию обновлять при изменениях.
- Перед коммитом — `/check`.
