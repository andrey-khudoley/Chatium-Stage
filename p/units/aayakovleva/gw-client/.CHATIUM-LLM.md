# Проект gw-client (Payments Client)

## Что это

Универсальный клиент платёжных гейтвеев (многогейтвейная архитектура с 2026-05-28; ранее — `sbp-client`, моно-клиент LifePay). Одно Chatium-приложение совмещает: хранилище секретов магазинов LifePay, Lava.Top и GetCourse, серверную прокладку `POST /api/lp/invoke` (исторический URL) к выбранному upstream gateway, журналы в Heap, приёмники webhook от LifePay и Lava.Top. Upstream-гейтвеи: `p/saas/gw/lifepay` (контур `bills_v1`), `p/saas/gw/lavatop` (контур `invoices_v1`), `p/saas/gw/gc` (динамический каталог операций).

## Ключевые инварианты (для LLM)

- **Многогейтвейность.** Тело `POST /api/lp/invoke` обязано содержать `gatewayId: 'lifepay' | 'lavatop' | 'gc'`. Без него — 400 `INVOKE_GATEWAY_REQUIRED`. Неизвестное значение — 400 `INVOKE_GATEWAY_UNKNOWN`. Для `gatewayId: 'gc'` дополнительно обязателен `httpMethod` (`'GET'` или `'POST'`); имя операции проходит `validateGcOpName` (SSRF-защита). Каталог LifePay/Lava.Top — статический (`shared/gatewayContract.ts`); каталог GC — динамический (`OPERATIONS_BY_GATEWAY.gc = []`, заполняется SSR-загрузчиком `lib/gateway/gcOperationsLoader.ts`). Операция, отсутствующая в каталоге — 400 `INVOKE_OP_UNKNOWN`.
- **UI-каталог операций.** Декларативное описание статических операций LifePay/Lava.Top в терминах формы — в `shared/operationsClientCatalog.ts` (`OPERATIONS_CLIENT_CATALOG`). GC-операции — динамические: тип `GcOperationEntry` с полем `argsTree?: ArgsTreeNode`; `argsTree` парсится из `GET /v1/operations` GC-гейтвея в `lib/gateway/gcOperationsLoader.ts`. Хелперы группировки/валидации/сборки `args` — в `shared/operationsClientForm.ts`; `groupOperationsForUi(gcOperations?)` принимает опциональный список GC-операций. При добавлении новой операции LifePay/Lava.Top — синхронно править каталог; для GC достаточно включить операцию на стороне `p/saas/gw/gc`.
- **Иерархическая форма GC.** Типы и хелперы (`buildFormRows`, `buildArgsObject`, `buildFieldErrors`, `jsonPlaceholder`, `FormRow`, `FormGroup`, `FormLeaf`, `LeafInput`) — кросс-проектный реэкспорт через `shared/gcArgsForm.ts` из `p/saas/gw/gc/shared/requestTestForm.ts`. Computed-блок и `prepareGcSubmit` — в `pages/sbpHomePageMixin.ts`; рендер иерархии — в `components/home/HomeGcRequestForm.vue`. Изменение схемы полей на стороне GC-гейтвея не требует правок клиента.
- **Диспатчер.** Точка входа диспатча — `lib/gateway/invokeDispatcher.ts → invokeByGateway(ctx, gatewayId, op, args, meta?)`. Выбирает клиент: `invokeLifepayGateway`, `invokeLavatopGateway` или `invokeGcGateway`. Для GC `meta.httpMethod` обязателен — диспетчер проверяет явно. Общие хелперы — в `lib/gateway/invokeResult.ts`.
- **Главная `/` — это панель оператора** (`pages/HomePage.vue`, компонент `ClientHomePage`), защищена `requireRealUser(ctx)` + `requireInternalAccess(ctx)` в `index.tsx`. Аноним → редирект на `/s/auth/signin`; авторизованный без гранта → `/web/forbidden` (403); Admin или активный грант → доступ. Старый `/web/panel` редиректит на `/` (легаси-совместимость).
- `guardInternalApi(ctx)` (= requireRealUser + requireInternalAccess) первой строкой всех `/api/lp/*`. Admin Chatium-аккаунта проходит `requireInternalAccess` всегда; не-Admin — только при активной записи в `panel_access` (ADR 0003, реализовано 2026-05-24).
- Доступ для не-Admin выдаётся через одноразовые пригласительные ссылки: `api/access/generate-invite` → `/web/access/invite?token=` → `api/access/consume-invite` (под `runWithExclusiveLock`). Переход по URL не расходует инвайт.
- Новые каталоги (ADR 0003): `lib/access/` (requireInternalAccess, invites, constants, apiGuard), `api/access/` (6 эндпоинтов), `web/access/invite/`, `web/forbidden/`, `shared/accessPages.tsx`, `pages/InviteAcceptPage.vue`.
- В журналах LifePay-операций (`request_log.argsRedacted`/`rawResponseBody`, `webhook_log.rawBody`/`rawQuery`/`email`) хранятся **сырые** payload — клиент является оператором персональных данных по 152-ФЗ. Структурная гигиена (циклы/несериализуемое/усечение > 64KB) — через `shared/prepareRawLog.prepareRawLog`. Маскирующие утилиты `shared/redact.ts` и `shared/redactRaw.ts` оставлены в коде с тестами, но в production-пути не используются.
- Webhook `/web/webhook` — анонимный, единственная аутентификация — токен в query. MD5-подпись webhook **не** проверяется (LifePay её не публикует).
- Серверные ретраи в прокладке **запрещены**: один входящий → один исходящий вызов к gateway.
- `requestId` берётся из заголовка `X-Gateway-Request-Id` ответа gateway, не генерируется заново.
- Секреты (`lp_apikey`, `lp_login`, `lp_webhook_token`) **не** попадают в `argsRedacted`, в `writeServerLog.payload`, в ответы аналитики и в журналы.
- В Vue (`.vue`) **запрещены** импорты `tables/`, `repos/`, `lib/` — данные только через SSR-пропсы и fetch к собственным API.
- Heap-схема не выражает unique constraint; дедупликация webhook — через `runWithExclusiveLock` из `@app/sync`.
- QR-код во вкладке «Создать запрос» — рендерится только если у выбранной операции в каталоге задан `paymentUrlPath` и в ответе по этому пути есть строка-URL (`LifePay.createBill`, `Lava.Top.createInvoice`). Используется CDN `cdnjs.cloudflare.com/ajax/libs/qrcode/1.5.1/qrcode.min.js` (UMD-сборка, выставляет `window.QRCode.toCanvas`); при недоступности — текст paymentUrl.

## Логирование

Без изменений из шаблона. Сервер: `lib/logger.lib.ts` → syslog RFC 5424. Клиент: `shared/logger`, `browserRemoteLogger` (пакетная отправка на POST /api/logger/browser). Уровень `log_level` (Debug/Info/Warn/Error/Disable). Секреты автоматически не попадают (`shouldIncludePayload` при level != Debug). PII в LifePay-журналах **не маскируется** — клиент-оператор, см. инвариант выше.

## Настройки lib

- `lib/settings.lib.ts` — геттеры, валидаторы, SETTING_KEYS, DEFAULTS, типы (`DateFilter` и т.д.). Включает ключи и геттеры GC: `gc_base_url`, `gc_test_school_api_key`, `gc_test_school_host`, `gc_enabled`, валидатор `isValidGcSchoolHost`.
- `lib/settings.mutations.ts` — `setSetting` (запись/сброс); re-export из `settings.lib.ts` сохраняет обратную совместимость импортов. Маскирует `gc_test_school_api_key` в exit-логе (наряду с `lava_test_apikey`, `lava_webhook_secret`).

## Критические правила

- Не хардкодить URL: использовать `config/routes.tsx` (`PROJECT_ROOT = 'p/units/aayakovleva/gw-client'`, `withProjectRoot`, `withProjectRootAndSubroute`).
- В Vue (`*.vue`) запрещены импорты `tables/`, `repos/`, `lib/`, `pagecss/` — только `shared/*` с `// @shared`.
- CSS-файлы из `pagecss/` импортируются только в TSX-роутах через `<style>{...}</style>`.
- Документацию обновлять при изменениях.
- Перед коммитом — `/check`.
