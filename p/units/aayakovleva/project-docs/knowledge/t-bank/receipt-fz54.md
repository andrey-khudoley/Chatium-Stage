# Т-Банк Эквайринг — Чек ФЗ-54 (объект Receipt)

## Когда нужен Receipt

Объект `Receipt` обязателен, если у терминала подключена онлайн-касса (облачная или собственная). Без него транзакция может проходить, но фискальный чек не будет выбит — нарушение ФЗ-54.

Если онлайн-касса **не подключена** к терминалу — `Receipt` передавать не нужно.

---

## Структура объекта Receipt

Передаётся в запросе `Init` (и в `Confirm`, `Cancel` для частичных операций).

> ⚠️ `Receipt` **не входит** в расчёт токена SHA-256.

```json
"Receipt": {
  "FfdVersion": "1.2",
  "Email": "buyer@example.com",
  "Phone": "+79001234567",
  "Taxation": "usn_income",
  "Items": [
    {
      "Name": "Онлайн-курс «Маркетинг с нуля»",
      "Price": 149900,
      "Quantity": 1.000,
      "Amount": 149900,
      "Tax": "none",
      "PaymentMethod": "full_payment",
      "PaymentObject": "service"
    }
  ]
}
```

### Поля верхнего уровня

| Поле         | Тип    | Описание                                                                  |
| ------------ | ------ | ------------------------------------------------------------------------- |
| `FfdVersion` | string | Версия ФФД: `"1.05"` или `"1.2"` (уточнить у оператора кассы)             |
| `Email`      | string | Email покупателя для отправки чека (или Phone — одно из двух обязательно) |
| `Phone`      | string | Телефон покупателя (формат `+7...`)                                       |
| `Taxation`   | string | Система налогообложения (см. таблицу ниже)                                |
| `Items`      | array  | Массив позиций чека (хотя бы одна)                                        |

### Значения поля `Taxation`

| Значение             | Система налогообложения    |
| -------------------- | -------------------------- |
| `osn`                | Общая система              |
| `usn_income`         | УСН «Доходы»               |
| `usn_income_outcome` | УСН «Доходы минус расходы» |
| `patent`             | Патентная                  |
| `envd`               | ЕНВД (устаревает)          |
| `esn`                | ЕСХН                       |

---

### Структура объекта Items[]

| Поле            | Тип     | Обяз. | Описание                                         |
| --------------- | ------- | ----- | ------------------------------------------------ |
| `Name`          | string  | ✅    | Наименование товара/услуги (до 128 символов)     |
| `Price`         | integer | ✅    | Цена за единицу в **копейках**                   |
| `Quantity`      | decimal | ✅    | Количество (например, `1.000`)                   |
| `Amount`        | integer | ✅    | Итоговая сумма = Price × Quantity в **копейках** |
| `Tax`           | string  | ✅    | Ставка НДС (см. таблицу ниже)                    |
| `PaymentMethod` | string  | ❌\*  | Признак способа расчёта (см. таблицу)            |
| `PaymentObject` | string  | ❌\*  | Признак предмета расчёта (см. таблицу)           |
| `Ean13`         | string  | ❌    | Штрих-код товара                                 |

\*Обязательны при ФФД 1.2.

### Значения поля `Tax`

| Значение | НДС                     |
| -------- | ----------------------- |
| `none`   | Без НДС                 |
| `vat0`   | 0%                      |
| `vat10`  | 10%                     |
| `vat20`  | 20%                     |
| `vat110` | Расчётная ставка 10/110 |
| `vat120` | Расчётная ставка 20/120 |

### Значения поля `PaymentMethod`

| Значение          | Описание                                     |
| ----------------- | -------------------------------------------- |
| `full_prepayment` | Полная предоплата                            |
| `prepayment`      | Частичная предоплата                         |
| `advance`         | Аванс                                        |
| `full_payment`    | Полный расчёт (стандарт для оказанных услуг) |
| `partial_payment` | Частичный расчёт и кредит                    |
| `credit`          | Передача в кредит                            |
| `credit_payment`  | Оплата кредита                               |

Для онлайн-курсов обычно используется `full_payment` или `full_prepayment`.

### Значения поля `PaymentObject`

| Значение                | Описание                                 |
| ----------------------- | ---------------------------------------- |
| `commodity`             | Товар                                    |
| `excise`                | Подакцизный товар                        |
| `job`                   | Работа                                   |
| `service`               | **Услуга** (для онлайн-курсов, подписок) |
| `gambling_bet`          | Ставка азартной игры                     |
| `gambling_prize`        | Выигрыш                                  |
| `lottery`               | Лотерейный билет                         |
| `lottery_prize`         | Выигрыш лотереи                          |
| `intellectual_activity` | Результат интеллектуальной деятельности  |
| `payment`               | Платёж (аванс, задаток и т.д.)           |
| `agent_commission`      | Агентское вознаграждение                 |
| `composite`             | Составной предмет расчёта                |
| `another`               | Иное                                     |

## Типичный пример для онлайн-школы

```json
"Receipt": {
  "FfdVersion": "1.2",
  "Email": "student@example.com",
  "Taxation": "usn_income",
  "Items": [
    {
      "Name": "Доступ к онлайн-курсу «Название курса»",
      "Price": 500000,
      "Quantity": 1.000,
      "Amount": 500000,
      "Tax": "none",
      "PaymentMethod": "full_payment",
      "PaymentObject": "service"
    }
  ]
}
```

> **Важно:** `Amount` в `Items` должна совпадать с `Amount` в запросе `Init`. Если несколько позиций — сумма `Items[].Amount` должна равняться общей сумме платежа.

---

## Чек при возврате (Cancel)

При вызове метода `Cancel` тоже можно передать `Receipt` — тогда будет выбит чек возврата. Если `Receipt` не передаётся при Cancel — чек возврата не выбивается (нарушение ФЗ-54 при полном возврате).

---

## ФФД 1.05 vs ФФД 1.2

|                   | ФФД 1.05                   | ФФД 1.2                    |
| ----------------- | -------------------------- | -------------------------- |
| `FfdVersion`      | `"1.05"` или не передавать | `"1.2"` — обязателен       |
| `PaymentMethod`   | Необязателен               | Обязателен                 |
| `PaymentObject`   | Необязателен               | Обязателен                 |
| Когда применяется | Старые кассы               | Актуальный стандарт с 2023 |
