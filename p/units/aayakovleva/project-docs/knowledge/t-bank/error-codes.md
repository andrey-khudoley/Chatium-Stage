# Т-Банк Эквайринг — Коды ошибок

Поле `ErrorCode` присутствует в каждом ответе API. `"0"` означает успех.

## Общие коды

| Код    | Описание                                |
| ------ | --------------------------------------- |
| `0`    | Успех                                   |
| `99`   | Транзакция отклонена (общая ошибка)     |
| `100`  | Неверный параметр запроса               |
| `191`  | Ошибка токена (неверная подпись)        |
| `194`  | Неверный статус для выполнения операции |
| `1001` | Отказ эмитента без пояснений            |
| `1002` | Заблокированная карта                   |
| `1003` | Украденная карта                        |
| `1004` | Отказ эмитента — общий                  |
| `1005` | Карта не найдена                        |
| `1006` | Недостаточно средств                    |
| `1007` | Превышен лимит операций                 |
| `1008` | Срок карты истёк                        |
| `1009` | Неверный CVV                            |
| `1010` | Превышен дневной лимит                  |
| `1012` | Операция недопустима для данной карты   |
| `1013` | Технический сбой у эмитента             |
| `1015` | Неверный PIN (для POS)                  |
| `1030` | Ошибка 3-D Secure                       |
| `1051` | Недостаточно средств                    |
| `9999` | Неизвестная ошибка                      |

> ⚠️ Полный актуальный список — только в официальной документации: [Коды ошибок EACQ](https://www.tinkoff.ru/kassa/develop/api/errorslist/)

## Как интерпретировать ошибку

1. `Success: false` + `ErrorCode: "0"` — не бывает (если Success=false, ErrorCode ≠ 0).
2. Поле `Message` содержит человекочитаемое сообщение об ошибке.
3. Поле `Details` — дополнительные технические детали.
