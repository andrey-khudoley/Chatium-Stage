# Chatium

Chatium - low-code платформа для быстрой разработки веб-сервисов, ботов, мобильных приложений, лендингов, личных кабинетов и внутренних инструментов. Код выполняется в облаке платформы: разработчик пишет файлы в аккаунте Chatium, а изменения становятся доступны без отдельной настройки хостинга и ручного деплоя.

## Смысл платформы

Главная идея Chatium - дать возможность быстро собрать рабочий продукт вокруг пользовательских сценариев: страница, API, база данных, авторизация, файлы, уведомления, платежи, интеграции и мобильный доступ живут в одной среде.

Платформа особенно полезна, когда нужно не просто сверстать страницу, а быстро сделать прикладной сервис: кабинет ученика, мини-приложение, внутреннюю админку, интеграционный слой, бота, аналитику или интерфейс поверх внешних API.

## Основные возможности

- Веб-страницы и API-роуты на TypeScript.
- Клиентский интерфейс на Vue, HTML, CSS и JavaScript.
- File-based routing: структура файлов определяет адреса страниц и API.
- Авторизация пользователей и проверка ролей.
- Внутренняя база данных Heap для хранения прикладных данных.
- Работа с файлами: изображения, видео, документы и статические материалы.
- Отложенные задачи и фоновые jobs.
- Вызовы внешних API через HTTP-клиент.
- WebSocket и real-time обновления.
- Уведомления, мессенджеры и фиды сообщений.
- Платежи, подписки и in-app purchases.
- Аналитика workspace, трафика и событий.
- AI-агенты и инструменты для них.
- Интеграции с GetCourse, AmoCRM, Notion, Toggl, Bizon365 и другими сервисами.

## Как думать о Chatium

Chatium можно воспринимать как облачную среду разработки и исполнения приложений: в одном аккаунте лежит код, данные, роуты, интерфейсы, фоновые задачи и интеграции. Это не просто конструктор страниц, а платформа для небольших и средних продуктовых систем, где важна скорость сборки и тесная связь с бизнес-процессами.

## Важные ограничения

- Серверная инфраструктура управляется платформой, ее нельзя настраивать как обычный VPS.
- Стек и доступные библиотеки ограничены средой Chatium.
- Секреты, токены и приватные ключи нельзя хранить в обычных markdown-заметках или публичном коде.
- Для проектов важно соблюдать соглашения платформы: относительные URL, file-based routing, разделение клиентского и серверного кода.

## Экосистема и инструменты платформы

### Ботан

**Ботан** — встроенный AI-ассистент Chatium, который пишет код прямо в среде платформы. Помогает создавать и редактировать TypeScript/Vue-файлы в аккаунте Chatium без ручного написания кода. Когда в проектах упоминается «инструкции для Ботана» или «дать ТЗ Ботану» — имеется в виду именно этот инструмент, а не человек.
