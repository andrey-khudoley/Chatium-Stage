# Decisions. Реестр ADR проекта

Реестр архитектурных решений по проекту «Платёжная страница GetCourse для школы Яковлевых» в формате lightweight ADR. Каждое решение фиксируется в момент принятия и не переписывается ретроспективно: при изменении создаётся новый ADR со ссылкой на предыдущий.

> **Где лежат тела ADR.** При переносе документации в воркспэйс проектные ADR размещены рядом с кодом тех модулей, к которым относятся (а не в этой папке). Этот файл — сводный реестр-указатель. Исходная нумерация ваулта (проектная серия 0001/0003) сохранена в заголовках файлов; в модульных сериях `docs/ADR/` они получили следующий свободный номер 0003.

## Шаблон

Каждый ADR оформляется по структуре:

- **Контекст** — что произошло, почему нужно решать.
- **Варианты** — минимум два, с явными pros/cons.
- **Решение** — выбранный вариант или статус «открыто».
- **Последствия** — что меняется в архитектуре, в коде, в коммерческих условиях.
- **Связанные документы** — ссылки на проектную и сервисную документацию.
