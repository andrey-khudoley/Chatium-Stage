# Ольга Шавковская — платежная страница GetCourse

## Статус

**Активен, стадия реализации на Chatium (без переезда на VDS).** Скоуп согласован, цена (60 тыс.) подтверждена. Дедлайн перед клиентом — **2026-05-25** (понедельник), но реальный объём работы резко сократился, цель — успеть раньше.

**Снят блокер с multipart.** 18.05 разработчики Chatium (Artur Eshenbrener) выкатили патч: платформа теперь поддерживает `multipart/form-data` нативно. Подтверждено примером ([play-mlncf.chatium.ru/s/ide/index.tsx?purl=/index](https://play-mlncf.chatium.ru/s/ide/index.tsx?purl=%2Findex)). Это снимает причину для переезда серверной части на собственный VDS. Документация webhook-proxy (`docs/webhook-proxy/`) помечена как отменённая, VDS остаётся арендованным «на всякий», но в боевом сценарии не нужен.

**Расширение скоупа (18.05, после первоначального блокера LifePay).** Клиент попросил добавить автоматизацию оплат **ОТП-банка** (сейчас сверка вручную). Поддержка ОТП подтвердила: технически возможно через API + webhooks, разработка на стороне клиента, документация по запросу. Договорная упаковка ОТП — открытый вопрос (доп.соглашение к Договору №1 или отдельный контракт; пока не обсуждалась).

> ⚠️ **ОТП — вне скоупа этого проекта (на 24.05.2026).** Работа над автоматизацией ОТП → GetCourse **не ведётся**: нет спецификации, ADR, плана и кода. Это потенциальный отдельный трек (после LifePay, отдельным договором/эпиком). В воркспэйсе по ОТП есть только справочный материал — [knowledge/otp-bank](knowledge/otp-bank/README.md). Текущий проект ограничен интеграцией LifePay/СБП и раскладкой способов оплаты.

## Цель

Собрать рабочую платежную схему в GetCourse и настроить страницу оплаты так, чтобы нужные способы оплаты отображались в правильном порядке и раскрывались блоками.

## Согласованный скоуп (финальный, после созвона 08.05)

1. Интеграция LifePay (СБП, комиссия 0.4% vs 3% GetCourse).
2. Логика платёжной страницы:
   - **Видны сразу:** СБП (LifePay) + ОТП Банк + оплата из-за границы (Lava Top)
   - **Скрыты внизу (серым):** GetCourse Тинькофф, GetCourse Сбер, прочие банки, сплиты, карты РФ
   - **Условие переключения:** сумма заказа до 50 000 руб. → все методы видны; от 50 000 руб. → только флагманские + альтернативы снизу
3. Компоновка страницы (функциональный дизайн — Андрей делает сам, без кастомных цветов и брендинга).

## Решение по LifePay (принято 13.05.2026)

Основной путь интеграции — прямая работа через старый API `POST https://api.life-pay.ru/v1/bill` с `method: "sbp"`, кабинет `my.life-pay.ru`. У клиента всё подключено, разработка не зависит от внешних сроков. Комиссия СБП 0.4% совпадает с согласованной 08.05.

Запасной путь — ECOM API через эквайринг `home.life-pay.ru` (вариант A в архитектуре). Активируется отдельным ADR 0002, если ответ инженеров LifePay покажет выгодные условия (комиссия ≤ 0.4%, быстрое подключение). Заявка инженерам передана 12.05.2026 22:11, ответ ожидается в течение 24 рабочих часов.

Полное обоснование в [ADR 0001 — LifePay API choice](../../../saas/gw/lifepay/docs/ADR/0003-lifepay-api-choice.md). Архитектурный разбор в [payment-scheme](../sbp-client/docs/architecture/payment-scheme.md). Карта кабинетов и сценариев в [cabinets](knowledge/lifepay/cabinets.md).
