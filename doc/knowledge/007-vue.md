## Основные концепции

**Vue в Chatium** — Vue.js 3.5.13 с Composition API для создания интерактивных пользовательских интерфейсов.

### Ключевые особенности

- **Vue 3.5.13** — актуальная версия
- **Composition API** — современный подход (`<script setup>`)
- **SSR** — серверный рендеринг
- **Глобальный ctx** — доступ к контексту без props
- **Реактивность** — автоматическое обновление UI

### Важные отличия

- ❌ Не используйте `Vue.createApp()` — делает фреймворк
- ❌ Не используйте Options API — используйте Composition API
- ✅ Используйте `<script setup>` для компонентов
- ✅ ctx доступен глобально — не прокидывайте через props

## Структура Vue компонента

### Базовый шаблон

```vue
<template>
  <!-- HTML разметка -->
  <div class="container">
    <h1>{{ title }}</h1>
    <button @click="handleClick">Нажми меня</button>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'

// Реактивные данные
const title = ref('Привет, мир!')

// Функции
function handleClick() {
  title.value = 'Кнопка нажата!'
}

// Хуки жизненного цикла
onMounted(() => {
  // Выполнится после монтирования
})
</script>

<style scoped>
.container {
  padding: 20px;
}
</style>
```

### Три секции

**1. Template** — HTML разметка:
- Обязательная секция
- Mustache синтаксис: `{{ variable }}`
- Директивы Vue: `v-if`, `v-for`, `@click`

**2. Script setup** — логика компонента:
- `<script setup>` — Composition API
- Импорты, переменные, функции
- Всё автоматически доступно в template

**3. Style** — стили компонента:
- `<style scoped>` — изолированные стили
- Обычный CSS/SCSS

## Глобальный контекст ctx

### Важная особенность

В Chatium `ctx` доступен **глобально** в любом Vue компоненте без props:

### Локальный tsconfig.json для Vue-модулей

При использовании Vue-компонентов в модуле необходимо создать локальный `tsconfig.json`, который наследует от корневого и переопределяет настройки для корректной работы с Vue.

**Пример конфигурации:**

```json
{
  "extends": "../../../../tsconfig.json",
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ESNext", "DOM", "DOM.Iterable"],
    "skipLibCheck": true,
    "jsx": "preserve",
    "strict": false,
    "noUncheckedIndexedAccess": false
  },
  "include": [
    "**/*.ts",
    "**/*.tsx",
    "**/*.vue"
  ],
  "exclude": [
    "node_modules"
  ]
}
```

**Что делает этот конфиг:**
- `"extends"` — наследует paths для `vue` и `@vue/*` из корневого tsconfig, что позволяет TypeScript находить типы Vue
- `"jsx": "preserve"` — предотвращает ошибки парсинга в Vue-шаблонах
- `"strict": false` — отключает строгую проверку типов для Vue-компонентов
- `"moduleResolution": "bundler"` — современный режим разрешения модулей для Vue 3

**⚠️ Важно при переносе модуля:**

Путь в `"extends"` — относительный. При переносе модуля в другую папку необходимо обновить количество `../` в пути, чтобы он вёл к корневому `tsconfig.json`.

**Ошибки, указывающие на проблему с tsconfig:**

1. **`Module '"vue"' has no exported member 'ref'`** — TypeScript не может найти типы Vue. Проверьте путь в `extends`.

2. **`Module '"vue"' has no exported member 'onMounted'`** — аналогично, проблема с разрешением типов Vue.

3. **`Cannot read file '.../tsconfig.json'`** — неверный путь в `extends`. Пересчитайте количество `../` от текущей папки до корня проекта.

4. **`Failed to write the global types file`** (Warning 404 от Volar) — предупреждение IDE, можно игнорировать если код работает. Корневой tsconfig содержит `vueCompilerOptions.globalTypesPath` для подавления.

**Как исправить при переносе модуля:**

Посчитайте уровень вложенности от новой папки до корня и обновите путь:
- `saas/analytics/telegram/channel/` → `../../../../tsconfig.json` (4 уровня)
- `saas/my-module/` → `../../tsconfig.json` (2 уровня)
- `doc/my-module/sub/` → `../../../tsconfig.json` (3 уровня)

### Подавление ошибки TypeScript для ctx

TypeScript не знает о глобальной переменной `ctx`, поэтому выдаёт ошибку `Cannot find name 'ctx'`. Чтобы подавить её, добавьте в начало `<script setup>`:

```typescript
declare const ctx: any
```

**Пример:**

```vue
<template>
  <div v-if="ctx.user">
    Привет, {{ ctx.user.displayName }}!
  </div>
  <div v-else>
    <a :href="loginUrl">Войти</a>
  </div>
</template>

<script setup>
declare const ctx: any

import { computed } from 'vue'

// ctx доступен глобально - не нужно импортировать!

const loginUrl = computed(() => 
  `/s/auth/signin?back=${encodeURIComponent(window.location.pathname)}`
)
</script>
```

### Доступные свойства ctx

```typescript
ctx.user           // Текущий пользователь (может быть null)
ctx.account        // Текущий аккаунт
ctx.session        // Сессия
ctx.lang           // Язык ('ru', 'en', etc.)
ctx.t()            // Функция локализации
```

**Пример**:

```vue
<template>
  <div>
    <p>Язык: {{ ctx.lang }}</p>
    <p v-if="ctx.user">ID: {{ ctx.user.id }}</p>
    <p>{{ ctx.t('welcome.message') }}</p>
  </div>
</template>
```

## @shared-route и метод .run()

### Что такое @shared-route?

Декоратор `// @shared-route` делает API-маршрут доступным как на бекенде, так и на фронтенде Vue компонентах.

### Объявление маршрута

```typescript
// api/products.ts
// @shared-route
export const apiProductsListRoute = app.get('/list', async (ctx, req) => {
  const products = await ProductsTable.findAll(ctx, { limit: 100 })
  return products
})

// @shared-route
export const apiProductsCreateRoute = app.post('/create', async (ctx, req) => {
  const product = await ProductsTable.create(ctx, req.body)
  return product
})
```

### Метод .run()

Вызывает маршрут с передачей контекста и опционального тела запроса.

**Важно:** `ctx` — это глобальная переменная, автоматически доступная в любом Vue компоненте. Не нужно её передавать через props или вычислять.

**TypeScript:** Чтобы подавить ошибку `Cannot find name 'ctx'`, добавьте в начало `<script setup>`:

```typescript
declare const ctx: any
```

### Использование в Vue компонентах

```vue
<script setup>
declare const ctx: any

import { ref, onMounted } from 'vue'
import { apiProductsListRoute, apiProductsCreateRoute } from '../api/products'

const products = ref([])

// ✅ GET без параметров
onMounted(async () => {
  products.value = await apiProductsListRoute.run(ctx)
})

// ✅ GET с параметрами пути
async function loadProduct(id: string) {
  const product = await apiProductCardRoute({ id }).run(ctx)
  return product
}

// ✅ GET с query параметрами
async function searchProducts(query: string) {
  const filtered = await apiProductsListRoute.query({ search: query }).run(ctx)
  return filtered
}

// ✅ POST с телом запроса
async function createProduct(data: any) {
  const created = await apiProductsCreateRoute.run(ctx, {
    name: data.name,
    price: data.price
  })
  return created
}
</script>
```

### Типы вызовов

| Сценарий | Синтаксис |
|----------|-----------|
| GET без параметров | `route.run(ctx)` |
| GET с query | `route.query({key: val}).run(ctx)` |
| POST | `route.run(ctx, {body})` |
| С path параметром | `route({param: val}).run(ctx)` |
| Комбинированно | `route({id}).query({filter}).run(ctx, {body})` |

### Ключевые правила

**✅ Хорошие практики:**
- Всегда экспортируй константу маршрута
- Используй `.run()` вместо hardcoded URL
- Импортируй маршруты перед использованием
- При параметрах: вызови маршрут как функцию, затем `.run()`
- Добавляй `declare const ctx: any` в начало `<script setup>`

**❌ Плохие практики:**
- Hardcoding URL адресов (`'/api/products'`)
- Использование fetch без маршрутов
- Забывать импортировать маршруты
- Передавать параметры в `.query()` вместо функции маршрута

## Реактивность

### ref() и reactive()

**ref()** — для примитивных значений:

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)
const name = ref('John')
const isActive = ref(true)

// Доступ к значению через .value
function increment() {
  count.value++
}

function changeName() {
  name.value = 'Jane'
}
</script>

<template>
  <!-- В template .value не нужен -->
  <div>
    <p>Count: {{ count }}</p>
    <p>Name: {{ name }}</p>
    <button @click="increment">+</button>
  </div>
</template>
```

**reactive()** — для объектов:

```vue
<script setup>
import { reactive } from 'vue'

const user = reactive({
  name: 'John',
  age: 30,
  email: 'john@example.com'
})

function updateUser() {
  user.name = 'Jane'
  user.age = 25
  // .value не нужен для reactive
}
</script>

<template>
  <div>
    <p>{{ user.name }}, {{ user.age }}</p>
    <button @click="updateUser">Обновить</button>
  </div>
</template>
```

### computed()

Вычисляемые свойства — кешируются и пересчитываются только при изменении зависимостей:

```vue
<script setup>
import { ref, computed } from 'vue'

const firstName = ref('John')
const lastName = ref('Doe')

// Вычисляется автоматически при изменении firstName или lastName
const fullName = computed(() => {
  return `${firstName.value} ${lastName.value}`
})

// Computed с setter
const age = ref(30)
const ageInMonths = computed({
  get: () => age.value * 12,
  set: (months) => { age.value = months / 12 }
})
</script>

<template>
  <div>
    <p>{{ fullName }}</p>
    <p>Age: {{ age }} years ({{ ageInMonths }} months)</p>
  </div>
</template>
```

### watch() и watchEffect()

**watch()** — следить за конкретными значениями:

```vue
<script setup>
import { ref, watch } from 'vue'

const count = ref(0)
const message = ref('')

// Следить за count
watch(count, (newValue, oldValue) => {
  message.value = `Count changed from ${oldValue} to ${newValue}`
})

// Следить за несколькими значениями
const firstName = ref('John')
const lastName = ref('Doe')

watch([firstName, lastName], ([newFirst, newLast]) => {
  // Вызывается при изменении любого
})
</script>
```

**watchEffect()** — автоматически следить за всеми используемыми значениями:

```vue
<script setup>
import { ref, watchEffect } from 'vue'

const search = ref('')
const results = ref([])

watchEffect(async () => {
  // Автоматически следит за search
  if (search.value.length > 2) {
    results.value = await performSearch(search.value)
  }
})
</script>
```

## Жизненный цикл

### Основные хуки

```vue
<script setup>
import { 
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted
} from 'vue'

onBeforeMount(() => {
  // Перед монтированием компонента
})

onMounted(() => {
  // После монтирования — самый частый
  // Загрузка данных, инициализация
})

onBeforeUpdate(() => {
  // Перед обновлением DOM
})

onUpdated(() => {
  // После обновления DOM
})

onBeforeUnmount(() => {
  // Перед размонтированием
  // Очистка таймеров, подписок
})

onUnmounted(() => {
  // После размонтирования
})
</script>
```

**Типичный паттерн - получение данных с сервера (SSR)**:

```vue
<script setup>
import { ref } from 'vue'

// ✅ Данные загружаются на сервере и передаются через props
const props = defineProps({
  initialData: { type: Array, default: () => [] }
})

const data = ref(props.initialData)
</script>

<template>
  <div v-if="data.length === 0">Нет данных</div>
  <div v-else>
    <div v-for="item in data" :key="item.id">
      {{ item }}
    </div>
  </div>
</template>
```

**Типичный паттерн - загрузка на клиенте через fetch()**:

```vue
<script setup>
import { ref, onMounted } from 'vue'

const props = defineProps({
  apiUrls: { type: Object, required: true }
})

const data = ref([])
const loading = ref(true)

onMounted(async () => {
  try {
    // ✅ URL из props (получен на сервере через .url())
    const response = await fetch(props.apiUrls.dataList)
    const result = await response.json()
    
    if (result.success) {
      data.value = result.data
    }
  } catch (error) {
    console.error('Error:', error)
  } finally {
    loading.value = false
  }
})
</script>

<template>
  <div v-if="loading">Загрузка...</div>
  <div v-else>{{ data }}</div>
</template>
```

### Вызов API из Vue компонентов

#### Использование @shared-route и метода .run()

**✅ РЕКОМЕНДУЕТСЯ**: Используйте `@shared-route` и метод `.run()` для вызова API из Vue компонентов.

**Что такое @shared-route?**

Декоратор `// @shared-route` делает API-маршрут доступным как на бекенде, так и на фронтенде Vue компонентах.

**Объявление маршрута:**

```typescript
// api/products.ts
// @shared-route
export const apiProductsListRoute = app.get('/list', async (ctx, req) => {
  const products = await ProductsTable.findAll(ctx, { limit: 100 })
  return products
})

// @shared-route
export const apiProductsCreateRoute = app.post('/create', async (ctx, req) => {
  const product = await ProductsTable.create(ctx, req.body)
  return product
})
```

**Использование в Vue компонентах:**

```vue
<script setup>
declare const ctx: any  // ctx доступен глобально в Vue компонентах

import { ref, onMounted } from 'vue'
import { apiProductsListRoute, apiProductsCreateRoute } from '../api/products'

const products = ref([])
const loading = ref(false)

// ✅ Загрузка данных через .run()
async function loadProducts() {
  loading.value = true
  try {
    // ✅ Используем .run() с глобальным ctx
    products.value = await apiProductsListRoute.run(ctx)
  } catch (error) {
    console.error('Error:', error)
  } finally {
    loading.value = false
  }
}

// ✅ Создание продукта через .run()
async function createProduct(data) {
  try {
    // ✅ POST запрос через .run() с телом запроса
    const created = await apiProductsCreateRoute.run(ctx, {
      name: data.name,
      price: data.price
    })
    products.value.push(created)
  } catch (error) {
    console.error('Error:', error)
  }
}

onMounted(() => {
  loadProducts()
})
</script>

<template>
  <div>
    <div v-if="loading">Загрузка...</div>
    <div v-else>
      <div v-for="product in products" :key="product.id">
        {{ product.name }}
      </div>
    </div>
  </div>
</template>
```

**Примеры различных типов вызовов:**

```vue
<script setup>
declare const ctx: any

import { ref, onMounted } from 'vue'
import { 
  apiProductsListRoute, 
  apiProductCardRoute,
  apiProductsCreateRoute 
} from '../api/products'

// Реактивные переменные для хранения данных
const products = ref([])
const product = ref(null)
const filtered = ref([])

const result = ref(null)

// ✅ GET без параметров - загрузка при монтировании

  } catch (e) {

  } finally {
    loading.value = false
  }
})

// ✅ GET с параметрами пути - функция для загрузки по требованию
async function loadProduct(id: string) {
  try {
    product.value = await apiProductCardRoute({ id }).run(ctx)
  } catch (e) {
    error.value = e.message || 'Ошибка загрузки продукта'
  }
}

// ✅ GET с query параметрами - функция для поиска
async function searchProducts(search: string) {
  try {
    filtered.value = await apiProductsListRoute.query({ search }).run(ctx)
  } catch (e) {
    error.value = e.message || 'Ошибка поиска'
  }
}

// ✅ POST с телом запроса - функция для создания
async function createProduct(data: { name: string; price: number }) {
  try {
    created.value = await apiProductsCreateRoute.run(ctx, {
      name: data.name,
      price: data.price
    })
    // После создания можно обновить список
    products.value = await apiProductsListRoute.run(ctx)
  } catch (e) {
    error.value = e.message || 'Ошибка создания продукта'
  }
}

// ✅ Комбинированный вызов - функция для загрузки с деталями
async function loadProductWithDetails(id: string) {
  try {
    result.value = await apiProductCardRoute({ id })
      .query({ include: 'details' })
      .run(ctx)
  } catch (e) {
    error.value = e.message || 'Ошибка загрузки деталей'
  }
}
</script>
```

**Ключевые правила:**

✅ **Хорошие практики:**
- Всегда экспортируй константу маршрута
- Используй `.run()` вместо hardcoded URL
- Импортируй маршруты перед использованием
- При параметрах: вызови маршрут как функцию, затем `.run()`
- Добавляй `declare const ctx: any` в начало `<script setup>`
- Обёртывай async вызовы в `onMounted()` для загрузки при монтировании или в async функции для вызова по требованию
- Используй `ref()` для хранения результатов API вызовов (реактивность)
- Всегда обрабатывай ошибки с try/catch

❌ **Плохие практики:**
- Hardcoding URL адресов (`'/api/products'`)
- Использование fetch без маршрутов
- Забывать импортировать маршруты
- Передавать параметры в `.query()` вместо функции маршрута
- **Использовать top-level `await` в `<script setup>`** — это выполняется при загрузке модуля, а не при монтировании компонента, что приводит к общему состоянию между экземплярами и отсутствию реактивности

**Таблица типов вызовов:**

| Сценарий | Синтаксис |
|----------|-----------|
| GET без параметров | `route.run(ctx)` |
| GET с query | `route.query({key: val}).run(ctx)` |
| POST | `route.run(ctx, {body})` |
| С path параметром | `route({param: val}).run(ctx)` |
| Комбинированно | `route({id}).query({filter}).run(ctx, {body})` |

#### Альтернативный способ: fetch() с URL из props (устаревший)

⚠️ **УСТАРЕВШИЙ ПОДХОД**: Если по каким-то причинам нельзя использовать `@shared-route`, можно использовать `fetch()` с URL, полученным на сервере через `.url()` и переданным через props. Но это менее удобно и не рекомендуется.

```vue
<script setup>
import { ref } from 'vue'

const products = ref([])
const loading = ref(false)

// ⚠️ Устаревший подход - используйте .run() вместо этого
const props = defineProps({
  apiUrls: { type: Object, required: true }
})

async function loadProducts() {
  loading.value = true
  try {
    const response = await fetch(props.apiUrls.productsList)
    const result = await response.json()
    
    if (result.success) {
      products.value = result.products
    }
  } catch (error) {
    console.error('Error:', error)
  } finally {
    loading.value = false
  }
}
</script>
```

### 🚨 КРИТИЧЕСКИЕ ОШИБКИ при работе с роутами в Vue

⚠️ **Самая частая ошибка** — использование хардкода URL вместо роут-объектов!

#### ❌ ОШИБКА 1: Хардкод URL в computed

```vue
<!-- ❌ НЕПРАВИЛЬНО - НИКОГДА ТАК НЕ ДЕЛАЙТЕ! -->
<script setup>
import { computed } from 'vue'

// ❌ ПЛОХО - хардкод пути
const customFieldsUrl = computed(() => '/dev/amocrm-connector/customFields')
</script>

<template>
  <a :href="customFieldsUrl">Дополнительные поля</a>
</template>
```

**Почему это плохо:**
- Ломается при переименовании файлов
- Ломается при изменении структуры проекта
- Нет типизации и проверки в IDE
- Сложно рефакторить

```vue
<!-- ✅ ПРАВИЛЬНО -->
<script setup>
import { computed } from 'vue'
import { customFieldsPageRoute } from '../customFields'

// ✅ ПРАВИЛЬНО - используем роут-объект
const customFieldsUrl = computed(() => customFieldsPageRoute.url())
</script>

<template>
  <a :href="customFieldsUrl">Дополнительные поля</a>
</template>
```

#### ❌ ОШИБКА 2: Забыли вызвать .url()

```vue
<!-- ❌ НЕПРАВИЛЬНО -->
<script setup>
import { customFieldsPageRoute } from '../customFields'
</script>

<template>
  <!-- ❌ ПЛОХО - передаётся объект роута, а не строка! -->
  <a :href="customFieldsPageRoute">Ссылка</a>
</template>

<!-- ✅ ПРАВИЛЬНО -->
<script setup>
import { customFieldsPageRoute } from '../customFields'
</script>

<template>
  <!-- ✅ ПРАВИЛЬНО - вызываем .url() -->
  <a :href="customFieldsPageRoute.url()">Ссылка</a>
</template>
```

#### ❌ ОШИБКА 3: Хардкод в методе навигации

```vue
<script setup>

  window.location.href = '/dev/amocrm-connector/oauthSettings'

```

#### ❌ ОШИБКА 4: Конкатенация URL с параметрами

```vue
<script setup>
import { ref } from 'vue'

const productId = ref('123')

// ❌ ПЛОХО - конкатенация строк
```

## Работа с данными

### 🚨 КРИТИЧЕСКОЕ ПРАВИЛО: Клиент vs Сервер

**ВАЖНО!** Vue компоненты выполняются **на клиенте** в браузере, а не на сервере!

#### ❌ ЧТО НЕЛЬЗЯ ДЕЛАТЬ В VUE КОМПОНЕНТАХ:

**1. НЕЛЬЗЯ импортировать Heap таблицы:**
```vue
<!-- ❌ ОШИБКА: NS_ERROR_CORRUPTED_CONTENT -->
<script setup>
import { AnalyticsDatasets } from '../tables/datasets.table'

// ❌ Heap таблицы НЕ РАБОТАЮТ на клиенте!
const datasets = await AnalyticsDatasets.findAll(ctx, {})
</script>
```

**Почему**: Heap таблицы — это серверная логика, недоступная в браузере.

**2. НЕЛЬЗЯ использовать `.run()` для НЕ-@shared-route маршрутов:**
```vue
<!-- ❌ ОШИБКА: маршрут не помечен как @shared-route -->
<script setup>
declare const ctx: any
import { regularRoute } from '../api/data'  // без @shared-route

onMounted(async () => {
  // ❌ Не сработает, если маршрут не помечен @shared-route!
  const result = await regularRoute.run(ctx, {})
})
</script>
```

**Почему**: Только маршруты с декоратором `@shared-route` доступны на клиенте. Обычные маршруты работают только на сервере.

**3. НЕЛЬЗЯ использовать хардкод URL даже в fetch():**
```vue
<!-- ❌ ОШИБКА: Хардкод URL -->
<script setup>
async function saveData() {
  // ❌ НЕТ! Хардкод пути!
  const response = await fetch('/dev/partnership/api/datasets/create', {
    method: 'POST',
    body: JSON.stringify(data)
  })
}
</script>
```

**Почему**: При рефакторинге или переименовании файлов путь сломается. URL должен быть получен через `.url()` на сервере и передан как prop.

```vue
<!-- IndexPage.vue (КЛИЕНТ) -->
<script setup>
const props = defineProps({
  initialDatasets: { type: Array, default: () => [] },
  apiUrls: { type: Object, required: true }
})

// Удаление через HTML форму
function deleteDataset(datasetId) {
  if (!confirm('Удалить датасет?')) return
  
  // ✅ Создаём форму для POST запроса
  const form = document.createElement('form')
  form.method = 'POST'
  form.action = props.apiUrls.delete  // ✅ URL из props (получен через .url())
  
  const input = document.createElement('input')
  input.type = 'hidden'
  input.name = 'id'
  input.value = datasetId
  
  const redirectInput = document.createElement('input')
  redirectInput.type = 'hidden'
  redirectInput.name = '_redirect'
  redirectInput.value = props.apiUrls.indexPage  // ✅ URL из props
  
  form.appendChild(input)
  form.appendChild(redirectInput)
  document.body.appendChild(form)
  form.submit()  // → POST → Удаление → Редирект → Страница перезагружается
}
</script>
```

| Операция | Сервер (.tsx) | Клиент (Vue) |
|----------|---------------|--------------|
| **Heap таблицы** | ✅ `AnalyticsDatasets.findAll(ctx, {})` | ❌ Недоступно |
| **API роуты .run() (@shared-route)** | ✅ `apiRoute.run(ctx, {})` | ✅ `apiRoute.run(ctx, {})` (ctx доступен глобально) |
| **API роуты .run() (обычные)** | ✅ `apiRoute.run(ctx, {})` | ❌ Только @shared-route маршруты |

| **HTML формы** | — | ✅ `form.action = props.apiUrl` или через `.url()` |
| **Props** | ✅ Передаём из .tsx | ✅ Получаем через defineProps |
| **URL генерация** | ✅ `route.url()` | ✅ `route.url()` для генерации ссылок |
| **Хардкод URL** | ❌ ЗАПРЕЩЕНО | ❌ ЗАПРЕЩЕНО |

## Интеграция внешних скриптов

### Встройка форм GetCourse

При интеграции форм GetCourse (или других виджетов с `<script>` тегами) в Vue компоненты, важно правильно управлять жизненным циклом элементов.

**Проблема**: Если просто добавить `<script>` в template, при каждом переключении компонента iframe будет перезагружаться, что ломает работу формы.

**Решение**: Создать контейнер на уровне страницы и переносить форму между компонентами без перезагрузки.

#### Пример: форма GetCourse в модальном окне

Если у вас есть скрипт формы GetCourse:
```html
<script id="00f40648400ce99b076d230a679ea99aa7a0ed51" 
        src="https://efimova-magic.ru/pl/lite/widget/script?id=1454514">
</script>
```

**Шаг 1**: Разместите контейнер в серверном коде страницы (в body):

```tsx
// pages/myPage.tsx (серверная часть)
export default app.html('/my-page', async (ctx, req) => {
  return (
    <html>
      <head>
        <title>Моя страница</title>
      </head>
      <body>
        <div id="app"></div>
        
        {/* Контейнер для всех форм GetCourse */}
        <div id="allGetcourseForms">
          <div id="getcourseForm1" style={{ display: 'none' }}>
            <script 
              id="00f40648400ce99b076d230a679ea99aa7a0ed51" 
              src="https://efimova-magic.ru/pl/lite/widget/script?id=1454514"
            />
          </div>
        </div>
      </body>
    </html>
  )
})
```

**Шаг 2**: В Vue компоненте перенесите форму в нужное место:

```vue
<template>
  <!-- Контейнер для формы -->
  <div ref="getcourseWidgetContainerRef" />
</template>

<script setup>
import { ref, onMounted, onBeforeUnmount } from 'vue'

const getcourseWidgetContainerRef = ref(null)

onMounted(() => {
  // Находим форму в глобальном контейнере
  const gcForm = document.getElementById('getcourseForm1')
  
  if (gcForm && getcourseWidgetContainerRef.value) {
    // Переносим форму в компонент
    getcourseWidgetContainerRef.value.appendChild(gcForm)
    // Показываем форму
    gcForm.style.display = 'block'
  }
})

// Важно: возвращаем форму обратно при размонтировании компонента
onBeforeUnmount(() => {
  const gcForm = document.getElementById('getcourseForm1')
  const allFormsContainer = document.getElementById('allGetcourseForms')
  
  if (gcForm && allFormsContainer) {
    // Переносим форму обратно в глобальный контейнер
    allFormsContainer.appendChild(gcForm)
    // Скрываем форму
    gcForm.style.display = 'none'
  }
})
</script>
```

**Шаг 3**: Использование в модальном окне:

```vue
<template>
  <div>
    <button @click="showModal = true">Открыть форму</button>
    
    <!-- Модальное окно с формой -->
    <Transition name="fade">
      <div v-show="showModal" class="modal-overlay" @click="showModal = false">
        <div class="modal-content" @click.stop>
          <button @click="showModal = false" class="close-btn">×</button>
          
          <h2>Заполните форму</h2>
          
          {/* Контейнер для формы GetCourse */}
          <div ref="getcourseWidgetContainerRef" />
        </div>
      </div>
    </Transition>
  </div>
</template>

<script setup>
import { ref, onMounted, onBeforeUnmount } from 'vue'

const showModal = ref(false)
const getcourseWidgetContainerRef = ref(null)

onMounted(() => {
  const gcForm = document.getElementById('getcourseForm1')

onBeforeUnmount(() => {

  }

</script>

<style scoped>
```

#### Ключевые моменты

✅ **Используйте v-show вместо v-if** для модального окна:
- Модальное окно должно быть отрендерено сразу
- `v-show` просто скрывает элемент, сохраняя его в DOM
- Это предотвращает перезагрузку iframe формы

✅ **Перемещайте форму, не клонируйте**:
- `appendChild()` перемещает элемент, а не копирует
- Iframe сохраняет своё состояние при перемещении

✅ **Всегда возвращайте форму обратно**:
- В `onBeforeUnmount` верните форму в глобальный контейнер
- Это позволит другим компонентам использовать её

✅ **Добавляйте анимации через Transition**:
```vue
<Transition name="fade">
  <div v-show="showModal">
    <!-- Модальное окно -->
  </div>
</Transition>
```

#### Несколько форм на странице

Если форм несколько, создайте для каждой отдельный контейнер:

```tsx
<div id="allGetcourseForms">
  <div id="getcourseForm1" style={{ display: 'none' }}>
    <script id="..." src="...form1..." />
  </div>
  <div id="getcourseForm2" style={{ display: 'none' }}>
    <script id="..." src="...form2..." />
  </div>
</div>
```

И работайте с каждой формой отдельно в компонентах.

## Настройка TypeScript для Vue проектов

При создании нового проекта с Vue компонентами необходимо настроить TypeScript для корректной работы IDE.

### Проблема

Без правильной настройки TypeScript в IDE появляются ошибки на HTML-элементы (`html`, `div`, `span` и т.д.) в JSX файлах:
- `Cannot find name 'html'`
- `Cannot find name 'div'`

**Важно**: Эти ошибки влияют только на IDE — код работает корректно, так как Chatium обрабатывает JSX самостоятельно.

### Решение: Создать два файла в корне модуля

**Важно**: Файлы создаются в корне **модуля** (рядом с `index.tsx`), а не в корне всего проекта. Один проект может содержать множество модулей, и каждому модулю нужны свои файлы конфигурации.

Пример структуры:
```
project/
├── saas/
│   └── analytics/
│       └── telegram/
│           └── channel/           ← корень модуля
│               ├── index.tsx      ← индекс-страница
│               ├── tsconfig.json  ← конфигурация TypeScript
│               ├── jsx.d.ts       ← определения JSX
│               └── pages/
│                   └── HomePage.vue
└── doc/
    └── knowledge-app/             ← другой модуль
        ├── index.tsx
        ├── tsconfig.json
        └── jsx.d.ts
```

#### 1. `tsconfig.json` — упрощённая конфигурация TypeScript

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ESNext", "DOM", "DOM.Iterable"],
    "rootDir": ".",
    "baseUrl": ".",
    "paths": {
      "/*": ["./*"]
    },
    "allowJs": false,
    "allowSyntheticDefaultImports": true,
    "experimentalDecorators": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "jsx": "preserve",
    "strict": false,
    "noUncheckedIndexedAccess": false
  },
  "include": [
    "**/*.ts",
    "**/*.tsx",
    "**/*.vue"
  ],
  "exclude": [
    "node_modules"
  ]
}
```

**Ключевые настройки:**
- `"jsx": "preserve"` — сохраняет JSX для обработки Chatium
- `"strict": false` — отключает строгие проверки типов
- `"skipLibCheck": true` — пропускает проверку типов в библиотеках
- `"moduleResolution": "bundler"` — современный режим разрешения модулей

#### 2. `jsx.d.ts` — определения типов для JSX элементов


- `[elemName: string]: any` — разрешает любые кастомные элементы

### Чек-лист при создании нового модуля

- [ ] Создать `tsconfig.json` в корне модуля (рядом с `index.tsx`)
- [ ] Создать `jsx.d.ts` в корне модуля (рядом с `index.tsx`)
- [ ] Убедиться, что `paths` в tsconfig содержит нужные алиасы
- [ ] Перезапустить TypeScript сервер в IDE (Cmd/Ctrl + Shift + P → "TypeScript: Restart TS Server")

## Решение проблемы с Vue типами в модулях

### Проблема

В Chatium Vue установлен на корневом уровне проекта (`node_modules/@vue/*`), а не как прямая зависимость каждого модуля. Это вызывает ошибки TypeScript и предупреждения Volar:

**Ошибки:**
- `Module '"vue"' has no exported member 'ref'`
- `Module '"vue"' has no exported member 'onMounted'`

**Предупреждение:**
- `Failed to write the global types file` - Volar не может найти Vue как прямую зависимость

### Решение: Локальная структура Vue в модуле

Создайте минимальную структуру `node_modules/vue` в корне модуля с stub-типами.

#### 1. Создайте `vue-shim.d.ts` в корне модуля

```typescript
declare module '*.vue' {
  const component: any
  export default component
}

declare module 'vue' {
  export function ref<T>(value: T): { value: T }
  export function reactive<T extends object>(obj: T): T
  export function computed<T>(getter: () => T): { value: T }
  export function watch<T>(source: any, callback: (newVal: T, oldVal: T) => void): void
  export function watchEffect(effect: () => void): void
  export function onMounted(callback: () => void): void
  export function onBeforeMount(callback: () => void): void
  export function onBeforeUpdate(callback: () => void): void
  export function onUpdated(callback: () => void): void
  export function onBeforeUnmount(callback: () => void): void
  export function onUnmounted(callback: () => void): void
  export function defineProps<T>(): T
  export function defineEmits<T>(): T
  export function withDefaults<T, D>(props: T, defaults: D): T & D
  
  export type Ref<T> = { value: T }
  export type ComputedRef<T> = { value: T }
  export type DefineComponent = any
}
```

#### 2. Создайте структуру `node_modules/vue/`

**`node_modules/vue/package.json`:**
```json
{
  "name": "vue",
  "version": "3.5.13",
  "main": "./dist/vue.runtime.esm-bundler.js",
  "module": "./dist/vue.runtime.esm-bundler.js",
  "types": "./index.d.ts"
}
```

**`node_modules/vue/index.d.ts`:**
```typescript
// Полные type definitions для Vue 3.5.13

export function reactive<T extends object>(obj: T): T
export function computed<T>(getter: () => T): { readonly value: T }
export function computed<T>(options: {
  get: () => T
  set: (value: T) => void
}): { value: T }
export function watch<T>(
  source: T | (() => T),
  callback: (newVal: any, oldVal: any) => void,
  options?: any
): void
export function watchEffect(effect: () => void): void
export function onMounted(callback: () => void): void
export function onBeforeMount(callback: () => void): void
export function onBeforeUpdate(callback: () => void): void
export function onUpdated(callback: () => void): void
export function onBeforeUnmount(callback: () => void): void
export function onUnmounted(callback: () => void): void
export function defineProps<T = any>(): T
export function defineEmits<T = any>(): T
export function withDefaults<T, D>(props: T, defaults: D): T & D
export function defineExpose<T = any>(exposed?: T): void
export function useSlots(): any
export function useAttrs(): any

export type Ref<T = any> = { value: T }
export type ComputedRef<T = any> = { readonly value: T }
export type UnwrapRef<T> = T

```

**`node_modules/vue/dist/vue.runtime.esm-bundler.js`:**
```javascript
// Stub runtime для Chatium
// Реальный Vue предоставляется инфраструктурой Chatium
export * from '../index.d.ts'
```

#### 3. Обновите `package.json` модуля

```json
{
  "name": "your-module-name",
  "version": "1.0.0",
  "private": true,
  "dependencies": {
    "vue": "3.5.13"
  }
}
```

#### 4. Обновите `tsconfig.json`

Убедитесь, что включён `vue-shim.d.ts`:

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ESNext", "DOM", "DOM.Iterable"],
    "jsx": "preserve",
    "skipLibCheck": true,
    "strict": false
  },
  "include": [
    "**/*.ts",
    "**/*.tsx",
    "**/*.vue",
    "vue-shim.d.ts"
  ]
}
```

### Как это работает

1. **TypeScript** использует `vue-shim.d.ts` и `node_modules/vue/index.d.ts` для проверки типов
2. **Volar** видит Vue как установленную зависимость через `package.json`
3. **Система сборки Chatium** использует реальный Vue из корневого `node_modules`
4. Нет конфликтов, всё работает корректно

### После настройки

1. Перезапустите языковые серверы:
   - `Ctrl+Shift+P` → `TypeScript: Restart TS Server`
   - `Ctrl+Shift+P` → `Vue: Restart Vue Server`

2. Или перезагрузите окно IDE:
   - `Ctrl+Shift+P` → `Developer: Reload Window`

### Важно

⚠️ **НЕ запускайте `npm install`** в директории модуля - это перезапишет созданную вручную структуру `node_modules/vue`!

⚠️ **Файлы в `node_modules/vue` создаются вручную** и не должны управляться npm.

### Чек-лист для нового Vue модуля

- [ ] Создать `vue-shim.d.ts` в корне модуля
- [ ] Создать `node_modules/vue/package.json`
- [ ] Создать `node_modules/vue/index.d.ts` с полными типами
- [ ] Создать `node_modules/vue/dist/vue.runtime.esm-bundler.js` (stub)
- [ ] Добавить `vue: "3.5.13"` в `package.json` модуля
- [ ] Добавить `vue-shim.d.ts` в `include` в `tsconfig.json`
- [ ] Перезапустить языковые серверы
