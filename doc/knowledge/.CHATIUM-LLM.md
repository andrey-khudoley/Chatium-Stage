## API Routes

### Backend Routes

**Documents API** (`api/docs.ts`):
- `GET /knowledge-app/api/docs~list?limit=...` - List documents (prefix auto-applied) [Admin only]
- `GET /knowledge-app/api/docs~get?filename=...` - Get document content (by filename only) [**Public**]
- `POST /knowledge-app/api/docs~put` - Upload/update document (body: `{filename, markdown}`) [Admin only]
- `POST /knowledge-app/api/docs~delete` - Delete document (body: `{filename}`) [Admin only]

**Important**: All document routes now use **filename** parameter instead of full keys.  
The prefix `usage=external/service=docs/` is automatically added server-side.

**Settings API** (`api/settings.ts`):
- `GET /knowledge-app/api/settings~get?key=...` - Get setting value
- `POST /knowledge-app/api/settings~save` - Save setting

**Theme API** (`api/theme.ts`):
- `GET /knowledge-app/api/theme~get` - Get default theme
- `POST /knowledge-app/api/theme~save` - Save default theme

**Main Routes** (`index.tsx`):
- `GET /knowledge-app/` - Document list (DocsListPage) [Admin only]
- `GET /knowledge-app~view?filename=000-summ.md` - View document (DocViewPage) [**Public**]
- `GET /knowledge-app~edit?filename=000-summ.md` - Edit document (DocEditPage) [Admin only]
- `GET /knowledge-app~create` - Create new document (DocEditPage) [Admin only]

**Test Routes** (`tests/`):
- `GET /knowledge-app/tests` - Interactive test page with UI [Admin only]
- `GET /knowledge-app/tests/ai` - Automated tests with JSON output [Admin only]
- `GET /knowledge-app/tests/api/run-tests~list` - Get test list
- `POST /knowledge-app/tests/api/run-tests~run-single` - Run single test

## Data Flow

### Load Documents
1. User opens `/knowledge-app`
2. `onMounted` loads settings and documents
3. `listDocsRoute.run()` calls backend API
4. **Backend adds prefix** `usage=external/service=docs/` automatically
5. Backend proxies to Yandex API with full keys
6. **Response transformed**: full keys → filenames only
7. Documents filtered and sorted
8. Displayed in modern table with gradient links

### View Document
1. User clicks document link with `filename=000-summ.md`
2. Navigate to `/knowledge-app~view?filename=000-summ.md`
3. Skeleton loader displays
4. `getDocRoute.query({ filename }).run()` calls backend
5. **Backend converts**: `000-summ.md` → `usage=external/service=docs/000-summ.md`
6. Backend proxies to Yandex API with full key
7. marked.js renders markdown
8. Smooth fade-in animation

### Edit Document
1. User clicks edit button with `filename=000-summ.md`
2. Navigate to `/knowledge-app~edit?filename=000-summ.md`
3. Content loads with skeleton loader
4. Split editor/preview layout with helpful hints
5. Live preview updates on input
6. On save: `putDocRoute.run({ filename, markdown })`
7. **Backend converts** filename to full key before Yandex API call

### Create Document
1. User clicks "Создать"
2. Navigate to `/knowledge-app~create`
3. Empty editor with file name input (no prefix shown)
4. Preview shows as user types
5. On save: `putDocRoute.run({ filename, markdown })`
6. **Backend adds prefix** automatically
7. Auto-redirect to list after success

## Key Architecture Decisions

### Document Key Management

**Problem**: URLs were too long and complex:
```
https://s.chtm.khudoley.tech/knowledge-app~view?key=usage%3Dexternal%2Fservice%3Ddocs%2F000-summ.md
```

**Solution**: Simplified to use filenames only in URLs:
```
https://s.chtm.khudoley.tech/knowledge-app~view?filename=000-summ.md
```

**Implementation**:
1. **Client Side** (Browser): Works only with filenames (`000-summ.md`)
2. **Server Side** (Chatium): Converts filename ↔ full key
3. **Yandex API**: Receives full keys (`usage=external/service=docs/000-summ.md`)

**Benefits**:
- ✅ Clean, short URLs
- ✅ User-friendly interface
- ✅ No changes needed in Yandex infrastructure
- ✅ All complexity hidden in backend
- ✅ Easy to share document links

**Key Functions** (`api/docs.ts`):
```typescript
const DOCS_PREFIX = 'usage=external/service=docs/'

function getFullKey(filename: string): string {
  return DOCS_PREFIX + filename
}

function getFilename(fullKey: string): string {
  return fullKey.substring(DOCS_PREFIX.length)
}
```

### Direct API Calls Pattern

**Change**: Removed `DocsApiClient` wrapper class in favor of direct API calls.

**Before**:
```typescript
const client = new DocsApiClient('', '')
const result = await client.list(PREFIX)
```

**After**:
```typescript
const result = await listDocsRoute.run(ctx)
```

**Benefits**:
- ✅ Less code
- ✅ No unnecessary abstractions
- ✅ Direct access to Chatium routing
- ✅ Better type safety
- ✅ Easier debugging

### Dynamic Routing After Folder Relocation

**Problem**: After moving the app from `/knowledge-app` to `/doc/knowledge-app`, all hardcoded paths broke:

```typescript
// ❌ BROKEN - Hardcoded paths
window.location.href = '/knowledge-app~create'
const url = `/knowledge-app~view?filename=${filename}`
return ctx.resp.redirect('/knowledge-app')
```
