## Основные концепции

**i18n (Internationalization)** — встроенная система переводов в Chatium для поддержки мультиязычных приложений.

### Ключевые компоненты

- **ctx.lang** — текущий язык пользователя
- **ctx.t()** — функция перевода
- **YAML файлы** — хранилище переводов (`*.lang.yml`)

### Принципы работы

1. Переводы хранятся в YAML файлах
2. Язык определяется автоматически или вручную
3. Функция `ctx.t()` возвращает перевод по ключу
4. Поддержка параметров и плюралов

## Работа с языками

### Получение текущего языка

```typescript
// Получить текущий язык
const currentLang = ctx.lang // 'ru', 'en', 'de', и т.д.

// Проверка конкретного языка
const isRussian = ctx.lang === 'ru'
const isEnglish = ctx.lang === 'en'
```

**В Vue компонентах**:

```vue
<template>
  <div>
    Текущий язык: {{ ctx.lang }}
    
    <div v-if="ctx.lang === 'ru'">
      Русская версия
    </div>
    <div v-else-if="ctx.lang === 'en'">
      English version
    </div>
  </div>
</template>
```

### Функция перевода ctx.t()

```typescript
// Простой перевод
const title = ctx.t('page.title')
const welcome = ctx.t('welcome.message')

// Перевод с параметрами
const greeting = ctx.t('welcome.name', { name: 'John' })
const itemCount = ctx.t('cart.count', { count: 5 })

// Fallback значение, если ключ не найден
const fallback = ctx.t('nonexistent.key', 'Default text')
```

**Использование в бэкенде**:

```typescript
export const apiGreetingRoute = app.get('/greeting', async (ctx, req) => {
  const userName = req.query.name || 'Guest'
  
  return {
    message: ctx.t('welcome.message', { name: userName }),
    language: ctx.lang
  }
})
```

## Файлы переводов

### Создание YAML файлов

Создайте файлы с расширением `.lang.yml` в корне проекта:

**Русский (lang/ru.yml)**:

```yaml
page:
  title: "Главная страница"
  subtitle: "Добро пожаловать"
  description: "Это главная страница приложения"

welcome:
  message: "Привет, {{name}}!"
  button: "Начать"

cart:
  empty: "Корзина пуста"
  count_zero: "Нет товаров"
  count_one: "{{count}} товар"
  count_few: "{{count}} товара"
  count_many: "{{count}} товаров"

buttons:
  save: "Сохранить"
  cancel: "Отмена"
  delete: "Удалить"
  
errors:
  userNotFound: "Пользователь с ID {{id}} не найден"
  networkError: "Ошибка сети: {{error}}"
```

**Английский (lang/en.yml)**:

```yaml
page:
  title: "Main page"
  subtitle: "Welcome"
  description: "This is the main application page"

welcome:
  message: "Hello, {{name}}!"
  button: "Start"

cart:
  empty: "Cart is empty"
  count_zero: "No items"
  count_one: "{{count}} item"
  count_other: "{{count}} items"

buttons:
  save: "Save"
  cancel: "Cancel"
  delete: "Delete"

errors:
  userNotFound: "User with ID {{id}} not found"
  networkError: "Network error: {{error}}"
```

### Структура файлов

```
/
├── lang/
│   ├── ru.yml
│   ├── en.yml
│   └── de.yml
├── pages/
├── components/
├── api/
└── ...
```

**Рекомендации**:
- Один файл на язык
- Группируйте переводы по смыслу (page, cart, buttons, errors)
- Используйте вложенные ключи для структуризации

## Использование в Vue компонентах

### Переводы в template

```vue
<template>
  <div>
    <h1>{{ $t('page.title') }}</h1>
    <p>{{ $t('page.subtitle') }}</p>
    
    <!-- С параметрами -->
    <p>{{ $t('welcome.message', { name: userName }) }}</p>
    
    <button>{{ $t('welcome.button') }}</button>
    
    <!-- Условная логика -->
    <div v-if="cartItems === 0">
      {{ $t('cart.empty') }}
    </div>
    <div v-else>
      {{ $t('cart.count', { count: cartItems }) }}
    </div>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'

const userName = computed(() => ctx.user?.displayName || 'Guest')
const cartItems = ref(0)
</script>
```

### Переводы в script

```vue
<script setup>
import { ref, computed } from 'vue'

const showNotification = (message) => {
  alert(message)
}

const handleSave = () => {
  showNotification(ctx.t('messages.saved'))
}

const handleError = (error) => {
  showNotification(ctx.t('messages.error', { error: error.message }))
}

// Реактивный текущий язык
const currentLanguage = computed(() => ctx.lang)
const isRussian = computed(() => ctx.lang === 'ru')

// Динамический перевод с вычислением
const dynamicMessage = computed(() => {
  return ctx.t('status.message', { 
    status: statusValue.value,
    time: new Date().toLocaleTimeString()
  })
})
</script>
```

## Использование на бэкенде

### В API роутах

```typescript
// @shared-route
export const apiUserDataRoute = app.get('/user-data', async (ctx, req) => {
  const userId = req.query.userId
  const user = await getUserById(ctx, userId)
  
  if (!user) {
    return { 
      success: false,
      error: ctx.t('errors.userNotFound', { id: userId })
    }
  }
  
  return {
    success: true,
    user: {
      ...user,
      greeting: ctx.t('welcome.welcomeBack', { name: user.displayName })
    }
  }
})
```

**Локализованные email**:

```typescript
export const sendWelcomeEmailRoute = app.post('/send-welcome', async (ctx, req) => {
  const { userId } = req.body
  const user = await getUserById(ctx, userId)
  
  const subject = ctx.t('email.welcome.subject', { 
    appName: 'MyApp' 
  })
  
  const body = ctx.t('email.welcome.body', {
    name: user.displayName,
    link: 'https://example.com'
  })
  
  await sendEmail(ctx, {
    to: user.confirmedEmail,
    subject,
    body
  })
  
  return { success: true }
})
```

### В отложенных задачах

```typescript
const notificationJob = app.job('/send-notification', async (ctx, params) => {
  const { userId, type } = params
  
  // ctx.lang будет соответствовать языку пользователя
  const message = ctx.t(`notifications.${type}`, {
    name: params.name,
    date: new Date().toLocaleDateString()
  })
  
  await sendNotification(ctx, userId, message)
  
  ctx.account.log('Notification sent', {
    level: 'info',
    json: { userId, type, language: ctx.lang }
  })
})
```

## Плюралы и формы слов

### Примеры для русского языка

В русском языке есть три формы множественного числа:

```yaml
# lang/ru.yml
items:
  count: "{{count}} предмет|{{count}} предмета|{{count}} предметов"

messages:
  new_messages: "У вас {{count}} новое сообщение|{{count}} новых сообщения|{{count}} новых сообщений"

time:
  hours_ago: "{{count}} час назад|{{count}} часа назад|{{count}} часов назад"
  minutes_ago: "{{count}} минуту назад|{{count}} минуты назад|{{count}} минут назад"
```

**Правила для русского**:
- 1, 21, 31, ... → первая форма (1 предмет)
- 2-4, 22-24, 32-34, ... → вторая форма (2 предмета)
- 5-20, 25-30, ... → третья форма (5 предметов)

### Примеры для английского языка

В английском только две формы:

```yaml
# lang/en.yml
items:
  count: "{{count}} item|{{count}} items"

messages:
  new_messages: "You have {{count}} new message|You have {{count}} new messages"

time:
  hours_ago: "{{count}} hour ago|{{count}} hours ago"
  minutes_ago: "{{count}} minute ago|{{count}} minutes ago"
```

### Использование плюралов

```vue
<template>
  <div>
    <p>{{ $t('items.count', { count: 1 }) }}</p>   <!-- 1 предмет -->
    <p>{{ $t('items.count', { count: 2 }) }}</p>   <!-- 2 предмета -->
    <p>{{ $t('items.count', { count: 5 }) }}</p>   <!-- 5 предметов -->
    <p>{{ $t('items.count', { count: 21 }) }}</p>  <!-- 21 предмет -->
  </div>
</template>
```

**На бэкенде**:

```typescript
export const getItemsCountRoute = app.get('/items/count', async (ctx, req) => {
  const count = await Items.count(ctx)
  
  return {
    count,
    message: ctx.t('items.count', { count })
  }
})
```

## Определение и переключение языка

### Автоматическое определение

Chatium автоматически определяет язык из:

1. **Настроек пользователя** (приоритет)
2. **Языка браузера** (Accept-Language)
3. **Языка по умолчанию** (из .chatiumrc)

```typescript
// Язык определяется автоматически при каждом запросе
export const homeRoute = app.html('/', async (ctx, req) => {
  ctx.account.log('User language', {
    level: 'info',
    json: { lang: ctx.lang }
  })
  
  // ctx.lang уже установлен
  return <div>{ctx.t('page.title')}</div>
})
```

### Ручная установка языка

```typescript
// Обновление языка пользователя
await ctx.user.updateExtendedInfo(ctx, {
  lang: 'en'
})
```

**API роут для смены языка**:

```typescript
export const changeLanguageRoute = app.post('/change-language', async (ctx, req) => {
  const { language } = req.body
  
  if (!['ru', 'en', 'de'].includes(language)) {
    return { 
      success: false, 
      error: ctx.t('errors.invalidLanguage')
    }
  }
  
  await ctx.user.updateExtendedInfo(ctx, {
    lang: language
  })
  
  return { 
    success: true,
    message: ctx.t('settings.languageChanged')
  }
})
```

### Компонент переключения языка

```vue
<template>
  <div class="language-switcher">
    <select v-model="selectedLanguage" @change="changeLanguage">
      <option value="ru">Русский 🇷🇺</option>
      <option value="en">English 🇬🇧</option>
      <option value="de">Deutsch 🇩🇪</option>
    </select>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { apiChangeLanguageRoute } from '../api/settings'

const selectedLanguage = ref(ctx.lang)

async function changeLanguage() {
  try {
    const result = await apiChangeLanguageRoute.run(ctx, {
      language: selectedLanguage.value
    })
    
    if (result.success) {
      // Перезагружаем страницу для применения нового языка
      window.location.reload()
    } else {
      alert(result.error)
    }
  } catch (error) {
    ctx.account.log('Language change error', {
      level: 'error',
      json: { error: error.message }
    })
  }
}
</script>

<style scoped>
.language-switcher select {
  padding: 0.5rem;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 1rem;
}
</style>
```

### Условные переводы

```yaml
# lang/ru.yml
welcome:
  guest: "Добро пожаловать, гость!"
  user: "С возвращением, {{name}}!"
  admin: "Добро пожаловать, администратор {{name}}!"
  premium: "Здравствуйте, {{name}}! Спасибо за премиум подписку!"
```

```typescript
const getWelcomeMessage = (user) => {
  if (!user) {
    return ctx.t('welcome.guest')
  }
  
  if (user.is('Admin')) {

  }
  
  if (user.isPremium) {
    return ctx.t('welcome.premium', { name: user.displayName })
  }
  
  return ctx.t('welcome.user', { name: user.displayName })
}
```

**В Vue**:

```vue
<template>

    <h1 v-else-if="ctx.user.is('Admin')">

    </h1>
    <h1 v-else>
      {{ $t('welcome.user', { name: ctx.user.displayName }) }}
```

## Настройка workspace

Настройте язык по умолчанию в `.chatiumrc`:

```json
{
  "i18n": {
    "defaultLang": "ru",
    "supportedLangs": ["ru", "en", "de"],
    "fallbackLang": "en"
  }
}
```

**Параметры**:
- `defaultLang` — язык по умолчанию для новых пользователей
- `supportedLangs` — список поддерживаемых языков
- `fallbackLang` — резервный язык, если перевод не найден

## Лучшие практики

### Именование ключей

✅ **Используйте описательные ключи**:
```yaml
# Хорошо
page.profile.title: "Профиль пользователя"
cart.checkout.button: "Оформить заказ"

# Плохо
title: "Профиль"
button: "Оформить"
```

✅ **Группируйте по функциональности**:
```yaml
# Группировка по разделам
auth:
  login: "..."
  register: "..."
  
cart:
  add: "..."
  remove: "..."
```

### Организация переводов

✅ **Синхронизируйте все языки**:
- Добавляя ключ в один файл, добавьте во все остальные
- Используйте одинаковую структуру во всех файлах

✅ **Используйте параметры вместо конкатенации**:
```yaml
# Хорошо
welcome: "Привет, {{name}}! Сегодня {{date}}"

# Плохо (не используйте конкатенацию в коде)
# welcome_part1: "Привет, "
# welcome_part2: "! Сегодня "
```

### Обработка отсутствующих переводов

✅ **Предоставляйте fallback**:
```typescript
const title = ctx.t('page.title', 'Default Title')
```

✅ **Логируйте отсутствующие ключи**:
```typescript
const translation = ctx.t('missing.key')
if (!translation) {
  ctx.account.log('Missing translation', {
    level: 'warn',
    json: { key: 'missing.key', lang: ctx.lang }
  })
}
```

### Тестирование

✅ **Проверяйте UI на всех языках**:
- Убедитесь, что текст помещается в элементы
- Проверьте длинные переводы (особенно немецкий)

✅ **Тестируйте плюралы**:
```typescript
// Тестируйте все формы
for (let i = 0; i <= 25; i++) {
  ctx.account.log(`Count ${i}:`, ctx.t('items.count', { count: i }))
}
```

### Производительность

✅ **Кешируйте редко меняющиеся переводы**:
```vue
<script setup>
import { computed } from 'vue'

// Вычисляется один раз
const pageTitle = computed(() => ctx.t('page.title'))
</script>
```

✅ **Не вызывайте ctx.t() в циклах**:
```vue
<script setup>
// Хорошо - перевод вне цикла
const itemLabel = ctx.t('items.label')

const items = computed(() => {
  return products.value.map(p => ({
    ...p,
    label: itemLabel // Используем готовый перевод
  }))
})
</script>
```
