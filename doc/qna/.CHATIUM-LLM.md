# QnA Project Documentation

## Описание
Workspace QnA - это внутренняя knowledge base с документацией по Chatium-специфичным функциям и практикам.

## Структура проекта

### Файлы документации
- **001-run.md** - Руководство по `@shared-route` декоратору и методу `.run()`
  - Как объявлять API маршруты с доступом на фронте и бекенде
  - Примеры использования в Vue компонентах и серверном коде
  - Правила и лучшие практики

- **003-fingerprint.md** - Серверный фингерпринт для дедупликации
  - Генерация уникального ID клиента на основе IP, User-Agent, языка браузера
  - Альтернатива клиентскому счётчику (без JS загрузки)
  - Логика дедупликации кликов за окно времени (5 сек)
  - Примеры с LinkClicks таблицей

### Серверные файлы
- **index.tsx** - Entry point приложения (минималистичная страница)

## Архитектура

### Назначение
Knowledge base для разработчиков Chatium - быстрый доступ к примерам и объяснениям частых задач.

### Технологии
- Vue 3 (фронтенд компоненты, если будут добавлены)
- TypeScript (примеры кода)
- Heap Tables (хранение данных)

## Добавляемые документы
При добавлении новой документации:
1. Создай файл `NNN-topic.md` (где NNN - номер)
2. Включай примеры кода, таблицы, good/bad practices
3. Ориентируйся на 1200-2000 символов
4. Обновляй этот файл (.CHATIUM-LLM.md)

## Важные замечания
- Все примеры кода ориентированы на реальный контекст Chatium
- Не используются стандартные nodejs модули (crypto, fs, http)
- Документация практическая, а не теоретическая
