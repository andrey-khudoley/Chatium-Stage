# Проект assistant

## Что это
Проект `p/assistant` в Chatium с минимальными страницами и документацией. Общие стили: `styles.tsx` → `formControlStyles` (фокус, autofill), `customScrollbarStyles`, `mobileSafeAreaStyles` и `VIEWPORT_META_CONTENT` (viewport-fit, safe-area для body/модалок; шапка/футер дополнительно в `Header`/`AppFooter`). Выпадающие списки в модалках задач — `components/JnCrtSelect.vue` (панель через `Teleport` в `body`, `fixed`, непрозрачный фон); модалка задачи — класс `crt-form-panel` (сканлайны); дети модалки с `z-index` — `> *:not(.jn-crt-select)`. Главная с карточками; «Мой журнал» (`./web/journal/`, `JournalPage.vue`, блокнот — `JournalNotebookPane.vue`, API `api/journal/notes/`; вкладки журнала — `?tab=month|week|day|habits` (блокнот без параметра), SSR `journalTabInitial`; вкладка «День» — `JournalDayPane.vue`, задачи «В работе», модалка редактирования по клику на название (`api/tasks/items/update`), `daySortOrder`, API `api/tasks/items/reorder-day`, `release-day`). «Управление задачами» (`./web/tasks/`, `TasksPage.vue`) — иерархия клиент → проект → задача (у проекта и у задачи опционально «Детали» в Heap и UI; служебное поле `context` в Heap у проекта и задачи без выдачи в API; у задачи бывшее поле `description` заменено на `details`/`context`) (имя клиента в сайдбаре только подпись; красная подсветка клиента при выбранном его проекте — акцент слева через inset box-shadow; DnD порядка клиентов/проектов за рукоять, `api/tasks/clients/reorder`, `api/tasks/projects/reorder` (UX DnD не идеален, сценарий рабочий — см. architecture); в модалках кнопки `width: auto`), кнопка «Отметить» (переключение «К выполнению» ↔ «В работе»; при «В работе» — красная подсветка), кнопка «Сформулировать задачу» (AI-генерация через `api/tasks/ai-formulate`, настройки в админке), query `?client=` / `?project=`, Heap и API `api/tasks/` (tree, CRUD, reorder, reorder-day, release-day, ai-formulate), DTO в `lib/tasks-types.ts`; обновление проекта по имени — без `clientId` в теле (иначе подставлялся бы клиент из выделения в сайдбаре). Heap: `Table.update(ctx, { id, ... })`, не три аргумента. Прелоадер: `shared/preloader.ts` — после `load` и `document.fonts.ready` — `boot-static-ready`, резерв 12 с / скрытие 15 с; `shared/bootUi.ts` — первая строка `// @shared`, шрифты (до 10 с), nextTick, rAF, `hideBootLoader`. AI-настройки: дефолты в `config/prompts.tsx` (системный промпт AI-формулирования: опора на факты из контекста, без выдуманных taskId, ответ только JSON без Markdown-обёртки, без длинного тире U+2014 в текстах; **критическое правило разделения** — события с датами/действия → create_task, общая инфо о проекте → update_project, с примерами (few-shot); список моделей `AVAILABLE_AI_MODELS` — от худшей к лучшей для этой задачи, см. комментарий в файле), UI в `components/admin/AiSettings.vue` импортирует те же константы из `config/prompts`, сохранение через `api/settings/save`.
