# Проект assistant

## Что это
Проект `k/assistant` в Chatium с минимальными страницами и документацией. Общие стили: `styles.tsx` → `formControlStyles` (фокус, autofill), `customScrollbarStyles`, `mobileSafeAreaStyles` и `VIEWPORT_META_CONTENT` (viewport-fit, safe-area для body/модалок; шапка/футер дополнительно в `Header`/`AppFooter`). Выпадающие списки в модалках задач — `components/JnCrtSelect.vue` (панель через `Teleport` в `body`, `fixed`, непрозрачный фон); модалка задачи — класс `crt-form-panel` (сканлайны); дети модалки с `z-index` — `> *:not(.jn-crt-select)`. Главная с карточками быстрых разделов: «Мой журнал», «Мои задачи» и «Инструменты» — ссылки с переходом; «Диалоги», «PARA», «Финансы», «Сервисы», «Библиотека» — action-кнопки, которые запускают только `triggerGlitch` без навигации и без бизнес-логики; сетка карточек адаптирована под 8 элементов (desktop 4 колонки, tablet 2, mobile 1). «Инструменты» (`./web/tools/`, `ToolsPage.vue`) сейчас содержат карточку перехода в `Pomodoro`. 

**Pomodoro (премиум-версия)**: `./web/pomodoro/`, `PomodoroPage.vue` — личный таймер с премиальным UX: динамическая тема (градиенты фона `theme-work`/`theme-rest`/`theme-long_rest`, смена CSS-переменных `--pomodoro-phase-color`), SVG-дуга с градиентами (`PomodoroTimerDial`, `linearGradient`, filter `glow`, pulse-анимация при `running`), моноширинный таймер, звук при смене фазы — пять пресетов Web Audio API (`lib/pomodoro-phase-sounds.ts`, поле `phaseChangeSound` 1–5 в Heap и в настройках), браузерные уведомления (Web Notifications API, запрос разрешения при mount), вибрация на мобильных (Vibration API), обновление `document.title` с таймером в неактивной вкладке; выбор задачи через `PomodoroTaskSelector` (API `api/tasks/in-progress`, dropdown с задачами статуса `in_progress`); автопилот — `autoStartRest`, `autoStartNextCycle` (настройки в `PomodoroSettingsModal`, рамка — `inset box-shadow`, не `border` с `clip-path`, сохранение в Heap, применение в `lib/pomodoro.lib.ts` при переходах фаз); статистика за день — `tasksCompletedToday` (инкремент при завершении work-фазы), визуализация в `.stat-card` (количество помидорок, время работы/отдыха); хранение состояния в Heap `tables/pomodoro-state.table.ts` (в т.ч. `autoStartRest`, `autoStartNextCycle`, `phaseChangeSound`, `tasksCompletedToday`), а полный журнал сегментов запусков — в Heap `tables/pomodoro-launches.table.ts` (каждый старт/продолжение, фаза, задача, источник запуска, причина завершения, длительность сегмента); бизнес-логика `lib/pomodoro.lib.ts` (автопилот влияет на `pauseAfter` при смене фаз, `runWithExclusiveLock`, автоматическое открытие/закрытие сегментов лога), репозитории `repos/pomodoro.repo.ts` и `repos/pomodoro-launches.repo.ts`, API `api/pomodoro/*` (`state/get`, `control`, `settings/save`, `assign-task`), `api/tasks/in-progress`. В `Header.vue` при активном Pomodoro показывается таймер и кнопка pause/resume (синхронизация через `pomodoroStateGetUrl` + `pomodoroControlUrl`). В `TasksPage.vue` есть действие привязки задачи к Pomodoro (`api/pomodoro/assign-task`) и передача URL Pomodoro API в шапку страницы задач.

«Мой журнал» (`./web/journal/`, `JournalPage.vue`, блокнот — `JournalNotebookPane.vue`, API `api/journal/notes/`; вкладки журнала — `?tab=month|week|tasks|day|habits` (блокнот без параметра), SSR `journalTabInitial`; в левом меню постоянный список вкладок расположен выше условного тулбара «Новая заметка», чтобы при переключении не смещались постоянные элементы; вкладка «Задачи» (`tasks`) — `JournalDayPane.vue`, задачи «В работе», модалка редактирования по клику на название (`api/tasks/items/update`), `daySortOrder`, API `api/tasks/items/reorder-day`, `release-day`; вкладка «День» (`day`) — `JournalDayInDevelopmentPane.vue` с 4 сегментами (`Ночь`, `Утро`, `День`, `Вечер`) и сохранением фиксации/разблокировки по `dayKey` (граница дня в 05:00, `api/journal/day/get|save`, `tables/journal-day-entries.table.ts`, `lib/journal-day-key.ts`); кнопки вкладок в левом меню всегда имеют видимую левую границу, активная вкладка выделяется акцентным цветом). «Управление задачами» (`./web/tasks/`, `TasksPage.vue`) — на десктопе три колонки (дерево, задачи, чат с AI: `components/tasks/TasksAiChatPanel.vue`, фиды `tasks-ai-chat-*`, сброс `tasks-ai-chat-reset`, история в `@app/feed`, вебсокет + polling; порядок сообщений в ленте нормализуется `shared/tasks-ai-chat-message-order.ts` — хронология старые→новые); на мобильном чат из FAB в шторке; иерархия клиент → проект → задача (у проекта и у задачи опционально «Детали» в Heap и UI; служебное поле `context` в Heap у проекта и задачи без выдачи в API; у задачи бывшее поле `description` заменено на `details`/`context`) (имя клиента в сайдбаре только подпись; красная подсветка клиента при выбранном его проекте — акцент слева через inset box-shadow; DnD порядка клиентов/проектов за рукоять, `api/tasks/clients/reorder`, `api/tasks/projects/reorder` (UX DnD не идеален, сценарий рабочий — см. architecture); в модалках кнопки `width: auto`), кнопка «Отметить» (переключение «К выполнению» ↔ «В работе»; при «В работе» — красная подсветка), чат с AI (системный промпт `getAiFormulateSystemPrompt` + `TASKS_AI_CHAT_JSON_APPENDIX`; перед completion в system подмешивается свежий блок проекта/задач из Heap, включая служебные `context` у проекта и задач (`buildTaskAiChatProjectContextBlock`) с явной меткой, что это служебный контекст; последнее сообщение пользователя отправляется отдельным user-блоком «ТЕКУЩЕЕ СООБЩЕНИЕ ПОЛЬЗОВАТЕЛЯ», чтобы модель не путала контекст с новым запросом; ответ модели — JSON с `reply`, `actions`, `summary`; рамка проекта — `update_project.context` (не дублировать список задач), по задачам — `update_task`/`create_task` с полным слитым `context` при необходимости; применение в Heap — `tasks-ai-formulate-apply.ts` из completion; в ленту пишется `reply`; отдельного роута `ai-formulate` нет; для reorder новых задач в том же ответе — плейсхолдеры `$new:0`, `$new:1`, …; настройки в админке), query `?client=` / `?project=`, Heap и API `api/tasks/` (tree, CRUD, reorder, reorder-day, release-day, tasks-ai-chat-*), DTO в `lib/tasks-types.ts`; обновление проекта по имени — без `clientId` в теле (иначе подставлялся бы клиент из выделения в сайдбаре). Heap: `Table.update(ctx, { id, ... })`, не три аргумента. Прелоадер: `shared/preloader.ts` — после `load` и `document.fonts.ready` — `boot-static-ready`, резерв 12 с / скрытие 15 с; `shared/bootUi.ts` — первая строка `// @shared`, шрифты (до 10 с), nextTick, rAF, `hideBootLoader`. AI-настройки: дефолты в `config/prompts.tsx` (системный промпт AI-формулирования / чата с задачами: массив действий (create_task с priority и декомпозицией шагов, update_task с title/priority, delete_task, reorder_tasks, update_project), опора на факты из контекста, без выдуманных taskId, ответ только JSON без Markdown-обёртки, без длинного тире U+2014 в текстах; **критическое правило разделения** — события с датами/действия → create_task, рамка проекта (не дублирование списка задач) → update_project, с примерами (few-shot); список моделей `AVAILABLE_AI_MODELS` — от худшей к лучшей для этой задачи, см. комментарий в файле), UI в `components/admin/AiSettings.vue` импортирует те же константы из `config/prompts`, сохранение через `api/settings/save`.

## Критические правила
- Не хардкодить URL, использовать `config/routes.tsx`.
- Документацию обновлять при любых изменениях структуры/логики.
- Структуру и стили левого меню журнала держать в `components/journal/JournalNav.vue`; `JournalPage.vue` не должен содержать прямую разметку `nav.journal-nav`.
- В `JournalPage.vue` для вкладки `tasks` допускается только переход по переданному `tasksPageUrl` (кнопка `Все задачи`), без хардкода путей.
- Динамические кнопки в `JournalPage.vue` (`Новая заметка` и `Все задачи`) должны рендериться после `journal-nav-divider`; для action-кнопок использовать кастомный `:focus-visible` в цветах темы (без системной жёлтой обводки браузера).
- Для кнопок вкладок журнала (`.journal-nav-btn`) также использовать кастомный `:focus-visible` в цветах темы и не оставлять браузерный системный outline.
- Порядок вкладок в левом меню журнала фиксированный: `tasks -> day -> week -> month -> habits -> notebook` (по лейблам: `Задачи, День, Неделя, Месяц, Привычки, Блокнот`).
- Для Pomodoro любые мутации состояния выполнять только через `lib/pomodoro.lib.ts` под `runWithExclusiveLock`; запись в `tables/pomodoro-state` напрямую из API/страниц не делать.
- Модули `lib/pomodoro-types.ts`, `lib/pomodoro-phase-sounds.ts` и `lib/pomodoro-stats-day.ts`, импортируемые из Vue, должны начинаться с `// @shared` (иначе ошибка сборки про отсутствие shared file mark).
- Для интеграции Pomodoro в шапке страницы передавать `pomodoroStateGetUrl` и `pomodoroControlUrl` в `Header.vue`; без этих URL таймер/кнопка не активируются; запросы дополнять `statsDayKey` (см. `lib/pomodoro-stats-day.ts`) для корректного сброса дневной статистики в 05:00 локально.

- В `styles.tsx` (`formControlStyles`) добавлено глобальное low-specificity правило для текстовых контролов (`input` кроме checkbox/radio/range/color/file, `textarea`, `select`): на `:focus/:focus-visible` отключается системный `outline` и browser `box-shadow`, чтобы по всему проекту не появлялась нативная жёлтая обводка; компонентные фокус-стили остаются приоритетными.
- В `components/JnCrtSelect.vue` селект больше не зависит от внешнего класса `jn-input`: у `.jn-crt-select__trigger` заданы собственные базовые стили поля (`padding`, `border`, `background`, `color`, `font-size`, фокус-обводка), что устраняет «сломанный» вид в scoped-модалке `JournalDayPane`; у `.jn-crt-select__value` сохранены `display: block` + наследование `line-height`/`letter-spacing` для стабильного рендера текста.
- Для аналитики Pomodoro добавлена таблица `tables/pomodoro-launches.table.ts` и репозиторий `repos/pomodoro-launches.repo.ts`; в `lib/pomodoro.lib.ts` лог сегментов обновляется на всех переходах состояния: `start/resume/pause/stop`, автопереходы фаз, смена задачи во время `running`, а также recovery-закрытие «зависшего» сегмента при несогласованном состоянии.
- Для `web/tools/index.tsx` и `web/pomodoro/index.tsx` обязательно задавать базовые CSS-переменные темы (`:root`), фон `html/body`, шрифт и подключение `customScrollbarStyles`/`mobileSafeAreaStyles`, иначе страницы визуально выглядят как «белый экран».
- В `ToolsPage.vue` и `PomodoroPage.vue` `app-layout` должен иметь `min-height: 100vh`, чтобы `content-wrapper` с `flex-1` не схлопывался.
- `PomodoroPage.vue` должен показывать fallback-состояние загрузки до получения `state` (не оставлять пустой `main` при ожидании API).
- Для API `api/pomodoro/state/get.ts` и `api/pomodoro/settings/save.ts` корректный импорт `pomodoro.lib` — `../../../lib/pomodoro.lib`; путь `../../../../lib/pomodoro.lib` приводит к runtime `HTTP 500` на странице `/web/pomodoro`.
- **Импорт таблиц Heap**: используй `import TableName from '../tables/table-name.table'` (default import), а не `import { TableName } from ...` (named import). Неправильный импорт приводит к ошибке «Не удалось получить состояние» в runtime.
- Контроль Pomodoro: `start` — рабочая фаза и `cyclesCompleted=0` (на странице — только из `stopped`); `resume` — из `paused` или `awaiting_continue`; `pause` — из `running`; `skip` — пропуск фазы (на странице — только при `running`); `reset` — полная остановка с обнулением циклов (`stopped`, полный work); при пропуске во время отсчёта в суммы и задачу не доначисляется остаток фазы; `stop` — в API сохранён, в UI страницы Pomodoro не выводится. После естественного окончания фазы по таймеру статус `awaiting_continue`, `phaseEndsAtMs` — момент окончания фазы (начало секундомера овертайма на клиенте).
- Текущая UX-семантика `PomodoroPage`: compact layout, основной таймер отображается как SVG-дуга (`PomodoroTimerDial`), настройки вынесены в отдельную модалку (`PomodoroSettingsModal`), inline-поля на странице отсутствуют; черновик настроек в модалке синхронизируется с `modelValue` только при открытии (не при каждом poll), иначе числовые поля сбрасывались бы к сохранённым значениям.
- Для устойчивости UI Pomodoro использовать защиту от гонок: блокировка повторного control-действия (`actionPending`), приоритет более свежего `serverNowMs`, принудительный `refresh` после каждого control, обновление состояния при `focus`/`visibilitychange`.
- **Премиум-функционал Pomodoro**: динамическая тема через классы `.theme-work`/`.theme-rest`/`.theme-long_rest` на `.app-layout` (градиенты фона, переменная `--pomodoro-phase-color`); SVG-градиенты на дуге (`linearGradient` в `PomodoroTimerDial`), мягкое статичное свечение `filter: url(#glow)` без пульсации в `running` (чтобы не мигал белый blur-фон); звук при смене фазы — пять пресетов (`phaseChangeSound` 1–5, `lib/pomodoro-phase-sounds.ts`, выбор в модалке настроек); браузерные уведомления (Web Notifications API, запрос разрешения); вибрация (Vibration API, паттерн `[200, 100, 200]`); обновление `document.title` с таймером и эмодзи фазы в фоновой вкладке; выбор задачи через `PomodoroTaskSelector` (`api/tasks/in-progress`); настройки `autoStartRest`/`autoStartNextCycle`/`pauseAfter*` в Heap сохраняются, но автоматический переход к следующей фазе по таймеру отключён — после окончания фазы `awaiting_continue` и овертайм до «Продолжить»/«Пропустить»; статистика за день (`.stat-card`, `tasksCompletedToday` инкрементируется при завершении work-фазы по таймеру или при пропуске работы); Vue `transition` для смены фаз.
- **Тюнинг мигания таймера**: в `PomodoroTimerDial.vue` мигание в `status=paused` сделано едва заметным: `paused-blink` и `time-paused-blink` замедлены до `2.6s`, минимум opacity повышен (дуга `0.5`, время `0.88`), `dial-scanline-flicker` ослаблен (диапазон `0.36..0.42`).
- **Устранение белого blur-пульса**: в `PomodoroTimerDial.vue` удалена анимация `progress-pulse` и связанный `drop-shadow` для `status=running`; интенсивность SVG-фильтров снижена (`glow: 3`, `glow-strong: 4`), чтобы убрать заметное мигание ореола под дугой.
- **CRT-стиль Pomodoro**: `web/pomodoro/index.tsx` включает полный набор CRT-слоёв (geometric-bg с виньеткой и SVG-сеткой, scanlines body::after, bezel body::before, crt-power-on анимация), идентичный `index.tsx` главной. Все элементы UI используют `clip-path` polygon для пиксельных углов (2px–6px), `repeating-linear-gradient` для scanlines overlay, `text-shadow` с RGB chromatic aberration. Phase bar (`.pomodoro-phase-bar`) — индикатор фазы с пульсирующей точкой, cycle dots прогресса, кнопка настроек. Кнопки `.pomo-btn` с вариантами `--primary`/`--secondary`/`--ghost`/`--danger`: scanlines `::before`, underline `::after` на hover. `PomodoroTimerDial` — 60 tick-марок, `glow-strong` filter, blink-анимация при паузе. `PomodoroSettingsModal` — секции с иконками, grid числовых полей, кастомные чекбоксы/select. `PomodoroTaskSelector` — scanlines overlay на toggle и dropdown.
- **Кнопки управления Pomodoro**: под таймером `.pomodoro-actions` → панель `.pomodoro-actions__panel`, класс раскладки `pomodoroActionPanelLayout`: `--n1` (только «Старт» при `stopped`, по центру), `--n2` (две колонки: при `running` — «Пауза» + «Пропустить»; при `paused` / `awaiting_continue` — «Продолжить» + «Сбросить`, действие `reset`). Классы `.pomo-btn`: `--primary`/`--secondary`/`--ghost`/`--danger`; подписи uppercase; `:focus-visible`; `actionPending`; safe-area в `.pomodoro-shell`.
- **Формат времени Pomodoro на клиенте**: `lib/pomodoro-types.ts` → `formatPomodoroSecondsDisplay` — до 59:59 — `MM:SS`, с 1 ч — `H:MM:SS` (статистика `.stat-cell`, таймер, `document.title`, таймер в `Header.vue`).
