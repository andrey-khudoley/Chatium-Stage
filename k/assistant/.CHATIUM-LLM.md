# Проект assistant

## Что это
Проект `k/assistant` в Chatium с минимальными страницами и документацией. Общие стили: `styles.tsx` → `formControlStyles` (фокус, autofill), `customScrollbarStyles`, `mobileSafeAreaStyles` и `VIEWPORT_META_CONTENT` (viewport-fit, safe-area для body/модалок; шапка/футер дополнительно в `Header`/`AppFooter`). Выпадающие списки в модалках задач — `components/JnCrtSelect.vue` (панель через `Teleport` в `body`, `fixed`, непрозрачный фон); модалка задачи — класс `crt-form-panel` (сканлайны); дети модалки с `z-index` — `> *:not(.jn-crt-select)`. Главная с карточками быстрых разделов: «Мой журнал», «Мои задачи» и «Инструменты» — ссылки с переходом; «Диалоги», «PARA», «Финансы», «Сервисы», «Библиотека» — action-кнопки, которые запускают только `triggerGlitch` без навигации и без бизнес-логики; сетка карточек адаптирована под 8 элементов (desktop 4 колонки, tablet 2, mobile 1). Раздел «Инструменты» (`./web/tools/`, `ToolsPage.vue`) содержит одну карточку `Таймеры`, которая ведёт на `./web/timers`; внутри `PomodoroPage` реализованы вкладки `Помидор / Таймер / Секундомер`. Для вкладок `Таймер/Секундомер` используется `components/pomodoro/FocusClockPane.vue` (селектор задачи, модалка настроек, счётчики, фокус-чекбокс, запись в `POST /api/tools/focus-log`).

**Pomodoro (премиум-версия)**: `./web/timers/`, `PomodoroPage.vue` — личный таймер с премиальным UX: динамическая тема (градиенты фона `theme-work`/`theme-rest`/`theme-long_rest`, смена CSS-переменных `--pomodoro-phase-color`), SVG-дуга с градиентами (`PomodoroTimerDial`, `linearGradient`, filter `glow`, pulse-анимация при `running`), моноширинный таймер, звук при смене фазы — пять пресетов Web Audio API (`lib/pomodoro-phase-sounds.ts`, поле `phaseChangeSound` 1–5 в Heap и в настройках), браузерные уведомления (Web Notifications API, запрос разрешения при mount), вибрация на мобильных (Vibration API), обновление `document.title` с таймером в неактивной вкладке; выбор задачи через `PomodoroTaskSelector` (API `api/tasks/in-progress`, dropdown с задачами статуса `in_progress`); поведение после завершения фазы настраивается отдельно для работы/отдыха/длинного отдыха в `PomodoroSettingsModal` с 3 режимами (`auto`/`pause`/`overtime`), совместимость с legacy `afterLongRest=stop` сохранена; статистика за день — `tasksCompletedToday` (инкремент при завершении work-фазы), визуализация в `.stat-card` (количество помидорок, время работы/отдыха); хранение состояния в Heap `tables/pomodoro-state.table.ts` (в т.ч. `autoStartRest`, `autoStartNextCycle`, `pauseAfter*`, `afterLongRest`, `phaseChangeSound`, `tasksCompletedToday`), а полный журнал сегментов запусков — в Heap `tables/pomodoro-launches.table.ts` (каждый старт/продолжение, фаза, задача, источник запуска, причина завершения, длительность сегмента); бизнес-логика `lib/pomodoro.lib.ts` (режимы завершения фаз, `runWithExclusiveLock`, автоматическое открытие/закрытие сегментов лога), репозитории `repos/pomodoro.repo.ts` и `repos/pomodoro-launches.repo.ts`, API `api/pomodoro/*` (`state/get`, `control`, `settings/save`, `assign-task`), `api/tasks/in-progress`. В `Header.vue` для `/web/timers` работает tool-widget в `header-clock`: при первом клике по часам выбирается `Помидор/Таймер/Секундомер`, после выбора доступны `Запуск/Пауза/Сброс`; pomodoro управляется через `pomodoroControlUrl`, timer использует текущие настройки длительности из `localStorage` (`assistant:focus-clock-settings:timer`), которые задаются на вкладке `Таймер` в `FocusClockPane.vue`. В `TasksPage.vue` есть действие привязки задачи к Pomodoro (`api/pomodoro/assign-task`) и передача URL Pomodoro API в шапку страницы задач.

«Мой журнал» (`./web/journal/`, `JournalPage.vue`, блокнот — `JournalNotebookPane.vue` (split-view: `NotebookFolderSidebar` + список заметок `NotebookNoteCard`, `NotebookFilterBar`, мультиселект + `NotebookBulkBar`, полностраничный `NotebookNoteEditor`; кнопка «Сохранить» в редакторе сохраняет без закрытия, к списку — «Назад к списку»; поле даты в редакторе вводится в формате `dd/mm/yyyy`, при сохранении конвертируется в ISO), API `api/journal/notes/` + `api/journal/categories/` + `api/journal/folders/`; Heap-таблицы: `journal-notes` (расширена: folderId, categoryIds, linkedTaskId, linkedProjectId, linkedClientId, noteDate, isArchived, sortOrder), `notebook-categories`, `notebook-folders`; репозитории: `journal-notes.repo` (расширен: reorder, bulk-операции, архивация), `notebook-categories.repo`, `notebook-folders.repo`; вкладки журнала — `?tab=month|week|tasks|day|habits` (блокнот без параметра), SSR `journalTabInitial`; в левом меню постоянный список вкладок, кнопка «Новая заметка» убрана из навигации; вкладка «Задачи» (`tasks`) — `JournalDayPane.vue`, задачи «В работе», модалка редактирования по клику на название (`api/tasks/items/update`), `daySortOrder`, API `api/tasks/items/reorder-day`, `release-day`; вкладка «День» (`day`) — `JournalDayInDevelopmentPane.vue` с 4 сегментами (`Ночь`, `Утро`, `День`, `Вечер`) и сохранением фиксации/разблокировки по `dayKey` (граница дня в 05:00, `api/journal/day/get|save`, `tables/journal-day-entries.table.ts`, `lib/journal-day-key.ts`); кнопки вкладок в левом меню всегда имеют видимую левую границу, активная вкладка выделяется акцентным цветом). «Управление задачами» (`./web/tasks/`, `TasksPage.vue`) — на десктопе три колонки (дерево, задачи, чат с AI: `components/tasks/TasksAiChatPanel.vue`, фиды `tasks-ai-chat-*`, сброс `tasks-ai-chat-reset`, история в `@app/feed`, вебсокет + polling; порядок сообщений в ленте нормализуется `shared/tasks-ai-chat-message-order.ts` — хронология старые→новые); на мобильном чат из FAB в шторке; иерархия клиент → проект → задача (у проекта и у задачи опционально «Детали» в Heap и UI; служебное поле `context` в Heap у проекта и задачи без выдачи в API; у задачи бывшее поле `description` заменено на `details`/`context`) (имя клиента в сайдбаре только подпись; красная подсветка клиента при выбранном его проекте — акцент слева через inset box-shadow; DnD порядка клиентов/проектов за рукоять, `api/tasks/clients/reorder`, `api/tasks/projects/reorder` (UX DnD не идеален, сценарий рабочий — см. architecture); в модалках кнопки `width: auto`), кнопка «Отметить» (переключение «К выполнению» ↔ «В работе»; при «В работе» — красная подсветка), чат с AI (системный промпт `getAiFormulateSystemPrompt` + `TASKS_AI_CHAT_JSON_APPENDIX`; перед completion в system подмешивается свежий блок проекта/задач из Heap, включая служебные `context` у проекта и задач (`buildTaskAiChatProjectContextBlock`) с явной меткой, что это служебный контекст; последнее сообщение пользователя отправляется отдельным user-блоком «ТЕКУЩЕЕ СООБЩЕНИЕ ПОЛЬЗОВАТЕЛЯ», чтобы модель не путала контекст с новым запросом; ответ модели — JSON с `reply`, `actions`, `summary`; рамка проекта — `update_project.context` (не дублировать список задач), по задачам — `update_task`/`create_task` с полным слитым `context` при необходимости; применение в Heap — `tasks-ai-formulate-apply.ts` из completion; в ленту пишется `reply`; отдельного роута `ai-formulate` нет; для reorder новых задач в том же ответе — плейсхолдеры `$new:0`, `$new:1`, …; настройки в админке), query `?client=` / `?project=`, Heap и API `api/tasks/` (tree, CRUD, reorder, reorder-day, release-day, tasks-ai-chat-*), DTO в `lib/tasks-types.ts`; обновление проекта по имени — без `clientId` в теле (иначе подставлялся бы клиент из выделения в сайдбаре). Heap: `Table.update(ctx, { id, ... })`, не три аргумента. Прелоадер: `shared/preloader.ts` — после `load` и `document.fonts.ready` — `boot-static-ready`, резерв 12 с / скрытие 15 с; `shared/bootUi.ts` — первая строка `// @shared`, шрифты (до 10 с), nextTick, rAF, `hideBootLoader`. AI-настройки: дефолты в `config/prompts.tsx` (системный промпт AI-формулирования / чата с задачами: массив действий (create_task с priority и декомпозицией шагов, update_task с title/priority, delete_task, reorder_tasks, update_project), опора на факты из контекста, без выдуманных taskId, ответ только JSON без Markdown-обёртки, без длинного тире U+2014 в текстах; **критическое правило разделения** — события с датами/действия → create_task, рамка проекта (не дублирование списка задач) → update_project, с примерами (few-shot); список моделей `AVAILABLE_AI_MODELS` — от худшей к лучшей для этой задачи, см. комментарий в файле), UI в `components/admin/AiSettings.vue` импортирует те же константы из `config/prompts`, сохранение через `api/settings/save`.

## Критические правила
- Не хардкодить URL, использовать `config/routes.tsx`.
- Документацию обновлять при любых изменениях структуры/логики.
- Структуру и стили левого меню журнала держать в `components/journal/JournalNav.vue`; `JournalPage.vue` не должен содержать прямую разметку `nav.journal-nav`.
- В `JournalPage.vue` для вкладки `tasks` допускается только переход по переданному `tasksPageUrl` (кнопка `Все задачи`), без хардкода путей.
- Динамические кнопки в `JournalPage.vue` (`Новая заметка` и `Все задачи`) должны рендериться после `journal-nav-divider`; для action-кнопок использовать кастомный `:focus-visible` в цветах темы (без системной жёлтой обводки браузера).
- Для кнопок вкладок журнала (`.journal-nav-btn`) также использовать кастомный `:focus-visible` в цветах темы и не оставлять браузерный системный outline.
- Порядок вкладок в левом меню журнала фиксированный: `tasks -> day -> week -> month -> habits -> notebook` (по лейблам: `Задачи, День, Неделя, Месяц, Привычки, Блокнот`).
- Для Pomodoro любые мутации состояния выполнять только через `lib/pomodoro.lib.ts` под `runWithExclusiveLock`; запись в `tables/pomodoro-state` напрямую из API/страниц не делать.
- Модули `lib/pomodoro-types.ts`, `lib/pomodoro-phase-sounds.ts` и `lib/pomodoro-stats-day.ts`, импортируемые из Vue, должны начинаться с `// @shared` (иначе ошибка сборки про отсутствие shared file mark).
- Для интеграции Pomodoro в шапке страницы передавать `pomodoroStateGetUrl` и `pomodoroControlUrl` в `Header.vue`; без этих URL таймер/кнопка не активируются; запросы дополнять `statsDayKey` (см. `lib/pomodoro-stats-day.ts`) для корректного сброса дневной статистики в 05:00 локально.
- Чтобы включить новый виджет `header-clock` только на нужной странице, передавать в `Header.vue` проп `enableToolClockWidget`; в проекте он включён на `PomodoroPage.vue`, на остальных страницах шапка остаётся в режиме обычных часов.
- Внутреннее состояние виджета (`выбранный режим`, `status` и тайминги локальных `timer/stopwatch`) сохраняется в `localStorage` (`assistant:header-clock-widget:v1`), поэтому после reload режим и время продолжают корректно отображаться; параллельный запуск запрещён — при старте одного инструмента остальные ставятся на паузу.
- При первом рендере `Header.vue` автоматически переключается на активный инструмент, если обнаружен `running`/`awaiting_continue` Pomodoro или `running` локальный timer/stopwatch; если активного инструмента нет, остаётся выбранный пользователем режим. Логика применяется на всех страницах, где используется `Header`.
- Выбор режима в `header-clock` оформлен как выпадающий вертикальный picker под часами (`Часы/Помидор/Таймер/Секундомер`), чтобы не ломать header по ширине на мобильных и узких экранах; сохранены иконки, активное состояние (`header-tool-picker-btn--active`), `aria-pressed`, `focus-visible`, закрытие по `Escape` и по клику вне picker-блока.
- Подложка `header-tool-picker` должна быть непрозрачной (`var(--color-bg-secondary)`), чтобы элементы контента не просвечивали за выпадающим меню.

- Вставка из буфера в `WysiwygEditor` работает в режиме smart-paste: если доступен `text/html`, вставляется готовая разметка; если есть только `text/plain`, при признаках Markdown выполняется базовая конвертация в HTML (headers/lists/blockquote/code/hr + inline `**`/`*`/`` ` ``/links), иначе используется построчная вставка как обычного текста.
- В Markdown-ветке smart-paste пустые строки не теряются: каждая пустая строка из буфера превращается в `<p><br></p>`, чтобы визуально сохранялись двойные переносы между блоками.

- При переключении `WYSIWYG -> HTML` source-строка дополнительно нормализуется функцией `normalizeSourceHtml`: удаляются только ведущие `\r/\n` в начале, чтобы в `textarea` не появлялась пустая первая строка, если браузер отдал `innerHTML` с техническим переносом.
- В `WysiwygEditor` усилена синхронизация при переключении `WYSIWYG / HTML`: введён буфер `lastKnownHtml` и fallback-приоритеты источников (`sourceHtml` -> `modelValue` -> `lastKnownHtml`), чтобы при быстрых переключениях не терялось содержимое из-за промежуточной пустой строки.
- В `WysiwygEditor` учтён жизненный цикл `v-if` для `contenteditable`: при возврате из `HTML` в `WYSIWYG` применение `innerHTML` делается в `nextTick`, когда `editorRef` уже заново создан в DOM; иначе контент мог «пропадать» из-за ранней попытки записи.
- При paste приоритет источника динамический: если `text/plain` содержит Markdown-признаки или двойные переносы (`\n\n`), используется plain-text конвертация (чтобы сохранить структуру и пустые строки), даже если в буфере есть `text/html`; в остальных случаях сохраняется вставка из `text/html`.

Кнопки `.pomo-btn` с вариантами `--primary`/`--secondary`/`--ghost`/`--danger`: scanlines `::before`, underline `::after` на hover. `PomodoroTimerDial` — 60 tick-марок, `glow-strong` filter, blink-анимация при паузе. `PomodoroSettingsModal` — секции с иконками, grid числовых полей, кастомные чекбоксы/select. `PomodoroTaskSelector` — scanlines overlay на toggle и dropdown.
- **Кнопки управления Pomodoro**: под таймером `.pomodoro-actions` → панель `.pomodoro-actions__panel`, класс раскладки `pomodoroActionPanelLayout`: `--n1` (только «Старт» при `stopped`, по центру), `--n2` (две колонки: при `running` — «Пауза» + «Пропустить»; при `paused` / `awaiting_continue` — «Продолжить» + «Сбросить`, действие `reset`). Классы `.pomo-btn`: `--primary`/`--secondary`/`--ghost`/`--danger`; подписи uppercase; `:focus-visible`; `actionPending`; safe-area в `.pomodoro-shell`.
- **Формат времени Pomodoro на клиенте**: `lib/pomodoro-types.ts` → `formatPomodoroSecondsDisplay` — до 59:59 — `MM:SS`, с 1 ч — `H:MM:SS` (статистика `.stat-cell`, таймер, `document.title`, таймер в `Header.vue`).
