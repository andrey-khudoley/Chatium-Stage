# Проект assistant

## Что это
Проект `k/assistant` в Chatium с минимальными страницами и документацией. Общие стили: `styles.tsx` → `formControlStyles` (фокус, autofill), `customScrollbarStyles`, `mobileSafeAreaStyles` и `VIEWPORT_META_CONTENT` (viewport-fit, safe-area для body/модалок; шапка/футер дополнительно в `Header`/`AppFooter`). Выпадающие списки в модалках задач — `components/JnCrtSelect.vue` (панель через `Teleport` в `body`, `fixed`, непрозрачный фон); модалка задачи — класс `crt-form-panel` (сканлайны); дети модалки с `z-index` — `> *:not(.jn-crt-select)`. Главная с карточками быстрых разделов: «Мой журнал», «Мои задачи» и «Инструменты» — ссылки с переходом; «Диалоги», «PARA», «Финансы», «Сервисы», «Библиотека» — action-кнопки, которые запускают только `triggerGlitch` без навигации и без бизнес-логики; сетка карточек адаптирована под 8 элементов (desktop 4 колонки, tablet 2, mobile 1). «Инструменты» (`./web/tools/`, `ToolsPage.vue`) сейчас содержат карточку перехода в `Pomodoro`. 

**Pomodoro (премиум-версия)**: `./web/pomodoro/`, `PomodoroPage.vue` — личный таймер с премиальным UX: динамическая тема (градиенты фона `theme-work`/`theme-rest`/`theme-long_rest`, смена CSS-переменных `--pomodoro-phase-color`), SVG-дуга с градиентами (`PomodoroTimerDial`, `linearGradient`, filter `glow`, pulse-анимация при `running`), моноширинный таймер, звуковые уведомления (Web Audio API, синтезированный тон при смене фазы), браузерные уведомления (Web Notifications API, запрос разрешения при mount), вибрация на мобильных (Vibration API), обновление `document.title` с таймером в неактивной вкладке; выбор задачи через `PomodoroTaskSelector` (API `api/tasks/in-progress`, dropdown с задачами статуса `in_progress`); автопилот — `autoStartRest`, `autoStartNextCycle` (настройки в `PomodoroSettingsModal`, сохранение в Heap, применение в `lib/pomodoro.lib.ts` при переходах фаз); статистика за день — `tasksCompletedToday` (инкремент при завершении work-фазы), визуализация в `.stat-card` (количество помидорок, время работы/отдыха); хранение состояния в Heap `tables/pomodoro-state.table.ts` (добавлены `autoStartRest`, `autoStartNextCycle`, `tasksCompletedToday`), бизнес-логика `lib/pomodoro.lib.ts` (автопилот влияет на `pauseAfter` при смене фаз, `runWithExclusiveLock`), репозиторий `repos/pomodoro.repo.ts`, API `api/pomodoro/*` (`state/get`, `control`, `settings/save`, `assign-task`), `api/tasks/in-progress`. В `Header.vue` при активном Pomodoro показывается таймер и кнопка pause/resume (синхронизация через `pomodoroStateGetUrl` + `pomodoroControlUrl`). В `TasksPage.vue` есть действие привязки задачи к Pomodoro (`api/pomodoro/assign-task`) и передача URL Pomodoro API в шапку страницы задач.

«Мой журнал» (`./web/journal/`, `JournalPage.vue`, блокнот — `JournalNotebookPane.vue`, API `api/journal/notes/`; вкладки журнала — `?tab=month|week|day|habits` (блокнот без параметра), SSR `journalTabInitial`; в левом меню постоянный список вкладок расположен выше условного тулбара «Новая заметка», чтобы при переключении не смещались постоянные элементы; вкладка «День» — `JournalDayPane.vue`, задачи «В работе», модалка редактирования по клику на название (`api/tasks/items/update`), `daySortOrder`, API `api/tasks/items/reorder-day`, `release-day`; кнопки вкладок в левом меню всегда имеют видимую левую границу, активная вкладка выделяется акцентным цветом). «Управление задачами» (`./web/tasks/`, `TasksPage.vue`) — на десктопе три колонки (дерево, задачи, чат с AI: `components/tasks/TasksAiChatPanel.vue`, фиды `tasks-ai-chat-*`, сброс `tasks-ai-chat-reset`, история в `@app/feed`, вебсокет + polling; порядок сообщений в ленте нормализуется `shared/tasks-ai-chat-message-order.ts` — хронология старые→новые); на мобильном чат из FAB в шторке; иерархия клиент → проект → задача (у проекта и у задачи опционально «Детали» в Heap и UI; служебное поле `context` в Heap у проекта и задачи без выдачи в API; у задачи бывшее поле `description` заменено на `details`/`context`) (имя клиента в сайдбаре только подпись; красная подсветка клиента при выбранном его проекте — акцент слева через inset box-shadow; DnD порядка клиентов/проектов за рукоять, `api/tasks/clients/reorder`, `api/tasks/projects/reorder` (UX DnD не идеален, сценарий рабочий — см. architecture); в модалках кнопки `width: auto`), кнопка «Отметить» (переключение «К выполнению» ↔ «В работе»; при «В работе» — красная подсветка), чат с AI (системный промпт `getAiFormulateSystemPrompt` + `TASKS_AI_CHAT_JSON_APPENDIX`; перед completion в system подмешивается свежий блок проекта/задач из Heap, включая служебные `context` у проекта и задач (`buildTaskAiChatProjectContextBlock`) с явной меткой, что это служебный контекст; последнее сообщение пользователя отправляется отдельным user-блоком «ТЕКУЩЕЕ СООБЩЕНИЕ ПОЛЬЗОВАТЕЛЯ», чтобы модель не путала контекст с новым запросом; ответ модели — JSON с `reply`, `actions`, `summary`; рамка проекта — `update_project.context` (не дублировать список задач), по задачам — `update_task`/`create_task` с полным слитым `context` при необходимости; применение в Heap — `tasks-ai-formulate-apply.ts` из completion; в ленту пишется `reply`; отдельного роута `ai-formulate` нет; для reorder новых задач в том же ответе — плейсхолдеры `$new:0`, `$new:1`, …; настройки в админке), query `?client=` / `?project=`, Heap и API `api/tasks/` (tree, CRUD, reorder, reorder-day, release-day, tasks-ai-chat-*), DTO в `lib/tasks-types.ts`; обновление проекта по имени — без `clientId` в теле (иначе подставлялся бы клиент из выделения в сайдбаре). Heap: `Table.update(ctx, { id, ... })`, не три аргумента. Прелоадер: `shared/preloader.ts` — после `load` и `document.fonts.ready` — `boot-static-ready`, резерв 12 с / скрытие 15 с; `shared/bootUi.ts` — первая строка `// @shared`, шрифты (до 10 с), nextTick, rAF, `hideBootLoader`. AI-настройки: дефолты в `config/prompts.tsx` (системный промпт AI-формулирования / чата с задачами: массив действий (create_task с priority и декомпозицией шагов, update_task с title/priority, delete_task, reorder_tasks, update_project), опора на факты из контекста, без выдуманных taskId, ответ только JSON без Markdown-обёртки, без длинного тире U+2014 в текстах; **критическое правило разделения** — события с датами/действия → create_task, рамка проекта (не дублирование списка задач) → update_project, с примерами (few-shot); список моделей `AVAILABLE_AI_MODELS` — от худшей к лучшей для этой задачи, см. комментарий в файле), UI в `components/admin/AiSettings.vue` импортирует те же константы из `config/prompts`, сохранение через `api/settings/save`.

## Критические правила
- Не хардкодить URL, использовать `config/routes.tsx`.
- Документацию обновлять при любых изменениях структуры/логики.
- Структуру и стили левого меню журнала держать в `components/journal/JournalNav.vue`; `JournalPage.vue` не должен содержать прямую разметку `nav.journal-nav`.
- В `JournalPage.vue` для вкладки `day` допускается только переход по переданному `tasksPageUrl` (кнопка `Все задачи`), без хардкода путей.
- Динамические кнопки в `JournalPage.vue` (`Новая заметка` и `Все задачи`) должны рендериться после `journal-nav-divider`; для action-кнопок использовать кастомный `:focus-visible` в цветах темы (без системной жёлтой обводки браузера).
- Для кнопок вкладок журнала (`.journal-nav-btn`) также использовать кастомный `:focus-visible` в цветах темы и не оставлять браузерный системный outline.
- Порядок вкладок в левом меню журнала фиксированный: `day -> week -> month -> habits -> notebook` (по лейблам: `День, Неделя, Месяц, Привычки, Блокнот`).
- Для Pomodoro любые мутации состояния выполнять только через `lib/pomodoro.lib.ts` под `runWithExclusiveLock`; запись в `tables/pomodoro-state` напрямую из API/страниц не делать.
- Для интеграции Pomodoro в шапке страницы передавать `pomodoroStateGetUrl` и `pomodoroControlUrl` в `Header.vue`; без этих URL таймер/кнопка не активируются.

## Актуализация 2026-03-23
- Для `web/tools/index.tsx` и `web/pomodoro/index.tsx` обязательно задавать базовые CSS-переменные темы (`:root`), фон `html/body`, шрифт и подключение `customScrollbarStyles`/`mobileSafeAreaStyles`, иначе страницы визуально выглядят как «белый экран».
- В `ToolsPage.vue` и `PomodoroPage.vue` `app-layout` должен иметь `min-height: 100vh`, чтобы `content-wrapper` с `flex-1` не схлопывался.
- `PomodoroPage.vue` должен показывать fallback-состояние загрузки до получения `state` (не оставлять пустой `main` при ожидании API).
- Для API `api/pomodoro/state/get.ts` и `api/pomodoro/settings/save.ts` корректный импорт `pomodoro.lib` — `../../../lib/pomodoro.lib`; путь `../../../../lib/pomodoro.lib` приводит к runtime `HTTP 500` на странице `/web/pomodoro`.
- **Импорт таблиц Heap**: используй `import TableName from '../tables/table-name.table'` (default import), а не `import { TableName } from ...` (named import). Неправильный импорт приводит к ошибке «Не удалось получить состояние» в runtime.
- Контроль Pomodoro должен соблюдать серверную семантику действий: `start` всегда запускает/перезапускает рабочую фазу (`phase=work`, полный `workMinutes`) из любого состояния; `resume` действует только из `paused`; `pause` — только из `running`; `stop` переводит в `stopped`.
- Текущая UX-семантика `PomodoroPage`: compact layout, основной таймер отображается как SVG-дуга (`PomodoroTimerDial`), настройки вынесены в отдельную модалку (`PomodoroSettingsModal`), inline-поля на странице отсутствуют.
- Для устойчивости UI Pomodoro использовать защиту от гонок: блокировка повторного control-действия (`actionPending`), приоритет более свежего `serverNowMs`, принудительный `refresh` после каждого control, обновление состояния при `focus`/`visibilitychange`.
- **Премиум-функционал Pomodoro**: динамическая тема через классы `.theme-work`/`.theme-rest`/`.theme-long_rest` на `.app-layout` (градиенты фона, переменная `--pomodoro-phase-color`); SVG-градиенты на дуге (`linearGradient` в `PomodoroTimerDial`), filter `glow`, pulse-анимация при `running`; звук при смене фазы (Web Audio API, синтезированный тон 880Hz); браузерные уведомления (Web Notifications API, запрос разрешения); вибрация (Vibration API, паттерн `[200, 100, 200]`); обновление `document.title` с таймером и эмодзи фазы в фоновой вкладке; выбор задачи через `PomodoroTaskSelector` (`api/tasks/in-progress`); автопилот (`autoStartRest`, `autoStartNextCycle` влияют на `pauseAfter` в `lib/pomodoro.lib.ts`); статистика за день (`.stat-card`, `tasksCompletedToday` инкрементируется при завершении work-фазы); Vue `transition` для смены фаз.
- **Кнопки управления Pomodoro**: блок `.pomodoro-actions` под таймером, класс `.pomodoro-btn` с вариантами `.primary`/`.secondary`; динамическая видимость кнопок в зависимости от `state.status` (v-if для pause/resume/stop); иконки Font Awesome (`fa-play`, `fa-pause`, `fa-rotate-right`, `fa-stop`); кастомный `:focus-visible` через `box-shadow` с `--pomodoro-phase-color`; защита от гонок через `actionPending`; touch-friendly размеры (min-height: 44–48px); адаптивная сетка на мобильных (grid 2 колонки при <400px); safe-area отступы через CSS-переменные.
