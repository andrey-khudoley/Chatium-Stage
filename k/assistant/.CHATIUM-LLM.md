# Проект assistant

## Что это
Проект `k/assistant` в Chatium с минимальными страницами и документацией. Общие стили: `styles.tsx` → `formControlStyles` (фокус, autofill), `customScrollbarStyles`, `mobileSafeAreaStyles` и `VIEWPORT_META_CONTENT` (viewport-fit, safe-area для body/модалок; шапка/футер дополнительно в `Header`/`AppFooter`). Выпадающие списки в модалках задач — `components/JnCrtSelect.vue` (панель через `Teleport` в `body`, `fixed`, непрозрачный фон); модалка задачи — класс `crt-form-panel` (сканлайны); дети модалки с `z-index` — `> *:not(.jn-crt-select)`. Главная с карточками быстрых разделов: «Мой журнал» и «Мои задачи» — ссылки с переходом, «Диалоги», «Инструменты», «PARA», «Финансы», «Сервисы», «Библиотека» — action-кнопки, которые запускают только `triggerGlitch` без навигации и без бизнес-логики; сетка карточек адаптирована под 8 элементов (desktop 4 колонки, tablet 2, mobile 1). «Мой журнал» (`./web/journal/`, `JournalPage.vue`, блокнот — `JournalNotebookPane.vue`, API `api/journal/notes/`; вкладки журнала — `?tab=month|week|day|habits` (блокнот без параметра), SSR `journalTabInitial`; в левом меню постоянный список вкладок расположен выше условного тулбара «Новая заметка», чтобы при переключении не смещались постоянные элементы; вкладка «День» — `JournalDayPane.vue`, задачи «В работе», модалка редактирования по клику на название (`api/tasks/items/update`), `daySortOrder`, API `api/tasks/items/reorder-day`, `release-day`; кнопки вкладок в левом меню всегда имеют видимую левую границу, активная вкладка выделяется акцентным цветом). «Управление задачами» (`./web/tasks/`, `TasksPage.vue`) — на десктопе три колонки (дерево, задачи, чат с AI: `components/tasks/TasksAiChatPanel.vue`, фиды `tasks-ai-chat-*`, сброс `tasks-ai-chat-reset`, история в `@app/feed`, вебсокет + polling; порядок сообщений в ленте нормализуется `shared/tasks-ai-chat-message-order.ts` — хронология старые→новые); на мобильном чат из FAB в шторке; иерархия клиент → проект → задача (у проекта и у задачи опционально «Детали» в Heap и UI; служебное поле `context` в Heap у проекта и задачи без выдачи в API; у задачи бывшее поле `description` заменено на `details`/`context`) (имя клиента в сайдбаре только подпись; красная подсветка клиента при выбранном его проекте — акцент слева через inset box-shadow; DnD порядка клиентов/проектов за рукоять, `api/tasks/clients/reorder`, `api/tasks/projects/reorder` (UX DnD не идеален, сценарий рабочий — см. architecture); в модалках кнопки `width: auto`), кнопка «Отметить» (переключение «К выполнению» ↔ «В работе»; при «В работе» — красная подсветка), чат с AI (системный промпт `getAiFormulateSystemPrompt` + `TASKS_AI_CHAT_JSON_APPENDIX`; перед completion в system подмешивается свежий блок проекта/задач из Heap, включая служебные `context` у проекта и задач (`buildTaskAiChatProjectContextBlock`) с явной меткой, что это служебный контекст; последнее сообщение пользователя отправляется отдельным user-блоком «ТЕКУЩЕЕ СООБЩЕНИЕ ПОЛЬЗОВАТЕЛЯ», чтобы модель не путала контекст с новым запросом; ответ модели — JSON с `reply`, `actions`, `summary`; рамка проекта — `update_project.context` (не дублировать список задач), по задачам — `update_task`/`create_task` с полным слитым `context` при необходимости; применение в Heap — `tasks-ai-formulate-apply.ts` из completion; в ленту пишется `reply`; отдельного роута `ai-formulate` нет; для reorder новых задач в том же ответе — плейсхолдеры `$new:0`, `$new:1`, …; настройки в админке), query `?client=` / `?project=`, Heap и API `api/tasks/` (tree, CRUD, reorder, reorder-day, release-day, tasks-ai-chat-*), DTO в `lib/tasks-types.ts`; обновление проекта по имени — без `clientId` в теле (иначе подставлялся бы клиент из выделения в сайдбаре). Heap: `Table.update(ctx, { id, ... })`, не три аргумента. Прелоадер: `shared/preloader.ts` — после `load` и `document.fonts.ready` — `boot-static-ready`, резерв 12 с / скрытие 15 с; `shared/bootUi.ts` — первая строка `// @shared`, шрифты (до 10 с), nextTick, rAF, `hideBootLoader`. AI-настройки: дефолты в `config/prompts.tsx` (системный промпт AI-формулирования / чата с задачами: массив действий (create_task с priority и декомпозицией шагов, update_task с title/priority, delete_task, reorder_tasks, update_project), опора на факты из контекста, без выдуманных taskId, ответ только JSON без Markdown-обёртки, без длинного тире U+2014 в текстах; **критическое правило разделения** — события с датами/действия → create_task, рамка проекта (не дублирование списка задач) → update_project, с примерами (few-shot); список моделей `AVAILABLE_AI_MODELS` — от худшей к лучшей для этой задачи, см. комментарий в файле), UI в `components/admin/AiSettings.vue` импортирует те же константы из `config/prompts`, сохранение через `api/settings/save`.

## Критические правила
- Не хардкодить URL, использовать `config/routes.tsx`.
- Документацию обновлять при любых изменениях структуры/логики.
- Структуру и стили левого меню журнала держать в `components/journal/JournalNav.vue`; `JournalPage.vue` не должен содержать прямую разметку `nav.journal-nav`.
- В `JournalPage.vue` для вкладки `day` допускается только переход по переданному `tasksPageUrl` (кнопка `Все задачи`), без хардкода путей.
- Динамические кнопки в `JournalPage.vue` (`Новая заметка` и `Все задачи`) должны рендериться после `journal-nav-divider`; для action-кнопок использовать кастомный `:focus-visible` в цветах темы (без системной жёлтой обводки браузера).
- Для кнопок вкладок журнала (`.journal-nav-btn`) также использовать кастомный `:focus-visible` в цветах темы и не оставлять браузерный системный outline.
- Порядок вкладок в левом меню журнала фиксированный: `day -> week -> month -> habits -> notebook` (по лейблам: `День, Неделя, Месяц, Привычки, Блокнот`).
