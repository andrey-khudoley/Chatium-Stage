# Architecture

## Назначение
Базовый шаблон проекта Chatium с минимальным набором страниц и документации.

## Ограничения платформы
- Серверная инфраструктура предоставляется Chatium.
- Нельзя менять стек и зависимости.
- Деплой — автоматически при пуше.

## Основные сценарии
- Открыть главную страницу.
- Авторизоваться и попасть в профиль.
- Открыть админку (только роль Admin).
- Использовать быстрые карточки на главной: «Мой журнал», «Мои задачи» и «Инструменты» переходят в разделы, карточки «Диалоги», «PARA», «Финансы», «Сервисы», «Библиотека» запускают только глобальный glitch-эффект без навигации.
- На вкладке «Неделя» в журнале перелистывать недели кнопками `← Пред.` / `След. →`; возврат к текущей неделе вынесен в отдельную подстроку (`К текущей неделе`) рядом с датой начала недели, чтобы не нарушать центрирование заголовка.

## Роутинг
- `index.tsx` — главная (SSR + Vue), единственный роут в корне.
- `web/admin/index.tsx` — админка, `requireAccountRole('Admin')`.
- `web/profile/index.tsx` — профиль, `requireRealUser()`.
- `web/journal/index.tsx` — страница «Мой журнал» (`JournalPage.vue`): компактное вертикальное меню слева (порядок вкладок: Задачи, День, Неделя, Месяц, Привычки, Блокнот) и панель справа с динамической подгрузкой Vue-компонентов (`components/journal/*Pane.vue`); при ширине ≤900px меню и панель складываются в колонку, вкладки — горизонтальный скролл. Навигация выделена в отдельный компонент `components/journal/JournalNav.vue`, чтобы структура DOM и стили меню были централизованы. Порядок в меню: сначала постоянный список вкладок, затем (если нужен) разделитель `journal-nav-divider`, и только после него динамический тулбар (`Новая заметка` для `notebook` или `Все задачи` для `tasks`); это фиксирует единый визуальный ритм меню. Кнопка `Все задачи` ведёт на общий список задач (`tasksPageUrl`). Для action-кнопок меню и кнопок вкладок задан кастомный `:focus-visible` в цветах темы (без системной жёлтой обводки). Активная вкладка синхронизируется с адресной строкой: `?tab=month|week|tasks|day|habits` (для «Блокнот» параметр не добавляется), `replaceState` при переключении, SSR читает `req.query.tab` и передаёт `journalTabInitial`, чтобы при возврате «назад» со страницы задач открывалась та же вкладка. Вкладка «Блокнот»: Heap-таблица заметок, SSR передаёт список заголовков и URL API (list/create/get/update/delete) для авторизованного пользователя; кнопка «Новая заметка» открывает модальный редактор в `JournalNotebookPane` (название и текст, сохранение через POST create). Вкладка «Задачи» (`tasks`): задачи со статусом «В работе» (дерево задач на SSR, `JournalDayPane` — порядок, ссылки на клиента/проект на странице задач с query-параметрами, клик по названию задачи открывает модальное редактирование с сохранением через POST `/api/tasks/items/update`, кнопка «В очередь» вызывает POST `/api/tasks/items/release-day`). Вкладка «День» (`day`): `JournalDayInDevelopmentPane.vue` с четырьмя многострочными сегментами (`Ночь`, `Утро`, `День`, `Вечер`) и кнопкой фиксации/разблокировки у каждого сегмента; запись хранится в Heap по `dayKey` (`YYYY-MM-DD`) с границей суток в 05:00 (`api/journal/day/get|save`, `lib/journal-day-key.ts`), старые дни не перезаписываются. Вкладка «Неделя» (`week`): `JournalWeekPane.vue` с верхним блоком общего плана `Неделя #N` (weekly-summary: textarea + `Зафиксировать`/`Редактировать`) и 7 карточками дней (`Понедельник..Воскресенье`) ниже; у каждого дня есть дата, отдельная фиксация и поле ввода. Для визуального выравнивания weekly-summary с дневными карточками summary использует ту же высотную структуру шапки (caption + скрытый спейсер строки даты), поэтому textarea начинается на одной высоте с полями Пн–Вс. Хранение: дневные записи недели — `tables/journal-week-entries.table.ts` (`api/journal/week/get|save`), общий weekly-summary — `tables/journal-week-summary.table.ts` (`api/journal/week/save-summary`); вычисление ключей недели — `lib/journal-week-key.ts`. В кнопках вкладок журнала левая граница всегда видима (неактивные — в цвете border, активная — accent). Месяц / Привычки — заглушка «В разработке». Страница доступна без обязательной авторизации.
- `web/tasks/index.tsx` — страница «Управление задачами» (`TasksPage.vue`): боковая панель с иерархией «клиент → проекты» (широкая колонка; вверху — вторичная по акценту кнопка «Новый клиент», новый проект — иконкой в строке клиента; название клиента не переключает контекст по клику, красная подсветка строки клиента — если выбран проект этого клиента; порядок клиентов и проектов в сайдбаре — drag-and-drop за рукоять, сохранение `sortOrder` через POST `/api/tasks/clients/reorder` и `/api/tasks/projects/reorder`) и основная область с таблицей задач в визуальном стиле журнала; на узких экранах (до 640px) таблица превращается в карточки с подписями полей (`data-label`); при ширине viewport **до 1023px** на `documentElement` вешается класс `tasks-page-html-mobile` (увеличен базовый `font-size` для `rem`, крупнее модалки и поля ввода **16px**); режим **master–detail**: сначала полноширинный список клиентов/проектов, после выбора проекта — полноширинный экран задач с кнопкой «Проекты» назад (не две колонки одновременно); DnD порядка на мобильном отключён; **оговорка по DnD:** поведение нативного drag-and-drop в браузере/встроенном клиенте доведено не до конца (иногда индикатор и фактический дроп расходятся или нужен повтор жеста), но смена порядка доступна и сценарий не блокируется; SSR передаёт дерево задач и URL API (`api/tasks/`) для авторизованного пользователя; без входа — подсказка. Модальные формы создания/редактирования, удаление с подтверждением, смена приоритета/статуса/проекта, кнопки «вверх/вниз» и POST reorder для порядка задач в проекте; кнопка «Отметить» переключает статус: не «В работе» → «В работе» (для дневного списка в журнале), повторный клик — обратно в «К выполнению»; при «В работе» кнопка подсвечена красным; поддержка `?client=` и `?project=` в URL для перехода на клиента/проект. При переименовании проекта в POST `api/tasks/projects/update` передаются `id`, `name` и при необходимости `details` (без `clientId`), чтобы проект не привязывался к клиенту, выбранному в сайдбаре под другим проектом; у проекта в Heap есть также служебное поле `context`, в API/UI не отображается. На десктопе (viewport ≥1024px) справа третья колонка — чат с AI (`components/tasks/TasksAiChatPanel.vue`): фид Chatium на пару пользователь+проект (`tables/task-ai-chat-feeds`, `getChat`, `feedMessages*Handler`), история в фиде передаётся в `startCompletion` как диалог, ответы ассистента — сообщения с `data.assistant`; обновления в реальном времени через `getOrCreateBrowserSocketClient` + `messages_socket_id` и периодический `messages/changes`; кнопка сброса истории — `tasks-ai-chat-reset`. На мобильном чат в нижней шторке (FAB).
- `web/tools/index.tsx` — страница «Инструменты» (`ToolsPage.vue`): список утилит, сейчас одна карточка `Pomodoro` с переходом в отдельный маршрут; в head задаются переменные темы (`:root`), фон `html/body`, scrollbar/safe-area styles для корректного рендера.
- `web/pomodoro/index.tsx` — страница Pomodoro (`PomodoroPage.vue`): SSR передаёт URL API (`state/get`, `control`, `settings/save`), UI построен как compact-layout с графическим таймером (SVG-дуга в `components/pomodoro/PomodoroTimerDial.vue`), отдельным модальным окном настроек (`components/pomodoro/PomodoroSettingsModal.vue`, в т.ч. пресеты звука смены фазы 1–5 и «Прослушать») и панелью действий (`.pomodoro-actions__panel`: все кнопки в одной CSS Grid по классу раскладки `pomodoroActionPanelLayout`) по статусам. Шапка получает `pomodoroStateGetUrl` и `pomodoroControlUrl` для общего индикатора Pomodoro в `Header.vue`; страница показывает fallback загрузки до получения состояния. Панель кнопок по статусам: `stopped` — только «Старт»; `running` — «Пауза» и «Пропустить»; `paused` / `awaiting_continue` — «Продолжить» и «Сбросить» (действие `reset`: остановка, `phase=work`, полный `workMinutes`, `cyclesCompleted=0`). Семантика API: `start` — запуск/перезапуск рабочей фазы (`phase=work`, отсчёт от `workMinutes`, `cyclesCompleted=0`; на странице кнопка только при `stopped`), `resume` — из `paused` (продолжить отсчёт) или из `awaiting_continue` (перейти к следующей фазе после овертайма), `pause` — только из `running`, `skip` — пропуск текущей фазы (на странице — только во время отсчёта `running`; на сервере по-прежнему доступен из `awaiting_continue`), `reset` — полный сброс в `stopped` с обнулением циклов; в суммы работы/отдыха и в учёт времени по задаче попадает только фактически прошедшее время (накопление через `tick` до действия), неотработанный остаток фазы не добавляется. По окончании работы или отдыха по таймеру статус `awaiting_continue`: на клиенте показывается секундомер овертайма (`phaseEndsAtMs` — момент окончания фазы), до «Продолжить» следующая фаза не стартует. Счётчики дневной статистики (`.stat-cell`: помидоры, работа, отдых) сбрасываются при смене ключа периода в **05:00 по локальному времени браузера** (`statsDayKey` в запросах к `state/get`, `control`, `settings/save`, `assign-task`; см. `lib/pomodoro-stats-day.ts`; без ключа — fallback `Europe/Moscow`); таймер и `cyclesCompleted` при этом не сбрасываются. В серверной логике ведётся журнал сегментов запусков (`pomodoro-launches`) с разметкой фаз, задач, источника запуска и причины завершения сегмента.
- `web/tests/index.tsx` — страница тестов, `requireRealUser()`.
- `web/login/index.tsx` — вход (редирект на системный `/s/auth/signin`).

## Стили UI
- `styles.tsx`: `customScrollbarStyles` (тёмный скроллбар), `mobileSafeAreaStyles` (viewport-fit, отступы `env(safe-area-inset-*)` для body и модалок `.jn-modal-overlay` / `.logout-modal-overlay`), `formControlStyles` (фокус и отключение жёлтого autofill у `.jn-input` / `.jn-textarea` / `.settings-input`), константа `VIEWPORT_META_CONTENT` для `<meta name="viewport">`. Подключаются в `<head>` у главной (`index.tsx`) и у роутов `web/*`. В компонентах `Header.vue` / `AppFooter.vue` дополнительно учтены safe-area для шапки и подвала.
- `styles.tsx`: `customScrollbarStyles` (тёмный скроллбар), `mobileSafeAreaStyles` (viewport-fit, отступы `env(safe-area-inset-*)` для body и модалок `.jn-modal-overlay` / `.logout-modal-overlay`), `formControlStyles` (глобальный reset системного `outline`/`box-shadow` для текстовых `input/textarea/select`, чтобы не показывалась нативная жёлтая рамка; плюс фокус и отключение жёлтого autofill у `.jn-input` / `.jn-textarea` / `.settings-input`), константа `VIEWPORT_META_CONTENT` для `<meta name="viewport">`. Подключаются в `<head>` у главной (`index.tsx`) и у роутов `web/*`. В компонентах `Header.vue` / `AppFooter.vue` дополнительно учтены safe-area для шапки и подвала.
- Выпадающие списки в модалках задач — `components/JnCrtSelect.vue` (панель в `Teleport` → `body`, `position: fixed`, непрозрачный базовый слой фона; не нативный `<select>`). Триггер селекта имеет собственный базовый набор стилей поля (`padding`, `border`, `background`, `color`, `font-size`, focus-ring), чтобы не зависеть от внешних scoped-стилей `jn-input`; для значения фиксирована типографика (`line-height` и наследование `letter-spacing`), чтобы текст не смещался по вертикали. Модалка задачи с классом `crt-form-panel` — сканлайны на фоне и на полях ввода; для `z-index` дочерних элементов модалки используется селектор `> *:not(.jn-crt-select)`.

## Разделение слоёв

Принцип разделения ответственности при работе с данными (см. [ADR-0002](ADR/0002-settings-heap-and-layered-api.md)):

| Слой | Каталог | Ответственность |
| --- | --- | --- |
| **Таблицы** | `tables/` | Схемы Heap (поля, типы). Только определение структуры данных. |
| **Репозитории** | `repos/` | Работа с БД: CRUD, запросы. Никакой бизнес‑логики, только вызовы Heap API. |
| **Бизнес‑логика** | `lib/` | Правила, дефолты, валидация значений, вычисления. Вызывает репозитории. |
| **API** | `api/` | HTTP‑эндпоинты, парсинг и первичная валидация запросов, проверка прав. Вызывает lib. |

Поток данных: `HTTP → API → lib → repos → Heap`.

## Структура каталогов
- `config/` — маршруты и `PROJECT_ROOT`.
- `web/` — браузерные роуты модулей (admin, profile, journal, tasks, tools, pomodoro, tests, login).
- `pages/` — Vue‑страницы (минимальные, в т.ч. JournalPage, TasksPage, ToolsPage, PomodoroPage).
- `components/` — переиспользуемые Vue‑компоненты (Header, AppFooter, GlobalGlitch, LogoutModal); подкаталог `components/journal/` — панели разделов журнала (`JournalNotebookPane` — список, раскрытие текста, модал создания/редактирования, удаление; остальные — заглушки).
- `api/` — API‑эндпоинты (получение и валидация входных данных). File-based: один файл — один эндпоинт с `/`. Примеры: `api/settings/list.ts`, `api/journal/notes/`, `api/tasks/tree/get.ts`, `api/pomodoro/`, `api/logger/log.ts`, `api/tests/list.ts`.
- `tables/` — Heap‑таблицы (settings, logs, journal-notes, task-clients, task-projects, task-items, pomodoro-state, pomodoro-launches).
- `repos/` — репозитории (settings, logs, journal-notes, tasks, pomodoro, pomodoro-launches; logs.repo — findBeforeTimestamp для пагинации).
- `lib/` — бизнес‑логика (settings.lib, logger.lib, tasks-types — DTO задач без Heap, pomodoro.lib — конечный автомат фаз, синхронизация времени под lock и запись сегментов запусков Pomodoro).
- `shared/` — общий код (preloader: `completeSequence` после `load` (и таймаут 3 с); безопасное добавление строк в терминал; затем `document.fonts.ready` и `boot-static-ready`; резерв через 12 с / принудительное скрытие через 15 с при зависании; `bootUi.ts` — шрифты (таймаут 10 с), nextTick, rAF, `hideBootLoader`; фон оверлея `#0a0a0a`; logLevel для клиента, logger syslog RFC 5424, createComponentLogger, setLogSink/LogEntry, logEmergency…logDebug с порогом).
- `docs/` — документация проекта.

## Интеграции
- Внешние сервисы: нет.
- Внутренние SDK: стандартные модули Chatium.
