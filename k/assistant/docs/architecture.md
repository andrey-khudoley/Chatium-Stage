# Architecture

## Назначение
Базовый шаблон проекта Chatium с минимальным набором страниц и документации.

## Ограничения платформы
- Серверная инфраструктура предоставляется Chatium.
- Нельзя менять стек и зависимости.
- Деплой — автоматически при пуше.

## Основные сценарии
- Открыть главную страницу.
- Авторизоваться и попасть в профиль.
- Открыть админку (только роль Admin).
- Использовать быстрые карточки на главной: «Мой журнал», «Мои задачи» и «Инструменты» переходят в разделы, карточки «Диалоги», «PARA», «Финансы», «Сервисы», «Библиотека» запускают только глобальный glitch-эффект без навигации.
- На вкладке «Неделя» в журнале перелистывать недели кнопками `← Пред.` / `След. →`; возврат к текущей неделе вынесен в отдельную подстроку (`К текущей неделе`) рядом с датой начала недели, чтобы не нарушать центрирование заголовка.
- В редакторе заметки блокнота использовать кнопку `Экспорт` для трёх сценариев: скачать содержимое как `TXT`, скачать как `MD` и открыть окно печати для сохранения в `PDF`.
- В WYSIWYG-редакторе заметки при вставке plain text сохраняются пустые строки: двойные переносы (`\n\n`) не схлопываются и остаются как отдельные абзацы.
- В WYSIWYG-редакторе заметки поддерживается smart-paste: при вставке Markdown из `text/plain` выполняется базовое форматирование в HTML; при наличии `text/html` в буфере используется готовая разметка источника.
- В Markdown-ветке smart-paste пустые строки также сохраняются, чтобы между блоками не терялись визуальные разделители абзацев.
- Вставка HTML при paste выполняется через прямую DOM-вставку (Range API, `insertHtmlAtCursor`) вместо `execCommand('insertHTML')`, чтобы браузер не удалял пустые абзацы. Пустые строки помечены классом `wy-blank-line` c визуальной высотой через `padding` (`height: 0; overflow: hidden; font-size: 0; line-height: 0`).
- Если курсор внутри блока кода (`pre[data-language]` / вложенный `<code>`), paste обрабатывается отдельно: в буфер берётся только plain text (при отсутствии `text/plain` — текст из `text/html` через временный `div`), вставка через Range внутрь `<code>` (`insertPlainTextIntoCodeBlockAtSelection`), без `insertHtmlAtCursor` — иначе та функция поднимала бы точку вставки до корня `contenteditable` и создавала узлы *после* `<pre>`.
- Для диагностики проблем форматирования в `WysiwygEditor` добавлен переключатель режимов `WYSIWYG / HTML`: можно открыть исходник текущей заметки, сверить и вручную поправить HTML, затем вернуться в визуальный режим.
- В режиме `HTML` исходник заметки не должен начинаться с пустой строки: при переходе из `WYSIWYG` редактор отбрасывает технические ведущие переводы строки (`\r/\n`) в source-представлении.
- Переключение `WYSIWYG / HTML` в `WysiwygEditor` работает с устойчивой синхронизацией буфера (`sourceHtml`/`modelValue`/последнее известное HTML): при быстром переключении режимов контент заметки не должен пропадать из-за перезаписи пустым значением.
- При переключении обратно в `WYSIWYG` HTML применяется только после `nextTick`: ветка с `contenteditable` рендерится через `v-if`, поэтому `editorRef` доступен только после обновления DOM; это предотвращает потерю контента из-за ранней записи в несуществующий узел.
- Стратегия источника для paste: если `text/plain` содержит Markdown или явные пустые строки, используется plain-text ветка (даже при наличии `text/html`), чтобы не терять переносы из-за внешних HTML-фрагментов с нестандартной разметкой.
- В редакторе есть нормализация `wy-blank-line`: как только в «технически пустом» абзаце появляется текст, класс удаляется автоматически, чтобы контент не был скрыт стилями пустой строки.
- **Блок кода с подсветкой синтаксиса** реализован в `WysiwygEditor.vue` через highlight.js v11.11.1, загружаемый с CDN `cdnjs.cloudflare.com` (тема `atom-one-dark`). HTML-структура: `<pre data-language="..."><code class="hljs">...</code></pre>`. Язык хранится в атрибуте `data-language` на `<pre>` — сохраняется в HTML-строке заметки и восстанавливается при загрузке. Подсветка загружается лениво (JS + CSS при первом использовании). Floating-пикер языка (`Teleport` в `body`) при фокусе внутри блока кода: позиция `fixed` по `rect.left`/`rect.top` активного `pre`, смещение вверх на высоту пикера + зазор (`translateY(calc(-100% - 0.3rem))`) — левый верх над блоком; 25 языков. Последний выбранный в пикере язык пишется в `localStorage` (`assistant-journal-preferred-code-lang`, только значения из списка) и подставляется как язык по умолчанию при вставке нового блока кода с панели и для fenced-блоков Markdown без указания языка при smart-paste. Подсветка обновляется при: смене языка, уходе фокуса из блока, вводе текста (дебаунс 700ms с сохранением позиции курсора через character-offset). При вставке Markdown с fenced code blocks язык парсится и подсвечивается автоматически. Экспорт в Markdown включает язык после ` ``` `. CSS: акцентная левая рамка на `<pre data-language>`, свечение при фокусе, `.hljs` background overridden на transparent.

## Роутинг
- `index.tsx` — главная (SSR + Vue), единственный роут в корне.
- `web/admin/index.tsx` — админка, `requireAccountRole('Admin')`.
- `web/profile/index.tsx` — профиль, `requireRealUser()`.
- `web/journal/index.tsx` — страница «Мой журнал» (`JournalPage.vue`): компактное вертикальное меню слева (порядок вкладок: Задачи, День, Неделя, Месяц, Привычки, Блокнот) и панель справа с динамической подгрузкой Vue-компонентов (`components/journal/*Pane.vue`); при ширине ≤900px меню и панель складываются в колонку, вкладки — горизонтальный скролл. Навигация выделена в отдельный компонент `components/journal/JournalNav.vue`, чтобы структура DOM и стили меню были централизованы. Порядок в меню: сначала постоянный список вкладок, затем (если нужен) разделитель `journal-nav-divider`, и только после него динамический тулбар (`Новая заметка` для `notebook` или `Все задачи` для `tasks`); это фиксирует единый визуальный ритм меню. Кнопка `Все задачи` ведёт на общий список задач (`tasksPageUrl`). Для action-кнопок меню и кнопок вкладок задан кастомный `:focus-visible` в цветах темы (без системной жёлтой обводки). Активная вкладка синхронизируется с адресной строкой: `?tab=month|week|tasks|day|habits` (для «Блокнот» параметр не добавляется), `replaceState` при переключении, SSR читает `req.query.tab` и передаёт `journalTabInitial`, чтобы при возврате «назад» со страницы задач открывалась та же вкладка. Вкладка «Блокнот»: полнофункциональный ноутбук с трёхпанельной архитектурой — `NotebookFolderSidebar` (боковая панель папок: создание, переименование, удаление, drag-and-drop порядок, архивация), `JournalNotebookPane` (основная панель: список карточек заметок `NotebookNoteCard`, фильтр-бар `NotebookFilterBar` для поиска/категории/архива, `NotebookBulkBar` для массовых операций с мультиселектом — перемещение, архивация, удаление, назначение категории), `NotebookNoteEditor` (полностраничный редактор вместо модального окна — название, текст, выбор папки, категорий, даты, привязки к задаче/проекту/клиенту). SSR передаёт расширенные данные (заметки + папки + категории) и URL для всех API (notes: list/create/get/update/delete/reorder/archive/move/bulk; categories: list/create/update/delete; folders: create/update/delete/reorder/archive). Heap-таблицы: `journal-notes` (расширена: folderId, categoryIds, linkedTaskId, linkedProjectId, linkedClientId, noteDate, isArchived, sortOrder), `notebook-categories` (name, color, sortOrder), `notebook-folders` (name, color, sortOrder, isArchived). Вкладка «Задачи» (`tasks`): задачи со статусом «В работе» (дерево задач на SSR, `JournalDayPane` — порядок, ссылки на клиента/проект на странице задач с query-параметрами, клик по названию задачи открывает модальное редактирование с сохранением через POST `/api/tasks/items/update`, кнопка «В очередь» вызывает POST `/api/tasks/items/release-day`). Вкладка «День» (`day`): `JournalDayInDevelopmentPane.vue` с четырьмя многострочными сегментами (`Ночь`, `Утро`, `День`, `Вечер`) и кнопкой фиксации/разблокировки у каждого сегмента; запись хранится в Heap по `dayKey` (`YYYY-MM-DD`) с границей суток в 05:00 (`api/journal/day/get|save`, `lib/journal-day-key.ts`), старые дни не перезаписываются. Вкладка «Неделя» (`week`): `JournalWeekPane.vue` с верхним блоком общего плана `Неделя #N` (weekly-summary: textarea + `Зафиксировать`/`Редактировать`) и 7 карточками дней (`Понедельник..Воскресенье`) ниже; у каждого дня есть дата, отдельная фиксация и поле ввода. Для визуального выравнивания weekly-summary с дневными карточками summary использует ту же высотную структуру шапки (caption + скрытый спейсер строки даты), поэтому textarea начинается на одной высоте с полями Пн–Вс. Хранение: дневные записи недели — `tables/journal-week-entries.table.ts` (`api/journal/week/get|save`), общий weekly-summary — `tables/journal-week-summary.table.ts` (`api/journal/week/save-summary`); вычисление ключей недели — `lib/journal-week-key.ts`. В кнопках вкладок журнала левая граница всегда видима (неактивные — в цвете border, активная — accent). Вкладка «Месяц» (`month`): `JournalMonthPane.vue` с двумя режимами — календарь месяца и детальный просмотр дня. Календарь: навигация по месяцам (`← Пред.` / `МЕСЯЦ ГОД` / `След. →`), кнопка `К текущему месяцу`, 7-колоночная сетка с индикаторами (количество завершённых задач, точка фокуса Pomodoro на ячейке дня). При клике на день открывается детальный вид: заголовок с датой/днём недели, навигация по дням, список завершённых задач (по `updatedAt` из Heap), суммарное время фокуса Pomodoro с кнопкой `Показать детали` (заглушка «В разработке»), план на день из вкладки «Неделя» и дневные записи из вкладки «День» (ночь/утро/день/вечер). Данные загружаются через `api/journal/month/data` + существующие `api/journal/day/get` и `api/journal/week/get`. Привычки — заглушка «В разработке». Страница доступна без обязательной авторизации.
- `web/tasks/index.tsx` — страница «Управление задачами» (`TasksPage.vue`): боковая панель с иерархией «клиент → проекты» (широкая колонка; вверху — вторичная по акценту кнопка «Новый клиент», новый проект — иконкой в строке клиента; название клиента не переключает контекст по клику, красная подсветка строки клиента — если выбран проект этого клиента; порядок клиентов и проектов в сайдбаре — drag-and-drop за рукоять, сохранение `sortOrder` через POST `/api/tasks/clients/reorder` и `/api/tasks/projects/reorder`) и основная область с таблицей задач в визуальном стиле журнала; на узких экранах (до 640px) таблица превращается в карточки с подписями полей (`data-label`); при ширине viewport **до 1023px** на `documentElement` вешается класс `tasks-page-html-mobile` (увеличен базовый `font-size` для `rem`, крупнее модалки и поля ввода **16px**); режим **master–detail**: сначала полноширинный список клиентов/проектов, после выбора проекта — полноширинный экран задач с кнопкой «Проекты» назад (не две колонки одновременно); DnD порядка на мобильном отключён; **оговорка по DnD:** поведение нативного drag-and-drop в браузере/встроенном клиенте доведено не до конца (иногда индикатор и фактический дроп расходятся или нужен повтор жеста), но смена порядка доступна и сценарий не блокируется; SSR передаёт дерево задач и URL API (`api/tasks/`) для авторизованного пользователя; без входа — подсказка. Модальные формы создания/редактирования, удаление с подтверждением, смена приоритета/статуса/проекта, кнопки «вверх/вниз» и POST reorder для порядка задач в проекте; кнопка «Отметить» переключает статус: не «В работе» → «В работе» (для дневного списка в журнале), повторный клик — обратно в «К выполнению»; при «В работе» кнопка подсвечена красным; поддержка `?client=` и `?project=` в URL для перехода на клиента/проект. При переименовании проекта в POST `api/tasks/projects/update` передаются `id`, `name` и при необходимости `details` (без `clientId`), чтобы проект не привязывался к клиенту, выбранному в сайдбаре под другим проектом; у проекта в Heap есть также служебное поле `context`, в API/UI не отображается. На десктопе (viewport ≥1024px) справа третья колонка — чат с AI (`components/tasks/TasksAiChatPanel.vue`): фид Chatium на пару пользователь+проект (`tables/task-ai-chat-feeds`, `getChat`, `feedMessages*Handler`), история в фиде передаётся в `startCompletion` как диалог, ответы ассистента — сообщения с `data.assistant`; обновления в реальном времени через `getOrCreateBrowserSocketClient` + `messages_socket_id` и периодический `messages/changes`; кнопка сброса истории — `tasks-ai-chat-reset`. На мобильном чат в нижней шторке (FAB).
- `web/tools/index.tsx` — страница «Инструменты» (`ToolsPage.vue`): список утилит, сейчас одна карточка `Pomodoro` с переходом в отдельный маршрут; в head задаются переменные темы (`:root`), фон `html/body`, scrollbar/safe-area styles для корректного рендера.
- `web/pomodoro/index.tsx` — страница Pomodoro (`PomodoroPage.vue`): SSR передаёт URL API (`state/get`, `control`, `settings/save`), UI построен как compact-layout с графическим таймером (SVG-дуга в `components/pomodoro/PomodoroTimerDial.vue`), отдельным модальным окном настроек (`components/pomodoro/PomodoroSettingsModal.vue`, в т.ч. пресеты звука смены фазы 1–5 и «Прослушать») и панелью действий (`.pomodoro-actions__panel`: все кнопки в одной CSS Grid по классу раскладки `pomodoroActionPanelLayout`) по статусам. Шапка получает `pomodoroStateGetUrl` и `pomodoroControlUrl` для общего индикатора Pomodoro в `Header.vue`; страница показывает fallback загрузки до получения состояния. Панель кнопок по статусам: `stopped` — только «Старт»; `running` — «Пауза» и «Пропустить»; `paused` / `awaiting_continue` — «Продолжить» и «Сбросить» (действие `reset`: остановка, `phase=work`, полный `workMinutes`, `cyclesCompleted=0`). Семантика API: `start` — запуск/перезапуск рабочей фазы (`phase=work`, отсчёт от `workMinutes`, `cyclesCompleted=0`; на странице кнопка только при `stopped`), `resume` — из `paused` (продолжить отсчёт) или из `awaiting_continue` (перейти к следующей фазе после овертайма), `pause` — только из `running`, `skip` — пропуск текущей фазы (на странице — только во время отсчёта `running`; на сервере по-прежнему доступен из `awaiting_continue`), `reset` — полный сброс в `stopped` с обнулением циклов; в суммы работы/отдыха и в учёт времени по задаче попадает только фактически прошедшее время (накопление через `tick` до действия), неотработанный остаток фазы не добавляется. По окончании работы или отдыха по таймеру статус `awaiting_continue`: на клиенте показывается секундомер овертайма (`phaseEndsAtMs` — момент окончания фазы), до «Продолжить» следующая фаза не стартует. Счётчики дневной статистики (`.stat-cell`: помидоры, работа, отдых) сбрасываются при смене ключа периода в **05:00 по локальному времени браузера** (`statsDayKey` в запросах к `state/get`, `control`, `settings/save`, `assign-task`; см. `lib/pomodoro-stats-day.ts`; без ключа — fallback `Europe/Moscow`); таймер и `cyclesCompleted` при этом не сбрасываются. В серверной логике ведётся журнал сегментов запусков (`pomodoro-launches`) с разметкой фаз, задач, источника запуска и причины завершения сегмента.
- `web/tests/index.tsx` — страница тестов, `requireRealUser()`.
- `web/login/index.tsx` — вход (редирект на системный `/s/auth/signin`).

## Стили UI
- `styles.tsx`: `customScrollbarStyles` (тёмный скроллбар), `mobileSafeAreaStyles` (viewport-fit, отступы `env(safe-area-inset-*)` для body и модалок `.jn-modal-overlay` / `.logout-modal-overlay`), `formControlStyles` (фокус и отключение жёлтого autofill у `.jn-input` / `.jn-textarea` / `.settings-input`), константа `VIEWPORT_META_CONTENT` для `<meta name="viewport">`. Подключаются в `<head>` у главной (`index.tsx`) и у роутов `web/*`. В компонентах `Header.vue` / `AppFooter.vue` дополнительно учтены safe-area для шапки и подвала.
- `styles.tsx`: `customScrollbarStyles` (тёмный скроллбар), `mobileSafeAreaStyles` (viewport-fit, отступы `env(safe-area-inset-*)` для body и модалок `.jn-modal-overlay` / `.logout-modal-overlay`), `formControlStyles` (глобальный reset системного `outline`/`box-shadow` для текстовых `input/textarea/select`, чтобы не показывалась нативная жёлтая рамка; плюс фокус и отключение жёлтого autofill у `.jn-input` / `.jn-textarea` / `.settings-input`), константа `VIEWPORT_META_CONTENT` для `<meta name="viewport">`. Подключаются в `<head>` у главной (`index.tsx`) и у роутов `web/*`. В компонентах `Header.vue` / `AppFooter.vue` дополнительно учтены safe-area для шапки и подвала.
- Выпадающие списки в модалках задач — `components/JnCrtSelect.vue` (панель в `Teleport` → `body`, `position: fixed`, непрозрачный базовый слой фона; не нативный `<select>`). Триггер селекта имеет собственный базовый набор стилей поля (`padding`, `border`, `background`, `color`, `font-size`, focus-ring), чтобы не зависеть от внешних scoped-стилей `jn-input`; для значения фиксирована типографика (`line-height` и наследование `letter-spacing`), чтобы текст не смещался по вертикали. Модалка задачи с классом `crt-form-panel` — сканлайны на фоне и на полях ввода; для `z-index` дочерних элементов модалки используется селектор `> *:not(.jn-crt-select)`.

## Разделение слоёв

Принцип разделения ответственности при работе с данными (см. [ADR-0002](ADR/0002-settings-heap-and-layered-api.md)):

| Слой | Каталог | Ответственность |
| --- | --- | --- |
| **Таблицы** | `tables/` | Схемы Heap (поля, типы). Только определение структуры данных. |
| **Репозитории** | `repos/` | Работа с БД: CRUD, запросы. Никакой бизнес‑логики, только вызовы Heap API. |
| **Бизнес‑логика** | `lib/` | Правила, дефолты, валидация значений, вычисления. Вызывает репозитории. |
| **API** | `api/` | HTTP‑эндпоинты, парсинг и первичная валидация запросов, проверка прав. Вызывает lib. |

Поток данных: `HTTP → API → lib → repos → Heap`.

## Структура каталогов
- `config/` — маршруты и `PROJECT_ROOT`.
- `web/` — браузерные роуты модулей (admin, profile, journal, tasks, tools, pomodoro, tests, login).
- `pages/` — Vue‑страницы (минимальные, в т.ч. JournalPage, TasksPage, ToolsPage, PomodoroPage).
- `components/` — переиспользуемые Vue‑компоненты (Header, AppFooter, GlobalGlitch, LogoutModal); подкаталог `components/journal/` — панели разделов журнала (`JournalNotebookPane` — split-view: боковая панель папок + список заметок, инлайн-управление категориями, фильтр-бар, мультиселект, массовые операции, полностраничный редактор вместо модала; `NotebookNoteCard` — карточка заметки с чекбоксом, меткой категории, превью; `NotebookFolderSidebar` — дерево папок с CRUD и DnD; `NotebookFilterBar` — поиск, фильтр по категории, архив; `NotebookBulkBar` — панель массовых действий; `NotebookNoteEditor` — полностраничный редактор заметки с WYSIWYG-редактором (`WysiwygEditor` + `WysiwygToolbar`: contenteditable, жирный/курсив/подчёркнутый/зачёркнутый, цвет текста, H1/H2/H3/абзац, списки с вложенностью, выравнивание, ссылки с href/target, таблицы с добавлением/удалением строк/столбцов, HR, inline code, code block, цитата, undo/redo, очистка форматирования; контент — HTML-строка с обратной совместимостью plain text); остальные — заглушки).
- `api/` — API‑эндпоинты (получение и валидация входных данных). File-based: один файл — один эндпоинт с `/`. Примеры: `api/settings/list.ts`, `api/journal/notes/`, `api/journal/categories/`, `api/journal/folders/`, `api/tasks/tree/get.ts`, `api/pomodoro/`, `api/logger/log.ts`, `api/tests/list.ts`.
- `tables/` — Heap‑таблицы (settings, logs, journal-notes, notebook-categories, notebook-folders, task-clients, task-projects, task-items, pomodoro-state, pomodoro-launches).
- `repos/` — репозитории (settings, logs, journal-notes, notebook-categories, notebook-folders, tasks, pomodoro, pomodoro-launches; logs.repo — findBeforeTimestamp для пагинации).
- `lib/` — бизнес‑логика (settings.lib, logger.lib, tasks-types — DTO задач без Heap, pomodoro.lib — конечный автомат фаз, синхронизация времени под lock и запись сегментов запусков Pomodoro).
- `shared/` — общий код (preloader: `completeSequence` после `load` (и таймаут 3 с); безопасное добавление строк в терминал; затем `document.fonts.ready` и `boot-static-ready`; резерв через 12 с / принудительное скрытие через 15 с при зависании; `bootUi.ts` — шрифты (таймаут 10 с), nextTick, rAF, `hideBootLoader`; фон оверлея `#0a0a0a`; logLevel для клиента, logger syslog RFC 5424, createComponentLogger, setLogSink/LogEntry, logEmergency…logDebug с порогом).
- `docs/` — документация проекта.
