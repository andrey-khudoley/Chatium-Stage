# Architecture

## Назначение
Базовый шаблон проекта Chatium с минимальным набором страниц и документации.

## Ограничения платформы
- Серверная инфраструктура предоставляется Chatium.
- Нельзя менять стек и зависимости.
- Деплой — автоматически при пуше.

## Основные сценарии
- Открыть главную страницу.
- Авторизоваться и попасть в профиль.
- Открыть админку (только роль Admin).
- Использовать быстрые карточки на главной: «Мой журнал», «Мои задачи» и «Инструменты» переходят в разделы, карточки «Диалоги», «PARA», «Финансы», «Сервисы», «Библиотека» запускают только глобальный glitch-эффект без навигации.
- На странице задач менять статус напрямую из таблицы по клику на ячейку «Статус» через выпадающее меню (`К выполнению / В работе / Завершено / Отменено`) без открытия модалки редактирования.
- На странице задач использовать переключатель `Архив`: в основном списке отображаются только активные задачи, а задачи со статусами `Завершено` и `Отменено` попадают в архивный список.
- В модалке задачи задавать дату/время события в формате `dd/mm/yyyy hh:mm` (24 часа) и срок напоминания в минутах (по умолчанию 15); поле события автоподставляет разделители при вводе (`/`, пробел, `:`), при наступлении момента напоминания на открытой вкладке с задачами срабатывают браузерное уведомление (`Notification`) и звуковой сигнал (внешний audio URL).
- Чат с AI на странице задач поддерживает установку `eventAtMs` и `reminderMinutesBefore` для `create_task`/`update_task` в JSON-actions.
- На вкладке «Задачи» в журнале кнопка «В помидор» и на странице «Управление задачами» кнопка «Добавить в pomodoro» привязывают задачу (`api/pomodoro/assign-task`) и отправляют в шапку событие `assistant:focus-task-selected`. На **странице задач** дополнительно: если после привязки Pomodoro ещё не в статусе `running`, вызывается `api/pomodoro/control` (`start` или `resume`), чтобы таймер реально запустился на сервере (эндпоинт `assign-task` только выставляет `currentTaskId`). В `detail` события передаётся `pomodoroSessionStartedFromTasksPage`, чтобы `Header.vue` не вызывал повторный `start` (иначе сессия сбросилась бы) и вместо этого сделал `syncPomodoro`. Для журнала по-прежнему: при выбранном инструменте (`Помидор/Таймер/Секундомер`) шапка запускает/продолжает остановленный инструмент; при режиме `Часы` — тройное мигание `header-clock`.
- В `Header.vue` для виджета таймеров в режимах «Помидор» и «Таймер» блок `header-clock` показывает слева-направо подложку прогресса (`.header-clock-progress`, насыщенный градиент от акцента + свечение); для секундомера и обычных часов заливки нет.
- На вкладке «Неделя» в журнале перелистывать недели кнопками `← Пред.` / `След. →`; возврат к текущей неделе вынесен в отдельную подстроку (`К текущей неделе`) рядом с датой начала недели, чтобы не нарушать центрирование заголовка.
- В редакторе заметки блокнота использовать кнопку `Экспорт` для трёх сценариев: скачать содержимое как `TXT`, скачать как `MD` и открыть окно печати для сохранения в `PDF`.
- Кнопка «Сохранить» в `NotebookNoteEditor` сохраняет заметку и **не** закрывает редактор; к списку заметок возвращает «Назад к списку». После первого сохранения новой заметки родитель (`JournalNotebookPane`) переключает режим с создания на редактирование по полученному `id`, чтобы следующие сохранения шли через update.
- В списке заметок блокнота (`NotebookNoteCard`): мультиселект — клик по телу карточки (включая строку метаданных под заголовком), открытие редактора — клик по заголовку; отдельного чекбокса нет. Справа колонка действий: для неархивной заметки — «В архив», для архивной — «Вернуть» и «Удалить» (`journalNotesArchiveUrl`, `journalNotesDeleteUrl`).
- В инбоксе (`JournalInboxPane`): та же схема — клик по телу карточки переключает выделение, клик по заголовку открывает редактор; при выборе одной или нескольких заметок показывается `NotebookBulkBar` без перемещения в папки (архив / из архива; **массовое «Удалить»** — только если все выделенные заметки архивные, иначе кнопка скрыта). Одиночное удаление — у архивной карточки рядом с «Вернуть». API: `inboxNotesArchiveUrl`, `inboxNotesDeleteUrl`. Кнопка «Архив» в тулбаре переключает режим: по умолчанию в списке только неархивные заметки, при активной кнопке — только архивные.
- В блокноте (`JournalNotebookPane` + `NotebookFilterBar`): кнопка «Архив» работает так же — в активном состоянии в списке остаются только заметки с `isArchived`, без смешения с обычными. В `NotebookBulkBar` массовое «Удалить» показывается только если все выделенные заметки архивные (та же логика, что в инбоксе).
- В WYSIWYG-редакторе заметки при вставке plain text сохраняются пустые строки: двойные переносы (`\n\n`) не схлопываются и остаются как отдельные абзацы.
- В WYSIWYG-редакторе заметки поддерживается smart-paste: при вставке Markdown из `text/plain` выполняется базовое форматирование в HTML; при наличии `text/html` в буфере используется готовая разметка источника.
- В Markdown-ветке smart-paste пустые строки также сохраняются, чтобы между блоками не терялись визуальные разделители абзацев.
- Вставка HTML при paste выполняется через прямую DOM-вставку (Range API, `insertHtmlAtCursor`) вместо `execCommand('insertHTML')`, чтобы браузер не удалял пустые абзацы. Пустые строки помечены классом `wy-blank-line` c визуальной высотой через `padding` (`height: 0; overflow: hidden; font-size: 0; line-height: 0`).
- Если курсор внутри блока кода (`pre[data-language]` / вложенный `<code>`), paste обрабатывается отдельно: в буфер берётся только plain text (при отсутствии `text/plain` — текст из `text/html` через временный `div`), вставка через Range внутрь `<code>` (`insertPlainTextIntoCodeBlockAtSelection`), без `insertHtmlAtCursor` — иначе та функция поднимала бы точку вставки до корня `contenteditable` и создавала узлы *после* `<pre>`.
- Для диагностики проблем форматирования в `WysiwygEditor` добавлен переключатель режимов `WYSIWYG / HTML`: можно открыть исходник текущей заметки, сверить и вручную поправить HTML, затем вернуться в визуальный режим.
- В режиме `HTML` исходник заметки не должен начинаться с пустой строки: при переходе из `WYSIWYG` редактор отбрасывает технические ведущие переводы строки (`\r/\n`) в source-представлении.
- Переключение `WYSIWYG / HTML` в `WysiwygEditor` работает с устойчивой синхронизацией буфера (`sourceHtml`/`modelValue`/последнее известное HTML): при быстром переключении режимов контент заметки не должен пропадать из-за перезаписи пустым значением.
- При переключении обратно в `WYSIWYG` HTML применяется только после `nextTick`: ветка с `contenteditable` рендерится через `v-if`, поэтому `editorRef` доступен только после обновления DOM; это предотвращает потерю контента из-за ранней записи в несуществующий узел.
- Стратегия источника для paste: если `text/plain` содержит Markdown или явные пустые строки, используется plain-text ветка (даже при наличии `text/html`), чтобы не терять переносы из-за внешних HTML-фрагментов с нестандартной разметкой.
- В редакторе есть нормализация `wy-blank-line`: как только в «технически пустом» абзаце появляется текст, класс удаляется автоматически, чтобы контент не был скрыт стилями пустой строки.
- **Блок кода с подсветкой синтаксиса** реализован в `WysiwygEditor.vue` через highlight.js v11.11.1, загружаемый с CDN `cdnjs.cloudflare.com` (тема `atom-one-dark`). HTML-структура: `<pre data-language="..."><code class="hljs">...</code></pre>`. Язык хранится в атрибуте `data-language` на `<pre>` — сохраняется в HTML-строке заметки и восстанавливается при загрузке. Подсветка загружается лениво (JS + CSS при первом использовании). Floating-пикер языка (`Teleport` в `body`) при фокусе внутри блока кода: позиция `fixed` по `rect.left`/`rect.top` активного `pre`, смещение вверх на высоту пикера + зазор (`translateY(calc(-100% - 0.3rem))`) — левый верх над блоком; 25 языков. Последний выбранный в пикере язык пишется в `localStorage` (`assistant-journal-preferred-code-lang`, только значения из списка) и подставляется как язык по умолчанию при вставке нового блока кода с панели и для fenced-блоков Markdown без указания языка при smart-paste. Подсветка обновляется при: смене языка, уходе фокуса из блока, вводе текста (дебаунс 700ms с сохранением позиции курсора через character-offset). При вставке Markdown с fenced code blocks язык парсится и подсвечивается автоматически. Экспорт в Markdown включает язык после ` ``` `. CSS: акцентная левая рамка на `<pre data-language>`, свечение при фокусе, `.hljs` background overridden на transparent.

## Роутинг
- `index.tsx` — главная (SSR + Vue), единственный роут в корне.
- `web/admin/index.tsx` — админка, `requireAccountRole('Admin')`.
- `web/profile/index.tsx` — профиль, `requireRealUser()`.
- `web/journal/index.tsx` — страница «Мой журнал» (`JournalPage.vue`): компактное вертикальное меню слева (порядок вкладок: Инбокс, Блокнот, Задачи, День, Неделя, Месяц, Привычки) и панель справа с динамической подгрузкой Vue-компонентов (`components/journal/*Pane.vue`); экземпляры панелей кэшируются через `<KeepAlive :max="8">` внутри `<Transition name="journal-view">` (анимация только opacity, без `transform`, чтобы снизить артефакты со `sticky`/таблицами при `overflow: hidden` у `.journal-panel`); при ширине ≤900px меню и панель складываются в колонку, вкладки — горизонтальный скролл. Навигация выделена в отдельный компонент `components/journal/JournalNav.vue`, чтобы структура DOM и стили меню были централизованы. Порядок в меню: сначала постоянный список вкладок, затем (если нужен) разделитель `journal-nav-divider`, и только после него динамический тулбар (`Все задачи` для `tasks`); это фиксирует единый визуальный ритм меню. Кнопка `Все задачи` ведёт на общий список задач (`tasksPageUrl`). Для action-кнопок меню и кнопок вкладок задан кастомный `:focus-visible` в цветах темы (без системной жёлтой обводки). Активная вкладка синхронизируется с адресной строкой: `?tab=notebook|month|week|tasks|day|habits` (для «Инбокс» параметр не добавляется — открытие по умолчанию), `replaceState` при переключении, SSR читает `req.query.tab` и передаёт `journalTabInitial`, чтобы при возврате «назад» со страницы задач открывалась та же вкладка. Вкладки «Блокнот» (`notebook`) и «Инбокс» (`inbox`) используют один UI-компонент `JournalNotebookPane` с разным `variant` и разными API: **блокнот** — Heap `journal-notes` и `api/journal/notes/*`; **инбокс** — отдельная таблица `inbox-notes` и `api/journal/inbox/*` (список/создание/get/update/archive/delete). Упрощённый интерфейс: plain text, превью по первой строке (`title`), архивация, «Показать архив». WYSIWYG в UI не подключается. SSR: `journalNotesInitial` + `inboxNotesInitial` и соответствующие URL. Heap-таблицы: `journal-notes`, `inbox-notes`, `notebook-categories`, `notebook-folders`. Вкладка «Задачи» (`tasks`): задачи со статусом «В работе» (дерево задач на SSR, `JournalDayPane` — порядок, ссылки на клиента/проект на странице задач с query-параметрами, клик по названию задачи открывает модальное редактирование с сохранением через POST `/api/tasks/items/update`, кнопка «В очередь» вызывает POST `/api/tasks/items/release-day`). Вкладка «День» (`day`): `JournalDayInDevelopmentPane.vue` с четырьмя многострочными сегментами (`Ночь`, `Утро`, `День`, `Вечер`) и кнопкой фиксации/разблокировки у каждого сегмента; запись хранится в Heap по `dayKey` (`YYYY-MM-DD`) с границей суток в 05:00 (`api/journal/day/get|save`, `lib/journal-day-key.ts`), старые дни не перезаписываются. Вкладка «Неделя» (`week`): `JournalWeekPane.vue` с верхним блоком общего плана `Неделя #N` (weekly-summary: textarea + `Зафиксировать`/`Редактировать`) и 7 карточками дней (`Понедельник..Воскресенье`) ниже; у каждого дня есть дата, отдельная фиксация и поле ввода. Для визуального выравнивания weekly-summary с дневными карточками summary использует ту же высотную структуру шапки (caption + скрытый спейсер строки даты), поэтому textarea начинается на одной высоте с полями Пн–Вс. Хранение: дневные записи недели — `tables/journal-week-entries.table.ts` (`api/journal/week/get|save`), общий weekly-summary — `tables/journal-week-summary.table.ts` (`api/journal/week/save-summary`); вычисление ключей недели — `lib/journal-week-key.ts`. В кнопках вкладок журнала левая граница всегда видима (неактивные — в цвете border, активная — accent). Вкладка «Месяц» (`month`): `JournalMonthPane.vue` с двумя режимами — календарь месяца и детальный просмотр дня. Календарь: навигация по месяцам (`← Пред.` / `МЕСЯЦ ГОД` / `След. →`), кнопка `К текущему месяцу`, 7-колоночная сетка в рамке на фоне вторичного фона с индикаторами (количество завершённых задач, компактная подпись времени фокуса Pomodoro/tools на ячейке дня, полное значение — во всплывающей подсказке); под сеткой легенда в тех же визуальных стилях показывает итоги за выбранный месяц (всего завершённых задач и суммарное время фокуса). При клике на день открывается детальный вид: заголовок с датой/днём недели и нижней границей, навигация по дням, блоки контента — карточки с шапкой (`jm-section-header`) и телом (`jm-section-body`); у «Завершённых задач» при наличии списка показывается числовой бейдж, каждая задача — отдельная карточка с акцентной полосой слева и разделителем под заголовком для меты «клиент · проект»; далее — время фокуса, план дня из «Недели», дневные записи из «Дня» (ночь/утро/день/вечер). Данные загружаются через `api/journal/month/data` + существующие `api/journal/day/get` и `api/journal/week/get`. Вкладка «Привычки» (`habits`): `JournalHabitsPane.vue` — таблица «привычка + 7 колонок (Пн–Вс)», граница «сегодня» и смена недели с 05:00 (`lib/journal-habits-time.ts`, тот же принцип суток, что `computeJournalDayKeyLocal`), хранение в Heap `journal-habits-week` (`api/journal/habits/get|save`), по прошлым неделям — только просмотр; строки по завершённым неделям не удаляются (для аналитики). Страница доступна без обязательной авторизации.
- `web/tasks/index.tsx` — страница «Управление задачами» (`TasksPage.vue`): боковая панель с иерархией «клиент → проекты» (широкая колонка; вверху — вторичная по акценту кнопка «Новый клиент», новый проект — иконкой в строке клиента; название клиента не переключает контекст по клику, красная подсветка строки клиента — если выбран проект этого клиента; порядок клиентов и проектов в сайдбаре — drag-and-drop за рукоять, сохранение `sortOrder` через POST `/api/tasks/clients/reorder` и `/api/tasks/projects/reorder`) и основная область с таблицей задач в визуальном стиле журнала; на узких экранах (до 640px) таблица превращается в карточки с подписями полей (`data-label`); при ширине viewport **до 1023px** на `documentElement` вешается класс `tasks-page-html-mobile` (увеличен базовый `font-size` для `rem`, крупнее модалки и поля ввода **16px**); режим **master–detail**: сначала полноширинный список клиентов/проектов, после выбора проекта — полноширинный экран задач с кнопкой «Проекты» назад (не две колонки одновременно); DnD порядка на мобильном отключён; **оговорка по DnD:** поведение нативного drag-and-drop в браузере/встроенном клиенте доведено не до конца (иногда индикатор и фактический дроп расходятся или нужен повтор жеста), но смена порядка доступна и сценарий не блокируется; SSR передаёт дерево задач и URL API (`api/tasks/`) для авторизованного пользователя; без входа — подсказка. Модальные формы создания/редактирования, удаление с подтверждением, смена приоритета/статуса/проекта, кнопки «вверх/вниз» и POST reorder для порядка задач в проекте; статус также можно менять напрямую в таблице по клику на ячейку `Статус` (выпадающее меню). В шапке таблицы есть переключатель `Активные/Архив`: задачи со статусами `done` и `cancelled` скрываются из общего списка и показываются в архиве. Кнопка «Отметить» переключает статус: не «В работе» → «В работе» (для дневного списка в журнале), повторный клик — обратно в «К выполнению»; при «В работе» кнопка подсвечена красным; поддержка `?client=` и `?project=` в URL для перехода на клиента/проект. При переименовании проекта в POST `api/tasks/projects/update` передаются `id`, `name` и при необходимости `details` (без `clientId`), чтобы проект не привязывался к клиенту, выбранному в сайдбаре под другим проектом; у проекта в Heap есть также служебное поле `context`, в API/UI не отображается. На десктопе (viewport ≥1024px) справа третья колонка — чат с AI (`components/tasks/TasksAiChatPanel.vue`): фид Chatium на пару пользователь+проект (`tables/task-ai-chat-feeds`, `getChat`, `feedMessages*Handler`), история в фиде передаётся в `startCompletion` как диалог, ответы ассистента — сообщения с `data.assistant`; обновления в реальном времени через `getOrCreateBrowserSocketClient` + `messages_socket_id` и периодический `messages/changes`; кнопка сброса истории — `tasks-ai-chat-reset`. На мобильном чат в нижней шторке (FAB).
- `web/tools/index.tsx` — страница «Инструменты» (`ToolsPage.vue`) содержит одну карточку `Таймеры` с переходом на `web/timers`.

Панель кнопок по статусам: `stopped` — только «Старт»; `running` — «Пауза» и «Пропустить»; `paused` / `awaiting_continue` — «Продолжить» и «Сбросить» (действие `reset`: остановка, `phase=work`, полный `workMinutes`, `cyclesCompleted=0`). Семантика API: `start` — запуск/перезапуск рабочей фазы (`phase=work`, отсчёт от `workMinutes`, `cyclesCompleted=0`; на странице кнопка только при `stopped`), `resume` — из `paused` (продолжить отсчёт) или из `awaiting_continue` (перейти к следующей фазе после овертайма), `pause` — только из `running`, `skip` — пропуск текущей фазы (на странице — только во время отсчёта `running`; на сервере по-прежнему доступен из `awaiting_continue`), `reset` — полный сброс в `stopped` с обнулением циклов; в суммы работы/отдыха и в учёт времени по задаче попадает только фактически прошедшее время (накопление через `tick` до действия), неотработанный остаток фазы не добавляется. По окончании работы или отдыха по таймеру статус `awaiting_continue`: на клиенте показывается секундомер овертайма (`phaseEndsAtMs` — момент окончания фазы), до «Продолжить» следующая фаза не стартует. Счётчики дневной статистики (`.stat-cell`: помидоры, работа, отдых) сбрасываются при смене ключа периода в **05:00 по локальному времени браузера** (`statsDayKey` в запросах к `state/get`, `control`, `settings/save`, `assign-task`; см. `lib/pomodoro-stats-day.ts`; без ключа — fallback `Europe/Moscow`); таймер и `cyclesCompleted` при этом не сбрасываются. В серверной логике ведётся журнал сегментов запусков (`pomodoro-launches`) с разметкой фаз, задач, источника запуска и причины завершения сегмента.

## Разделение слоёв

Принцип разделения ответственности при работе с данными (см. [ADR-0002](ADR/0002-settings-heap-and-layered-api.md)):

| Слой | Каталог | Ответственность |
| --- | --- | --- |
| **Таблицы** | `tables/` | Схемы Heap (поля, типы). Только определение структуры данных. |
| **Репозитории** | `repos/` | Работа с БД: CRUD, запросы. Никакой бизнес‑логики, только вызовы Heap API. |
| **Бизнес‑логика** | `lib/` | Правила, дефолты, валидация значений, вычисления. Вызывает репозитории. |
| **API** | `api/` | HTTP‑эндпоинты, парсинг и первичная валидация запросов, проверка прав. Вызывает lib. |

Поток данных: `HTTP → API → lib → repos → Heap`.

## Структура каталогов
- `config/` — маршруты и `PROJECT_ROOT`.
- `web/` — браузерные роуты модулей (admin, profile, journal, tasks, tools, pomodoro, tests, login).
- `pages/` — Vue‑страницы (минимальные, в т.ч. JournalPage, TasksPage, ToolsPage, PomodoroPage).
- `components/` — переиспользуемые Vue‑компоненты (Header, AppFooter, GlobalGlitch, LogoutModal); подкаталог `components/journal/` — панели разделов журнала (`JournalNotebookPane` — полный блокнот: папки, категории, `NotebookNoteEditor`; `JournalInboxPane` — упрощённый инбокс: список + textarea, архив, мультиселект и `NotebookBulkBar` без папок; `JournalHabitsPane` — таблица привычек, порядок строк на текущей неделе меняется HTML5 drag-and-drop за ручку в ячейке названия); остальные — заглушки и панели дня/недели/месяца/задач, где ещё не выделены отдельно.
- `api/` — API‑эндпоинты (получение и валидация входных данных). File-based: один файл — один эндпоинт с `/`. Примеры: `api/settings/list.ts`, `api/journal/notes/`, `api/journal/categories/`, `api/journal/folders/`, `api/tasks/tree/get.ts`, `api/pomodoro/`, `api/logger/log.ts`, `api/tests/list.ts`.
- `tables/` — Heap‑таблицы (settings, logs, journal-notes, notebook-categories, notebook-folders, task-clients, task-projects, task-items, pomodoro-state, pomodoro-launches).
- `repos/` — репозитории (settings, logs, journal-notes, notebook-categories, notebook-folders, tasks, pomodoro, pomodoro-launches; logs.repo — findBeforeTimestamp для пагинации).
- `lib/` — бизнес‑логика (settings.lib, logger.lib, tasks-types — DTO задач без Heap, pomodoro.lib — конечный автомат фаз, синхронизация времени под lock и запись сегментов запусков Pomodoro).
- `shared/` — общий код (preloader: `completeSequence` после `load` (и таймаут 3 с); безопасное добавление строк в терминал; затем `document.fonts.ready` и `boot-static-ready`; резерв через 12 с / принудительное скрытие через 15 с при зависании; `bootUi.ts` — шрифты (таймаут 10 с), nextTick, rAF, `hideBootLoader`; фон оверлея `#0a0a0a`; logLevel для клиента, logger syslog RFC 5424, createComponentLogger, setLogSink/LogEntry, logEmergency…logDebug с порогом).
- `docs/` — документация проекта.
