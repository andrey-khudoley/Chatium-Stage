# QnA Project Documentation

## Описание

Workspace QnA - это внутренняя knowledge base с документацией по Chatium-специфичным функциям и практикам.

## Структура проекта

### Файлы документации

- **001-run.md** - Руководство по `@shared-route` декоратору и методу `.run()`

  - Как объявлять API маршруты с доступом на фронте и бекенде
  - Примеры использования в Vue компонентах и серверном коде
  - Правила и лучшие практики

- **003-fingerprint.md** - Серверный фингерпринт для дедупликации

- **004-shared-route.md** - Различие между `// @shared` и `// @shared-route`

  - Когда использовать каждую директиву
  - Структура файлов и способы вызова

- **005-avoiding-circular-deps.md** - Избегание циклических зависимостей

  - Правильное использование `.url()` и `.run()`
  - Однонаправленная архитектура зависимостей
  - Антипаттерны и типичные ошибки
  - Генерация уникального ID клиента на основе IP, User-Agent, языка браузера
  - Альтернатива клиентскому счётчику (без JS загрузки)
  - Логика дедупликации кликов за окно времени (5 сек)
  - Примеры с LinkClicks таблицей

- **006-heap-filtering.md** - Фильтрация данных в Heap API

  - Поддерживаемые операторы: $lt, $lte, $gt, $gte, $and, $or, $not
  - Синтаксис фильтрации с примерами
  - Совместимость с разными типами полей (Number, String, Date, Money, RefLink, AnyKind)
  - Альтернативы для неподдерживаемых операторов ($eq, $ne, $in, $nin, $exists)
  - Оптимизация и рекомендации по производительности
  - 25+ реальных примеров использования

- **007-clarity-analytics.md** - Аналитика браузера: clrtUid, clrtSid и счётчик Clarity

  - Различие между clrtUid (браузер) и clrtSid (сессия)
  - Как выбрать переменную для идемпотентности визитов
  - Связь с полями `uid`, `sid`, `session_id`, `user_id` в Clickhouse
  - Практические примеры SQL запросов для аналитики
  - Рекомендации по дедупликации событий при отправке с фронтенда
  - Таблица сравнения всех идентификаторов пользователя

- **008-heap-reflink.md** - Работа с Heap.RefLink для связей между таблицами
  - Синтаксис определения RefLink полей
  - **onDelete поведение**: CASCADE, SET_NULL, NO_ACTION, RESTRICT
  - Выбор правильного поведения при удалении целевой записи
  - Создание и обновление записей со связями
  - Чтение и доступ к связанным данным (синхронно и асинхронно)
  - Методы RefLink объектов (.get(), .getTitle(), .getTargetTableRepo())
  - Batch loading для оптимизации (избежание N+1 запросов)
  - Фильтрация по RefLink полям
  - Типичные ошибки: циклические зависимости, передача объектов вместо ID
  - 4+ примера реальных случаев: заказы с клиентами, вложенные связи
  - Управление миграциями при удалении целевых таблиц

### Серверные файлы

- **index.tsx** - Entry point приложения (минималистичная страница)

## Архитектура

### Назначение

Knowledge base для разработчиков Chatium - быстрый доступ к примерам и объяснениям частых задач.

### Технологии

- Vue 3 (фронтенд компоненты, если будут добавлены)
- TypeScript (примеры кода)
- Heap Tables (хранение данных)

## Добавляемые документы

При добавлении новой документации:

1. Создай файл `NNN-topic.md` (где NNN - номер)
2. Включай примеры кода, таблицы, good/bad practices
3. Ориентируйся на 1200-2000 символов
4. Обновляй этот файл (.CHATIUM-LLM.md)

## Важные замечания

- Все примеры кода ориентированы на реальный контекст Chatium
- Не используются стандартные nodejs модули (crypto, fs, http)
- Документация практическая, а не теоретическая
