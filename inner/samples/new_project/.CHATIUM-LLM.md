# Индекс LLM‑документации

## Назначение

Этот файл указывает, где лежит проектная документация и где хранится история переписки с LLM.

## Документация проекта

- README: `./README.md`
- Архитектура: `./docs/architecture.md`
- Данные: `./docs/data.md`
- API: `./docs/api.md`
- Runbook: `./docs/run.md`
- ADR: `./docs/adr/`

## История LLM

- Каталог: `./docs/LLM/`
- Формат имен: `NNN-<agent>.md` (например, `001-codex.md`)
- Нумерация сквозная: 001, 002, 003…
- Текущий лог: `./docs/LLM/001-codex.md`

## Правила заполнения LLM‑логов

- **Каждый новый агент/сессия** получает отдельный файл.
- В файл добавляется **полная история** сообщений (user + assistant) с датами.
- Обновлять сразу после завершения диалога.
