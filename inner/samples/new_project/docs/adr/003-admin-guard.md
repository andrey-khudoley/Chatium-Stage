# ADR-003: Админ‑функции ограничены ролью Admin

**Статус:** принят
**Дата:** 2026-01-30

## Контекст

В проекте есть функции управления настройками и логами, которые должны быть доступны только администраторам.

## Решение

Все админ‑маршруты и UI защищаются ролью Admin:

- `requireAccountRole(ctx, 'Admin')` в API (`api/admin-settings.ts`, `api/admin-logs.ts`)
- UI `/admin` ориентирован на Admin‑роль

## Последствия

- Настройки/логи защищены от обычных пользователей.
- Нужен процесс выдачи роли Admin в окружении.

## Альтернативы

- Авторизация по отдельному списку пользователей (усложняет администрирование).
- Публичный доступ (неприемлемо по безопасности).

## Когда пересматривать

- Если появятся новые роли и потребуется более тонкая матрица доступов.
