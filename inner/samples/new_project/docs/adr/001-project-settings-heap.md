# ADR-001: Хранение настроек проекта в Heap (key‑value)

**Статус:** принят
**Дата:** 2026-01-30

## Контекст

Нужны глобальные настройки проекта (название, описание, уровень логирования, счётчики), доступные из разных частей системы и редактируемые через админ‑панель.

## Решение

Использовать Heap‑таблицу `ProjectSettings` как key‑value хранилище:

- ключ — `key` (string), значение — `value` (any)
- доступ и обновление через `createOrUpdateBy()`
- нормализация значений выполняется на уровне бизнес‑логики (`lib/settings.ts`, `lib/logging.ts`)

## Последствия

- Простая структура без миграций и сложных связей.
- Нужна явная нормализация/валидация значений в коде.
- Удобно добавлять новые настройки без изменения схемы.

## Альтернативы

- Отдельные таблицы под каждую настройку (слишком многословно).
- JSON‑конфиг файл (хуже для динамического обновления в рантайме).

## Когда пересматривать

- Если настроек станет много и потребуется строгая типизация/валидация на уровне схемы.
- Если появятся требования к истории изменений/аудиту.
