# ADR-002: Централизованное логирование с сохранением в Heap

**Статус:** принят
**Дата:** 2026-01-30

## Контекст

Нужно единое логирование для серверного кода и UI, плюс возможность просмотра логов в админ‑панели и live‑обновления.

## Решение

Использовать `shared/debug.ts` как единый логгер и сохранять логи в Heap:

- `Debug.*` → `setPersistLogCallback()` → `persistLog()`
- Хранилище: `ProjectLogs` (уровень, сообщение, код)
- Счётчики warn/error в `ProjectSettings`
- Авто‑trim старых логов при превышении лимита (1000 записей)
- WebSocket для live‑обновлений в админке

## Последствия

- Логи доступны в UI `/admin` и в Heap.
- Дополнительная нагрузка на запись в БД.
- Нужно следить за рекурсией (логирование внутри endpoints логов ограничено).

## Альтернативы

- Писать только в `console/ctx.log` (нет UI‑просмотра).
- Внешние лог‑сервисы (не предусмотрено шаблоном).

## Когда пересматривать

- Если логов станет существенно больше, чем 1000, или потребуется архивирование.
- Если появятся требования к централизованному лог‑сервису.
