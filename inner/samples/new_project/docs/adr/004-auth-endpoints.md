# ADR-004: Авторизация через Telegram OAuth и helper для пароля

**Статус:** принят
**Дата:** 2026-01-30

## Контекст

Нужно поддержать несколько способов входа в шаблонном проекте: через Telegram и через пароль (с серверным получением хеша).

## Решение

- Telegram OAuth через `api/auth-telegram.ts` (возвращает OAuth URL).
- Helper для пароля через `api/auth-password.ts` (возвращает hash по `it/ik/pwd`).

## Последствия

- Логика входа вынесена в API и используется из UI `/login`.
- Для пароля есть минимальный контракт без хранения пользовательских данных в проекте.

## Альтернативы

- Использовать только Telegram OAuth (ограничение по каналам входа).
- Реализовать полный auth‑flow в проекте (избыточно для шаблона).

## Когда пересматривать

- Если требуется корпоративная/SSO‑авторизация.
- Если меняется API провайдера авторизации.
