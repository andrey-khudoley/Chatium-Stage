# Partnership — шаблон проекта Chatium

Кратко: базовый шаблон Chatium‑проекта с готовыми страницами, админкой настроек, системой логирования и тестовым контуром. Нужен, чтобы быстро стартовать новый проект с рабочей инфраструктурой.

## Что делает

- HTML‑страницы: `/`, `/login`, `/profile`, `/admin`, `/tests`.
- Админка: управление настройками проекта (название, описание, уровень логирования).
- Система логов: `Debug → Heap (ProjectLogs)` + счётчики предупреждений/ошибок.
- API для авторизации: Telegram OAuth URL и хеш пароля.
- Тестовый контур: `/tests` и `/tests/ai` для smoke‑проверки.

## Что принципиально не делает

- Нет бизнес‑логики домена (заказы, платежи, каталоги и т.п.).
- Нет фоновых задач/джобов и cron‑процессов.
- Нет полноценного публичного REST/GraphQL API.
- Нет внешних интеграций кроме Telegram OAuth и hash‑endpoint.
- Нет production‑конфига деплоя/CI/CD (нужно добавить под проект).

## Стек и ключевые зависимости

- Chatium runtime (file‑based routing, `app.html`/`app.get`/`app.post`).
- Vue 3.5.13 для страниц и компонентов (`/pages`).
- TypeScript.
- Heap (`@app/heap`) как хранилище данных.
- Auth/roles (`@app/auth`, `@users/sdk/auth`).
- WebSocket (`@app/socket`) для live‑логов.
- TailwindCSS 3.4.16 и FontAwesome 6.7.2 (CDN), шрифт Share Tech Mono.

## Где лежит основная логика

- Серверная логика и настройки: `./lib`.
- API‑контракты: `./api`.
- UI страниц: `./pages`.
- Общие компоненты и Debug: `./shared`.
- Данные (Heap таблицы): `./tables`.
