# Архитектура федеративных чатов

## Общая концепция

Федеративный чат — это чат, который "живёт" в одном Chatium-аккаунте (хост), но доступен для подключения из других аккаунтов (клиентов). Участники из разных аккаунтов видят общую историю сообщений и могут взаимодействовать друг с другом в реальном времени.

## Ключевые принципы

### 1. Единый источник истины (Single Source of Truth)
- **Хост-инстанс** — единственный владелец данных фида
- Все сообщения сохраняются ТОЛЬКО в Feed API хост-инстанса
- Клиентские инстансы получают данные только для отображения, не хранят историю

### 2. Токен-based авторизация
- Каждый федеративный чат имеет уникальный federation token
- Токен даёт доступ ТОЛЬКО к конкретному feedId
- Токен используется для всех API запросов и WebSocket подключений

### 3. Изоляция WebSocket
- Для каждого федеративного чата создаётся отдельный WebSocket канал
- Канал привязан к паре `(feedId, federationToken)`
- Невозможно получить события из других чатов

### 4. Прокси-участники
- Удалённые участники представлены в хост-инстансе как "виртуальные" пользователи
- Каждый прокси-участник содержит информацию об исходном аккаунте

## Структура данных

### Таблица `chats` (расширение)

```typescript
{
  // Существующие поля...
  
  // Федеративные поля
  isFederated: boolean,              // Является ли чат федеративным
  federationToken: string | null,    // Токен доступа (генерируется на хосте)
  isHostInstance: boolean,           // true если это хост-инстанс, false если подключенный
  hostFederationUrl: string | null,  // URL хост-инстанса (для клиентских)
  federationEnabled: boolean,        // Включена ли федерация (можно отключить временно)
  
  // Метаданные
  federatedAt: Date | null,          // Когда чат стал федеративным
  connectedInstancesCount: number,   // Сколько инстансов подключено (обновляется хостом)
}
```

### Таблица `federation_connections` (новая)

Хранит информацию о подключенных инстансах (ТОЛЬКО на хост-инстансе):

```typescript
{
  id: string,
  chatId: string,                    // RefLink на chats
  instanceUrl: string,               // URL клиентского инстанса (например, https://other.chatium.ru)
  instanceDomain: string,            // Домен для отображения (например, other.chatium.ru)
  instanceName: string,              // Название инстанса (опционально)
  
  connectedAt: Date,                 // Когда подключился
  lastSeenAt: Date,                  // Последняя активность
  isActive: boolean,                 // Активно ли подключение
  
  // Статистика
  messagesCount: number,             // Сколько сообщений отправлено из этого инстанса
  participantsCount: number,         // Сколько участников из этого инстанса
}
```

### Таблица `federation_invitations` (новая, на хост-инстансе)

Хранит приглашения для подключения клиентских инстансов:

```typescript
{
  id: string,
  feedId: string,                    // ID федеративного чата
  invitationToken: string,           // 64-char hex token
  
  // Ограничения
  maxUses: number,                   // Максимум подключений
  usedCount: number,                 // Сколько использовано
  expiresAt: Date,                   // Срок действия
  
  // Безопасность
  allowedInstances: string[] | null, // Разрешённые домены
  createdBy: string,                 // ID создавшего админа
  
  // Статус
  status: 'active' | 'revoked' | 'expired',
  createdAt: Date,
  revokedAt: Date | null,
}
```

### Таблица `federation_participants` (новая)

Хранит информацию о том, из какого инстанса каждый участник (на ВСЕХ инстансах):

```typescript
{
  id: string,
  chatId: string,                    // RefLink на chats
  participantId: string,             // ID участника в Feed API
  
  // Для локальных участников
  isLocal: boolean,                  // true если из текущего аккаунта
  userId: string | null,             // UserRefLink (только для локальных)
  
  // Для удалённых участников
  remoteInstanceUrl: string | null,  // URL исходного инстанса
  remoteInstanceDomain: string | null, // Домен для отображения
  remoteUserId: string | null,       // ID пользователя в удалённом инстансе
  remoteDisplayName: string,         // Имя пользователя
  remoteAvatarUrl: string | null,    // Аватар пользователя
  
  joinedAt: Date,
  lastActiveAt: Date,
}
```

## Жизненный цикл федеративного чата

### 1. Создание (на хост-инстансе)

```
Администратор → Создаёт чат → Включает федерацию
                                    ↓
                          Генерируется federationToken
                                    ↓
                          Создаётся federation URL для подключения
                                    ↓
                          URL публикуется для других инстансов
```

**Формат federation URL (legacy, не рекомендуется):**
```
https://host.chatium.ru/projekt-chat/api/federation/connect?token=FEDERATION_TOKEN&feedId=FEED_ID
```

**Формат invitation (рекомендуется, out-of-band):**
```
federation-invite:INVITATION_TOKEN@host.chatium.ai
```

Пример: `federation-invite:a3f7b2d9e8c1...5f6a9b2c@chatium.ai`

### 2. Подключение (на клиентском инстансе) — Межсерверное установление

```
Администратор хоста → Создаёт invitation (out-of-band)
                              ↓
                    Передаёт код админу клиента (email/TG/QR)
                              ↓
Администратор клиента → Вводит код в свой инстанс
                              ↓
                    Его сервер запрашивает info (сервер-сервер)
                              ↓
                    Подтверждает подключение (сервер-сервер)
                              ↓
                    Получает federation token
                              ↓
                    Создаёт "зеркальный" чат локально
                              ↓
                    Подключение к WebSocket хоста
```

**Важно:** Администраторы работают только со своими инстансами. Все запросы к внешним хостам выполняются сервер-сервером, что позволяет работать при блокировках между кластерами.

Подробнее: [10-federation-establishment.md](10-federation-establishment.md)

### 3. Отправка сообщения

#### Из хост-инстанса:
```
Участник → Отправка → createFeedMessage → WebSocket broadcast
                                              ↓
                                    Всем локальным подписчикам
                                              ↓
                                    Всем федеративным каналам
```

#### Из клиентского инстанса:
```
Участник → Отправка → POST /federation/messages/send (на хост)
                                              ↓
                      Валидация токена + участника
                                              ↓
                      createFeedMessage на хосте
                                              ↓
                      WebSocket broadcast (как в п.1)
                                              ↓
                      Клиент получает своё сообщение через WebSocket
```

### 4. Получение обновлений

Все инстансы (хост + клиенты) подписаны на WebSocket канал:
```
federation-${feedId}-${federationToken}
```

При любом событии (новое сообщение, реакция, редактирование, удаление):
```
Хост → sendDataToSocket → WebSocket Gateway → Все подключенные инстансы
```

## Идентификация участников

### Формат отображения

**Локальный участник (из своего аккаунта):**
```
Иван Петров
```

**Удалённый участник:**
```
Мария Сидорова из other.chatium.ru
```

### Формат participantId в Feed API

**Локальный:**
```
local-{userId}
```

**Удалённый:**
```
remote-{instanceDomain}-{remoteUserId}
```

Пример: `remote-other.chatium.ru-user123`

### Создание прокси-участника на хосте

Когда клиентский инстанс отправляет сообщение:
1. Хост получает в теле запроса информацию об отправителе
2. Проверяет, есть ли такой прокси-участник
3. Если нет — создаёт через `createOrUpdateFeedParticipant` с виртуальным userId
4. Сохраняет mapping в `federation_participants`

## Безопасность и изоляция

### Принцип минимальных привилегий
- Федеративный токен даёт доступ ТОЛЬКО к одному feedId
- Невозможно получить список других чатов
- Невозможно получить участников других чатов
- Невозможно подключиться к WebSocket других чатов

### Валидация на каждом запросе
1. Проверка federationToken
2. Проверка соответствия feedId
3. Проверка, что чат действительно федеративный
4. Проверка, что федерация активна
5. Проверка прав участника

### Rate Limiting
- Ограничение запросов с одного инстанса: 100 req/min
- Ограничение на создание сообщений: 10 msg/min на участника
- Блокировка при превышении

## Обработка офлайна

### Хост-инстанс недоступен
- Клиентские инстансы показывают статус "Соединение потеряно"
- Отправка сообщений блокируется
- При восстановлении — автоматический reconnect WebSocket
- История синхронизируется через GET /messages

### Клиентский инстанс недоступен
- Хост продолжает работать нормально
- События накапливаются в WebSocket очереди
- При reconnect клиент получит пропущенные события

## Ограничения и требования

### Требования к хост-инстансу
- Должен быть публично доступен по HTTPS
- Стабильное соединение для WebSocket
- Достаточные ресурсы для обработки запросов от всех клиентов

### Требования к клиентскому инстансу
- Должен иметь возможность делать исходящие HTTPS запросы
- Должен поддерживать WebSocket клиент

### Ограничения
- Максимум 100 подключенных инстансов к одному чату
- Максимум 1000 участников во всех инстансах суммарно
- История сообщений: клиент получает только последние 1000 при подключении
- Прикрепленные файлы должны быть доступны публично (через Chatium Filestorage)
