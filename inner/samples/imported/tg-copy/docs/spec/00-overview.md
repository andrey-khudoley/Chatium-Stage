# 00. Обзор и соглашения

## Что строим

Мессенджер в стиле Telegram на Chatium: SPA на Vue 3 + серверные file-based роуты. Сообщения и участники — в Feed API (`@app/feed`); метаданные — в Heap (`@app/heap`); real-time — WebSocket (`@app/socket`). Базовый путь приложения — **`/tg/`**.

## Функциональные модули (что должно работать)

1. Группы, каналы, личные и публичные чаты; платные чаты с подписками.
2. Сообщения: текст (markdown), reply, пересылка, редактирование (автор), удаление, системные.
3. Вложения: изображения, видео, файлы, голосовые (waveform + транскрипция), видео-кружки.
4. Реакции (одна на пользователя), упоминания `@[Имя](userId)`, индикатор непрочитанных упоминаний.
5. Real-time: typing, read-receipts, live-доставка.
6. Папки, закрепление чатов (drag&drop), фильтры, закреплённые сообщения, inbox-бейджи.
7. Приглашения: username/email/phone/userId/ссылка.
8. Роли чата owner/admin/guest; модерация (мьют/бан); блокировки; приватность ЛС.
9. AI-агенты: личный чат с агентом, агент в группе (режимы all/admins/mention), тулы.
10. Подписки: тарифы с периодами, оплата (`@pay/sdk`), автопродление, ручная выдача.
11. Профиль, онбординг, темы, масштаб, PWA + push (FCM).

## Глоссарий

| Термин | Значение |
| --- | --- |
| **feedId** | id фида (= чата) в Feed API; первичный идентификатор чата во всём проекте |
| **chatId** | в части роутов/таблиц синоним feedId (`chat-plan-chats.feedId`, `subscriptions.chatId`) |
| **участник (participant)** | связь user↔feed; имеет `role` (`owner`/`admin`/`guest`) |
| **роль воркспейса** | `None`/`Staff`/`Admin`/`Developer`/`Owner` — `ctx.user.is(role)` / `requireAccountRole` |
| **бот-пользователь** | пользователь-агент, создаётся `createOrUpdateBotUser`, становится участником фида |
| **chainKey** | ключ цепочки сообщений агента: `chat_<feedId>_agent_<agentId>` |
| **inboxSubjectId** | субъект inbox фида для фильтрации бейджей; формат `chat-<id>` / `direct-<...>` |
| **appId** | маркер изоляции: все фиды создаются с `data: { appId: 'tg' }` |

## Соглашения спеки

- **Имена роутов** сохраняй как в спеке (`apiChatsListRoute` и т.п.) — на них завязаны клиентские вызовы.
- **Контракты** описаны как: метод+путь, авторизация, вход (params/body/query), выход, логика по шагам, WS-события, ошибки. Воспроизводи **функциональные** контракты 1:1.
- **Формат ответа**: успех большинства мутаций — `{ success: true, ... }`; чтения возвращают доменный объект напрямую.
- **WS-события** обязательны — UI на них завязан. Не пропускай рассылку.
- **Соответствие платформе обязательно.** Спецификация предписывает корректный по `inner/docs/` способ реализации (см. [01-platform.md](01-platform.md)), а НЕ воспроизведение исходного sample. Там, где описание контракта расходится со структурными правилами (организация файлов, URL, блокировки, ошибки, логирование) — приоритет у правил из 01.

## Обязательные правила соответствия (приоритет над поведением sample)

Полностью — в [01-platform.md](01-platform.md); статус по каждому пункту и план исправления — в [../diff.md](../diff.md). Кратко:

- **Один файл = один роут** (path `'/'`); браузерные — в `/web/`, API — в `/api/`. Несколько `app.*` в одном файле запрещено.
- **Никакого хардкода URL.** Все ссылки — через `config/routes.tsx` (`withProjectRoot`/`withProjectRootAndSubroute`); серверные редиректы — относительные `./`, `../`.
- Клиентские роуты помечать `// @shared-route`; в `.vue` не импортировать `tables/`/`repos/`/`lib/`.
- Логирование — только `ctx.account.log(...)`; каждая ветка с бизнес-смыслом логируется. `console.log` запрещён.
- Race conditions (`findOneBy`→`create`, read-modify-write) — `runWithExclusiveLock` из `@app/sync`.
- Ошибки — классы `@app/errors` (`NotFoundError`/`AccessDeniedError`/`ValidationError`), не `throw new Error`.
- Роль администратора — `requireAccountRole(ctx, 'Admin')` (с заглавной — это **корректно** по 003-auth; не `'admin'`). Проверка — первой строкой.
- Heap: `countBy` (не `findAll().length`), `where` (не JS-`filter`), `order [{f:'asc'}]`, методы `Money`, `RefLink` с `onDelete`, не объявлять/не передавать `id/createdAt/updatedAt`.
- Файлы ≤300–400 строк (крупные `.vue`/`api`-файлы — дробить). Tailwind 3.4.16, FontAwesome 6.7.2.

## Физические идентификаторы таблиц

Heap-таблицы объявляются с физическим id (`t_projekt_chat_<name>_<rnd>`). Суффиксы случайны — генерируй заново, но **зафиксируй id `chats` и `chat-subscription-plans`** перед созданием таблиц, ссылающихся на них через `RefLink` (см. [02-data-model.md](02-data-model.md)).
