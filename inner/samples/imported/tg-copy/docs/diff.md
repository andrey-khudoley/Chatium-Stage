# diff.md — соответствие приложения вашим условиям

Чеклист: что в приложении `tg-copy` **уже соответствует** правилам платформы Chatium (`inner/docs/`) и спецификации `docs/spec/`, и что **не реализовано / реализовано неправильно** и требует доработки.

**Легенда:** ✅ соответствует · ❌ не реализовано или сделано неправильно · ⚠️ частично / зависит от файла.

Приложение — импортированный sample (`@kunitcan/tg-copy`), созданный под другие условия. Контракты фич (что делают роуты) корректны и переиспользуемы; расходятся **структурные** правила и инфраструктура.

## 1. Роутинг и URL (002-routing, 006-arch)

| # | Правило | Статус | Сейчас в приложении | Действие |
|---|---|---|---|---|
| 1.1 | Один файл = один роут | ❌ | Много роутов в одном файле (`chats.ts` — 9, `messages.ts` — 6, `invites.ts` — 7, …) | Разнести каждый роут в отдельный файл `/api/<module>/...` (путь `'/'`), сохранив имена `export const ...Route` |
| 1.2 | `config/routes.tsx` (`PROJECT_ROOT`, `withProjectRoot`) обязателен | ❌ | Файла нет; каталога `/config/` нет | Создать `config/routes.tsx` |
| 1.3 | Никакого хардкода URL | ❌ | Хардкод `/tg/...` повсюду: `inboxUrl:'/tg/chat~...'`, `href="/tg/manifest.json"`, `fetch('/tg/api/chats~list')`, `successUrl:'/tg#/subscriptions'`, `back=/tg` | Все ссылки — `withProjectRoot(route.url())` / `withProjectRootAndSubroute(...)` |
| 1.4 | Серверные редиректы — относительные `./`, `../` | ⚠️ | `index.tsx` редиректит на `/s/auth/signin?back=/tg` (системный путь ок, но `back` хардкод) | `back` строить через `withProjectRoot` |
| 1.5 | Путь внутри файла `'/'` (минимум тильд) | ❌ | Подпути и `~` в URL (`chat~`, `api/<file>~<sub>`) | После 1.1 пути станут `'/'` |
| 1.6 | Вход `.body((s)=>...)` / `.query(...)` | ⚠️ | Большинство — через `.body()`; но `set-paid`, `read-mentions/mark` читают body без схемы | Привести к `.body()` |
| 1.7 | Браузерные роуты в `/web/`, API в `/api/` | ⚠️ | API в `/api/` ✅; SPA — один корневой `index.tsx` ✅; `invite.tsx` в корне | `invite.tsx` → `/web/invite/` (или оставить как доп. корневой роут согласно 006 «эндпоинт рядом с индексом») |
