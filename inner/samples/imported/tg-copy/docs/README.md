# Документация проекта `tg-copy`

Telegram-подобный мессенджер на платформе Chatium. SPA на Vue 3, серверная часть — file-based роуты Chatium поверх `@app/feed` (сообщения/участники), `@app/heap` (метаданные), `@app/socket` (real-time), `@pay/sdk` (подписки), `@ai-agents/sdk` (агенты), FCM (push).

## Спецификация для воссоздания

В каталоге [spec/](spec/) лежит **самодостаточная техническая спецификация**, приведённая в соответствие правилам платформы (`inner/docs/`): по ней ИИ-агент воссоздаёт функционально идентичный проект, оформленный **корректно** под эти правила. См. [spec/README.md](spec/README.md).

## Соответствие условиям — [diff.md](diff.md)

[diff.md](diff.md) — чеклист: что в текущем приложении **уже соответствует** правилам платформы, а что **не реализовано или сделано неправильно** (с приоритетами доработки). Документы `docs/01–08` описывают приложение **как есть** (включая отклонения, напр. хардкод `/tg/`); спецификация `spec/` и `diff.md` описывают **как должно быть**.

## Важные оговорки

- Проект — **импортированный sample** (`templatePath: @kunitcan/tg-copy`). Местами нарушает платформенные инварианты workspace (использует `console.log`, `requireAccountRole(ctx, 'Admin')` с заглавной буквы). Это поведение исходника; в спецификации оно зафиксировано как есть, с пометками.
- Базовый путь приложения — `/tg/`. Все ссылки, `inboxUrl`, манифест и push завязаны на него.
- Каталог [../fed-docs/](../fed-docs/) — проектная спека федерации; в коде она **не реализована** (0 ссылок на `federation`).
