## Описание проекта

Система управления календарем для записи клиентов на встречу. Аналог Calendly - позволяет администраторам управлять расписанием, а клиентам - выбирать удобное время для записи без необходимости авторизации.

## Архитектура

### Таблицы базы данных

#### 1. **appointments** - Записи клиентов

- `id`: ID записи (автоматический)
- `name`: Имя клиента
- `telegramId`: Telegram username или ID клиента (опциональное)
- `email`: Электронная почта (опциональное)
- `comments`: Комментарий клиента (опциональное)
- `appointmentDate`: Дата записи
- `appointmentTime`: Время записи (HH:MM)
- `duration`: Длительность встречи в минутах
- `status`: Статус записи:
  - `new`: новая запись
  - `confirmed`: подтверждена
  - `in_progress`: в работе
  - `completed`: завершена
  - `cancelled`: отменена (слот открывается для новой записи)
- `notes`: Заметки администратора

#### 2. **calendarSettings** - Настройки календаря

- `id`: ID настроек (обычно одна запись)
- `title`: Название календаря
- `workDaysOfWeek`: Рабочие дни (1-7, где 1=ПН, 7=ВС)
- `workStartTime`: Время начала работы (HH:MM)
- `workEndTime`: Время конца работы (HH:MM)
- `slotDurationMinutes`: Длительность одного слота в минутах (интервал между слотами)
- `noticeBeforeMinutes`: Минимум минут до записи
- `maxDaysAhead`: Количество дней вперед для записи
- `isActive`: Активна ли запись

##### Статусы записей и их цвета в интерфейсе:

- 🟡 **Новая** (new) - желтый
- 🟣 **Подтверждена** (confirmed) - фиолетовый
- 🔵 **В работе** (in_progress) - синий
- 🟢 **Завершена** (completed) - зелёный
- ⚪ **Отменена** (cancelled) - серый

#### 3. **adminUsers** - Администраторы календаря

- `id`: ID администратора
- `email`: Email администратора (используется для авторизации)
- `firstName`: Имя администратора
- `lastName`: Фамилия администратора
- `role`: Роль (staff, admin)
- `isActive`: Активен ли администратор
- `isOwner`: Является ли владельцем (первый администратор)

##### Иерархия ролей:

- 👑 **Owner** - первый создавший администратор:
  - Может менять роли всех администраторов (кроме себя)
  - Может деактивировать всех администраторов (кроме себя)
  - Сам неприкасаемый - никто не может изменить или деактивировать Owner
  - Имеет полный доступ ко всем функциям
- 🔐 **Admin** - администратор с расширенными правами:
  - Может менять роли только Staff
  - Может деактивировать только Staff
  - НЕ может изменять других Admin или Owner
  - Имеет полный доступ к календарю и записям
- 📝 **Staff** - сотрудник:
  - НЕ может менять роли
  - НЕ может деактивировать других администраторов
  - Имеет полный доступ к календарю и записям

### API Routes

#### Публичные API (для клиентов)

1. **GET /api/appointments/settings**

   - Получить текущие настройки календаря
   - Ответ: объект calendarSettings

2. **POST /api/appointments/schedule**

   - Получить расписание на несколько дней (для интерфейса типа Calendly)
   - Параметры: { startDate: string (YYYY-MM-DD), days: number (максимум 14), isAdmin?: boolean }
   - **Параметр isAdmin:** если true, игнорируются ограничения рабочих дней и минимального времени
   - Возвращает объект с расписанием для каждого дня
   - Ответ: { schedule: { [dateStr]: Array<{time: string, available: boolean, booked: boolean}> }, settings: calendarSettings }
   - Используется для отображения расписания с горизонтальной прокруткой по датам
   - **Важно:** слоты отмечаются как занятые на всю длительность встречи (slotDurationMinutes)
   - **Для клиентов:** нерабочие дни показывают слоты с `available: false, booked: false` (отображаются как "Недоступно")
   - **Для администраторов (isAdmin=true):** все дни показывают слоты как свободные или занятые, игнорируя рабочие дни

3. **POST /api/appointments/available-slots** (старый формат, сохранён для обратной совместимости)

   - Получить свободные слоты на определенную дату
   - Параметры: { date: string (YYYY-MM-DD) }
   - Логика:
     - Проверяет, активен ли календарь (`isActive = true`), если нет - возвращает пустой список
     - Исключаются слоты со статусом, отличным от "отменена"
     - Для сегодняшнего дня исключаются слоты, которые раньше чем (текущее время + noticeBeforeMinutes)
     - Время вычисляется в московской зоне (UTC+3)
     - **Важно:** учитывает длительность записей, используя slotDurationMinutes
   - Ответ: { slots: Array<{time: string, available: boolean}> } или пустой массив если календарь неактивен

4. **POST /api/appointments/create**
   - Создать новую запись клиента
   - Параметры: { name, telegramId (опциональная), email (опциональная), comments (опциональный), appointmentDate, appointmentTime }

   - Длительность встречи берется из настроек календаря (slotDurationMinutes)
   - Ответ: { success: boolean, appointment: Appointment } или { error: string }

   - Обновить настройки календаря
   - Параметры: Partial<CalendarSettings>

     - Только Owner может менять статус любого (кроме себя)
   - Ответ: AdminUser или { error: string }

10. **POST /api/appointments/delete-admin/:id**
    - Удалить администратора
    - Проверка прав:
      - Owner не может быть удален
      - Нельзя удалить себя

      - Admin может удалять только Staff

## Механика работы

### Расписание

1. Администратор устанавливает:

   - Рабочие дни недели (например, 1-5 = ПН-ПТ)
   - Часы работы (например, 09:00-18:00)
   - Длительность слота (например, 30 минут) - это интервал между слотами
   - Количество дней вперед для записи (например, 30 дней)
   - Минимум времени до записи в минутах (например, 60 минут)

2. Для каждой даты система:

   - Проверяет, является ли день рабочим
   - Генерирует список свободных слотов на день
   - Исключает уже забронированные слоты (с учетом их длительности)
   - **Для сегодняшнего дня**: исключает слоты, которые раньше чем (текущее московское время + noticeBeforeMinutes)
   - **Исключает слоты с отмененными записями**: они автоматически становятся доступными

3. Клиент:
   - Переходит на главную страницу `/`
   - **Видит расписание на 14 дней сразу** (времена слева, даты листаются вправо-влево)
   - **Видит цветовое обозначение:** зелёный = свободно, красный = занято, серый = недоступно
   - **Листает недели** кнопками "Назад" и "Вперед"
   - Кликает на свободный слот
   - Вводит имя и опционально Телеграм, почту, комментарий
   - Подтверждает запись
   - Новая запись автоматически получает статус "новая" и длительность из настроек календаря

### Управление статусами в админке

Администратор может менять статус записи:

- **Через модальное окно**: кликните на запись, откроется окно с деталями и выпадающим меню статусов

Доступные статусы:

1. **Новая** → **Подтверждена** — запись одобрена
2. **Подтверждена** → **В работе** — встреча идет
3. **В работе** → **Завершена** — встреча завершена
4. **Любой статус** → **Отменена** — отмена записи (слот снова открывается для клиентов)

**Важно:** При отмене записи (статус "отменена") соответствующий временной слот автоматически становится доступным для новой записи.

### Авторизация администраторов

- Администраторы авторизуются через email (используется встроенная система авторизации Chatium)
- После авторизации система проверяет наличие администратора в таблице adminUsers
- Доступ в админку выдается только администраторам
- При первой авторизации новый администратор автоматически добавляется в систему
- Проверка администратора происходит на клиенте (в AdminDashboard.vue) через API маршрут `/check-admin`
- Если пользователь не администратор — показывается сообщение об ошибке "Нет доступа"
- Если администратор — загружается интерфейс админки с тремя вкладками

### Временная зона

- Система использует **московское время (UTC+3)** для всех расчетов
- Минимальное время до записи рассчитывается на основе московского времени
- Это обеспечивает корректную работу независимо от часового пояса пользователя

## Механика расчета занятых слотов

Система корректно учитывает длительность встречи при расчете доступности слотов:

1. **Администратор устанавливает длительность слота:** 60 минут (это длительность всех встреч)
2. **На календаре уже есть запись:** 11:00
3. **Клиент выбирает:** 10:00
4. **Система создает запись:** на 60 минут (с 10:00 до 11:00)
5. **Следующие слоты становятся занятыми:** 10:00 занят новой записью

**Формула расчета:**

- Время слота: HH:MM
- Длительность встречи: slotDurationMinutes (из настроек)
- Слот занят если: есть запись на это время
- Длительность всех встреч: фиксированная, берется из настроек календаря
