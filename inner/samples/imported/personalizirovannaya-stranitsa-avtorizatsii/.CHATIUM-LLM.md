# Страница авторизации

## Структура файлов

### Основные файлы

- `/index.tsx` - Главная страница с проверкой авторизации и настройкой провайдеров
- `/Index.vue` - Основной компонент страницы авторизации с выбором метода
- `/sdk.ts` - SDK для работы с API авторизации (функции отправки/подтверждения кодов, авторизации по паролю)

### Интернационализация

- `/i18n/en.lang.yml` - Английские переводы (язык ключей)
- `/i18n/ru.lang.yml` - Русские переводы

### API

- `/api/password.ts` - API для получения хеша пароля с использованием соли

### Компоненты

- `/components/PhoneAuthForm.vue` - Форма авторизации по телефону (SMS + пароль)
- `/components/EmailAuthForm.vue` - Форма авторизации по email (код + пароль)

## Функциональность

### Поддерживаемые методы авторизации

1. **SMS** - отправка кода на телефон
2. **Email** - отправка кода на почту
3. **Password** - авторизация по паролю (для телефона и email)

### Особенности

- Автоматическая проверка доступных провайдеров
- Переключение между методами авторизации (телефон/email)
- Валидация вводимых данных
- Обработка ошибок с понятными сообщениями
- Responsive дизайн
- Поддержка нескольких языков (i18n) с использованием `ctx.t()`

## Безопасность

- Нормализация телефонных номеров и email
- Валидация данных на клиенте и сервере
- Использование встроенных API эндпоинтов Chatium

## API эндпоинты

Использует встроенные системные эндпоинты:

- `/s/auth/sms/send` - отправка SMS
- `/s/auth/sms/confirm` - подтверждение SMS
- `/s/auth/email/send` - отправка email
- `/s/auth/email/confirm` - подтверждение email
- `/s/auth/password` - авторизация по паролю

## Настройка

Провайдеры авторизации обрабатываются автоматически через `getEnabledAuthProviders(ctx)`. Система адаптируется под доступные методы авторизации.

## Интернационализация

Все текстовые строки интерфейса локализованы через систему `ctx.t()`. Язык интерфейса определяется автоматически на основе:

1. Языка пользователя (`ctx.user.lang`)
2. Языка мобильного приложения
3. Языка аккаунта по умолчанию
4. HTTP-заголовка `accept-language`
5. Английского языка (по умолчанию)

Поддерживаемые языки:

- Английский (en) - язык ключей
- Русский (ru)
