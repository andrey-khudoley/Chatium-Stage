# Личный сайт парикмахера с календарем записи

## Версия: 1.0.0

## Описание проекта

Современный лендинг для профессионального парикмахера с интегрированной системой онлайн-записи. Сайт включает портфолио работ, описание услуг, информацию о мастере и удобную форму бронирования с выбором даты и времени.

## Архитектура

### Структура файлов

```
/hairdresserBooking/
```

## Основные функции

### 1. Главная страница (HomePage.vue)

**Секции:**

- **Hero**: Полноэкранный баннер с призывом к действию
- **О мастере**: Информация о парикмахере, опыт, статистика
- **Услуги**: Карточки с описанием услуг, ценами и длительностью
- **Портфолио**: Галерея работ мастера
- **Запись онлайн**: Форма бронирования с календарем
- **Контакты**: Адрес, телефон, время работы, социальные сети
- **Футер**: Копирайт

### 2. Форма записи (BookingForm.vue)

**Поля:**

- Имя клиента (обязательно)
- Телефон (обязательно)
- Email (опционально)
- Услуга (выбор из списка)
- Дата (выбор из календаря, не ранее текущей даты)
- Время (динамический выбор доступных слотов)
- Примечания (опционально)

**Функционал:**

- Проверка доступного времени для выбранной даты
- Визуальное отображение занятых/свободных слотов
- Валидация обязательных полей
- Отображение сообщений об успехе/ошибке
- Автоочистка формы после успешной отправки

### 3. API для записей (api/bookings.ts)

**Роуты:**

#### `apiCreateBookingRoute` (POST /api/bookings/create)

Создание новой записи клиента.

**Входные данные:**

```typescript
{
  clientName: string
  clientPhone: string
  clientEmail?: string
  service: string
  date: string
  time: string
  notes?: string
}
```

**Действия:**

1. Валидация обязательных полей
2. Создание записи в таблице bookings со статусом 'pending'
3. Отправка уведомления администратору через `sendNotificationToAccountOwners`
4. Запись события через `writeWorkspaceEvent`

#### `apiGetAvailableTimesRoute` (POST /api/bookings/available-times)

Получение доступных временных слотов для выбранной даты.

**Входные данные:**

```typescript
{
  date: string
}
```

**Логика:**

- Запрос всех записей на выбранную дату (кроме отмененных)
- Формирование списка всех возможных слотов (9:00 - 19:00)
- Фильтрация занятых слотов
- Возврат списка доступных слотов

### 4. Таблица bookings

**Поля:**

- `clientName` (StringKind) - Имя клиента
- `clientPhone` (StringKind) - Телефон клиента
- `clientEmail` (StringKind) - Email клиента
- `service` (StringKind) - Выбранная услуга
- `date` (DateKind) - Дата записи
- `time` (StringKind) - Время записи
- `status` (StringKind) - Статус записи (pending, confirmed, cancelled)
- `notes` (StringKind) - Дополнительные примечания

## Интеграции

### Уведомления

При создании записи отправляется уведомление владельцу аккаунта через `@user-notifier/sdk` с полной информацией о клиенте и записи.

### Аналитика

События записи отслеживаются через `@start/sdk` с передачей UTM-меток для аналитики трафика.

### Метрики

Интегрирована Microsoft Clarity для анализа поведения пользователей.

## Временные слоты

**Рабочее время:** 9:00 - 20:00 (последняя запись в 19:00)

**Доступные слоты:**
09:00, 10:00, 11:00, 12:00, 13:00, 14:00, 15:00, 16:00, 17:00, 18:00, 19:00

**Логика бронирования:**

- Один клиент может занять один слот
- Нельзя записаться на прошедшие даты
- Занятые слоты автоматически скрываются

## Расширение функционала

### Возможные улучшения:

1. **Личный кабинет клиента**: просмотр своих записей, история посещений
2. **SMS-уведомления**: напоминания клиентам о записи
3. **Админ-панель**: управление записями, изменение статусов
4. **Отзывы клиентов**: секция с отзывами и рейтингом
5. **Онлайн-оплата**: возможность предоплаты услуг
6. **Программа лояльности**: бонусы для постоянных клиентов
7. **Блог**: статьи по уходу за волосами

## События для аналитики

**Регистрируемое событие:**

- `booking_created` - создание новой записи

**Параметры события:**

- Информация о клиенте (имя, телефон, email)
- Параметры записи (услуга, дата, время)
- UTM-метки для отслеживания источников трафика
