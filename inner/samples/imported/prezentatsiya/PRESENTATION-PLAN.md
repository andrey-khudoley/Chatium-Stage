# ЧАСТЬ 4. ПРОДУКТ 2: ЛЕНОЧКА (Процессные AI-агенты)

# ═══════════════════════════════════════

## Что такое процессные ИИ-агенты

> **Процессный ИИ-агент — это цифровой сотрудник, который работает 24/7.**
> Он не просто отвечает на вопросы (как ChatGPT). Он выполняет реальные бизнес-задачи: продаёт, консультирует, обрабатывает заявки, ведёт переписку, достигает целей.

### Почему «Леночка»

Это внутреннее название процессных агентов Chatium. Леночка — универсальный агент, который может быть кем угодно: продавцом, консультантом, ассистентом, менеджером по работе с клиентами.

---

## Как работают процессные агенты — техническая суть

### Архитектура агента

```
ИНСТРУКЦИИ (промпт)
    ↓
БАЗА ЗНАНИЙ (документы, FAQ, каталоги)
    ↓
ИНСТРУМЕНТЫ (tools) — доступ к БД, API, CRM, рассылкам
    ↓
ЦЕПОЧКИ ДИАЛОГОВ (chains) — контекст каждого клиента
    ↓
КАНАЛЫ КОММУНИКАЦИИ — Telegram, WhatsApp, VK, email, Авито, сайт
```

### Что делает агента «процессным» (а не просто чат-ботом)

| Чат-бот (ChatGPT и аналоги) | Процессный агент (Леночка)                              |
| --------------------------- | ------------------------------------------------------- |
| Отвечает на вопросы         | **Достигает целей**                                     |
| Не знает ваших клиентов     | **Видит историю клиента, его заказы, оплаты**           |
| Не может ничего сделать     | **Создаёт заказы, отправляет сообщения, обновляет CRM** |
| Забывает контекст           | **Помнит всю историю переписки с клиентом**             |
| Одинаковый для всех         | **Персонализированный под каждого клиента**             |
| Работает в одном канале     | **Работает в 8+ каналах одновременно**                  |
| Требует программиста        | **Создаётся Ботаном по вашему описанию**                |

---

## Что умеет Леночка (функционал)

### 🎯 Достижение целей

Леночка не просто реагирует — она проактивно работает на результат:

- Понимает цель (продать, записать, проконсультировать)
- Строит стратегию общения с каждым клиентом
- Следит за прогрессом и корректирует подход

### 🧰 Инструменты (Tools)

Агент имеет доступ к реальным бизнес-функциям:

- **Поиск по каталогу** — находит товары/услуги в вашей базе
- **Создание заказов** — оформляет заказ прямо в диалоге
- **Проверка оплаты** — видит статус платежа
- **Обновление CRM** — записывает данные клиента
- **Отправка сообщений** — пишет клиенту в Telegram, Email, WhatsApp
- **Запись на мероприятие** — бронирует место
- **Персональные рассылки** — отправляет индивидуальные сообщения

### 📚 База знаний

- Загружаете документы, FAQ, каталоги
- Агент изучает их и использует в ответах
- Обновляете базу — агент сразу знает новое

### 📝 Инструкции

- Пишете инструкцию естественным языком: «Ты — менеджер по продажам. Твоя задача — записать клиента на консультацию...»
- Агент следует инструкциям как живой сотрудник

### 🔗 Цепочки диалогов (Chains)

- Каждый клиент — отдельная цепочка
- Агент помнит всю историю общения
- Видит профиль клиента (имя, телефон, email, UTM-метки, теги)
- Может дополнять профиль новыми данными

## Как просто создать агента

> **Буквально сказать Ботану:**
> «Сделай мне агента-продавца для моего магазина электроники. Он должен помогать клиентам выбирать товары, оформлять заказы и отвечать на вопросы о доставке.»

1. Создаёт агента через `getOrCreateAgentForWorkspace`
2. Настраивает инструкции
3. Подключает инструменты (поиск товаров, создание заказов)
4. Подключает к Telegram-боту
5. Готово — агент работает

## 🏆 Метафора (КЛЮЧЕВАЯ для понимания)

> **Chatium — это как операционная система. Windows или macOS.**
> Внутри неё живут программы (Ботан, Леночка), которые работают ВМЕСТЕ. Они видят файлы друг друга, пользуются одними устройствами (модулями), общаются между собой.
>
> А Cursor, Claude Code, Lovable, n8n — это отдельные программы, скачанные с разных сайтов. Они не знают друг о друге. Они не имеют общей файловой системы. Чтобы они заработали вместе — вам нужно САМОМУ написать тысячи строк кода для интеграции.
>
> Представьте: вы купили принтер, монитор, клавиатуру и мышь от разных производителей, но у вас НЕТ компьютера. Каждое устройство работает само по себе, но вместе — никак. **Chatium — это компьютер. А Cursor, n8n, Lovable — это периферия без компьютера.**

### 💡 Ещё одна метафора (проще для аудитории)

> **Chatium — это город.**
> В нём есть дороги (инфраструктура), здания (сервисы), жители (агенты), полиция (авторизация), банк (платежи), почта (рассылки), больница (мониторинг).
>
> Cursor, Lovable, n8n — это отдельные дома, построенные посреди поля. Без дорог. Без электричества. Без водопровода. Можете ли вы жить в таком доме? Можно. Удобно ли? Нет. Масштабируемо ли? Точно нет.

## Детальный разбор

### Chatium vs Lovable

**Lovable** — это хороший генератор React-приложений для международного рынка. Но:

- ❌ Только веб (нет мобильных)
- ❌ Нет российских платежей
- ❌ Нет CRM, рассылок, автоматизаций
- ❌ Нет процессных AI-агентов
- ❌ Хостинг в США/EU (нарушение 152-ФЗ)
- ❌ Оплата только в $
- Для полноценного бизнеса нужно ещё 5–9 сервисов → $150–300+/мес в валюте

### Chatium vs Cursor / Claude Code

**Cursor и Claude Code** — это текстовые редакторы с ИИ. Они помогают ПИСАТЬ код. Но:

- ❌ У них нет инфраструктуры. Вы пишете код — а где он будет работать?
- ❌ Нужен свой сервер, своя БД, свой деплой, свои модули
- ❌ Код написанный в Cursor будет работать КРИВО, если вы попробуете перенести его в другую среду
- ❌ Они не знают вашу бизнес-логику, ваших клиентов, ваши данные

**Ключевое:** Код, написанный в Cursor вне инфраструктуры Chatium, — это как чертёж здания без земельного участка, без стройматериалов и без строителей. Красивый чертёж, но дома нет.

### Chatium vs n8n

**n8n** — отличный инструмент для автоматизаций. Но:

- ❌ Это ТОЛЬКО автоматизации. Нет сайтов, нет мобильных, нет CRM, нет агентов
- ❌ Нужен свой сервер для self-hosted
- ❌ Нет контекста бизнеса — просто перекладывает данные из одного API в другой
- ❌ Нет ИИ-агентов, которые общаются с клиентами

## Почему код из Cursor/Claude Code будет работать криво

> «Представьте, что вы наняли строителя, который строил дома в другом городе. Он приехал к вам и начал строить по своим правилам. Но здесь другой грунт, другие нормы, другие материалы. Его дом не пройдёт проверку и будет протекать.»

**Технически:** Код, написанный вне Chatium, не знает:

- Как работает `app.get()`, `app.post()` (файловый роутинг)
- Как устроены Heap Tables (ORM)
- Как использовать `@sender`, `@pay`, `@crm`, `@automations`
- Как работает `ctx` (глобальный контекст)
- Как устроена авторизация (`@app/auth`)
- Как работают WebSocket (`@app/socket`), Jobs, Storage

Это как разговаривать на другом языке. Ботан ГОВОРИТ на языке Chatium. Cursor — нет.
