# AI-Виджет чата для сайта

**Версия:** 1.0  
**Назначение:** Встраиваемый виджет чата с AI-агентом для любого сайта  
**Тип проекта:** Sender Widget + AI Agent Integration

---

## Описание

Это приложение представляет собой виджет чата с AI-агентом, который можно встроить на любой сайт через JavaScript-код. Виджет состоит из двух частей:

1. **Кнопка** (80x80px) - плавающая кнопка для открытия чата
2. **Попап чата** (370x620px) - полноценный интерфейс чата с AI-агентом

Виджет работает через iframe'ы, встраиваемые на сайт, и использует систему транспортов Chatium Sender для обработки сообщений и интеграции с AI-агентами.

### 🔵 `api/chats.ts` - API СООБЩЕНИЙ

**Роуты:**

#### 1. `apiChatSendMessageRoute` - POST `/send`

**Query:** `uid` (string) - уникальный идентификатор пользователя  
**Body:** `{ text: string, files?: Array<{hash, name, mime, size}> }`  
**Назначение:** Отправка сообщения от пользователя агенту

**Логика:**

1. Получает или создает пользователя через `requireAnyUser()`
2. Получает транспорт воркспейса
3. Вызывает `privateMessageRecieved()` из `@sender/sdk`
4. Отправляет сообщение в систему Sender для обработки агентом

#### 2. `apiChatMessagesRoute` - GET `/messages`

**Query:** `uid` (string)  
**Назначение:** Получение истории сообщений чата

**Логика:**

1. Получает или создает транспорт
2. Находит персону (person) по `externalId` = `uid`
3. Получает ID чата персоны
4. Загружает последние 100 сообщений через `findMessagesByChatId()`
5. Форматирует сообщения в формат `{id, role, content, createdAt, files}`

**Важно:**

- `role: 'user'` если сообщение от пользователя (createdBy === person.user)
- `role: 'assistant'` если сообщение от агента

### 🔵 `api/setup.ts` - API НАСТРОЙКИ

**Роут:** `apiCreateTransportRoute` - POST `/create-transport`

**Назначение:** Создание транспорта и привязка AI-агента

**Логика:**

1. Проверяет роль Admin
2. Получает идентификатор транспорта воркспейса
3. Создает или получает транспорт через `getOrCreateTransport()`
4. Создает AI-агента через `getOrCreateAgentForWorkspace()` с:
   - Инструкциями по умолчанию
   - Подключенным инструментом `sendMessageToChatTool`
   - Привязкой к созданному каналу

---

### 🟠 `transport/hook.ts` - ИНТЕГРАЦИЯ С SENDER

**Основные функции:**

#### 1. `getWorkspaceTransportIdentity(ctx)`

Возвращает идентификатор транспорта для текущего воркспейса:

```typescript
{
  id: workspace.id,
  key: "site-chat-with-gpt-transport-" + workspace.id,
  title: workspace.name,
  description: workspace.description
}
```

#### 2. `findWorkspaceTransport(ctx)`

Находит созданный транспорт в системе Sender по `externalId` и `externalKey`

#### 3. Account Hook `@sender/get-external-transports`

Регистрирует виджет как внешний транспорт в системе Sender:

- Заголовок: "Chatium Chat Widget"
- Иконка транспорта
- Действие `sendMessage` для отправки сообщений пользователю
- Компоненты UI для создания и настройки транспорта

#### 4. `sendMessageFunction`

Функция для отправки сообщения от агента пользователю:

- Получает `userId` из `body.chat.id`
- Отправляет WebSocket событие через `sendDataToSocket()`
- Триггерит обновление интерфейса чата

### 🟡 `setup.tsx` и `pages/SetupPage.vue` - ПЕРВОНАЧАЛЬНАЯ НАСТРОЙКА

**Назначение:** Страница, показываемая при первом запуске для создания транспорта

**SetupPage.vue содержит:**

- Кнопку "Создать транспорт"
- Ссылку на управление транспортом после создания
- Кнопку "Начать использовать" (активна после создания транспорта)

**Логика:**

1. При клике на "Создать транспорт" вызывается `apiCreateTransportRoute`
2. После создания показывается ссылка на `/app/sender/v2?tab=apps#/settings/channel/{id}`
3. При клике "Начать использовать" переходит на `indexPageRoute` (главная страница)

## Основные сценарии использования

### Сценарий 1: Изменение внешнего вида кнопки

**Файл:** `components/ButtonComponent.vue`

**Что можно изменить:**

- Размер кнопки (переменные `height`, `width`)
- Цвета градиента (`background: linear-gradient(...)`)
- Иконку/эмодзи (`.button__emoji`)
- Анимацию (`@keyframes button-emoji-wave`)
- Тень и эффекты (`box-shadow`, `transform`)

### Сценарий 2: Изменение стиля чата

**Файл:** `components/PopupComponent.vue`

**Что можно изменить:**

- CSS переменные цветов:
  - `--chat-background` - фон чата
  - `--chat-surface` - цвет поверхностей
  - `--chat-accent` - акцентный цвет
  - `--chat-border` - цвет границ
- Заголовок: `.chat-title`, `.chat-subtitle`
- Стили сообщений: `.chat-message--agent`, `.chat-message--user`
- Стили баблов: `.chat-message__bubble`

### Сценарий 3: Изменение логики отправки сообщений

**Файлы:**

- `components/PopupComponent.vue` (функция `sendMessage`)
- `api/chats.ts` (роут `apiChatSendMessageRoute`)

**Последовательность:**

1. Пользователь вводит текст/прикрепляет файлы
2. `MessageInput.vue` эмиттит событие `submit`
3. `PopupComponent.vue` обрабатывает в `sendMessage()`
4. Добавляет pending-сообщение в список
5. Вызывает `apiChatSendMessageRoute.run(ctx, {...})`
6. API отправляет в Sender через `privateMessageRecieved()`
7. После успеха - загружает обновленный список сообщений
8. Показывает индикатор "печатает..."
9. WebSocket получает событие о новом сообщении агента
10. Автоматически обновляет список

### Сценарий 4: Добавление нового функционала в чат

**Возможности:**

- Добавить кнопки быстрых ответов
- Добавить поддержку аудио/видео
- Добавить формы в чат
- Добавить оценку сообщений
- Добавить экспорт истории

**Где редактировать:** `components/PopupComponent.vue`, добавить новые элементы в template

## Советы по работе

### При изменении UI:

- Всегда помнить про ограничения размера iframe
- Тестировать в разных разрешениях (desktop, mobile)
- Проверять прозрачность фонов
- Учитывать контрастность текста

### При изменении логики:

- Sender обрабатывает сообщения асинхронно
- WebSocket нужен для realtime-обновлений
- UID пользователя - ключевой параметр для идентификации
- Транспорт должен быть создан перед использованием

### При работе с файлами:

- Сначала получить uploadUrl через API
- Затем загрузить файл напрямую в storage
- Получить hash файла
- Передать hash в сообщении
- Использовать `getThumbnailUrl()` для отображения
