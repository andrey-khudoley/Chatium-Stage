# Аналитика вовлеченности

**Версия:** 2.1.3  
**Дата обновления:** 2025-10-22

## Описание проекта

Универсальный шаблон для аналитики вовлеченности пользователей на лендингах. Отслеживает среднее время пребывания на страницах и глубину скроллинга с использованием автоматического сбора данных через `behaviour2_log`.

## Основные возможности

- 📊 **Статистика времени** - среднее, минимальное и максимальное время пребывания на лендингах
- 🔥 **Тепловая карта скроллинга** - визуализация глубины просмотра страниц пользователями
- 📅 **Гибкий выбор периода** - предустановленные периоды или произвольный диапазон дат
- 🚀 **Автоматический сбор данных** - не требуется установка дополнительных скриптов
- ⚙️ **Управление лендингами через UI** - добавление и активация/деактивация лендингов

## Архитектура

### Источники данных

#### Behavior Log (ClickHouse через `@traffic/sdk`)

- База данных: `chatium_ai.behaviour2_log`
- Поля:
  - `uid` - уникальный идентификатор сессии
  - `urlPath` - полный URL посещенной страницы
  - `view_total_duration` - общее время просмотра (мс)
  - `view_focused_duration` - время в фокусе (мс)
  - `scroll_distance` - максимальная глубина скролла
  - `dt` - дата события

**Преимущества:**

- Автоматический сбор данных системой
- Не требуется установка скриптов отслеживания
- Готовые метрики времени и скролла
- Оптимизировано для аналитики

### Конфигурация лендингов

Лендинги хранятся в таблице `tables/landings.table` с полями:

- `path` (StringKind) - путь к лендингу (например, `/pu-v1-1`)
- `title` (StringKind) - название лендинга (например, `ПУ v1-1`)
- `isActive` (BooleanKind) - активен ли лендинг для отслеживания

**Управление лендингами:**

1. Нажмите кнопку "Управление лендами" в правом верхнем углу
2. В модальном окне:
   - **Добавьте новый лендинг**: укажите путь и название, нажмите "Создать"
   - **Активируйте/деактивируйте**: используйте переключатели рядом с лендингами
3. Изменения применяются мгновенно - аналитика автоматически обновится

### API Routes

#### `/engagement-stats` (GET)

**Параметры:**

- `period`: 'today', 'yesterday', '7', '14', '30', '60', '90'
- `startDate`, `endDate`: для произвольного периода (формат: YYYY-MM-DD)

**Возвращает:** Статистику вовлеченности для каждого **активного** лендинга из таблицы

- Количество уникальных сессий (uid)
- Среднее время пребывания на лендинге (в секундах)
- Минимальное время сессии (>= 3 секунд)
- Максимальное время сессии
- Отформатированные значения времени

**Логика расчета:**

- Загружает активные лендинги из таблицы `landings` (где `isActive = true`)
- Источник данных: `chatium_ai.behaviour2_log`
- Поле: `view_total_duration` (миллисекунды)
- Для каждого уникального `uid` вычисляется среднее значение
- Преобразование: миллисекунды → секунды
- Форматирование: часы/минуты/секунды
- **Минимальное время: >= 3 секунд** (фильтр `view_total_duration >= 3000`)
- **Максимальное время: <= 2 часа** (фильтр `view_total_duration <= 7200000`)
- **Исключаются короткие визиты < 3 сек и аномально долгие > 2 часов** при расчете MIN/AVG/MAX

#### `/scroll-heatmap` (GET)

**Параметры:**

- `path`: путь к лендингу
- `period`: 'today', 'yesterday', '7', '14', '30', '60', '90' (опционально)
- `startDate`, `endDate`: для произвольного периода (формат: YYYY-MM-DD, опционально)

**Возвращает:** Тепловую карту скроллинга для конкретного лендинга

- Данные из `chatium_ai.behaviour2_log`
- Использует поле: `scroll_distance` (максимальная глубина скролла)
- **Использует тот же период, что выбран в основной аналитике**
- **Фильтр: scroll_distance > 0** (учитываются только пользователи с реальным скроллингом)
- Расчет процентилей: 25%, 50%, 75%
- **Всегда возвращает 4 диапазона:**
  - **0-25%**: пользователи до 25-го процентиля (голубой)
  - **25-50%**: между 25-м и 50-м процентилями (желтый)
  - **50-75%**: между 50-м и 75-м процентилями (оранжевый)
  - **75-100%**: выше 75-го процентиля (красный)
- Количество и процент пользователей для каждого диапазона

### API управления лендингами

#### `/landings` (GET)

Возвращает список всех лендингов из таблицы (активных и неактивных)

#### `/create-landing` (POST)

**Параметры:**

- `path` (string) - путь к лендингу, должен начинаться с `/`
- `title` (string) - название лендинга

**Возвращает:** Созданный лендинг (по умолчанию `isActive = true`)

**Валидация:**

- Путь должен начинаться с `/`
- Проверяется уникальность пути

#### `/toggle-landing` (POST)

**Параметры:**

- `id` (string) - ID лендинга
- `isActive` (boolean) - новое состояние активности

**Возвращает:** Обновленный лендинг

## Ключевые особенности

### 1. Автоматический сбор данных

- Данные собираются системой без установки скриптов
- Используется встроенная таблица `behaviour2_log`
- Полная информация о поведении пользователей

### 2. Точность метрик

- Минимальный порог времени: 3 секунды
- Максимальный порог времени: 2 часа (7200 секунд)
- Исключаются случайные клики, моментальные закрытия и аномалии (например, забытые вкладки)
- Реальная вовлеченность пользователей

### 3. Визуализация

- Цветовая градация для тепловой карты
- Интуитивно понятные графики
- Автоматические рекомендации на основе данных

### 4. Оптимизация

- Параллельная загрузка данных для всех лендингов
- Кэширование данных во фронтенде
- Эффективные SQL-запросы к ClickHouse

### 5. Часовой пояс

- Все даты в московском часовом поясе
- Для "вчера": endDate = сегодня 00:00:00

### 6. Единый период для всех метрик

- Тепловая карта использует тот же период, что выбран в основной аналитике
- При изменении периода автоматически обновляются все данные
- Поддержка произвольных диапазонов дат
- По умолчанию: 90 дней (если период не указан)

## Как использовать

### 1. Добавление лендингов

1. Откройте приложение и нажмите "Управление лендами" в правом верхнем углу
2. В форме добавления:
   - **Путь**: введите путь к лендингу (например, `/landing-1`)
   - **Название**: введите понятное название (например, `Лендинг 1`)
3. Нажмите "Создать"
4. Лендинг автоматически активируется и начнет отслеживаться

### 2. Управление активностью

- В модальном окне переключайте тумблеры рядом с лендингами
- Зеленый = активен (отслеживается в аналитике)
- Серый = неактивен (не отслеживается)
- Изменения применяются мгновенно

### 3. Запуск приложения

Перейдите по адресу вашего приложения - аналитика начнет работать автоматически.

### 4. Просмотр данных

- Выберите период (быстрый или произвольный)
- Нажмите "Обновить" для загрузки свежих данных
- Изучите summary cards с общей статистикой
- Просмотрите таблицу с детализацией по каждому лендингу
- Откройте тепловую карту для анализа глубины скроллинга

## Технические детали

### Расчет времени вовлеченности

```sql
-- Запрос для получения общего количества сессий
SELECT COUNT(DISTINCT uid) as sessions_count
FROM chatium_ai.behaviour2_log
WHERE
  startsWith(urlPath, 'https')
  AND (urlPath LIKE '%/landing%' OR urlPath LIKE '%/landing/%')
  AND dt >= subtractDays(today(), 30)

-- Запрос для MIN/AVG/MAX (только визиты >= 3 секунд и <= 2 часов)
SELECT
  AVG(view_total_duration) as avg_duration_ms,
  MIN(view_total_duration) as min_duration_ms,
  MAX(view_total_duration) as max_duration_ms
FROM chatium_ai.behaviour2_log
WHERE
  startsWith(urlPath, 'https')
  AND (urlPath LIKE '%/landing%' OR urlPath LIKE '%/landing/%')
  AND dt >= subtractDays(today(), 30)
  AND view_total_duration >= 3000
  AND view_total_duration <= 7200000
```

### Расчет тепловой карты

```sql
WITH percentiles AS (
  SELECT
    COUNT(DISTINCT uid) as total_users,
    quantile(0.25)(scroll_distance) as p25,
    quantile(0.50)(scroll_distance) as p50,
    quantile(0.75)(scroll_distance) as p75
  FROM chatium_ai.behaviour2_log
  WHERE urlPath LIKE '%/landing%'
    AND dt >= subtractMonths(today(), 1)
)

```
