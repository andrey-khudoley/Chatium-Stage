## Описание проекта

Веб-приложение для мастера маникюра с возможностью онлайн-записи на услуги. Проект состоит из двух основных частей:

1. **Главная страница** - лендинг с информацией о мастере, услугах, портфолио, отзывами и формой записи
2. **Чат с AI-агентом** - интеллектуальный помощник для консультаций и записи

## Таблицы данных

### Bookings (Записи на услуги)

Хранит информацию о записях клиентов на маникюр.

**Поля:**

- `clientName` (string) - Имя клиента
- `clientPhone` (string) - Телефон клиента
- `clientEmail` (string) - Email клиента
- `service` (string) - Название услуги
- `date` (Date) - Дата записи
- `time` (string) - Время записи
- `status` (string) - Статус записи (pending, confirmed, completed, cancelled)
- `notes` (string) - Дополнительные заметки

### Chats (Чаты)

Хранит историю чатов с AI-агентом.

**Поля:**

- `userId` (string) - ID пользователя
- `title` (string) - Название чата
- `messages` (array) - Массив сообщений
- `createdAt` (Date) - Дата создания

## API Endpoints

### Bookings API

- `GET /api/bookings/list` - Получить список записей
- `POST /api/bookings/create` - Создать новую запись
- `POST /api/bookings/update/:id` - Обновить запись (статус, дата, время)
- `POST /api/bookings/delete/:id` - Удалить запись

### Chats API

- `GET /api/chats/list` - Получить список чатов
- `POST /api/chats/create` - Создать новый чат
- `GET /api/chats/:id/messages` - Получить сообщения чата
- `POST /api/chats/:id/send` - Отправить сообщение

### Config API

- `GET /api/config/services` - Получить список услуг из конфигурации
- `GET /api/config/schedule` - Получить расписание работы
- `GET /api/config/timeslots/:date?` - Получить доступные слоты времени для указанной даты

### Setup API

- `POST /api/setup/create-transport` - Создать транспорт для работы с AI-агентом
- `GET /api/setup/check-transport` - Проверить наличие транспорта

## Конфигурация

### services.ts

TypeScript модуль с централизованной информацией об услугах:

- `id` - уникальный идентификатор услуги
- `name` - название услуги
- `description` - описание услуги
- `price` - цена в рублях
- `duration` - длительность в минутах
- `currency` - валюта (RUB)

### schedule.ts

TypeScript модуль с информацией о рабочем графике:

- `workingHours` - расписание по дням недели
  - Для каждого дня: `isOpen`, `start`, `end`, `breaks`
- `timeSlotDuration` - длительность одного слота в минутах (30)
- `bookingAdvanceDays` - за сколько дней можно записаться заранее (14)

## AI-агент

### Инструкции агента

Агент настроен как AI-помощник мастера маникюра и помогает клиентам:

- Узнать об услугах, ценах и длительности процедур
- Узнать о рабочем времени и днях работы
- Записаться на маникюр
- Получить консультацию по выбору дизайна

### Доступные инструменты

Агент использует следующие инструменты:

1. **sendMessageToChatTool** - отправка сообщений в чат
2. **getServicesTool** - получение списка услуг с описанием, ценами и длительностью
3. **getScheduleTool** - получение рабочего расписания и графика работы
4. **bookServiceTool** - запись клиента на услугу

### get-services

Инструмент для получения списка доступных услуг. Агент использует его когда пользователь спрашивает:

- О доступных услугах
- О ценах на услуги
- О длительности процедур

Возвращает форматированный список всех услуг с подробной информацией.

### get-schedule

Инструмент для получения рабочего расписания. Агент использует его когда пользователь спрашивает:

- О рабочих часах
- О днях работы
- Когда можно записаться

Возвращает полное расписание работы с указанием времени работы, выходных и перерывов.

### book-service

Инструмент для записи клиента на услугу. Агент использует его когда пользователь:

- Хочет записаться на маникюр
- Готов предоставить все необходимые данные

**Обязательные параметры:**

- `clientName` - имя клиента
- `clientPhone` - телефон клиента в формате +7XXXXXXXXXX
- `serviceId` - ID услуги из списка услуг (получается через get-services)
- `date` - дата записи в формате YYYY-MM-DD
- `time` - время записи в формате HH:MM

**Опциональные параметры:**

- `clientEmail` - email клиента
- `notes` - дополнительные пожелания или комментарии

**Процесс работы:**

1. Агент собирает всю необходимую информацию у клиента
2. Валидирует формат данных (дата, время, телефон)
3. Проверяет существование услуги по ID
4. Создает запись в таблице Bookings
5. Отправляет уведомление администратору
6. Подтверждает клиенту успешную запись

## Основной функционал

### Главная страница

- Презентация мастера и его работ
- Список предоставляемых услуг с ценами
- Галерея работ (портфолио)
- Отзывы клиентов
- Форма онлайн-записи
- Контактная информация
- Ссылка на админку для администраторов (в навигации)

### Чат с агентом

- Интеллектуальный AI-помощник
- Консультации по услугам
- Помощь в выборе подходящего дизайна
- Запись на услуги через чат
- История переписки
- Ссылка на админку и настройки для администраторов (в шапке)

### Форма записи

- Выбор услуги
- Выбор даты и времени
- Ввод контактных данных
- Дополнительные пожелания
- Уведомления о записи

- Просмотр всех записей с фильтрацией по статусу
- Подтверждение новых записей
- Перенос записей на другую дату/время
- Отмена записей
