# Chatium Documentation - Editor

## Version

4.12.0

Chatium Documentation - редактор документации с Markdown/MDX форматом, аналог Lovable/GitBook с иерархической структурой. Современный минималистичный дизайн с верхним хедером, боковой навигацией и чистым контентом. Поддерживает динамическую навигацию без перезагрузки страницы (SPA). **Единая таблица документов с типом (page или section). Root-страница отображается как логотип, ее дочерние разделы - как верхнее меню.** **Server-Side Rendering (SSR) с загрузкой данных на сервере для быстрой первой загрузки.** **Контроль публикации страниц - неопубликованные страницы видны только администраторам.** **Система настроек с выбором типа URL (slug-based, slug-only или id-based) с применением в навигации.** **Быстрое предзаполнение базы реальным контентом из текущей документации.** **Полная интернационализация через систему переводов ctx.t() с русским языком по умолчанию.** **Улучшенный welcome screen для пустой базы данных - логотип скрыт, контент по центру.**

## Architecture

### Components

- **components/Header.vue** - верхний хедер с логотипом root-документа, табами его дочерних секций, поиском и админ-контролами

  - **Отступ сверху `pt-6`** - визуальное разделение от верхней границы экрана
  - **Логотип** - обычная ссылка на главную страницу
    - **Клик на логотип** - переход на basePath с полной перезагрузкой страницы (не AJAX)
    - **cursor-pointer** - визуальный индикатор кликабельности
  - **Fallback** - если нет root-документа, показывается "Нет родительской страницы" с иконкой предупреждения
  - Отображает дочерние разделы root-документа как горизонтальное меню табов
  - Эмитит событие 'section-change' при выборе секции
  - Содержит поле поиска с выпадающим меню результатов
  - Эмитит событие 'navigate' при выборе результата поиска
  - Подсвечивает активную секцию (жирный черный текст + синяя нижняя граница)
  - **Админ-контролы (справа):**
    - **Кнопка "+ Add Section"** - видна только в режиме редактирования, эмитит `add-section`
    - **Кнопка "+ Add Page"** - видна только в режиме редактирования, эмитит `add-page`
    - **Переключатель Edit/View** - меняет `isViewMode`, эмитит `toggle-view-mode`

- **components/TreeNode.vue** - рекурсивный компонент для отображения узлов дерева
  - **Стрелки раскрытия/скрытия (справа от названия):**
    - Показываются только для **страниц** (type === 'page') с дочерними элементами
    - **Секции не имеют стрелок** - секции всегда раскрыты по умолчанию
    - Стрелка вправо (fa-chevron-right) - узел свернут
    - Стрелка вниз (fa-chevron-down) - узел раскрыт
    - Кликабельны - вызывают функцию `toggleExpand()`
    - Расположены после текста, перед админ-кнопками
  - **Локальный state:**
    - `isManuallyExpanded` - ref(true) - состояние раскрытия узла, по умолчанию раскрыт
    - Сохраняется независимо для каждого экземпляра компонента
  - **Условия отображения:**
    - **Страницы (type === 'page'):** `v-if="node.type === 'page' && (node.isPublished || !isViewMode || ctx.user?.is('Admin'))"`
      - Показываются если опубликованы ИЛИ режим редактирования ИЛИ пользователь админ
    - **Секции (type === 'section'):** `v-else-if="node.type === 'section' && (node.isPublished || !isViewMode || ctx.user?.is('Admin'))"`
      - Показываются если опубликованы ИЛИ режим редактирования ИЛИ пользователь админ
    - **В режиме View** неопубликованные документы полностью скрыты для обычных пользователей
  - **shouldShowChildren:**
    - Проверяет `isManuallyExpanded` - если false, дети не показываются
    - Для секций - показывает детей если раскрыто
    - Для страниц - показывает детей только если узел в currentPath И раскрыт

## Navigation Flow

```
// Навигация на главную:
Header logo (click)
  → <a href="/doc"> - нативная браузерная навигация
  → полная перезагрузка страницы
  → SSR рендеринг главной страницы

// Навигация по документам:
TreeNode (click)
  → emit('navigate', documentId)
  → Sidebar
  → emit('navigate', documentId)
  → MainPage
  → update currentDocumentId
  → load document path via apiDocumentByIdRoute
  → update URL via history.pushState
  → render EditorView with new documentId

// Навигация по секциям:
Header tab (click)
  → emit('section-change', sectionId)
  → MainPage
  → update activeSectionId
  → reset currentDocumentId
  → Sidebar filters tree based on active section

// Поиск документа:
Header search input (type >= 2 chars)
  → apiDocumentSearchRoute.run(ctx, {q: query})
  → Documents.searchBy() with full-text and embeddings search
  → Display results dropdown
  → click on result
  → selectResult(result)
  → emit('navigate', documentId)
  → MainPage (handleNavigate)
  → update currentDocumentId
  → update URL and render EditorView
```
