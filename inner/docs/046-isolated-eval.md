# Модуль @app/isolated-eval: Безопасное выполнение кода

Краткий справочник по модулю `@app/isolated-eval` — выполнение строки кода в изолированном окружении с передачей аргументов. Типизация: `node_modules/@app/isolated-eval/index.d.ts`.

## Назначение

**@app/isolated-eval** — выполнение произвольного кода (строка + массив аргументов) в изолированной среде. Используется платформой для сценариев, где нужно интерпретировать пользовательский или конфигурируемый код без доступа к полному контексту приложения. Не подходит для выполнения непроверенного пользовательского ввода без ограничений — изоляция снижает риски, но не заменяет валидацию и санитизацию.

---

## Экспорты

- **isolatedEval(ctx, code, args)** — выполнить код в изолированном окружении. Параметры: RichUgcCtx, строка кода, массив аргументов. Возвращает Promise с результатом выполнения.

Точная семантика изоляции и доступные глобалы см. в документации платформы и исходной реализации.
