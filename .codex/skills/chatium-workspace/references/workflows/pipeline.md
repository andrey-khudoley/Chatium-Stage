# /pipeline — автоматический выбор уровня конвейера

Твоя единственная задача в этой команде — **квалифицировать задачу, определить уровень 1–10 и запустить конвейер этого уровня** по плейбуку `.claude/agents/pp-orchestrator.md`. Сам код не пиши и фазы не выполняй — это сделает плейбук на выбранном уровне.

## Шаги

1. **Возьми задачу.** Текст после `/pipeline` ($ARGUMENTS). Если пусто — последняя формулировка из контекста чата.

2. **Оцени сложность по сигналам** (быстро, без агентов и без чтения всего проекта — максимум пара `Grep`/`Glob`, если масштаб неочевиден):
   - сколько файлов/слоёв затронет (1 / 1–3 / api+lib+ui / много);
   - есть ли платформенные тонкости Chatium (gateway, Heap-инварианты, роутинг, auth, деньги, race);
   - риск/необратимость (платежи, миграция данных, публичные эндпоинты, удаление кода);
   - чёткость постановки (готова / нужна формализация с вопросами);
   - объём генерации кода (правка строки / функция / модуль / переработка).

3. **Сопоставь с уровнем** (ориентир — §6b CLAUDE.md и матрица плейбука):
   | Уровень | Признаки задачи |
   |---|---|
   | **pp1–2** | опечатка, однострочник, переименование, правка константы/строки |
   | **pp3–4** | локальная фича/багфикс в 1–3 файлах, постановка ясна |
   | **pp5–6** | фича со связями (api+lib+ui), нужен план и ревью, есть платформенные касания |
   | **pp7–8** | сложная фича, платформенные тонкости, нужны гейты соответствия и Opus на рассуждении |
   | **pp9–10** | крупная/рискованная задача, миграция, многофайловая переработка, нужна максимальная уверенность/автономия |

   При сомнении между двумя уровнями выбирай **меньший** (дешевле; повысить всегда можно повтором). Не завышай уровень «на всякий случай».

4. **Объяви решение одной строкой** и сразу переходи к выполнению: `Квалификация: <1–2 фразы о сложности> → уровень **ppN**.`

5. **Запусти конвейер уровня N:** прими роль оркестратора в этом же чате и веди задачу строго по плейбуку `.claude/agents/pp-orchestrator.md` с выбранным **N** — точно так же, как это делает команда `/ppN`. Плейбук `pp-orchestrator` через Agent tool НЕ запускай (вложенный спавн сломается) — оркеструешь сам.

## Правила

- **Не задавай вопрос «какой уровень?»** — это и есть твоя работа здесь. (Уточняющие вопросы по самой задаче допустимы только если уровень предполагает фазу формализации с `+Q` и формализатор их вернул.)
- Если пользователь уже знает уровень — он вызовет `/ppN` напрямую; `/pipeline` для случая «выбери за меня».
- Профиль уровня (модели, шаги, параллелизм, лимит правок, метрики, отчёт) — целиком из матрицы плейбука. Ничего не добавляй и не выкидывай вне профиля выбранного N.
