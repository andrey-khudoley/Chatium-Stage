# Отчеты по трафику сайта

Приложение для анализа трафика сайта с использованием данных Chatium Analytics.

## Структура проекта

### Основные файлы
- `index.tsx` - главная точка входа приложения с проверкой конфигурации
- `pages/TrafficReports.vue` - основная страница с отчетами
- `pages/ConfigSetup.vue` - страница настройки конфигурации системы
- `api/traffic.ts` - API-роуты для получения данных трафика
- `api/config.ts` - API-роуты для управления конфигурацией

### Таблицы данных
- `tables/config.table` - таблица настроек системы:
  - serverAddress (StringKind) - адрес сервера
  - apiKey (StringKind) - API ключ

### Конфигурация системы

#### Первичная настройка
- При первом запуске система показывает форму настройки конфигурации
- Форма запрашивает адрес сервера и API ключ для подключения к аналитике
- После сохранения настроек происходит автоматическая перезагрузка страницы

#### API роуты конфигурации
- `GET /api/config/get` - получение текущей конфигурации
- `POST /api/config/save` - сохранение настроек конфигурации
- `GET /api/config/check` - проверка состояния конфигурации

### Функциональность

#### Фильтры
- **Дата события**: выбор периода от/до для анализа
- **URL события**: фильтрация по конкретному URL или его части

#### Отчеты (табы)
1. **Пользователи** - статистика регистрации пользователей с метриками по Telegram и группам
2. **Заказы** - детальная аналитика заказов:
   - Сводные метрики (количество заказов, сумма заказов, конверсия, количество оплат, сумма оплат, средний чек)
   - Таблица деталей заказов (ID заказа, сумма заказа, сумма оплаты)
   - Иерархическая группировка по годам/месяцам/дням
   - График суммы оплат по дням
3. **Последние посещения** - список последних 50 посещений с деталями
4. **Посетители по дням** - статистика посещений, уникальных пользователей и сессий по дням с интерактивным Chart.js графиком
5. **Самые посещаемые страницы** - топ-20 страниц по количеству посещений
6. **Страницы входа** - анализ первых страниц сессии пользователей

#### Визуализация данных
- **Chart.js графики**:
  - Линейный график посетителей по дням с переключаемыми метриками
  - График регистрации пользователей
  - График суммы оплат по дням
- Интерактивные элементы с hover-эффектами
- Цветовая легенда с возможностью включения/выключения линий
- Адаптивный дизайн

### API-роуты
- `GET /api/traffic/recent-visits` - последние посещения
- `GET /api/traffic/daily-stats` - статистика по дням
- `GET /api/traffic/popular-pages` - популярные страницы
- `POST /api/traffic/entry-pages` - страницы входа
- `GET /api/traffic/orders` - заказы с группировкой по дням
- `GET /api/traffic/orders-summary` - суммарные данные по заказам
- `GET /api/traffic/order-details` - детальный список заказов с ID
- `GET /api/traffic/order-status/:orderId` - статус конкретного заказа
- `GET /api/traffic/payments-by-date` - оплаты по дням
- `GET /api/traffic/users-registration` - регистрация пользователей
- `GET /api/traffic/total-users` - общее количество пользователей
- `GET /api/traffic/telegram-connected-users` - пользователи с подключенным Telegram
- `GET /api/traffic/telegram-disconnected-users` - пользователи с отключенным Telegram
- `GET /api/traffic/group-added-users` - пользователи в группе
- `GET /api/traffic/channel-subscribers` - подписчики канала

### Логика статусов заказов
Система использует приоритетную логику определения статусов:
1. Если есть событие `dealPaid` - статус автоматически "payed" (завершен)
2. Иначе берется последний статус из `dealStatusChanged`
3. Иначе берется начальный статус из `dealCreated`

### Используемые технологии
- **VueJS 3** с Composition API
- **Chart.js** для построения интерактивных графиков
- **Удаленные запросы к ClickHouse** через GetCourse MCP Server
- **ClickHouse** база данных `chatium_ai.access_log` и `chatium_ai.behaviour2_log`
- **Heap Tables** для хранения конфигурации системы

### Особенности
- Responsive дизайн для мобильных устройств
- Debounce для фильтров (500ms)
- Обработка ошибок с логированием
- Форматирование дат и времени для русской локали
- Динамическая конфигурация подключения через веб-интерфейс
- Интерактивная визуализация данных с Chart.js графиками
- Иерархическая группировка заказов с раскрывающимися секциями
- Детальная таблица заказов с ID, статусами и суммами
- Цветовые индикаторы статусов заказов
- Фильтрация по множественным статусам заказов
- Метрики конверсии и среднего чека
- Ленивая загрузка таблицы деталей заказов для оптимизации производительности
  - Таблица свёрнута по умолчанию
  - Данные загружаются только при разворачивании
  - Автоматически обновляется при изменении фильтров
