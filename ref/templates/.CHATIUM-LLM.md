# Проект: Примеры и шаблоны кода Chatium

## Архитектура проекта

Проект содержит исчерпывающую коллекцию примеров кода для разработки на платформе Chatium, соответствующих эталонным стандартам качества.

### Основные компоненты

- **`index.tsx`** - Входная точка приложения
- **`examples/`** - Директория с примерами кода, организованными по категориям

### Темная тема и контрастность ⚡ ВАЖНО

**Критическое правило:** Всегда указывайте стили для **обеих тем одновременно**!

```vue
<!-- ❌ ПЛОХО: текст невидим в темной теме -->
<div class="bg-white text-gray-900">Текст</div>

<!-- ✅ ХОРОШО: читаемо в обеих темах -->
<div class="bg-white dark:bg-gray-800 text-gray-900 dark:text-gray-100">Текст</div>
```

**Системный подход к карточкам:**

```vue
<!-- Невыбранная карточка -->
<div class="border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-800">
  <h4 class="text-gray-800 dark:text-gray-100">Заголовок</h4>
  <p class="text-gray-600 dark:text-gray-300">Описание</p>
</div>

<!-- Выбранная карточка -->
<div class="border-blue-400 bg-blue-50 dark:bg-blue-900/30">
  <h4 class="text-blue-900 dark:text-blue-200">Заголовок</h4>
  <p class="text-blue-700 dark:text-blue-300">Описание</p>
</div>
```

**См. полное руководство:** `doc/knowledge/019-design.md` → раздел "Работа с Tailwind dark: модификаторами"

## Стандарты кода

### Соглашения по кодированию

#### ✅ Что ДОЛЖНО быть в коде

1. **JSDoc документация** для всех функций и методов
   ```typescript
   /**
    * Описание функции
    * @param param1 Описание параметра
    * @returns Описание возвращаемого значения
    */
   export function myFunction(param1: string): string {
   ```

2. **Правильная обработка ошибок** вместо console.log/error
   ```typescript
   catch (error) {
     return {
       success: false,
       error: error instanceof Error ? error.message : 'Неизвестная ошибка'
     }
   }
   ```

3. **Использование route.run()** вместо fetch() и hardcoded URLs
   ```typescript
   const result = await contactFormRoute.run(ctx, { /* data */ })
   ```

4. **Type-safe код** с полной типизацией
   ```typescript
   export interface UserData {
     name: string
     email: string
   }
   ```

5. **Экспортируемые маршруты** через // @shared-route
   ```typescript
   // @shared-route
   export const myRoute = app.post('/path', async (ctx, req) => {
   ```

#### ❌ Что НЕЛЬЗЯ использовать

1. **console.log/console.error в серверном коде** - используй правильную обработку ошибок
   ```typescript
   // ❌ НЕПРАВИЛЬНО (в серверных роутах)
   export const myRoute = app.get('/test', async (ctx, req) => {
     console.log('Ошибка:', error)  // Серверный код!
   })
   
   // ✅ ПРАВИЛЬНО (в серверных роутах)
   export const myRoute = app.get('/test', async (ctx, req) => {
     throw new Error('Описание ошибки')  // Или return { error: '...' }
   })
   
   // ✅ ДОПУСТИМО (в клиентском JavaScript)
   <script>{`
     async function loadData() {
       try {
         const data = await fetch('/api/data')
       } catch (error) {
         console.error('Error loading data:', error)  // Клиентский код!
       }
     }
   `}</script>
   ```
   
   **Важно:** `console.*` допустимы только в клиентском JavaScript коде (внутри `<script>` тегов в браузере). В серверном коде используйте правильную обработку ошибок.

2. **alert()** - используй UI компоненты или показывай ошибку в UI
   ```typescript
   // ❌ НЕПРАВИЛЬНО
   alert('Заполните поле')
   
   // ✅ ПРАВИЛЬНО
   if (!value) {
     errorMessage.value = 'Заполните поле'
     return
   }
   ```

3. **Hardcoded URLs** - используй route.url()
   ```typescript
   // ❌ НЕПРАВИЛЬНО
   window.location.href = '/profile'
   
   // ✅ ПРАВИЛЬНО
   window.location.href = profileRoute.url()
   ```

4. **Node.js импорты** - используй Chatium API
   ```typescript
   // ❌ НЕПРАВИЛЬНО
   import { exec } from 'child_process'
   import { Buffer } from 'buffer'
   
   // ✅ ПРАВИЛЬНО
   import { request } from '@app/request'
   ```

5. **Недокументированные методы** - все методы должны иметь JSDoc
   ```typescript
   // ❌ НЕПРАВИЛЬНО
   export function process(data) { }
   
   // ✅ ПРАВИЛЬНО
   /**
    * Обрабатывает данные
    * @param data Исходные данные
    * @returns Обработанные данные
    */
   export function process(data: string): string { }
   ```

### Структурирование кода

1. **Групировка по функциональности** - группируй related функции
   ```typescript
   // ============== FILE UPLOAD ==============
   export const uploadFileRoute = app.post('/upload', ...)
   
   // ============== FILE DOWNLOAD ==============
   export const downloadFileRoute = app.get('/download', ...)
   ```

2. **Экспортируемые переменные в начале** - затем - функции/классы
   ```typescript
   export interface MyInterface { }
   export class MyClass { }
   export const myRoute = app.get(...)
   ```

3. **Helper функции в конце** - приватные функции ниже публичных
   ```typescript
   // Public
   export const publicFunction = () => { }
   
   // Helper
   function privateHelper() { }
   ```

### Файловая структура

- **Малые файлы** - один файл = один основной экспорт
- **Папки** - логически связанные функции в одной папке
- **Imports** - в начале файла, сгруппированы по типам
- **Exports** - явные экспорты с // @shared-route для маршрутов

### Примеры использования

#### Правильный API обработчик

```typescript
// @shared-route
/**
 * POST /api/users/create
 * Создает нового пользователя
 * @param body.name Имя пользователя
 * @param body.email Email пользователя
 * @returns Созданный пользователь
 */
export const createUserRoute = app.post('/users/create', async (ctx, req) => {
  try {
    const { name, email } = req.body
    
    if (!name || !email) {
      return {
        success: false,
        error: 'Требуются поля: name, email'
      }
    }
    
    const user = await Users.create(ctx, { name, email })
    
    return {
      success: true,
      user: user
    }
  } catch (error) {
    return {
      success: false,
      error: error instanceof Error ? error.message : 'Ошибка создания пользователя'
    }
  }
})
```

#### Правильный Vue компонент

```vue
<template>
  <div class="component">
    <button @click="handleClick" :disabled="loading">
      {{ loading ? 'Загрузка...' : 'Отправить' }}
    </button>
    <div v-if="result" :class="resultClass">
      {{ result.message }}
    </div>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { myRoute } from '../api/routes'

const loading = ref(false)
const result = ref(null)

/**
 * Обрабатывает клик по кнопке
 */
async function handleClick() {
  loading.value = true
  try {
    result.value = await myRoute.run(ctx, { })
  } catch (error) {
    result.value = {
      success: false,
      message: 'Ошибка выполнения'
    }
  } finally {
    loading.value = false
  }
}

const resultClass = {
  'bg-green-50': result.value?.success,
  'bg-red-50': !result.value?.success
}
</script>
```
