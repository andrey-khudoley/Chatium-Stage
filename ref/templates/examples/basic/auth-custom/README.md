# Кастомная форма авторизации

Полноценный пример создания кастомной страницы авторизации с поддержкой:
- SMS авторизации (отправка и подтверждение кода)
- Email авторизации (отправка и подтверждение кода)
- Авторизации по паролю (для телефона и email)
- Telegram OAuth
- Темная/светлая тема
- Адаптивный дизайн

## Структура

```
auth-custom/
├── index.tsx               # Главный роут с HTML страницей
├── LoginPage.vue           # Основной компонент формы входа
├── components/
│   ├── PhoneAuthForm.vue   # Форма авторизации по телефону
│   └── EmailAuthForm.vue   # Форма авторизации по email
├── sdk/
│   └── auth.ts             # SDK для работы с API авторизации
├── api/
│   ├── password.ts         # API для получения хеша пароля
│   └── telegram.ts         # API для Telegram OAuth
└── styles.ts               # Стили и CSS переменные
```

## Настройка Telegram OAuth

Чтобы работала авторизация через Telegram, нужно:

**Шаг 1:** Перейти по адресу `/app/store/~telegram-auth` в вашем Chatium workspace
**Шаг 2:** Установить дополнение "Telegram Auth"
**Шаг 3:** Нажать кнопку "Включить Telegram OAuth"
**Шаг 4:** Добавить нового провайдера "Telegram OAuth" и поставить галочку для активации

## Использование

1. Скопируйте всю папку `auth-custom` в свой проект
2. Адаптируйте стили в `styles.ts` под ваш дизайн
3. Настройте `back` URL в `index.tsx` (куда редиректить после входа)
4. Настройте Telegram OAuth в админке (если нужно)

## Провайдеры авторизации

Система автоматически определяет доступные провайдеры через `getEnabledAuthProviders(ctx)`:

- `'Sms'` - SMS авторизация
- `'Email'` - Email авторизация
- `'Password'` - Авторизация по паролю
- `'telegram-auth'` - Telegram OAuth

## Ключевые особенности

✅ Автоматическое определение доступных методов авторизации
✅ Пошаговая авторизация (ввод идентификатора → ввод кода)
✅ Возможность входа по паролю вместо кода

✅ Адаптивный дизайн

✅ Состояния загрузки

## Системные API endpoints

Примеры используют встроенные системные эндпоинты:

- `/s/auth/sms/send` - отправка SMS
- `/s/auth/sms/confirm` - подтверждение SMS
- `/s/auth/email/send` - отправка Email
- `/s/auth/email/confirm` - подтверждение Email
- `/s/auth/password` - авторизация по паролю
- `/s/auth/sign-out` - выход из системы
