# Отчеты по трафику сайта

Простое приложение для просмотра отчетов по трафику сайта с использованием данных из аналитики Chatium.

## Структура проекта

### Основные файлы
- `index.tsx` - главная точка входа приложения
- `pages/TrafficReports.vue` - основная страница с отчетами
- `pages/EventDetails.vue` - страница детального просмотра события
- `pages/GalleryPage.vue` - страница-галерея с описанием проекта (доступна по query параметру `gallery` без проверки прав)
- `pages/Statistics.vue` - компонент статистики для отображения общих метрик
- `components/Pagination.vue` - компонент пагинации для навигации по страницам
- `api/traffic.ts` - API-роуты для получения данных последних посещений с использованием `queryAi` из `@traffic/sdk`

### Функциональность

#### Фильтры
- **Дата события**: выбор периода от/до для анализа
- **URL события**: фильтрация по конкретному URL или его части
- **ID пользователя**: фильтрация по ID, имени или email пользователя
- Кнопка "Сбросить" для очистки поисковых фильтров (URL и ID пользователя)
- Поддержка отправки формы клавишей Enter в полях ввода
- **Пагинация**: навигация по страницам с настраиваемым количеством записей на странице (по умолчанию 20)

#### Вкладки
- **События**: основная таблица с детальным списком событий
- **Статистика**: обзор ключевых метрик (уникальные пользователи, общее количество посещений)

#### Отчеты
**Последние посещения** - список посещений с деталями и пагинацией:
- Время посещения
- URL страницы
- Информация о пользователе
- Данные об устройстве
- Email пользователя (если есть) с возможностью фильтрации по клику
- ID пользователя с возможностью фильтрации по клику
- Подсветка событий с URL начинающимся на "event://" легко-бежевым фоном
- Отображение общего количества найденных событий
- Пагинация для навигации по большим объемам данных
- Локация посетителя

**Детальный просмотр события** - страница с полной информацией о событии:
- Основная информация (время, URL, путь)
- Пользователь (ID, имя, тип)
- Техническая информация (User Agent, устройство, браузер, IP)
- Геолокация (страна, город, координаты)
- HTTP данные (метод, referer, язык, статус)
- UTM параметры (если есть)

**Статистика** - обзорная страница с ключевыми метриками:
- Общее количество посещений
- Количество уникальных пользователей  
- Среднее количество посещений на пользователя
- Период анализа
- Применяются те же фильтры что и для событий

### API-роуты
- `GET /api/traffic/recent-visits` - последние посещения
- `GET /api/traffic/event/:id` - детальная информация о событии
- `POST /api/traffic/event` - детальная информация о событии (ID передается через body)
- `GET /api/traffic/statistics` - получение статистики (уникальные пользователи и посещения)
- Используется `queryAi` из `@traffic/sdk` для запросов к базе данных аналитики

#### Параметры API для `/api/traffic/recent-visits`:
- `page` - номер страницы (по умолчанию 1)
- `limit` - количество записей на странице (по умолчанию 20)
- `GET /event` - страница детального просмотра события

### Особенности
- Responsive дизайн для мобильных устройств
- Debounce для фильтров (500ms)
- Обработка ошибок с логированием
- Форматирование дат и времени для русской локали
- Переходы между списком событий и детальным просмотром
- Кликабельные ссылки на resolved_user_id и email для автоматической фильтрации
- Поиск пользователей по частичному совпадению имени, фамилии или email
- Время в первой колонке отображается на разных строках с кликабельной ссылкой на времени
- Компонент пагинации с умным отображением номеров страниц
- При изменении фильтров происходит автоматический возврат на первую страницу
- Страница-галерея доступна по адресу `/?gallery` без требования прав администратора

### Компоненты
**Pagination.vue** - универсальный компонент пагинации:
- Отображение информации о количестве записей
- Навигация по страницам с кнопками "Первая", "Предыдущая", "Следующая", "Последняя"
- Умное отображение номеров страниц (показывает первую, текущую +/- 2, последнюю)

**Statistics.vue** - компонент статистики:
- Карточки с ключевыми метриками (уникальные пользователи, посещения)
- Автоматический расчет среднего количества посещений на пользователя
- Реагирует на изменения фильтров из родительского компонента
- Responsive дизайн с адаптацией под мобильные устройства

**GalleryPage.vue** - страница-галерея проекта:
- Доступна по query параметру `gallery` (например: `/?gallery`)
- Не требует прав администратора
- Светло-голубой градиентный фон
- Краткое описание проекта и его основных возможностей
- Карточки с иконками для каждой функции
- Адаптивный дизайн
