# Словарь для LLM: корень workspace s.chtm.khudoley.pro

Это **корень синхронизированного воркспэйса**, а не одно приложение с единым `index.tsx`.

- **Приложения** лежат в `p/` (у каждого свой вход и часто `docs/`, `.CHATIUM-LLM.md`).
- **Корневой `tsconfig.json`:** не охватывает весь `p/` и сэмплы в `inner/samples/` — только `inner/` (кроме samples), `zoom-agent-tool/`, `types/*.d.ts`, `vue-global.d.ts`; см. `README.md`.
- **Обзор проверок TypeScript:** `inner/docs/020-testing.md`; обязательный прогон для агентов — `.cursor/rules/ts-lint.mdc` (обычно `npm run ts-lint:touched -- <изменённые пути>`; полный воркспейс — `npm run ts-lint`).
- **Кратко о воркспэйсе:** корневой `README.md`.
- **Slash-команды Cursor** (появляются в списке при открытом этом воркспэйсе): каталог `.cursor/commands/` — например `/pipeline`, `/check`, `/toprod-assistant`, **`/pp`** (Parallel Pipeline в Multitask; см. `pp.md`). Глобальные команды из `~/.cursor/commands/` в списке воркспэйса могут не отображаться, если их нет здесь.
