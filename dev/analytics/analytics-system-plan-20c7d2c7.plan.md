# План разработки системы аналитики

## Ценностные потоки (Value Streams)

### VS-1: Просмотр исторических данных (приоритет 1)

Пользователь может просматривать и анализировать исторические события из ClickHouse через удобные дашборды с фильтрами, графиками и таблицами.

### VS-2: Управление пользовательскими событиями (приоритет 2)

Администратор может создавать, регистрировать и отслеживать пользовательские события (custom events) для специфичной бизнес-логики.

### VS-3: AI-ассистент для подписки на события (приоритет 3)

Пользователь взаимодействует с AI-агентом, который помогает настроить подписку на нужные события через интуитивный диалог и валидацию действий.

### VS-4: Real-time мониторинг событий (приоритет 4)

Пользователь получает обновления дашборда в реальном времени через WebSocket при поступлении новых событий.

## Архитектура системы

### Базовая структура

```
s.chtm.khudoley.tech/dev/analytics/
├── index.tsx                    # Главная точка входа
├── .dir.json                    # Конфигурация папки
├── .workspace.json              # Настройки workspace
├── .CHATIUM-LLM.md              # Основная документация
├── api/                         # API эндпоинты
│   ├── events.ts               # Работа с событиями
│   ├── custom-events.ts        # Пользовательские события
│   ├── dashboards.ts           # Управление дашбордами
│   └── permissions.ts          # Управление доступами
├── pages/                       # Vue компоненты страниц
│   ├── DashboardList.vue       # Список дашбордов
│   ├── DashboardView.vue       # Просмотр дашборда
│   └── AdminPanel.vue          # Админ-панель
├── components/                  # Переиспользуемые компоненты
│   ├── charts/                 # Компоненты графиков
│   ├── tables/                 # Компоненты таблиц
│   └── filters/                # Компоненты фильтров
├── tools/                      # Тулы для AI-агентов
│   ├── analytics-event-subscription.ts   # Тул поиска события в потоке
│   └── analytics-get-metric.ts # Тул для запроса информации по простым метрикам
└── tables/                      # Heap таблицы
    ├── dashboards.table        # Конфигурация дашбордов
    ├── custom-events.table     # Пользовательские события
    └── permissions.table       # Права доступа
```

## MVP (Волны 2-4) - Minimum Viable Product

**Цель**: Полнофункциональная система для просмотра данных и управления пользовательскими событиями.

**Результат**: Несколько готовых дашбордов, система custom events, базовая админка.

### Волна 2: Дашборды и визуализация

**Результат**: Набор предустановленных дашбордов с графиками и таблицами.

### Фичи:

- **F-2.1**: Главная страница со списком дашбордов
  - Страница `DashboardList.vue` с карточками дашбордов
  - Навигация между дашбордами
  - Иконки и описания дашбордов

- **F-2.2**: Универсальный шаблон дашборда
  - Компонент `DashboardView.vue` для отображения любого дашборда
  - Система компоновки компонентов (grid layout)
  - Общая шапка с фильтрами

- **F-2.3**: Компоненты визуализации
  - Компонент графика (Chart.js)
  - Компонент таблицы с сортировкой
  - Компонент метрик-карточек
  - Компонент фильтров по дате

- **F-2.4**: API для работы с событиями
  - Эндпоинты для разных типов событий (заказы, пользователи, платежи)
  - Агрегация данных по дням/месяцам
  - Фильтрация по параметрам

- **F-2.5**: Предустановленные дашборды
  - "Обзор событий" - общая статистика
  - "Заказы и платежи" - финансовая аналитика
  - "Пользователи" - регистрации, активность

### Волна 3: Пользовательские события (Custom Events)

**Результат**: Система создания и регистрации пользовательских событий.

### Фичи:

- **F-3.1**: Heap таблица для custom events
  - Схема таблицы: id, name, description, endpoint, created_at, created_by
  - CRUD операции через Heap

- **F-3.2**: API эндпоинты для custom events
  - `POST /api/custom-events/create` - создание события
  - `GET /api/custom-events/list` - список всех событий
  - `DELETE /api/custom-events/:id` - удаление события
  - `POST /event/reg/:id` - регистрация срабатывания события

- **F-3.3**: Админка для управления custom events
  - Страница списка пользовательских событий
  - Форма создания нового события
  - Валидация: событие должно существовать перед регистрацией
  - Отображение статистики срабатываний

- **F-3.4**: Интеграция custom events в дашборды
  - Возможность добавлять custom events в существующие дашборды
  - Отображение статистики по custom events

### Волна 4: Система доступов

**Результат**: Гибкая система управления правами доступа к дашбордам.

### Фичи:

- **F-4.1**: Heap таблицы для permissions
  - Таблица `dashboard_permissions`: dashboard_id, role, can_view, can_edit, can_delete
  - Таблица `dashboards`: id, name, description, config, owner, is_public

- **F-4.2**: Middleware проверки доступа
  - Функция `requireDashboardAccess(ctx, dashboardId, action)`
  - Проверка прав перед отображением дашборда
  - Проверка прав перед редактированием/удалением

- **F-4.3**: Админ-панель управления доступами
  - Страница `AdminPanel.vue`
  - Управление ролями пользователей
  - Назначение прав доступа к конкретным дашбордам
  - Просмотр аудит-лога изменений

- **F-4.4**: UI для управления доступами
  - Список пользователей с ролями
  - Матрица доступов (пользователь × дашборд)
  - Возможность массового назначения прав

### Волна 5: AI-агент для подписки на события

**Результат**: AI-ассистент помогает пользователю настроить подписку на нужные события.

### Фичи:

- **F-5.1**: AI-агент базовый функционал
  - Файл `agents/event-subscription.ts`
  - Интеграция с `@app/agents` (startCompletion, pushMessageToChain)
  - Базовый промпт для понимания запросов пользователя

- **F-5.2**: Tools для агента
  - Tool "findEventByDescription" - поиск события по описанию
  - Tool "subscribeToEvent" - подписка на событие
  - Tool "waitForEventTrigger" - ожидание срабатывания события

- **F-5.3**: Workflow подписки на события
  - Пользователь описывает желаемое событие в чате
  - Агент находит подходящее событие
  - Агент подписывается и ждёт первого срабатывания
  - Пользователь выполняет действие, которое вызывает событие
  - Агент получает подтверждение и показывает детали события
  - Пользователь подтверждает или отклоняет

- **F-5.4**: UI для AI-ассистента
  - Компонент чата с агентом
  - Отображение истории диалога
  - Визуализация найденного события
  - Кнопки подтверждения/отклонения

- **F-5.5**: Интеграция подписок с дашбордами
  - Сохранение подписки на событие для дашборда
  - Heap таблица `event_subscriptions`
  - Связь подписки с дашбордом

## Технические детали

### Используемые технологии

- **Frontend**: Vue 3 (Composition API), Chart.js
- **Backend**: Chatium framework, TypeScript
- **Database**: ClickHouse (события), Heap (конфигурация)
- **Real-time**: WebSocket (@app/socket)
- **AI**: Chatium Agents (@app/agents)
- **Стили**: TailwindCSS (по `001-standards.md`)

### Ключевые файлы из референса

- `ref/analitika-getkursa-extended/index.tsx` - структура главной страницы
- `ref/analitika-getkursa-extended/api/traffic.ts` - примеры SQL запросов
- `ref/analitika-getkursa-extended/pages/TrafficReports.vue` - UI компоненты

### Стандарты разработки

- Следовать `001-standards.md` для кода
- Использовать `007-vue.md` для Vue компонентов
- Применять `008-heap.md` для работы с БД
- Использовать `016-analytics.md` для работы с событиями
- Применять `010-agents.md` для AI-агента
- Использовать `014-socket.md` для WebSocket
