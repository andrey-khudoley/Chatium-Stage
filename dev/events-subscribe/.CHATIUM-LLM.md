# Events Subscribe System - Документация проекта

## Описание проекта

Веб-интерфейс для подписки и мониторинга событий из трафика и GetCourse в режиме реального времени.

## Архитектура

### Структура проекта

```
/
├── index.tsx                          # Главная страница
├── settings.tsx                       # Страница настроек подписок
├── monitor.tsx                        # Страница мониторинга событий
├── tables/
│   └── subscriptions.table           # Таблица подписок пользователей
├── api/
│   ├── subscriptions.ts              # API для управления подписками
│   └── events.ts                     # API для получения событий
├── pages/
│   ├── MainPage.vue                  # Главная страница
│   ├── SettingsPage.vue              # Страница настроек
│   └── MonitorPage.vue               # Страница мониторинга
└── shared/
    └── eventTypes.ts                 # Типы и константы событий
```

### Таблицы данных

#### subscriptions.table
Хранит подписки пользователей на события:
- `userId` (UserRefLinkKind) - пользователь
- `eventType` (String) - тип события (traffic/getcourse)
- `eventName` (String) - название события
- `isActive` (Boolean) - активна ли подписка

### API Routes

#### /api/subscriptions
- `GET /list` - получить список подписок текущего пользователя
- `POST /subscribe` - подписаться на одно событие
- `POST /unsubscribe` - отписаться от одного события
- `POST /subscribe-all` - массовая подписка (все / traffic / getcourse)
- `POST /unsubscribe-all` - массовая отписка (все / traffic / getcourse)
- `GET /available-events` - получить список доступных событий

#### /api/events
- `POST /stream` - начать стрим событий через websocket

#### События трафика (Traffic Events) - 21 тип

**Важно:** События трафика фильтруются следующим образом:
- Только события с непустым полем `action` (реальные действия пользователей)
- Только внешние URL (начинающиеся с `https`)
- Системные страницы (s.chtm, /dev/) автоматически исключаются
- События текущего пользователя скрыты
- `pageview` - Просмотр страницы
- `registration` - Регистрация
- `form_submit` - Отправка формы
- `button_click` - Клик по кнопке
- `link_click` - Клик по ссылке
- `video_play` - Воспроизведение видео
- `video_pause` - Пауза видео
- `video_complete` - Просмотр видео до конца
- `scroll` - Прокрутка страницы
- `download` - Скачивание файла
- `search` - Поиск
- `add_to_cart` - Добавление в корзину
- `remove_from_cart` - Удаление из корзины
- `checkout` - Оформление заказа
- `purchase` - Покупка
- `login` - Вход в аккаунт
- `logout` - Выход из аккаунта
- `share` - Поделиться контентом
- `comment` - Комментарий
- `like` - Лайк
- `custom_action` - Пользовательское действие

### Websocket для real-time мониторинга

Используется `@app/socket` для трансляции событий:
- Сервер отправляет события через `sendDataToSocket()`
- Клиент подписывается через `getOrCreateBrowserSocketClient()`
- Socket ID генерируется на основе userId для персонализации

### Фильтрация событий

**События текущего пользователя:**
- Автоматически исключаются события, где `user_id` совпадает с текущим пользователем
- Фильтрация происходит как на уровне SQL-запроса, так и на уровне JavaScript

**Системные страницы:**
- Исключаются события с URL, начинающихся с `https://s.chtm` (системные ресурсы)
- Исключаются события с URL `/dev/*` (страницы разработки)
- Показываются только события с внешних публичных страниц

### Авторизация

Все страницы защищены авторизацией через `requireRealUser(ctx)`.

## Используемые SDK

- `@traffic/sdk` - для запросов к данным трафика (`queryAi`)
- `@gc-mcp-server/sdk` - для запросов к данным GetCourse (`gcQueryAi`)
- `@app/socket` - для websocket соединений
- `@app/auth` - для авторизации
