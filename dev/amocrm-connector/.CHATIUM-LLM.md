# AmoCRM Connector

> **Веб-приложение для управления настройками интеграции с AmoCRM и работой с OAuth 2.0**

## 📋 Описание проекта

**AmoCRM Connector** — это современное веб-приложение на платформе Chatium для управления настройками интеграции с AmoCRM. Проект предоставляет интуитивный интерфейс для создания, редактирования и удаления настроек подключения к AmoCRM API, а также полноценную интеграцию с OAuth 2.0 для безопасного доступа к данным AmoCRM.

### Основные возможности

- ✅ **CRUD операции** — создание, чтение, обновление и удаление настроек
- 🔐 **OAuth 2.0 интеграция** — авторизация через AmoCRM с автоматическим обновлением токенов
- 🛡️ **Защита административных страниц** — доступ к настройкам OAuth только для администраторов
- 🤖 **Автоматическое обновление токенов** — джоб, который обновляет токен каждые 8 часов без участия пользователя
- 🔔 **Вебхуки AmoCRM** — автоматическая обработка обновлений сделок в реальном времени
- 🎯 **Настройка полей** — выбор соответствия полей AmoCRM через UI (Продукт, Тариф, Своя цена, Сформировать?)
- 🔄 **Идемпотентность** — защита от дублирования обработки одного события
- 📧 **Извлечение контактов** — автоматическое получение email основного контакта сделки
- 📝 **Подробная инструкция по настройке** — пошаговое руководство с копируемыми ссылками для настройки OAuth
- 📊 **Дополнительные поля** — просмотр всех кастомных полей из AmoCRM
- 🧪 **Тестирование** — проверка обработки вебхуков по ID сделки с визуализацией всех извлеченных данных
- 🎨 **Современный UI** — красивый интерфейс с градиентами и анимациями
- 🌓 **Темная/светлая тема** — автоматическое определение системной темы и ручное переключение
- 📈 **Статистика** — отображение статуса OAuth, количества сервисов и времени последнего обновления
- 💾 **Локальное хранилище** — сохранение предпочтений темы в localStorage
- 📱 **Адаптивный дизайн** — корректная работа на мобильных устройствах и десктопах
- ⚡ **Реактивность** — мгновенное обновление интерфейса на Vue 3 Composition API
- 💳 **Генерация ссылок на оплату** — автоматическое создание заказов GetCourse при обработке вебхуков
- 🔗 **Интеграция с Deal Manager** — отправка запросов к внешнему сервису для получения ссылок на оплату
- 🔄 **Повторная отправка** — возможность повторить неудачный запрос через UI и API
- 💬 **Комментарии в AmoCRM** — автоматическое добавление комментариев со ссылками на оплату в карточки сделок
- ✏️ **Редактируемый шаблон** — гибкая настройка текста комментария с поддержкой переменных
- 📝 **Логирование комментариев** — полная история всех добавленных комментариев с возможностью повторной отправки

## 🔐 OAuth 2.0 интеграция

### Архитектура OAuth

Проект реализует полноценную OAuth 2.0 интеграцию с AmoCRM:

1. **Хранение конфигурации** — в таблице `settings` с ключами:
   - `amocrm_subdomain` — поддомен AmoCRM
   - `amocrm_client_id` — Client ID интеграции
   - `amocrm_client_secret` — Client Secret
   - `amocrm_redirect_uri` — Redirect URI

2. **Хранение токенов** — в той же таблице `settings`:
   - `amocrm_access_token` — токен доступа
   - `amocrm_refresh_token` — токен обновления
   - `amocrm_expires_at` — время истечения токена
   - `amocrm_status` — статус подключения (active/offline/expired)

3. **Автоматическая проверка** — при каждом запросе проверяется срок действия токена

### Автоматическое обновление токенов

Проект включает систему автоматического обновления токенов через периодический джоб:

**Как это работает:**

```
Время 0:     Авторизация → Токен действителен до: 30 дней
Через 8ч:    Джоб автоматически обновляет токен → Новый токен на 30 дней
Через 16ч:   Джоб снова обновляет токен → Ещё один токен на 30 дней
Через 24ч:   ...и так далее бесконечно каждые 8 часов
```

**Компоненты системы:**

1. **Джоб** (`.functions/refreshAmoCRMToken.ts`)
   - Обновляет токен через AmoCRM API
   - Запускается каждые 8 часов автоматически
   - Логирует все операции
   - Планирует следующий запуск после выполнения
   - Сохраняет `taskId` в настройки для возможности отмены

2. **API endpoints**
   - `/start-token-refresh` — запуск автообновления
   - `/cancel-token-refresh` — отмена автообновления
   - `/debug-task-id` — отладочная информация о taskId

3. **Интерфейс управления**
   - "Включить автообновление" — запускает периодическое обновление
   - "Отменить автообновление" — останавливает джоб и очищает настройки
   - Отображение времени следующего запуска
   - Статус автообновления (включено/выключено)

**Преимущества:**
- ✅ Токен всегда актуален (обновление каждые 8 часов)
- ✅ Не нужно авторизовываться вручную каждый месяц
- ✅ Полностью автоматизированный процесс
- ✅ Детальное логирование для мониторинга
- ✅ Обработка ошибок (статус → offline при неудаче)
- ✅ Возможность отмены автообновления в любой момент
- ✅ Отображение времени следующего обновления

**Технические детали:**

- `taskId` от `scheduleJobAsap/scheduleJobAfter` сохраняется как **строка** в таблице настроек
- При отмене строка преобразуется обратно в **число** для `cancelScheduledJob`
- Используется `await` для всех методов планирования джобов (они возвращают Promise)
- Настройки хранятся в ключах: `amocrm_refresh_task_id` и `amocrm_next_refresh_at`

### API endpoints для OAuth

#### GET `/oauth/config`
Получение настроек OAuth конфигурации.

**Response:**
```json
{
  "success": true,
  "config": {
    "subdomain": "mycompany",
    "clientId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "clientSecret": "••••••••",
    "redirectUri": "https://example.com/oauth/callback"
  }
}
```

#### POST `/oauth/config`
Сохранение настроек OAuth.

**Request Body:**
```json
{
  "subdomain": "mycompany",
  "clientId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "clientSecret": "your_secret",
  "redirectUri": "https://example.com/oauth/callback"
}
```

#### GET `/oauth/status`
Получение текущего статуса подключения.

**Response:**
```json
{
  "success": true,
  "status": "active",
  "expiresAt": "2025-11-03T12:00:00.000Z"
}
```

Возможные статусы:
- `active` — токен действителен
- `expired` — токен истёк
- `offline` — подключение не настроено

#### GET `/oauth/auth-url`
Генерация URL для авторизации в AmoCRM.

**Response:**
```json
{
  "success": true,
  "authUrl": "https://mycompany.amocrm.ru/oauth?client_id=..."
}
```

#### POST `/oauth/exchange`
Обмен кода авторизации на токены доступа.

**Request Body:**
```json
{
  "code": "authorization_code_from_amocrm"
}
```

**Response:**
```json
{
  "success": true
}
```

После успешного обмена токены сохраняются в таблице настроек.

#### POST `/oauth/refresh`
Обновление токена доступа вручную.

**Response:**
```json
{
  "success": true
}
```

#### POST `/start-token-refresh`
Запуск автоматического обновления токена (джоб каждые 8 часов).

**Response:**
```json
{
  "success": true,
  "taskId": "6222078",
  "nextRefreshAt": "2025-11-10T19:59:57.000Z",
  "message": "Автоматическое обновление токенов запущено. Следующее обновление через 8 часов."
}
```

**Как работает:**
- Запускает джоб `refreshAmoCRMTokenJob` немедленно с помощью `await scheduleJobAsap()`
- Джоб автоматически обновляет токен через AmoCRM API
- После выполнения планирует следующий запуск через 8 часов
- Сохраняет `taskId` в настройки для возможности отмены
- Цикл повторяется бесконечно, обеспечивая постоянную актуальность токена
- Токен действителен ~1 месяц, обновление каждые 8 часов гарантирует отсутствие истечения

#### POST `/cancel-token-refresh`
Отмена автоматического обновления токена.

**Response (успех):**
```json
{
  "success": true,
  "message": "Автоматическое обновление токенов успешно отменено"
}
```

**Response (ошибка):**
```json
{
  "success": false,
  "error": "Не удалось отменить задачу: [причина]. Настройки очищены, но задача может остаться в планировщике."
}
```

**Как работает:**
- Получает сохранённый `taskId` из настроек
- Преобразует строку в число (если это число)
- Вызывает `cancelScheduledJob(ctx, taskId)` для отмены джоба
- Удаляет `taskId` и время следующего запуска из настроек
- Обрабатывает некорректные taskId (например, `"[object Promise]"` от старых версий)

#### GET `/debug-task-id`
Отладочная информация о сохранённом taskId (для диагностики).

**Response:**
```json
{
  "success": true,

    "valueType": "string",
    "valueLength": 7,
    "isValidFormat": true

    "value": "2025-11-10T19:59:57.000Z"
  }
}
```

#### GET `/custom-fields`
Получение списка дополнительных полей из AmoCRM.

**Response:**
```json
{
  "success": true,
  "fields": [
    {
      "id": 12345,

      "code": "EMAIL",
```

**Ошибки:**
- `error: "Необходима авторизация"` — токен отсутствует
- `error: "Токен истек", needRefresh: true` — нужно обновить токен

## 🔔 Вебхуки AmoCRM

### Архитектура вебхуков

Проект реализует полноценную интеграцию с вебхуками AmoCRM для автоматической обработки обновлений сделок в реальном времени:

1. **Программная подписка** — через API AmoCRM на события `update_lead` и `status_lead`
2. **Приём вебхуков** — публичный endpoint `/webhook` для получения уведомлений от AmoCRM
3. **Идемпотентная обработка** — защита от дублирования через таблицу `webhook-events`
4. **Настройка полей** — UI для выбора соответствия полей AmoCRM через select-списки
5. **Извлечение данных** — автоматическое получение сделки, контактов и email
6. **Логирование** — детальные логи обработки каждого события

### Настройки полей в таблице settings

Для работы вебхуков используются следующие ключи:

**Управление подпиской:**
- `webhook_destination` — URL для приёма вебхуков (полный)
- `webhook_status` — статус подписки (subscribed/unsubscribed)

**Соответствие полей AmoCRM:**
- `webhook_field_check` — field_id чекбокса "Сформировать?"
- `webhook_field_product` — field_id поля "Продукт" (select)
- `webhook_field_tariff` — field_id поля "Тариф" (select)
- `webhook_field_price` — field_id поля "Своя цена" (numeric)

### Таблица идемпотентности

**Таблица:** `webhook-events.table.ts`

**Поля:**
- `leadId` — ID сделки в AmoCRM
- `leadUpdatedAt` — timestamp обновления сделки
- `processedAt` — время обработки события
- `status` — статус обработки (processing/processed/skipped/error)
- `errorMessage` — сообщение об ошибке (опционально)

**Назначение:** Защита от повторной обработки одного события при дублировании вебхуков от AmoCRM.

### API endpoints для вебхуков

#### POST `/webhooks/subscribe`
Подписка на вебхуки AmoCRM (программно через API).

**Request Body:**
```json
{
  "webhookUrl": "https://s.chtm.khudoley.tech/dev/amocrm-connector/webhook"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Подписка на вебхук успешно создана"
}
```

**Как работает:**
- Отправляет POST запрос к AmoCRM API `/api/v4/webhooks`
- Подписывается на события: `update_lead`, `status_lead`
- Сохраняет статус подписки в настройки (`webhook_status: subscribed`)

#### GET `/webhooks/list`
Получение списка активных вебхуков из AmoCRM.

**Response:**
```json
{
  "success": true,
  "webhooks": [
    {
      "destination": "https://s.chtm.khudoley.tech/dev/amocrm-connector/webhook",
      "settings": ["update_lead", "status_lead"]
    }
  ]
}
```

**Автоматическая синхронизация статуса:**
- При каждом запросе автоматически синхронизирует статус вебхука в БД с реальным статусом в AmoCRM
- Поиск вебхука по паттерну `/dev/amocrm-connector/api/webhook` (работает даже если URL не был сохранен)
- Если вебхук найден в AmoCRM, но в БД статус `unsubscribed` — автоматически обновляет на `subscribed`
- Если вебхука нет в AmoCRM, но в БД статус `subscribed` — автоматически обновляет на `unsubscribed`
- Сохраняет URL вебхука в настройки (`webhook_destination`) при обнаружении

#### POST `/webhooks/unsubscribe`
Отписка от вебхуков AmoCRM.

**Request Body:**
```json
{
  "webhookUrl": "https://s.chtm.khudoley.tech/dev/amocrm-connector/webhook"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Отписка от вебхука успешно выполнена"
}
```

## 💳 Генерация ссылок на оплату (Payment Service)

### Архитектура Payment Service

Сервис автоматически создаёт заказы в GetCourse при обработке вебхуков AmoCRM и возвращает ссылки на оплату:

1. **Маппинг кодов предложений** — таблица соответствия продукт+тариф → ID предложения GetCourse
2. **Автоматическая обработка** — при получении вебхука система ищет соответствующий offerId
3. **Интеграция с Deal Manager** — отправка запроса к внешнему сервису для создания заказа
4. **Логирование** — сохранение всех операций с детальной информацией

### Таблица кодов предложений

**Таблица:** `payment-offers.table.ts`

**Поля:**
- `productValue` — строковое значение поля "Продукт" из AmoCRM (например, "Флагман")
- `tariffValue` — строковое значение поля "Тариф" из AmoCRM (например, "Тариф 1")
- `offerId` — ID предложения для GetCourse
- `offerName` — название предложения (для удобства)
- `description` — описание (опционально)

**Важно:** Комбинация productValue + tariffValue должна быть уникальной.

### Таблица логов

**Таблица:** `payment-logs.table.ts`

**Поля:**
- `leadId` — ID сделки AmoCRM
- `email` — email клиента
- `productValue` — название продукта
- `tariffValue` — название тарифа
- `customPrice` — своя цена из AmoCRM (опционально)
- `offerId` — ID предложения GetCourse
- `dealId` — ID созданного заказа GetCourse (опционально)
- `dealNumber` — номер заказа GetCourse (опционально)
- `paymentUrl` — ссылка на оплату (опционально)
- `status` — статус (success/error/pending)
- `errorMessage` — текст ошибки (опционально)
- `requestData` — данные запроса (опционально)
- `responseData` — данные ответа (опционально)

- 🐛 **Устранена ошибка API** — исправлены валидационные ошибки AmoCRM (`NotSupportedChoice`, `TooFew`, `InvalidType`)

**Техническая информация:**

AmoCRM API имеет строгие требования к очистке полей различных типов:
- **Select-поля** (список): требуют `values: [{ value: null }]` для сброса в состояние "Выбрать"
- **Numeric-поля** (число): требуют `values: [{ value: 0 }]` (строго числовой тип, не строка)
- **Checkbox-поля** (флажок): требуют `values: [{ value: false }]` для снятия галочки

**Неправильные подходы (вызывали ошибки):**
- ❌ `values: []` — ошибка `TooFew` (требуется ровно 1 элемент)
- ❌ `values: [{ value: '' }]` — для select: ошибка `NotSupportedChoice`
- ❌ `values: [{ value: '' }]` — для numeric: ошибка `InvalidType` (требуется число)

**Правильная реализация:**
```typescript
const customFieldsToReset = [
  {
    field_id: parseInt(fieldSettings.field_check),
    values: [{ value: false }]  // Чекбокс → выключен
  },
  {
    field_id: parseInt(fieldSettings.field_product),
    values: [{ value: null }]  // Select → "Выбрать"
  },
  {
    field_id: parseInt(fieldSettings.field_tariff),
    values: [{ value: null }]  // Select → "Выбрать"
  },
  {
    field_id: parseInt(fieldSettings.field_price),
    values: [{ value: 0 }]  // Numeric → 0 (пустое поле)
  }
]
```

**Где происходит сброс:**
- Функция: `resetLeadFields()` в `api/webhook.ts`
- Вызов: после успешной отправки комментария в `processCommentService()`
- Endpoint AmoCRM: `PATCH /api/v4/leads/{leadId}` с обновлением `custom_fields_values`

## 🎨 Правила дизайна (Design System)

### Общие принципы

**AmoCRM Connector** использует единую дизайн-систему для обеспечения консистентности интерфейса на всех страницах проекта.

#### Базовые правила:

1. **Темная тема по умолчанию** — все страницы используют темный фон `#0f172a`
2. **Единая типографика** — шрифт Inter для всех элементов
3. **CSS переменные** — все цвета через CSS Variables для поддержки светлой/темной темы
4. **Градиенты** — использование градиентов для создания глубины и визуального интереса
5. **Анимации** — плавные transitions для всех интерактивных элементов (0.2s ease)
6. **Адаптивность** — корректная работа на всех устройствах (mobile-first подход)
