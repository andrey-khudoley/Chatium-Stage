# Отчеты по трафику сайта

Приложение для анализа трафика сайта с использованием данных Chatium Analytics.

## Структура проекта

### Основные файлы
- `index.tsx` - главная точка входа приложения с проверкой конфигурации
- `pages/TrafficReports.vue` - основная страница с отчетами
- `pages/CloudSettingsPage.vue` - страница настройки облачных данных
- `api/traffic.ts` - API-роуты для получения данных трафика
- `api/settings.ts` - API-роуты для работы с настройками (CRUD)
- `api/test-settings.ts` - тестовый роут для проверки работы heap-таблицы
- `tables/settings.table.ts` - определение heap-таблицы настроек

### Таблицы данных
- `tables/settings.table.ts` - таблица настроек проекта в формате ключ-значение:
  - key (String) - ключ настройки
  - value (String) - значение настройки
  - Используется для хранения конфигурации системы и пользовательских настроек

### Конфигурация системы

#### Первичная настройка
- При первом запуске система показывает форму настройки конфигурации
- Форма запрашивает адрес сервера и API ключ для подключения к аналитике
- После сохранения настроек происходит автоматическая перезагрузка страницы

#### API роуты настроек
- `GET /api/settings~get?key=settingKey` - получение значения настройки по ключу
- `GET /api/settings~all` - получение всех настроек
- `POST /api/settings~save` - сохранение или обновление настройки (body: {key, value})
- `POST /api/settings~delete?key=settingKey` - удаление настройки по ключу

#### Работа с Heap-таблицей настроек

**Определение таблицы** (`tables/settings.table.ts`):
```typescript
export const TAnalitikaGetkursaCloudSettingsKv8 = Heap.Table(
  't_analitika_getkursa_cloud_settings_kv8',
  {
    key: Heap.String({ 
      customMeta: { title: 'Ключ' }, 
      searchable: { langs: ['ru', 'en'], embeddings: false } 
    }),
    value: Heap.String({ 
      customMeta: { title: 'Значение' }, 
      searchable: { langs: ['ru', 'en'], embeddings: false },
      defaultValue: ''
    }),
  }
)
```

**Основные методы работы с таблицей**:

1. **Создание или обновление настройки** (рекомендуемый способ):
```typescript
import TAnalitikaGetkursaCloudSettingsKv8 from './tables/settings.table'

// createOrUpdateBy автоматически создает или обновляет запись
const setting = await TAnalitikaGetkursaCloudSettingsKv8.createOrUpdateBy(
  ctx,
  'key',  // Поле для поиска уникальной записи
  { key: 'apiUrl', value: 'https://api.example.com' }
)
```

2. **Получение настройки по ключу**:
```typescript
const setting = await TAnalitikaGetkursaCloudSettingsKv8.findOneBy(ctx, { 
  key: 'apiUrl' 
})
console.log(setting?.value) // https://api.example.com
```

3. **Получение всех настроек**:
```typescript
const allSettings = await TAnalitikaGetkursaCloudSettingsKv8.findAll(ctx, {
  where: {}
})
allSettings.forEach(setting => {
  console.log(`${setting.key}: ${setting.value}`)
})
```

4. **Удаление настройки**:
```typescript
const setting = await TAnalitikaGetkursaCloudSettingsKv8.findOneBy(ctx, { 
  key: 'apiUrl' 
})
if (setting) {
  await TAnalitikaGetkursaCloudSettingsKv8.delete(ctx, setting.id)
}
```

**Типовые сценарии использования**:

Хранение конфигурации API:
```typescript
await TAnalitikaGetkursaCloudSettingsKv8.createOrUpdateBy(ctx, 'key', {
  key: 'gc_api_url',
  value: 'https://getkourse.ru/api/v1'
})
```

Хранение JSON конфигурации:
```typescript
// Сохранение
await TAnalitikaGetkursaCloudSettingsKv8.createOrUpdateBy(ctx, 'key', {
  key: 'report_filters',
  value: JSON.stringify({ period: 30, status: 'active' })
})

// Чтение
const setting = await TAnalitikaGetkursaCloudSettingsKv8.findOneBy(ctx, { 
  key: 'report_filters' 
})
const filters = JSON.parse(setting?.value || '{}')
```

Массовое создание настроек:
```typescript
const defaultSettings = [
  { key: 'theme', value: 'dark' },
  { key: 'language', value: 'ru' },
  { key: 'timezone', value: 'Europe/Moscow' }
]

for (const setting of defaultSettings) {
  await TAnalitikaGetkursaCloudSettingsKv8.createOrUpdateBy(
    ctx, 
    'key', 
    setting
  )
}
```

**Важные замечания**:
- ✅ Используйте `createOrUpdateBy` для создания/обновления настроек
- ✅ Метод `createOrUpdateBy` требует уникальности по указанному полю
- ✅ Все значения хранятся как строки (для сложных типов используйте JSON.stringify/parse)
- ✅ Требуется роль Admin для доступа ко всем API endpoints
- ✅ Таблица создается автоматически при первом использовании

### API-роуты

#### API настроек
- `GET /api/settings~get?key=settingKey` - получение значения настройки по ключу
  - Возвращает: `{ success: true, key, value }`
- `GET /api/settings~all` - получение всех настроек
  - Возвращает: `{ success: true, settings: [{key, value}, ...] }`
- `POST /api/settings~save` - сохранение или обновление настройки
  - Body: `{ key: string, value: string }`
  - Использует `createOrUpdateBy` для автоматического создания/обновления
  - Возвращает: `{ success: true, key, value, id }`
- `POST /api/settings~delete?key=settingKey` - удаление настройки по ключу
  - Находит запись по ключу и удаляет по ID
  - Возвращает: `{ success: true, message }`

#### API тестирования
- `GET /api/test-settings` - полный тест CRUD операций с настройками
  - Создает 2 тестовые настройки
  - Проверяет чтение
  - Обновляет настройки
  - Удаляет настройки
  - Проверяет удаление
- `GET /api/test-settings~cleanup` - очистка всех тестовых настроек
  - Удаляет все записи с ключами, начинающимися на `test_setting_`
  - Полезно для очистки дубликатов после экспериментов

#### API трафика
- `GET /api/traffic/recent-visits` - последние посещения
- `GET /api/traffic/daily-stats` - статистика по дням
- `GET /api/traffic/popular-pages` - популярные страницы
- `POST /api/traffic/entry-pages` - страницы входа
- `GET /api/traffic/orders` - заказы с группировкой по дням
- `GET /api/traffic/orders-summary` - суммарные данные по заказам
- `GET /api/traffic/order-details` - детальный список заказов с ID
- `GET /api/traffic/order-status/:orderId` - статус конкретного заказа
- `GET /api/traffic/payments-by-date` - оплаты по дням
- `GET /api/traffic/users-registration` - регистрация пользователей
- `GET /api/traffic/total-users` - общее количество пользователей
- `GET /api/traffic/telegram-connected-users` - пользователи с подключенным Telegram
- `GET /api/traffic/telegram-disconnected-users` - пользователи с отключенным Telegram
- `GET /api/traffic/group-added-users` - пользователи в группе
- `GET /api/traffic/channel-subscribers` - подписчики канала

### Логика статусов заказов
Система использует приоритетную логику определения статусов:
1. Если есть событие `dealPaid` - статус автоматически "payed" (завершен)
2. Иначе берется последний статус из `dealStatusChanged`
3. Иначе берется начальный статус из `dealCreated`
