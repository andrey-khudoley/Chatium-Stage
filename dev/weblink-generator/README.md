# Weblink Generator - Форма входа на вебинар

Проект содержит форму регистрации на вебинар с дизайном, соответствующим стилю сайта [key.sobolevarent.ru](https://key.sobolevarent.ru/).

## 🎨 Дизайн

Форма выполнена в стиле референсного сайта с использованием:
- **Фон**: Тёмно-оливковый (#4a5234)
- **Карточка формы**: Белая (#FFFFFF) с мягкими тенями
- **Текст**: Тёмно-серый/чёрный (#1A1A1A, #333333)
- **Акценты**: Бежевые оттенки (#D4B896, #F5E5D0)
- **Кнопки**: Тёмные (#222222) с белым текстом

```
weblink-generator/
```

## 🔧 Основные компоненты

### index.tsx
Создаёт HTML структуру страницы и подключает:
- Google Fonts (Inter)
- FontAwesome 6.7.2 (локальная версия)
- TailwindCSS 3.4.16 (локальная версия)
- CSS переменные и общие стили

### styles.tsx
Экспортирует три основных блока:

#### 1. `tailwindScript`
Конфигурация TailwindCSS с тёмным режимом.

#### 2. `cssVariables`
CSS-переменные для цветовой схемы:
- `--color-bg`: #4a5234 (фон страницы)
- `--color-bg-card`: #FFFFFF (фон карточки)
- `--color-text`: #1A1A1A (основной текст)
- `--color-primary`: #D4B896 (акцентный цвет)
- `--color-button-dark`: #222222 (тёмная кнопка)
- `--color-button-light`: #F5E5D0 (светлая кнопка)

#### 3. `commonStyles`
Общие стили компонентов:
- `.gradient-bg` - фон с градиентом и декоративными элементами
- `.card` - белая карточка с тенями и скруглёнными углами
- `.btn` и `.btn-primary` - стили кнопок
- `.input` - стили полей ввода
- `.input-error` - стили для полей с ошибками (красная рамка и тень)

### pages/LoginPage.vue
Vue компонент с адаптивной структурой:

**Структура компонента:**
- `.form-container` - основной контейнер с фоном
- `.form-wrapper` - обертка формы с ограничением ширины
- `.card` - карточка формы с grid-раскладкой
  - `.form-header` - левая декоративная панель (на десктопе) / верхняя часть (на мобильных)
  - `.form-content` - правая панель с полями и кнопками

**Функционал:**
- **Три поля ввода**: Имя, Email, Телефон
- **Валидация в реальном времени**: 
  - Имя - только кириллица и пробелы
  - Email - проверка формата
  - Телефон - международная валидация через intl-tel-input
- **Визуальная индикация ошибок**: Красная рамка и текст при некорректном вводе
- **Форматирование телефона**: Автоматическое по стандартам выбранной страны
- **Состояния**: Loading, Success, Error
- **Адаптивность**: Автоматическая перестройка макета по ориентации экрана
- **Оптимальные пропорции**: Двухколоночный макет 1000-1100px для гармоничного баланса
- **Анимация**: Плавные переходы и hover-эффекты

## 🎯 Функционал формы

### Поля ввода
1. **Имя** - обязательное текстовое поле
   - Валидация на этапе ввода (при потере фокуса и в реальном времени)
   - Разрешены только кириллические буквы (а-яА-ЯёЁ) и пробелы
   - Исключены латиница, цифры и спецсимволы
2. **Email** - обязательное поле с валидацией формата
   - Валидация на этапе ввода (при потере фокуса и в реальном времени)
   - Проверка соответствия формату email (name@domain.zone)
3. **Телефон** - обязательное поле с международным форматом и выбором страны

### Маска телефона (intl-tel-input v23.0.12)
- **Библиотека**: intl-tel-input подключена через CDN
- **Функции**:
  - Автоматический выбор страны с флагами
  - Автоматическое форматирование номера для выбранной страны
  - Валидация номера через встроенный метод `isValidNumber()`
  - Ограничение ввода только цифрами (keypress + paste фильтрация)
  - Предпочитаемые страны: Россия, Казахстан, Беларусь, Украина, США
  - По умолчанию: Россия (+7)

### Валидация

#### Валидация на этапе ввода (клиентская)
- **Имя**:
  - Проверка при потере фокуса (`@blur`)
  - Реальное время при наличии ошибки (`@input`)
  - Регулярное выражение: `/^[а-яёА-ЯЁ\s]+$/` (только кириллица и пробелы)
  - Визуальная индикация: красная рамка + текст ошибки
  
- **Email**:
  - Проверка при потере фокуса (`@blur`)
  - Реальное время при наличии ошибки (`@input`)
  - Регулярное выражение: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
  - Визуальная индикация: красная рамка + текст ошибки

- **Телефон**:
  - Автоматическое форматирование через intl-tel-input
  - Ограничение ввода только цифрами
  - Валидация номера через `isValidNumber()`

#### Валидация при отправке формы
- Проверка заполненности всех полей
- Повторная валидация имени (кириллица + пробелы)
- Повторная валидация email по регулярному выражению
- Проверка телефона через intl-tel-input (`isValidNumber()`)
- Получение полного международного номера через `getNumber()`

#### Сообщения об ошибках
- **Имя пустое**: "Пожалуйста, введите ваше имя"
- **Имя содержит недопустимые символы**: "Пожалуйста, используйте только кириллицу"
- **Email пустой**: "Пожалуйста, введите email"
- **Email некорректный**: "Пожалуйста, введите корректный email"
- **Телефон пустой**: "Пожалуйста, введите телефон"
- **Телефон некорректный**: "Пожалуйста, введите корректный номер телефона"

### Состояния
- **Loading**: Показывает spinner во время отправки
- **Success**: Зелёное уведомление об успешной отправке (исчезает через 2 сек)
- **Error**: Красное уведомление об ошибке (исчезает через 5 сек)
- **Field Error**: Красная рамка поля + текст ошибки под полем (при некорректном вводе)

### Добавление новых полей
В `pages/LoginPage.vue`:

```vue
<!-- Добавьте в форму -->
<div class="form-field">
  <label class="block text-sm font-semibold mb-2 flex items-center" 
         style="color: var(--color-text);">
    <i class="fas fa-icon mr-2" style="color: var(--color-text-secondary)"></i>
    Название поля
  </label>
  <input 
    v-model="form.fieldName" 
    type="text" 
    class="input" 
    :class="{ 'input-error': fieldNameError }"
    placeholder="Placeholder"
    required
    @blur="validateFieldName"
    @input="handleFieldNameInput"
  />
  <p v-if="fieldNameError" class="text-xs mt-1.5 ml-1" style="color: var(--color-danger);">
    {{ fieldNameError }}
  </p>
</div>
```

Не забудьте добавить реактивные переменные и функции валидации в `<script setup>`:

```javascript
const fieldNameError = ref(null)

function validateFieldName() {
  const value = form.value.fieldName.trim()
  
  if (!value) {
    fieldNameError.value = 'Пожалуйста, заполните поле'
    return false
  }
  
  // Добавьте свои правила валидации
  // ...
  
  fieldNameError.value = null
  return true
}

function handleFieldNameInput() {
  if (fieldNameError.value) {
    validateFieldName()
  }
}
```

## ⚠️ Известные проблемы

### Выпадающий список стран
**Проблема**: Выпадающий список стран может не адаптироваться автоматически к доступному пространству на экране (открытие вверх/вниз).

**Текущее состояние**:
- Список работает, но может открываться за пределы видимой области
- `overflow: hidden` убран с класса `.card` в `styles.tsx` для предотвращения обрезки
- Попытки использования `dropdownContainer: document.body` ломали работу библиотеки
- Кастомное JS-позиционирование также не дало результата

**Возможные решения для будущего**:
1. Использовать другую библиотеку (например, vue-tel-input)
2. Реализовать полностью кастомный компонент выбора страны
3. Настроить правильную работу `dropdownContainer` с учетом специфики Chatium платформы

## 📝 Важные замечания

### JSX в Chatium
В Chatium JSX **НЕ используйте** `dangerouslySetInnerHTML` для style тегов:

❌ **Неправильно:**
```tsx
<style dangerouslySetInnerHTML={{ __html: cssVariables }} />
```

✅ **Правильно:**
```tsx
<style>{cssVariables}</style>
```

### CSS Variables
Все цвета управляются через CSS-переменные, что позволяет:
- Легко менять цветовую схему
- Поддерживать консистентность дизайна
- Быстро адаптировать под другие проекты

### Локальные ресурсы
Все CDN заменены на локальные версии для работы в России:
- TailwindCSS: `/s/static/lib/tailwind.3.4.16.min.js`
- FontAwesome: `/s/static/lib/fontawesome/6.7.2/css/all.min.css`

## 🎨 Дизайн-система

### Отступы
- Внутри карточки: `3rem` (48px)
- Между полями: `1.25rem` (20px)
- Отступы полей ввода: `1rem 1.25rem`

### Скругления
- Карточка: `1.5rem` (24px)
- Поля ввода: `0.75rem` (12px)
- Кнопки: `1.25rem` (20px)
- Иконка: `1rem` (16px)

### Тени
- `--shadow-sm`: Лёгкая тень для полей
- `--shadow-md`: Средняя тень для кнопок
- `--shadow-lg`: Сильная тень для карточки
- `--shadow-xl`: Максимальная тень при hover

### Анимации
- Transitions: `0.3s ease`
- Hover на карточке: увеличение тени без подъема (для корректной работы выпадающего списка стран)
- Hover на кнопке: подъем `-2px` с увеличением тени

## 📱 Адаптивность

Форма полностью адаптирована под разные устройства и ориентации экрана:

### Портретная ориентация (мобильные устройства)
- Форма занимает почти весь экран с небольшими отступами (0.75rem)
- Вертикальное расположение всех элементов
- Иконка и заголовки центрированы сверху
- Все поля ввода расположены вертикально
- Адаптивные отступы для комфортного использования

### Альбомная ориентация (десктопы)
- **Макет с изображением спикера слева**:
  - **Левая панель**: Изображение спикера (340px, 380px на экранах >1200px) с бежевым градиентным оверлеем, реализовано через псевдоэлемент `::before`
  - **Правая панель**: Компактная форма с вертикальными полями, занимает всё оставшееся пространство
- **Оптимальная ширина для двухколоночного макета**:
  - Обычные экраны: max-width **1000px**
  - Большие экраны (>1200px): max-width **1100px**
  - Карточка имеет `width: 100%` для заполнения контейнера
- Внутренние отступы карточки: 3rem (3.5rem на больших экранах)
- Минимальная высота: 600px (650px на больших экранах)
- Изображение обрезается автоматически (background-size: cover), убирая прозрачные поля
- Футер прижат к низу правой панели

### Адаптивные брейкпоинты
- **< 640px**: Минимальные отступы, уменьшенные шрифты и размеры элементов
- **< 900px (landscape)**: Переключение в одноколоночный режим даже в альбомной ориентации
- **> 1200px (landscape)**: Увеличенные размеры элементов и отступы для комфортного просмотра

### Особенности реализации
- Использование `display: block` с левым padding для размещения изображения спикера
- Изображение спикера реализовано через псевдоэлемент `::before` с абсолютным позиционированием
- Media queries на основе `orientation` для определения типа экрана
- Flexbox для вертикального выравнивания контента
- Форма никогда не выходит за пределы видимой области экрана
- Левая панель HTML-разметки (`.form-header`) скрыта на десктопах, используется только на мобильных
