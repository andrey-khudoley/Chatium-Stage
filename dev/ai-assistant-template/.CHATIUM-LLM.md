**v2.8** (2025-11-05)
- 🎯 **ФИНАЛЬНОЕ ИСПРАВЛЕНИЕ**: Множественная отправка ответов - переход на явное добавление инструмента
  - **Проблема**: Несмотря на добавленную проверку в хук, агент всё ещё отправляет несколько копий ответа
  - **Корневая причина** (обнаружена после диагностики): Проверка в глобальном хуке ненадёжна, потому что:
    - Хук вызывается несколько раз во время обработки одного сообщения
    - `params.tools` может быть неполным списком
    - Даже с проверкой `params?.tools?.some()`, инструмент может добавляться параллельно
    - Система может вызывать хук асинхронно, создавая race condition
  - **Решение**: Архитектурный переход от глобального хука к явному добавлению:
    ```typescript
    // ✅ НОВАЯ АРХИТЕКТУРА
    // tools/sendChatResponsePodolyak.ts - только определение инструмента, БЕЗ хука
    export const sendChatResponsePodolyakTool = app
      .function('/send-chat-response-podolyak')
      .meta({ name: 'sendChatResponsePodolyak', ... })
      .handle(async (ctx, body) => { ... })
    
    // api/agent.ts - явное добавление в enabledTools
    import { sendChatResponsePodolyakTool } from '../tools/sendChatResponsePodolyak'
    
    const agent = await getOrCreateAgentForWorkspace(ctx, finalKey, {
      title: finalTitle,
      instructions: finalInst,
      enabledTools: [sendChatResponsePodolyakTool]  // ✅ Явное добавление = гарантия одного раза
    })
    ```
  - **Преимущества явного добавления**:
    - ✅ Инструмент добавляется РОВНО ОДИН РАЗ при создании агента
    - ✅ Нет race condition, нет асинхронных конфликтов
    - ✅ Явно видно что и где подключается
    - ✅ Нет глобальных побочных эффектов
    - ✅ Просто и надёжно
  - **Удаленные элементы**:
    - Убран глобальный хук `app.accountHook('@start/agent/tools')`
    - Убраны попытки проверки инструмента в lists
    - Упрощена архитектура
  - **Файлы изменены**:
    - `tools/sendChatResponsePodolyak.ts` - убран хук, упрощено
    - `api/agent.ts` - добавлен импорт инструмента и явное добавление в enabledTools
  - **Результат**: Инструмент добавляется один раз, агент отправляет ровно один ответ на каждый запрос

**v2.7** (2025-11-05)
- 🐛 **КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ**: Множественная регистрация инструмента через хук
  - **Проблема**: При каждом запросе агент генерировал и отправлял несколько копий одного и того же ответа
  - **Симптомы**:
    - На один запрос пользователя агент отправлял 2-3 одинаковых сообщения
    - С каждым новым запросом количество дубликатов увеличивалось
    - Индикатор "печатает..." мигал несколько раз
  - **Причина**: Хук `@start/agent/tools` вызывается множество раз:
    - При создании агента
    - При пробуждении агента  
    - При обработке каждого сообщения
    - При выполнении любой операции с агентом
    - Каждый вызов возвращал инструмент БЕЗ проверки, что он уже добавлен
    - Инструмент накапливался в списке `enabledTools`
    - Агент вызывал все копии инструмента последовательно
  - **Решение (Вариант 1)**: Добавлена проверка перед регистрацией:
    ```typescript
    app.accountHook('@start/agent/tools', async (ctx, params) => {
      // Проверяем, был ли инструмент уже добавлен
      const hasToolAlready = params?.tools?.some(
        t => t.meta?.name === 'sendChatResponsePodolyak'
      )
      
      if (!hasToolAlready) {
        return sendChatResponsePodolyakTool
      }
      
      return null  // Не добавляем повторно
    })
    ```
  - **Альтернативное решение (Вариант 2)**: Можно было убрать хук полностью и явно добавлять инструмент через `enabledTools` в `api/agent.ts`, но Вариант 1 безопаснее для существующих агентов
  - **Файлы изменены**:
    - `tools/sendChatResponsePodolyak.ts` - добавлена проверка в хук
    - `.CHATIUM-LLM.md` - обновлена документация с описанием проблемы и решения
  - **Техническая деталь**: 
    - Проверка выполняется через `params?.tools?.some()` - поиск в массиве существующих инструментов
    - Сравнение по `meta.name` - уникальному идентификатору инструмента
    - Добавлено логирование для отладки (✅ добавляем / ⏭️ пропускаем)
  - **Результат**: Агент теперь отправляет ровно один ответ на каждый запрос пользователя (БЫЛО ОШИБКОЙ, ИСПРАВЛЕНО В v2.8)

    - При выборе агента сначала проверяется наличие сохранённого chainKey
    - Если найден - используется сохранённый, иначе создаётся новый и сохраняется
    - При сбросе контекста новый chainKey автоматически заменяет старый в localStorage
  - **Файлы изменены**:

  - **Поведение**:
    - **Первый заход**: создаётся новый chainKey и сохраняется → показывается маркер выбора агента
    - **Обновление страницы**: загружается сохранённый chainKey → история полностью восстанавливается
    - **Смена агента**: загружается/создаётся chainKey для выбранного агента
    - **Сброс контекста**: создаётся новый chainKey и заменяет старый → начинается новый диалог
  - **Логирование**: Добавлены консольные логи `📦 Используем сохранённый chainKey` и `🆕 Создаём новый chainKey` для отладки
  - **Изоляция**: У каждого пользователя и каждого агента свой chainKey в localStorage
  - **Результат**: История диалога сохраняется между сессиями и обновлениями страницы
