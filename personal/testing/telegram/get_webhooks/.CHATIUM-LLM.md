# Telegram Webhooks Viewer

## Описание проекта

Приложение для просмотра и мониторинга событий (updates) из Telegram API через механизм long polling. Позволяет настроить один или несколько токенов Telegram-ботов и автоматически опрашивать API каждые 15 секунд, сохраняя все полученные события в базу данных.

## Архитектура

### Таблицы данных

#### `settings.table`
Хранит настройки приложения:
- `key` (StringKind) - ключ настройки
- `value` (StringKind) - значение настройки

Используемые ключи:
- `bot_tokens` - токены ботов (разделены переносами строк или запятыми)
- `is_active` - статус активности опроса ('true' / 'false')
- `active_job_task_id` - ID текущей активной задачи опроса

#### `webhooks.table`
Хранит события из Telegram:
- `botToken` (StringKind) - токен бота, от которого получено событие
- `updateId` (NumberKind) - уникальный ID обновления из Telegram
- `data` (AnyKind) - сырые данные события в формате JSON
- `receivedAt` (DateKind) - дата и время получения события

### API endpoints

#### GET `/api/settings`
Получение всех настроек приложения.
- Возвращает: `{ success: boolean, settings: Record<string, string> }`

#### POST `/api/settings`
Сохранение настроек.
- Параметры: `{ tokens: string, isActive?: boolean }`
- При включении `isActive=true` запускает job опроса
- Возвращает: `{ success: boolean, error?: string }`

#### GET `/api/webhooks?limit=50&offset=0`
Получение списка событий с пагинацией.
- Query параметры: `limit`, `offset`
- Возвращает список событий с ограниченным preview данных

#### GET `/api/webhooks/:id`
Получение полных данных одного события.
- Возвращает полный JSON события

### Jobs (фоновые задачи)

#### `pollTelegramWebhooksJob` (`/poll-telegram-webhooks`)
Основная задача опроса Telegram API:
1. Проверяет активность опроса через настройку `is_active`
2. Получает список токенов из настроек
3. Для каждого токена:
   - Определяет последний обработанный `update_id`
   - Вызывает `getUpdates` с offset = last_update_id + 1
   - Сохраняет новые события в таблицу `webhooks`
4. Запланирует следующий запуск через 15 секунд (если опрос активен)

**Важно**: Job самостоятельно планирует следующий запуск, создавая цикл опроса.

#### `cleanupOldWebhooksJob` (`/cleanup-old-webhooks`)
Очистка старых событий (старше 7 дней):
- Запускается ежедневно в 3:00
- Удаляет события старше 7 дней
- Автоматически планирует следующий запуск

### Страницы

#### Главная страница (`/` - `HomePage.vue`)
Отображает список событий Telegram:
- Таблица со списком событий (ID, токен бота, update_id, время)
- Пагинация (50 событий на страницу)
- Модальное окно для просмотра полного JSON события
- Автообновление каждые 5 секунд
- Ссылка на страницу настроек

#### Страница настроек (`/settings` - `SettingsPage.vue`)
Управление настройками:
- Текстовое поле для ввода токенов (через перенос строки или запятую)
- Переключатель автоматического опроса
- Кнопка сохранения настроек
- Уведомления об успехе/ошибке

### Инициализация (`init.ts`)

При старте приложения (`@start` hook):
1. Инициализирует таблицу webhooks
2. Планирует первый запуск задачи очистки на ближайшее 3:00

## Особенности реализации

1. **Long Polling**: Используется метод Telegram API `getUpdates` с параметром `timeout=10` секунд
2. **Отслеживание offset**: Для каждого бота отслеживается последний обработанный `update_id`
3. **Предотвращение дубликатов**: Перед сохранением проверяется, не существует ли уже событие с таким `update_id`
4. **Автоматическое продолжение**: Job сам планирует свой следующий запуск через 15 секунд
5. **Остановка опроса**: При выключении `isActive` удаляется `active_job_task_id`, что останавливает цепочку запусков

- FontAwesome - для иконок

## Логирование

Все ключевые операции логируются через `ctx.account.log()`:
- Запуск/остановка джобов
- Получение обновлений от Telegram
- Ошибки при сохранении событий
- Результаты очистки старых данных
